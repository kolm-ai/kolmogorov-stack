# Opt-in heavy extras: sub-2-bit + QAT calibration-time / training-time
# optimizers for the aqlm / quip / qat quantization lanes.
#
# These are PINNED to commit SHAs so the build is reproducible. They are heavy
# (CUDA build steps, large transitive deps) and are intentionally NOT in the
# base requirements.txt, so a plain `pip install -r requirements.txt` never
# pulls a multi-GB CUDA research repo.
#
# Install ONLY when you intend to run the turnkey sub-2-bit / QAT lanes:
#
#   KOLM_QUANT_OPTIMIZERS=1 pip install -r requirements-optimizers.txt
#
# Once installed, run_aqlm / run_quip / run_qat in scripts/quantize.py drive
# the optimizer via `python -m <module>` automatically when $*_REPO_PATH is
# unset. Operators who keep their own checkout still override via
# $AQLM_REPO_PATH / $QUIP_SHARP_REPO_PATH / $EFFICIENT_QAT_REPO_PATH (env wins).
#
# NOTE: the pinned SHAs below are placeholders that the operator pins to the
# exact upstream commit they have validated; replace before vendoring into a
# locked build. They are commented so CI never attempts a network git install
# unless the operator deliberately uncomments them.
