#### Moderate-severity advisories (2)

1. **qs** `6.11.1 - 6.15.1` — [GHSA-q8mj-m7cp-5q26](https://github.com/advisories/GHSA-q8mj-m7cp-5q26) — DoS: `qs.stringify` throws TypeError on null/undefined entries in comma-format arrays when `encodeValuesOnly` is set. CVSS 5.3. CWE-476.
2. **express** `4.21.0 - 4.22.1` — depends on vulnerable `qs`.

**Decision: retained at pinned 4.22.1 with no `npm audit fix`.**

Mitigation analysis:
- The kolm server (`server.js`, `src/router.js`) does NOT call `qs.stringify` with `encodeValuesOnly: true` AND `arrayFormat: 'comma'`. Express uses `qs` for request-body / querystring PARSING (`qs.parse`), not for the vulnerable `stringify` path.
- `npm audit fix` (non-`--force`) reports `fix available` but resolves to the same minor branch (4.22.x); no clean non-major upgrade closes the moderate. The advisory targets `qs.stringify`, not `qs.parse`.
- `npm audit fix --force` would jump express to a major-version bump (5.x). Per WC01 design constraint: **do not upgrade any dep major version without explicit user authorization.**
- W707-W835 plan does not mandate express 5 migration.

Action: **document mitigation here, ship pinned 4.22.1.** Re-evaluate if (a) `qs` ships a patched 6.x that hits the express 4.x branch, or (b) the user explicitly authorizes the express 5 migration as a separate wave.
