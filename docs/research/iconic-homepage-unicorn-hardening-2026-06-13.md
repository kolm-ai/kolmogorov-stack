## Product Read

The market is not missing another gateway, eval dashboard, data replication platform, or API manager. Those categories already own routing, tracing, scoring, connectors, streams, API policy, and admin surfaces. The product opening has to say what Kolm owns after those systems emit data: governed API behavior becomes a signed portable artifact, with eval/policy gates, runtime targets, and exportable proof.

Pioneer raises the bar on closed-loop improvement, but it is framed around training and verification loops. Kolm should beat that category position by showing the enterprise control layer around the loop: source declaration, opaque unknown-event intake, redaction, retention, eval gates, compile receipts, runtime targets, and governance export. The public site must make that control layer visible immediately.

Airbyte and Confluent are the data-plane pressure test. They show that broad "data in/out" claims only sound credible when the product distinguishes source, sink, stream, storage, connector, API, and operational write paths. Kolm copy should keep that distinction and avoid pretending to semantically understand every vendor payload without an adapter state.

## Image-2 Design Contract

The reference is strongest when it behaves like an infrastructure product, not a marketing hero:

- Thin sticky nav, mostly text, with one black primary action.
- Large left-aligned display headline with a narrow measure so the green line breaks become an identity mark.
- Paper background, low-contrast technical grid, no decorative blob system.
- A dark terminal/artifact panel as the product proof object, not a decorative card.
- Mobile should preserve hierarchy by stacking, wrapping terminal code, and avoiding cramped button rows.

## Implemented Hardening

- Removed the heavy above-fold CTA row from the homepage hero and left primary action in nav plus the later conversion band.
- Tightened desktop top rhythm to match the image-2 first viewport.
- Narrowed the homepage headline measure so the green line becomes a deliberate signature.
- Rewrote the hero lead around the control-to-artifact contract with `.kolm`, eval/policy gates, receipts, and exports.
- Made the home nav closer to the reference: text sign-in, black compact API-key CTA, no extra boxed secondary chrome.
- Wrapped mobile terminal code so long commands do not visually smash into the viewport.
- Reconciled the route proof count across static HTML, generated product graph, docs spec, and contract tests from `922` to `923`.
