## Executive summary

kolm is **two products on one stack**: (A) Agent Security Evidence / ASR (audit.kolm.ai) and (B) the model platform / Compiler (kolm.ai) - compile, distil, train, quantize, serve, sign. This document specs the platform stack end to end and audits each category against the June 2026 frontier.

**Headline finding.** The engine is, in most categories, *already at or near the frontier and unit-tested* - but a large fraction of that capability is **built and not wired into the live path**. The dominant, lowest-risk, highest-value work is therefore **wiring already-paid-for capability**, not inventing new methods. Twelve categories carry frontier-grade primitives (see Already-at-frontier per section); the recurring gap is emission/dispatch/registry-drift, not correctness.

**What is genuinely missing at the frontier (net-new, spec-gated):** GAD adversarial distillation (arXiv:2511.10643, the #1 black-box method), a shared boot-and-measure probe harness (flips estimated->tested across 5 categories), a pluggable real embedding provider (uncaps curation + routing), a real attestation cert-chain verifier (ends TOFU), a hosted managed distill+serve fleet (the biggest DX gap vs competitors), and an in-browser verify-then-run WebGPU demo.

**Counts:** 36 surgical-now improvements (test-gated, low/med risk), 9 cross-cutting bets (spec-followups), 12 confirmed already-at-frontier capabilities.

## Cross-cutting bets (spec-followups)

### Bet 1: Real measurement / accuracy-gate harness (boot-and-measure on GPU)

Categories: quantization, kv-cache, speculative-decoding, distillation, agent-security-eval

A shared 'probe-harness' that boots an artifact under a config and measures real numbers recurs as the missing piece across at least five categories: quantization post-quant accuracy floor (kscore drop vs baseline), KV-cache measure_kv_policy (compression_ratio/retained/evicted/peak_kv_mb/quality_delta to flip passport rows estimated->tested), spec-decode no-draft baseline speedup (vLLM acceptance/accepted-length is now parsed in W630), distillation method bake-off (SeqKD vs ROPD vs GAD with a teacher judge), and agent-security benign-utility/utility-under-attack ASR. Build ONE boot-and-measure load driver in apps/export/probe.py and reuse it; without it, all 'receipt is the proof' claims on these axes stay synthetic/estimated.

### Bet 2: Pluggable real semantic embedding (lexical -> learned)

Categories: synthetic-data-curation, llm-routing

Multiple curation/routing capabilities are silently capped because they run on the 256-d hash-bag lexical embedder. A single pluggable real-embedding provider behind embed() (env-selected, default unchanged) simultaneously lifts data-curation dedup/clustering/diversity/DSIR/label-error AND enables the calibrated meets-bar quality predictor for routing and per-model similarity discrimination. High-leverage single change touching many consumers.

### Bet 3: Frontier trainers that are entirely absent (the headline #1 methods)

Categories: distillation, moe-distill-quant, speculative-decoding

Three top-ranked frontier methods have zero code and warrant full specs with cost caps: GAD (Generative Adversarial Distillation, arXiv:2511.10643) the #1 black-box distillation method; MoE-teacher->dense-student collapse (krafton moe-to-dense, DO-ACP expert scoring + FFN merge + staged forward-KL) plus actually executing expert pruning; and EAGLE-3 draft-head auto-training from the same distillation data (SpecForge/vLLM speculators v0.3) so kolm-distilled models are self-accelerating instead of depending on 3 third-party heads. Each reuses existing GRPO/KD/distill cores.

### Bet 4: Privacy execution path (the honest stubs that never run)

Categories: confidential-compute, compile-api-to-model-competitors

The privacy story is architecturally complete but does not execute end-to-end: air-gap distill writes a queue that no worker consumes, cloud-distill is a no_pool_configured stub, and there is no confidential-GPU (H100/H200 CC) infra generated. Compounds with the hosted-fleet bet. Stand up the trainer-queue consumer (apps/trainer worker scanning ~/.kolm/airgap-distill-runs and a managed/bridged GPU pool over the existing Modal/RunPod/Together runner scaffolds), then add DP-SGD (Opacus + epsilon accountant) and GPU-TEE BYOC scripts with NRAS attestation capture.

### Bet 5: Proven-compute: real attestation crypto-chain verification

Categories: verifiable-inference, confidential-compute

Every TEE/GPU attestation path is shape-only/TOFU; no vendor cert chain is walked anywhere. Specs: a real NRAS verifier (validate token sig, walk NVIDIA device-identity chain to pinned root, bind nonce to output digest) registered via the existing pluggable seam; real TDX/SEV-SNP/Nitro cert-chain verification ending pure TOFU; and a Proven-Compute Receipt binding the Ed25519-signed artifact to a GPU-TEE attestation anchored in the transparency log. This is the only path from 'signed artifact + tamper-evident log' to 'proven compute'.

### Bet 6: In-browser verify-then-run (WebGPU) demo + signed weights digest

Categories: ondevice-inference, model-signing-standards

The flagship on-device demo (verify the model runs AND is the signed one, entirely in-browser) is unbuilt; the browser layer only checks JSON signatures. Spec: a WebGPU runner (WebLLM/LlamaWeb) that SHA-256s downloaded weight bytes in-browser and checks an Ed25519-signed weights_sha256 from the manifest BEFORE handing bytes to the engine. Requires the prerequisite bet of pinning+signing a weights SHA-256 in the manifest/artifact (the puller already computes the hash).

### Bet 7: Hosted managed distill+serve fleet + continuous-RL (competitive parity)

Categories: compile-api-to-model-competitors, confidential-compute

The single biggest DX gap vs every named competitor (OpenPipe/Together/Fireworks/Modal/Baseten): no Kolm-hosted training/inference fleet, so 'upload captures -> we distill -> download signed .kolm' cannot execute without the customer bringing a GPU. Pair with a continuous-RL loop (OpenPipe ART parity) re-distilling from production captures gated by the existing W808 regression gate and signed into superseding .kolm artifacts. Reuses existing runner scaffolds, CLOUD_METER_RATES, grpo/online_dpo.

### Bet 8: Per-expert mixed-precision quant + size-aware quality curves

Categories: moe-distill-quant, quantization

MoE quant is uniform-bit per group (byte-accounting only, not real per-expert precision); sub-2-bit expert methods (MC-MoE/GEMQ) are oracle-listed external_repo but never run; and FP4 oracle quality_loss is a flat scalar that cannot express small-model sensitivity (the research's named #1 NVFP4 risk). Specs: a real per-expert quant loop assigning bits by activation hotness (router fp16), a paramsB-dependent FP4 quality curve, and fusing the BATQuant calibration plan into the NVFP4 export instead of modelopt's trivial forward_loop.

### Bet 9: Registry/source-of-truth consolidation + benchmark fields

Categories: speculative-decoding, small-llm-students

Recurring drift: three speculative-decoding draft registries disagree; student-arch-recommender slugs are unlinked to real pullable backbones; FRONTIER_MODELS verified tier is a year stale; no registry carries benchmark scores (MMLU-Pro/IFEval/BFCL) so ranking is VRAM/license-only. Consolidate into single sources of truth with cross-file equality tests, refresh verified rows, and add a sourced/dated benchmark table to enable score-aware recommend().

- Small-LLM students: W610 closed the dependency chain - current 2026 student rows now exist before DEFAULT_MODEL points at Qwen3-4B, and current Qwen3/Phi-4 rows are promoted into FRONTIER_MODELS with exact HF model-card URLs.
- On-device WebGPU demo (bet) depends on first pinning+signing a weights SHA-256 in the model-weights manifest/artifact (bet/spec) — the 'is-the-signed-one' half is only cryptographically real once a signed weights digest exists for the browser to check.
- Quantization accuracy gate, KV 'tested' passport rows, spec-decode no-draft speedup baselines, and the distillation method bake-off ALL depend on the single shared boot-and-measure probe harness (bet). W630 closes vLLM acceptance/accepted-length parsing, but net acceleration still needs a paired baseline run. Building the harness once avoids four divergent measurement implementations; the spec deltas explicitly note this overlap.
- MoE-to-dense distillation (bet) depends on first executing expert pruning to a smaller checkpoint (spec) — pruning/merge is the prerequisite stage before the FFN-collapse + forward-KL recovery can run. Both reuse the existing distill.py compute_loss.
- Privacy execution: DP-SGD, confidential-GPU BYOC scripts, and the air-gap worker (all confidential-compute) compound with the hosted managed fleet (compile-api). The air-gap queue consumer and the hosted fleet both need the same trainer-queue + runner-pool plumbing — design one orchestrator over the existing Modal/RunPod/Together runner scaffolds rather than two.
- Model-signing: the build-time DSSE sidecar and the OMS file-manifest bundle (both surgical) both write into src/artifact.js using the per-member SHA-256 map already computed into manifest.hashes — sequence them in one artifact-build change to share that digest computation and avoid double iteration.
- No hard contradictions found across the 16 deltas; the recurring tension is breadth-of-honest-stubs vs depth-of-execution — many categories are 'built and tested but unwired,' so prioritize wiring (surgical-now) before net-new frontier trainers (bets), since wiring unlocks already-paid-for capability at low risk.
- MCP: W609 closed the durable store+signer prerequisite and production Merkle enqueue/stamp-back path. The remaining MCP work is no longer receipt durability; it is upstream MCP client execution/registry, mcp_tool_receipts lookup from the in-toto route, tool-metadata pinning, and per-tool policy.

**Frontier (June 2026 SOTA).** June 2026 SOTA for black-box (API-teacher, no-logits) distillation, per research/strategy-2026/raw-tech-frontier/tf1-distillation-sota.json:
1. GAD (Generative Adversarial Distillation), arXiv:2511.10643 (Ye et al, 2025-11-13, Microsoft) — #1 for black-box. Student=generator vs a co-evolving discriminator that acts as an on-policy reward model over teacher-vs-student TEXT. Closes most of the SeqKD gap (3-5+ pts); Qwen2.5-14B+GAD reaches GPT-5-Chat parity on LMSYS. Open-source artifacts (ytianzhu.github.io/Generative-Adversarial-Distillation).
2. GKD / on-policy KD, arXiv:2306.13649 (Agarwal et al, ICLR2024); MiniLLM reverse-KL arXiv:2306.08543 (Gu et al).
3. Best-of-N / rejection-sampling distill — BOND arXiv:2407.14622, BRTS arXiv:2605.09725 (2026); +2-4 pts on reasoning/alignment, combines with on-policy.
4. ROPD (Rubric-based On-Policy Distillation) arXiv:2605.07396 — induce per-prompt rubric from teacher TEXT, score student rollouts, GRPO; black-box, ~10x sample efficiency.
5. GrayKD multi-rationale injection (AAAI 2026); CoT/reasoning-path distillation w/ self-consistency filtering.
Startup recommendation: GAD + GKD/BoN give the biggest jump capturing real API traffic; all logit-free.

- AgentHarm (Gray Swan + UK AISI, arXiv:2410.09024, ICLR 2025): 110/440 malicious multi-step tasks, 11 harm categories; measures refusal vs tool-backed harmful execution + jailbreak robustness.
- Agent Security Bench / ASB (openreview V4y0CpX4hK): ~84% avg ASR across attacks, limited defense efficacy.
- Gray Swan Arena / ART benchmark (arXiv:2507.20526): massive dynamic red-teaming (272k+ attacks on 13 frontier models, none fully clean).
