# Whole-Stack SOTA Deep Dive

Date: 2026-06-17

This pass continues the same finalized-pass workflow at the full-stack frontier
layer. The machine-readable ledger is
`docs/whole-stack-sota-deep-dive-2026-06-17.json`, generated by
`scripts/build-stack-sota-deep-dive.mjs`.

## Scope

The ledger covers the 16 frontier categories from
`docs/STACK-TECH-SPEC-2026-06-15.md`:

- distillation
- MoE distill and quantization
- quantization
- KV cache
- speculative decoding
- fine-tuning frameworks
- synthetic data curation
- small LLM students
- on-device inference
- LLM routing
- MCP tool gateway receipts
- verifiable inference
- model signing standards
- confidential compute
- agent security evaluation
- compile API to model competitors

Each category is tied back to the backend atomic component ledger, product SOTA
readiness, frontier maps, math frontier, research atlas, buildbook, and
implementation spec.

## Review Standard

Every category must carry:

- as-built code evidence
- frontier delta and gap trace
- atomic component mapping
- readiness and claim scope
- improvement or invention path
- verification gate

This ledger does not turn open external gates into completed evidence. It makes
the state explicit: where the stack is locally aligned, where local wiring or
new methods remain, and where public packages, benchmarks, partners, or live
certifications are still required.

## External Freshness Probe

Because frontier stack claims move quickly, the generated ledger also records a
primary-source spot check for volatile areas before this pass is locked:

- distillation: GAD black-box on-policy distillation paper
  (`https://arxiv.org/abs/2511.10643`)
- quantization: NVIDIA TensorRT-LLM quantization docs and NVIDIA Model
  Optimizer repository
- speculative decoding: vLLM speculative decoding docs
- confidential compute: NVIDIA Attestation Suite docs and Intel Trust
  Authority GPU attestation docs

The probe is not treated as proof that local gaps are closed. It is used to keep
the improvement tracks aligned with current external references.

## Cross-Stack Invention Themes

1. Boot-and-measure harness for quality, latency, acceptance rate, cache
   retention, and regression gates.
2. Proof fabric across artifacts, audit reports, MCP calls, runtime passports,
   and compliance exports.
3. Hosted private training fleet with privacy gates, worker attestations, and
   artifact receipts.
4. Frontier method bakeoff for distillation, MoE, quantization, and student
   selection.
5. Cross-device verified runtime where signed weights are checked before
   WebGPU, WASM, mobile, or edge execution.

## Gate

`npm run verify:stack-sota` checks that the generated ledger is current and that
all 16 stack categories have local evidence, frontier gaps, atomic components,
improvement tracks, and verification commands. The gate is included in
`npm run verify:depth`.
