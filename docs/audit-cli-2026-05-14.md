## Top 10 gaps vs SOTA (prioritized)

1. **No `--json` / `--output` format flag** — uv, gh, wrangler all have this. Add to run/eval/logs/doctor. ~2 lines each.
2. **Inconsistent flag parsing** — two pickers (`pickFlag`, `flag`) with different null/empty semantics. Standardize to one. ~50 lines.
3. **No `--color` / `--no-color` + `NO_COLOR` env handling** — peers do this. Detect tty + env, gate formatters. ~20 lines.
4. **Exit codes inconsistent and undocumented** — define `EXIT.OK/BAD_ARGS/MISSING_PREREQ/EXECUTION`, use everywhere. ~100 lines.
5. **Help text lacks EXAMPLES sections** — gh has them. Add to each HELP[] entry. ~150 lines.
6. **No shell completion** — `kolm completion {bash,zsh,fish}` emitting a `_kolm()` function. ~80 lines.
7. **Error messages lack context/suggestions** — wrap `console.error` in `error(msg, hint)`. ~100 lines.
8. **No `--dry-run` on compile/deploy** — wrangler/vercel/gh have. ~30 lines.
9. **No `kolm upgrade`** — gh, vercel, uv self update. ~40 lines.
10. **Unknown flags silently ignored** — should fail with "flag not defined" + suggest closest. ~80 lines.

## Quick wins (under 20 lines each)
