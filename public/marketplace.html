What every entry ships with

  • sha256. The exact bytes of the artifact on disk are hashed and published in the catalog manifest. Re-hashing locally must match.
  • ed25519 receipt chain. Every recipe inside the .kolm carries the spec, seeds, train, eval, and recipe receipts. kolm verify walks the chain offline.
  • License. Apache-2.0 for every kolm-verified artifact. Third-party submissions declare their own license; the catalog filter respects it.
  • K-score. Measured against the artifact's holdout set at compile time. The number on this page is the same number the verifier sees.
  • Honest badges. HIPAA / BAA / GDPR appear only where the artifact's design supports the claim. A redactor that strips PHI gets HIPAA; a classifier that touches PII does not.

Catalog manifest

The marketplace ships a deterministic, signed manifest at /v1/marketplace/catalog.json. The signature algorithm is sha256-anchor over the canonical JSON of the manifest body. The ed25519 signature ships alongside via the sidecar without breaking the verifier contract.

$ curl -s https://kolm.ai/v1/marketplace/catalog.json | jq .spec
"kolm-marketplace-1"
$ curl -s https://kolm.ai/v1/marketplace/catalog.json | jq .signature_algo
"sha256-anchor"

Submit your own

Compile a recipe locally, run kolm marketplace publish ./your-artifact.kolm, and the catalog adds your submission to the manual review queue. We re-run the holdout, re-verify the receipt chain, and flip the Verified badge only after the K-score matches what the manifest claims. No back-channel promotions.
