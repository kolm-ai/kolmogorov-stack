The pipeline

Four stages, each producing an object an operator can inspect.

The strongest infra products make invisible workflow state concrete. Kolm's lifecycle is designed around operator-owned objects, not marketing promises — capture behavior, compile a signed artifact, compose specialists, then deploy to the runtime that fits.

01 / Capture

Route your model traffic.

Point your existing OpenAI- or Anthropic-compatible calls at Kolm. REST, streaming, webhooks, OTEL, MCP, A2A, GraphQL/RPC, queues, warehouse drops, browser events and logs enter as governed events — redacted before persistence.

02 / Compile

Sign captured behavior.

Examples, recipes, evaluator references, tokenizer metadata, hashes and a verifier receipt are packaged into one portable .kolm — signed with Ed25519, reproducible and inspectable layer by layer.

03 / Compose

Assemble specialists.

Eval gates decide what advances: human labels, failure taxonomy, protected slices, drift checks and regression replay compose the artifact from the behavior that actually passed.

04 / Deploy

Target an honest runtime.

Hosted GPU, self-hosted serving, local runner, portable inference, device edge, browser/WASM and restricted fleet targets each get fit evidence, caveats and a runtime recipe. Your data never leaves your hardware.

Operator workbench

The API Control Center turns coverage into a runbook.

Broad API/data collection only matters when an operator can move one source to one proof artifact without losing policy or evidence. The account contract exposes that runbook directly — register, control, improve, release.

Reg · register

Source and schema

Declare the source, owner, environment, connector, adapter manifest, schema hints and opaque-event policy.

Reg · control

Policy before egress

Set redaction, retention, provider allowlists, export destinations, budget, approvals and tenant-isolated cache keys.

Reg · improve

Failure to eval gate

Cluster failures, add labels, build protected regression slices and block promotion when replay degrades.

Reg · release

Artifact to proof export

Compile target receipts, generate runtime recipes and export governance packets to the systems buyers already use.
