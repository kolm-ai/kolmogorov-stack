01 / Lifecycle

Every step produces an object an operator can inspect.

The strongest infra products make invisible workflow state concrete. Kolm's lifecycle is designed around operator-owned objects, not marketing promises.

source
Declare the boundary. Workspace, project, environment, source, connector, credential, schema state, owner, and retention policy are registered before reuse.
/account/api-control-center
capture
Bring behavior into the ledger. REST, streaming, webhooks, OTEL, MCP, A2A, GraphQL/RPC, queues, warehouse drops, browser events, files, logs, and custom adapters enter as governed events.
/v1/route
policy
Constrain before learning. Redaction, provider routing, retention, data boundary, budget, rate limits, approvals, cache scope, and export allowlists travel with the event.
governance
eval
Turn failures into gates. Human labels, failure taxonomy, protected slices, drift checks, and regression replay decide whether a compile can advance.
/v1/evals
compile
Produce a signed artifact. The compile run emits a .kolm artifact, manifest, examples, evaluator references, hashes, target hints, and verifier receipt.
/v1/compile
target
Choose a runtime honestly. Hosted GPU, self-hosted serving, local runner, portable inference, device edge, browser/WASM, and restricted fleet targets get fit evidence and blockers.
/runtimes
export
Make proof portable. Receipts, manifests, eval reports, event streams, SIEM/log drains, warehouse exports, webhook callbacks, and governance packets remain useful outside the UI.
/v1/evidence
02 / Operator Workbench

The API Control Center turns coverage into a runbook.

Broad API/data collection only matters when an operator can move one source to one proof artifact without losing policy or evidence. The account contract exposes that runbook directly.

Inspect Control Center See Data Channels

Register

Source and schema

Control

Policy before egress

Set redaction, retention, provider allowlists, export destinations, budget, approvals, and tenant-isolated cache keys.

Compile target receipts, generate runtime recipes, and export governance packets to the systems buyers already use.

03 / Category Position

Kolm does not need to replace your stack. It has to connect the control loop.

Gateways route, workflow systems orchestrate, data planes move, eval tools score, trust systems govern, and runtimes execute. Kolm owns the transition from captured behavior into governed, signed, portable runtime artifacts.

gateway
Provider gateways and route controls. Keep their routing strength; promote their events into compiler inputs.
source + route
workflow
Workflow engines and approval systems. Keep their triggers and approvals; export receipts and callbacks back into the workflow layer.
approval + callback
data plane
Data movers, streams, and warehouses. Keep the movement layer; bind AI behavior policy, lineage, and export manifests to the moved data.
lineage
trust
GRC, API inventory, SIEM, and assurance systems. Keep control systems current; send governed packets, logs, and evidence instead of screenshots.
evidence
