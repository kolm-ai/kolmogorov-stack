An open, axis-by-axis benchmark for compiled models. Submit a JSONL of responses, get a K-Score per axis plus a composite. Licensed CC-BY-4.0 so anyone can re-run, re-score, and re-publish. 1. What KolmBench measures

KolmBench v1 measures a compiled model along three orthogonal K-Score axes (faithfulness, reliability, efficiency) split across seven task categories. Every task in the canonical pack maps to exactly one category so a model's per-category strengths and weaknesses are legible from the axis breakdown alone.

K-Score axes

F · Faithfulness

Does the output match the ground truth on the task? Token-level + structural agreement against the reference answer.

R · Reliability

Same inputs, same outputs? Variance across N=5 reruns and stability under minor prompt perturbations.

E · Efficiency

Tokens emitted per correct response + p50/p95 latency on a fixed-hardware reference rig. Cost-per-correct, not raw throughput.

Task categories (7)

  • reasoning — deduction chains, counterfactuals, syllogisms, causal/temporal/spatial puzzles.
  • coding — bug-fix, refactor, test-writing, SQL/regex/API design.
  • writing — summarize, rewrite-tone, outline, headlines, email reply.
  • analysis — entity extraction, table-from-text, classification, sentiment, anomaly spotting, claim isolation.
  • support — FAQ matching, empathic deflection, clarifying questions, status updates, escalation triggers.
  • math — algebra, geometry, probability, calculus, linear systems, combinatorics, number theory.
  • tool_use — multi-step planning, tool selection, argument formatting, result parsing, loop termination.

2. Scoring method

K-Score per axis is computed against the reference pack's ground truth. The composite K-Score is the harmonic mean across the three axes (so a model that wins on Efficiency but tanks on Faithfulness cannot game a high composite). Full algorithm in /docs/k-score.

Per-category scores roll up by averaging the composite over each category's tasks, then projecting the seven category scores back through the same harmonic-mean composite to give a single number you can rank against. Sortable on every column in the leaderboard table below.

3. How to submit

Submissions go through a GitHub Actions CI workflow that calls validateSubmission() from src/kolmbench.js on every PR that touches the submissions/ directory. The workflow comments validation pass/fail back on the PR; until the W807 reviewer-bot ships, human review is required to merge a submission into the leaderboard.

JSONL submission row schema

{
  "task_id": "reason-001-arithmetic-word",
  "response": "<model's response text>",
  "model": "your-model-slug-7b",
  "artifact_cid_or_null": "sha256:<hex64>"
}

Required: task_id + response. Optional: model (defaults to the submitter handle) + artifact_cid_or_null (sha256:<hex64> or pending_<id> or null). Each task_id may appear at most once per submission.

Submit a run

# 1. fork github.com/sneaky-hippo/kolmogorov-stack
# 2. add your run at submissions/<your-handle>-<date>.jsonl
# 3. open a PR; the workflow at .github/workflows/kolmbench-submission.yml
#    will validate the shape and comment results back on the PR
# 4. human reviewer merges (until W807 reviewer-bot ships)

CLI: kolm kolmbench validate <jsonl> runs the same validator locally. kolm kolmbench submit <jsonl> --confirm posts to the auth-gated POST /v1/kolmbench/submit endpoint (requires confirm:true as a spend-protection gate).
