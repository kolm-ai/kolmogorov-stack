Migration · 2026

OpenAI is sunsetting fine-tuning. Land here.

OpenAI's self-serve fine-tuning is closed to new users now, and existing users have until January 2027 before the API turns off. kolm is the open-source landing pad. Bring your fine_tuning_data.jsonl, walk out with a portable distilled model that runs on your GPU, your cloud, or your edge. Apache-2.0, signed receipts, no per-token bill.

Shutdown facts

What OpenAI announced.

Three sentences that explain why you are reading this page today.

1

Closed to new fine-tuning users — as of OpenAI's 2026 policy update. New accounts can no longer create fine-tuning jobs against the self-serve API.

2

Existing fine-tuning users have until January 2027 — after that the surface goes dark. Pre-existing custom models on the platform need a new home before the cutoff.

3

kolm is open-source (Apache 2.0), runs on your hardware, and exports portable artifacts: HuggingFace weights, GGUF (Q4_K_M / Q5_K_M / Q8_0), Ollama Modelfile, vLLM, TGI, and TRT-LLM. No vendor swap, no second lock-in.

Migration path

Five steps from OpenAI JSONL to a model you own.

Each step is one CLI call. The whole loop runs end-to-end on a $0.39/hr cloud GPU or a workstation 5090 in under an hour for a 50k-row corpus.

  1. Export your fine-tuning data from OpenAI.

    Download the JSONL you uploaded for any fine-tuning job. Both chat and completion formats are supported. Your evals carry over the same way.

    cp fine_tuning_data.jsonl ./data.jsonl

  2. Import the JSONL into a kolm capture corpus.

    The importer is pure-Node, zero-dep, and skips malformed rows with a per-row error log. You end up with the same training pairs in kolm capture format, ready for any distill spec.

    kolm import --from openai-finetune ./data.jsonl --out ./captures.jsonl

  3. Draft a distill spec — base model plus teachers.

    Pick a base (Qwen 2.5 7B, Llama 3.1 8B, Phi-3.5) and one or more teacher models. Multi-teacher council distillation is one flag — the OpenAI fine-tuning API only ever distilled from a single model.

    kolm distill spec --base qwen2.5-7b --teacher claude-4-7 --teacher gpt-5 --out spec.json

  4. Train the distilled student.

    Reads the spec, streams captures, writes a LoRA adapter (or a full merge) and a signed .kolm artifact. Resumable, deterministic, every step logged to a receipt chain.

    kolm distill train --spec spec.json --captures ./captures.jsonl

  5. Export to GGUF / Ollama / vLLM / TGI and deploy.

    One verb, every common target. The exported weights are yours — commit them, mirror them, run them on a Pi 5 or a Jetson Orin. No callbacks, no per-token meter.

    kolm export --to gguf --quant Q4_K_M ./run_*.kolm && kolm deploy --target ollama
Reproducible

The whole migration in six commands.

Copy this block into a terminal. Replace YOUR_KEY with the API key kolm hands you after kolm signup.

terminal · bash Copy & run 
# 1. Install the kolm CLI (Node 20+ or Python 3.11+).
$ npm i -g github:kolm-ai/kolm

# 2. Authenticate.
$ kolm login --key YOUR_KEY

# 3. Import your existing OpenAI fine-tuning JSONL.
$ kolm import --from openai-finetune ./fine_tuning_data.jsonl --out ./captures.jsonl

# 4. Draft a distill spec (multi-teacher council, your choice of base).
$ kolm distill spec --base qwen2.5-7b --teacher claude-4-7 --teacher gpt-5 --out spec.json

# 5. Train the student. Resumable; emits signed receipts at each step.
$ kolm distill train --spec spec.json --captures ./captures.jsonl

# 6. Export to a portable target and run it.
$ kolm export --to gguf --quant Q4_K_M ./run_latest.kolm
$ kolm run ./run_latest.kolm --prompt "hello, world"
Provable lineage

Every step emits a signed receipt.

No silent step in the loop — you can verify any artifact end-to-end.

Every distill run, every eval pair, and every inference call emits an Ed25519-signed receipt. A receipt commits to the inputs (capture hashes, teacher prompts, base model digest), the operation, and the outputs (model digest, eval scores, output hash). The chain is append-only and verifiable offline.

When you migrate off OpenAI you typically need to prove to your buyer, your security team, or your regulator that the new model behaves equivalently to the fine-tuning baseline. The receipt chain is that proof — not a screenshot, not a spreadsheet.

Full schema and verification recipe at /docs/receipts. The Ed25519 public key is published in the kolm CLI and in CI; verification needs no network.
