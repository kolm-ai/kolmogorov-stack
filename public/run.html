01commands

The run loop.

A short command set. Each maps to shipping code. Each emits a receipt.

$ npm i -g github:kolm-ai/kolm
$ kolm config base https://kolm.ai
$ kolm login

$ kolm inspect support-writer.kolm
 manifest.json 12 KB 1a92c4...aa
 recipes.json 7.2 MB ab02f1...7e
 receipt.json 312 B verified
$ kolm eval support-writer.kolm
$ kolm run support-writer.kolm '{"input":"draft a reply"}'
$ kolm serve --mcp --port 7464

02surface map

One artifact. Many surfaces.

A single signed .kolm file flows from compile to CLI to native runtime: the same bytes everywhere.

CLI · Node

Local inspect, eval, run, and MCP serve. Cross-platform: macOS, Linux, Windows. Covered end-to-end by the test suite in this repo.

Hosted · HTTPS

Compile, registry, verified wrap, receipt verification, readiness, and tenant-scoped auth ship through the Express API at kolm.ai.

Native · mobile

iOS and Android adapters benchmarked on named hardware before each release; latency, memory, battery, and store-policy posture published in the matrix.

03target requirements

What a platform target must prove.

signature

Verify locally.

The adapter must reject manifest, layer, and receipt mismatches before execution.

policy

Declare egress.

The runtime must expose its network behavior and fail closed when the artifact forbids egress.

benchmark

Measure devices.

Each target needs p50/p95 latency, memory, battery, thermal, and app-size data on named hardware.

fallback

Document degradation.

Teams must know exactly what happens when a device cannot run the selected model path.
