kolm.ai for Cursor

One command wires kolm into Cursor as an MCP server. Every compiled .kolm artifact appears as a tool the editor can call. The installer merges into your existing ~/.cursor/mcp.json without overwriting other servers.

Install command

copy · paste · run
node scripts/install-mcp.cjs cursor

Already have kolm installed globally? Run kolm install cursor instead. Pass --dry-run to preview the JSON without touching disk.

target config: ~/.cursor/mcp.json

What this does

The installer reads ~/.cursor/mcp.json (if present), parses the JSON, merges in a single mcpServers.kolm entry that points to kolm serve --mcp, and writes atomically using a tmp-file + rename. Other servers in the same file are preserved byte-for-byte. Re-running is a no-op when the kolm block is already correct.

The exact block injected:

{
 "mcpServers": {
 "kolm": {
 "command": "kolm",
 "args": ["serve", "--mcp"],
 "env": {}
 }
 }
}

If an existing mcpServers.kolm block differs from the canonical one, the installer prints a diff and refuses to overwrite. Pass --force to replace it.

After install

step 1

Reload Cursor MCP

Cmd+Shift+P, then "MCP: Reload". Cursor picks up the new server without a full restart.

step 2

See kolm tools appear

Open the MCP panel. Every .kolm artifact under your kolm.yaml is listed as a callable tool.

step 3

Call from chat

Ask Cursor to run a compiled task. It will route to the local kolm server, sign the call, and write a receipt.

Cross-platform paths

  • macOS & Linux: ~/.cursor/mcp.json
  • Windows: %USERPROFILE%\.cursor\mcp.json

Troubleshooting

If the installer prints "refusing to overwrite", you already have a different mcpServers.kolm entry. Inspect the diff, then either edit by hand or rerun with --force. The installer never crashes on an unknown platform: it prints the JSON block and the best-guess path so you can paste manually.
