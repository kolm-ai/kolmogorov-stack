Two products. One signed artifact.

The Wrapper is the OpenAI-compatible gateway you drop in front of any production app. The Studio is the lab where you distill, quantize, and sign your own specialists. Same proof chain. Use either independently or together.

Product 1 · Wrapper

Route every model behind your app.

Apps, agents, internal tools, and production APIs that already call model providers.

Keep the OpenAI-compatible call shape. Kolm adds routing, budgets, privacy policy, receipts, and capture controls so every successful call can become future training signal — and the bridge into the Studio.

  • OpenAI-compatible routing across provider and local model endpoints
  • PII/PHI controls before data becomes training material
  • Signed receipts and tenant-fenced captures per response
  • Budgets, policy, replay, and cost controls in one operator path

01 · Gateway

One URL for every model API.

Flip one env var. Your app keeps using the OpenAI SDK, Anthropic SDK, LangChain, Vercel AI SDK, whatever. Behind the scenes, kolm routes to the right provider, redacts PII, retries, fails over, and signs every approved response into a tenant-fenced capture log.

  • OpenAI + Anthropic Messages, drop-in compatible
  • vLLM, SGLang, TRT-LLM, TGI, llama.cpp, Fireworks, Together
  • Per-namespace rate limits, retries, failover
  • PII redaction at the edge (text, image, audio)
  • Audit log + signed receipt per response
Gateway docs → All integrations 
# turn it on by changing one env var.
$ export OPENAI_BASE_URL=https://api.kolm.ai/v1
$ export OPENAI_API_KEY=$KOLM_API_KEY

# the OpenAI SDK is unchanged.
>>> from openai import OpenAI
>>> client = OpenAI()
>>> client.chat.completions.create(
...     model="claude-sonnet-4-6",
...     messages=[{"role":"user","content":"..."}]
... )
[ok] routed to anthropic
[ok] response captured: cap_8f3a1...
[ok] receipt signed: rcp_92c4...

02 · Capture

Your private dataset, built automatically.

Every approved gateway response is hashed, timestamped, and stored in a tenant-fenced JSONL store. Filter by namespace, tag, score, model, latency. Export as your own training set. The data never crosses a tenant boundary, ever.

  • One JSONL row per response: input, output, scores, traces
  • Tenant-fenced, defense-in-depth at every read
  • Approve / reject / tag from CLI, TUI, or web
  • Export as .kolm-dataset.jsonl for distill
Capture docs → Open your captures 
# list captures by namespace and score.
$ kolm capture list --namespace support --min-score 0.8

  2,341 approved · 17 tagged "regression"

# inspect one.
$ kolm capture show cap_8f3a1
  input:   "my order #1834 is stuck on..."
  output:  "I see #1834 shipped on..."
  model:   claude-sonnet-4-6
  score:   0.91 (consistency)
  redact:  3 PII spans removed
  receipt: rcp_92c4... [ok] verified

# export as a training set.
$ kolm capture export --namespace support \
    --out support-dataset.jsonl

03 · Distill

Turn frontier calls into a specialist.

Pick a teacher (Claude, GPT-4, DeepSeek-R1) and an open student (Qwen, Llama, Mistral, Phi). Kolm samples the captures, trains, evaluates on a frozen seed-split, and ships a smaller model that does your specific job. Off-policy or on-policy. Preference-tuned or not.

  • Teachers: any model your gateway can route to
  • Students: open models from HF, or your own
  • Frozen eval split: not leaked into training
  • K-score baseline at compile time, drift later
  • Resume / retry / fallback teacher built in
Distill docs → Training presets 
# compile your captures into a specialist.
$ kolm distill \
    --namespace support \
    --teacher claude-sonnet-4-6 \
    --student qwen2.5-7b-instruct \
    --eval-split 0.15

  [ok] seed split: 1,990 train · 351 eval
  [ok] distilling on captures (k=8)
  [ok] step 2000/2000 · loss 0.241
  [ok] K-score @ eval: 0.847
  [ok] bake-off vs teacher: 0.918 Jaccard

  Ready to compile.

04 · Compile

One signed file. Every backend.

A .kolm packs the weights, the spec, the receipts, the K-score, the seed split, and an Ed25519 signature into one sealed file. Quantize to INT4, INT8, or BF16. The same artifact runs on every supported runtime, no rebuild, no per-target export step.

  • INT4 / INT8 / BF16, same file format, different size class
  • NF4 + double quant via bitsandbytes for INT4
  • Ed25519 sidecar binds weights to spec
  • Optional confidential-compute attestation
  • WASM, native C, native Rust SDKs ship targets too
Compile docs → .kolm file spec 
# compile + quantize in one command.
$ kolm compile \
    --from support-2026-05-22 \
    --quantize int4

  [ok] loading qwen2.5-7b-instruct (BF16, 14.2 GB)
  [ok] quantizing NF4 + double quant
  [ok] packing weights + spec + receipts
  [ok] signing Ed25519
  [→] support-2026-05-22.kolm (5.2 GB)

# verify it yourself, locally.
$ kolm verify support-2026-05-22.kolm
  [ok] signature valid
  [ok] spec to weights binding intact
  [ok] receipts: 2,341 captures, all checksum match

05 · Runtime

Cloud, browser, phone, VPC, air-gap.

A .kolm is portable by construction. The runtime verifies the sig, picks the best backend for the device, vLLM on a server, llama.cpp on a laptop, MLX on a Mac, WebGPU in a browser, and emits a signed receipt for every response. Air-gap deploys ship the entire runtime in one binary.

  • vLLM, SGLang, TRT-LLM, TGI for GPU servers
  • llama.cpp, candle, MLX for laptops + Macs
  • WebGPU + WASM for browsers
  • iOS + Android via the mobile SDK
  • One sealed binary for air-gap / classified
Runtime docs → Air-gap deploy 
# pick a runtime, or let kolm pick.
$ kolm run support-2026-05-22.kolm \
    "snail climbs 3ft, slips 2ft. 10ft wall?"

  [ok] verified signature
  [ok] backend: llama.cpp (CPU)
  [ok] loaded INT4 (5.2 GB RAM)
  [→] Day 8.
  (24.5 tok/s · receipt rcp_4d2b...)

# or run it as an OpenAI-compatible server.
$ kolm serve support-2026-05-22.kolm --port 8080
  listening on http://localhost:8080/v1

06 · Registry

Verify any kolm in two seconds.

The registry is the public ledger for the whole stack. Every compiled .kolm gets a receipt; every receipt links back to the captures it learned from, the teacher it learned from, and the spec it was compiled against. Anyone with the receipt ID can verify the response is real.

  • Public queryable: /v1/registry/export
  • Private per-tenant registry on Team + Enterprise
  • Embed a "verify this" badge on your site
  • SLSA-style supply chain for AI artifacts
