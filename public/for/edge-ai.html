The problem

Why frontier models don’t go to the edge.

Disk + RAM

A 70B BF16 model is 140 GB. Your Pi has 8.

Even the smallest hosted-tier behaviour you depend on weighs more than a phone’s entire storage. Quantization is not optional — and ad-hoc int4 conversion silently degrades whatever your evaluator was holding the model to.

Connectivity

Hosted endpoints assume a cell tower.

Robotics, in-flight, factory floor, rural clinic, submarine. The user is not online. The model must complete the task locally and reconcile later. Streaming over LTE is not the answer.

Latency

A 300ms round trip is unusable in a control loop.

Closed-loop robotics need ~10ms decisions. Hosted LLMs don’t exist on that budget. A distilled INT4 specialist on a $99 Jetson returns the first token in 18ms.

Verification

You can’t hash a model that lives behind an API.

Safety review wants a fixed binary they can sign, ship, and verify on first boot. Kolm’s .kolm format is that binary: weights + eval + receipt in one self-describing file.

What kolm changes

From teacher to on-device artifact.

Kolm’s SOTA quantize matrix has shipped INT4 across 0.5B / 3B / 7B / 32B on RTX 5090 via bitsandbytes NF4 + double, verified end-to-end. The compile target picks the right runtime: Core ML for iOS, NNAPI for Android, WebGPU for browser, llama.cpp for Pi.

Step 01

Distill the teacher you already trust

Point kolm at the cloud model you already use. The student size is set by the device class — 32B INT4 for Pi 5, 7B INT4 for laptops, 0.5B INT4 for microcontrollers.

$ kolm distill --teacher gpt-4o \
  --student qwen2.5-3b --quantize int4 \
  --eval evals/edge-cases.jsonl
Step 02

Compile per target runtime

Same .kolm file, different compile target. Kolm emits MLX for Apple silicon, Core ML for iOS, NNAPI for Android, WASM + WebGPU for browser, GGUF for llama.cpp.

$ kolm compile --target coreml ios-build.kolm
$ kolm compile --target webgpu web-build.kolm
$ kolm compile --target gguf pi-build.kolm
Step 03

Verify on first boot

The device-side SDK checks the Ed25519 receipt before loading the weights. Tampered artifacts refuse to run. Update flow ships a new signed file; the runtime atomically swaps it.

$ kolm verify /opt/kolm/specialist.kolm
ok · ed25519 · eval-hash matches · loading...
