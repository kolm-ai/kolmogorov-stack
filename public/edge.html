Target profile catalog

Hardware presets that ship with the compiler.

Eight presets cover ~95% of edge requests. BYO profile for the rest via kolm config set target-profiles.<name> ....

Jetson

Jetson Orin NX / Nano

16 GB shared GPU/CPU. Compiles to GGUF Q4_K_M with vLLM-on-Jetson runtime. Sustains 30 tok/s on a 7B at 15 W power cap.

kolm compile --target-profile jetson-orin-nx
Raspberry Pi

Raspberry Pi 5 (8 GB)

ARM Cortex-A76, no GPU. Compiles to GGUF Q3_K_M + llama.cpp ARM-NEON runtime. Use for 1-3B specialists on intermittent power.

kolm compile --target-profile raspi-5-8gb
Phone

iPhone (A17) / Pixel 8

Compiles to GGUF Q4_K_S + Core ML (iOS) or NNAPI (Android). Ships inside your app. No model auto-update without your release cycle.

kolm compile --target-profile iphone-a17
MacBook

MacBook M-series

Unified memory + Metal. Compiles to MLX or GGUF Q5_K_M. Used as a dev box AND the runtime when the use case fits one machine.

kolm compile --target-profile macbook-m3
Browser

WebAssembly

WASM SIMD runtime. Compiles to a downloadable .wasm bundle < 200 MB. Ships inside a static site, no server.

kolm compile --target-profile browser-wasm
Industrial

Industrial PC / NVIDIA IGX

Fanless, ECC RAM, 24 V power. Compiles to GGUF Q4_K_M with watchdog timer + crash-safe restart hook.

kolm compile --target-profile igx-orin
What a profile actually sets

No magic. Just defaults you would have set anyway.

A profile is a JSON blob shipped with the compiler. Inspect any of them with kolm inspect target-profile <name>.

Quant format
GGUF Q4_K_M / Q5_K_M / Q3_K_M / MLX / Core ML / WASM

Chosen by peak-memory headroom against the model size you supply.

Runtime
llama.cpp / vLLM / MLX / Core ML / WebAssembly

The runtime that ships on or builds for that device class.

KV-cache pattern
Shard 10x or full attention

Edge devices typically use Shard kv-cache to fit a longer context inside small VRAM. See Shard write-up.

Power & thermal hints
nice-level / power cap / batch size

Used by kolm serve to set realistic concurrency for the target.

Verify on real hardware

Profile defaults compile fast. Real device measurements decide.

Add the device with kolm devices add, then run a Pareto sweep before committing the build.

1

Register the device

in: SSH host · out: device id

kolm devices add edge-01 --type ssh --host 10.0.0.7

2

Sweep quants

in: artifact + device · out: Pareto plot

kolm test-quants out.kolm --device edge-01

3

Roll out

in: winning quant · out: deployed

kolm deploy out.kolm --device edge-01 --port 8766
