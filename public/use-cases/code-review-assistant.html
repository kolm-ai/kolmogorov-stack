Internal code review assistant

How does kolm handle internal code review assistant?

kolm captures your review interactions, distills a specialist that learns your conventions, and quantizes it for local serving — so PR review runs on your own hardware at fixed cost with a signed receipt, and your source never has to leave on every call. The result is a model you own and run on your own hardware, trained on your real workload, with a signed build receipt instead of a recurring per-call API bill.

The problem

What makes internal code review assistant expensive today?

A frontier-model code reviewer is costly per pull request and sends proprietary source to an external API. The reviews are generic rather than tuned to your codebase conventions. That cost compounds with volume and ties a sensitive workload to an external API, which is exactly the situation an owned, compiled specialist is designed to replace once the traffic pattern is well understood.

The kolm flow

How does the kolm pipeline solve it?

Point your review tool's LLM at the kolm gateway. Captured (diff, review) pairs feed a distillation run; kolm compiles a quantized specialist you serve in CI or on a workstation. Each stage runs inside your own environment, so capture, distillation, and compilation happen without handing the workload to a third party, and the final artifact is a portable, signed model.

The outcome

What do you end up with?

You get a review assistant you own, tuned to your conventions, served locally at fixed cost, with a signed receipt proving exactly which weights were compiled. Because the artifact is portable and signed, you can move it between a laptop, a VPC, or an air-gapped network, and prove at any point exactly which weights and method produced it.

FAQ

Can it run in CI?
Yes. The compiled artifact serves an OpenAI-compatible endpoint via kolm serve, which a CI job can call locally.
Does my source go to a third party?
With a self-hosted gateway and a compiled local model, inference happens inside your boundary. PII and secrets are redacted on capture.
How big is the model?
You choose the base and quantization. The verified matrix records sizes from 0.44 GB (a 0.5B model INT4) up to 17.9 GB (a 32B model INT4).
