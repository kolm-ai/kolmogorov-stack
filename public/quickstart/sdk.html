kolm.ai / quickstart / sdk

The SDK quickstart for kolm.ai · the AI compiler in your language

Drop-in replacement for the OpenAI SDK in TypeScript and Python. Same call shape, plus capture metadata, compile triggers, and a verify() that replays the receipt chain offline. Change the import, keep your code.

target time: 3 minutes · supported runtimes: node 18+, deno 1.40+, bun 1.0+, python 3.10+ · one-line import swap

Install

$ git clone https://github.com/sneaky-hippo/kolm-stack
$ npm i file:./kolm-stack/sdk/node
local package linked as @kolm/kolm-sdk
$ pip install git+https://github.com/sneaky-hippo/kolm-stack@main#subdirectory=sdk/python
Successfully installed kolm-0.2.0

Registry note: the Node package is not published yet, and the PyPI kolm name is occupied by an unrelated project. These commands install from the Kolm source checkout.

Five-step flow

step 1Swap the importimport Kolm from '@kolm/kolm-sdk'

The SDK ships an OpenAI-compatible client surface so the body of your existing handler does not change. Pass captureNamespace to start collecting prompt+response pairs into your tenant's capture store.

import Kolm from '@kolm/kolm-sdk';
const kolm = new Kolm({ apiKey: process.env.KOLM_KEY, captureNamespace: 'summarizer' });
const res = await kolm.chat.completions.create({
 model: 'qwen-3.5-27b',
 messages: [{ role: 'user', content: 'Summarize this PR' }],
});
// res.kolm.capture_id -> cap_018a3e...
// res.kolm.durable -> true
// res.kolm.namespace -> 'summarizer'
from kolm import Kolm
kolm = Kolm(api_key=os.environ['KOLM_KEY'], capture_namespace='summarizer')
res = kolm.chat.completions.create(
 model='qwen-3.5-27b',
 messages=[{'role': 'user', 'content': 'Summarize this PR'}],
)
# res.kolm.capture_id -> cap_018a3e...
# res.kolm.durable -> True
# res.kolm.namespace -> 'summarizer'
step 2Stream the live capture tailkolm.captures.stream()

Open an SSE stream on your namespace and inspect what is actually landing. Async iteration in TS, generator in Python.

for await (const cap of kolm.captures.stream({ namespace: 'summarizer' })) {
 console.log(cap.id, cap.latency_us, cap.input.slice(0, 60));
}
for cap in kolm.captures.stream(namespace='summarizer'):
 print(cap.id, cap.latency_us, cap.input[:60])
step 3Promote to a compile jobkolm.distill.fromCaptures()

Triggers the same /v1/distill/from-captures route the CLI uses. Returns a job handle you can await.

const job = await kolm.distill.fromCaptures({ namespace: 'summarizer', tier: '3090', recipeClass: 'specialist' });
const artifact = await job.wait();
console.log(artifact.id, artifact.k_score);
job = kolm.distill.from_captures(namespace='summarizer', tier='3090', recipe_class='specialist')
artifact = job.wait()
print(artifact.id, artifact.k_score)
step 4Verify before runkolm.artifacts.verify()

Replays the receipt chain offline against the artifact bytes. Throws on the first broken link. Never run an artifact you cannot verify.

const ok = await kolm.artifacts.verify('./summarizer.kolm');
console.log(ok.signatures_valid, ok.drift_within_tolerance);
ok = kolm.artifacts.verify('./summarizer.kolm')
print(ok.signatures_valid, ok.drift_within_tolerance)
step 5Run offlinekolm.run()

Hand the SDK a verified .kolm and call .run() with no network. Dispatches to the runtime baked into the artifact, JS, WASM, native, GGUF, or ONNX.

const r = await kolm.run('./summarizer.kolm', { prompt: 'Summarize this PR:...' });
console.log(r.output);
r = kolm.run('./summarizer.kolm', prompt='Summarize this PR:...')
print(r.output)
