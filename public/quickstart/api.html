The HTTP API quickstart for kolm.ai · the AI compiler

Five curl commands. Every kolm.ai surface is reachable as REST over Bearer-keyed HTTP. JSON in, JSON out, signed receipts on the way back. If your stack speaks HTTP you already speak kolm.ai.

target time: 2 minutes · prerequisites: a shell with curl · auth: Bearer ks_...

Five-step flow

step 1Provision an anonymous workspacePOST /v1/anon/bootstrap

No signup form. POST returns a Bearer key valid for the anonymous workspace quota. Claim the workspace later with POST /v1/anon/claim.

$ curl -s https://kolm.ai/v1/anon/bootstrap -X POST | tee /tmp/k.json
{
 "anon_token": "kao_8e2a4...c1d7",
 "tenant_id": "anon_yc8e9f2a",
 "kind": "anon",
 "quota": 1000,
 "expires_at": "2026-06-19T00:00:00.000Z"
}
$ export KOLM_KEY=$(jq -r .anon_token /tmp/k.json)
step 2Capture a prompt+response pairPOST /v1/capture/log

Push the observation directly. The response carries x-kolm-capture-durable: true only after the write reached durable storage. If that header is missing or false, the row did not persist and your client must retry, the receipt cannot lie.

$ curl -s https://kolm.ai/v1/capture/log -i \
 -H "authorization: Bearer $KOLM_KEY" -H "content-type: application/json" \
 -d '{"namespace":"summarizer","items":[{"input":"Summarize this PR:...","output":"This PR refactors...","latency_us":8421}],"model":"qwen-3.5-27b"}'
HTTP/2 201
x-kolm-capture-durable: true
x-kolm-namespace: summarizer
x-kolm-capture-id: cap_018a3e...
{ "persisted": 1, "total": 1, "durable": true }
step 3Compile from capturesPOST /v1/distill/from-captures

Promote a namespace once it has crossed the readiness threshold. The job runs async; poll the returned job_id via GET /v1/jobs/:id.

$ curl -s https://kolm.ai/v1/distill/from-captures -X POST \
 -H "authorization: Bearer $KOLM_KEY" -H "content-type: application/json" \
 -d '{"namespace":"summarizer","tier":"3090","recipe_class":"specialist"}'
{ "job_id": "job_w8e2", "status": "running", "watch": "/v1/jobs/job_w8e2" }
step 4List + download the artifactGET /v1/artifacts & GET /v1/artifacts/:id/download

Every completed compile job becomes an artifact under your tenant. Pull the metadata first, then the bytes.

$ curl -s https://kolm.ai/v1/artifacts -H "authorization: Bearer $KOLM_KEY" | jq '.artifacts[0]'
{ "id": "art_summ_v3", "recipe_class": "specialist", "k_score": 0.91, "download_url": "/v1/artifacts/art_summ_v3/download" }
$ curl -sLO https://kolm.ai/v1/artifacts/art_summ_v3/download -H "authorization: Bearer $KOLM_KEY"
step 5Replay against the capturesPOST /v1/replay

Close the loop: replay the same captured prompts against the new artifact, compare per-row k_score + latency delta + cost delta. Same screen, same one click, observe→compile→replay.

$ curl -s https://kolm.ai/v1/replay -X POST \
 -H "authorization: Bearer $KOLM_KEY" -H "content-type: application/json" \
 -d '{"namespace":"summarizer","artifact_id":"art_summ_v3","limit":20}'
{ "rows": 20, "k_mean": 0.91, "latency_delta_us": -184000, "cost_delta_micro_usd": -2400 }

The full list lives in /docs/api. The fifteen most-used endpoints, one row per concern.

MethodPathWhat it does
POST/v1/anon/bootstrapprovision an anonymous workspace + Bearer key
GET/v1/accounttenant identity + plan + quota
POST/v1/capture/logpush a batch of prompt+response observations
GET/v1/capture/streamSSE live tail of captures for a namespace
GET/v1/capture/healthcapture driver liveness + durability flag
POST/v1/distill/from-captureskick off a compile job from a namespace
POST/v1/compilecompile from an explicit recipe spec
GET/v1/jobs/:idpoll job state
GET/v1/artifactslist completed artifacts in your tenant
GET/v1/artifacts/:idartifact metadata + receipt chain summary
GET/v1/artifacts/:id/downloadartifact bytes (zip)
POST/v1/replayreplay captures against an artifact + diff
POST/v1/receipts/verifyoffline-replay the receipt chain
POST/v1/loop/tryanonymous demo of the capture receipt envelope

One-config-change for existing SDKs

Point your existing OpenAI / Anthropic / OpenRouter SDK at kolm.ai (or the local daemon) and every call gets captured, redacted, costed, and event-store­d, no code changes. Drop these two env vars into your shell and your SDK keeps working.

# Hosted (cloud capture, signed receipts):
$ export OPENAI_BASE_URL=https://kolm.ai/v1
$ export ANTHROPIC_BASE_URL=https://kolm.ai

# Local daemon (offline, your own machine):
$ kolm connect start --detach
$ export OPENAI_BASE_URL=http://127.0.0.1:8787/v1
$ export ANTHROPIC_BASE_URL=http://127.0.0.1:8787
$ export OPENROUTER_BASE_URL=http://127.0.0.1:8787/v1

# Now your existing OpenAI / Anthropic / OpenRouter code just works:
$ python -c "from openai import OpenAI; print(OpenAI().chat.completions.create(model='gpt-4o-mini', messages=[{'role':'user','content':'hi'}]).choices[0].message.content)"

Surface covered (server and daemon, same envelope): /v1/chat/completions, /v1/responses, /v1/embeddings, /v1/audio/transcriptions, /v1/audio/speech, /v1/messages (Anthropic), /v1/openrouter/v1/chat/completions (with HTTP-Referer + X-Title), GET /v1/models. Every call emits a canonical event row in your lake.
