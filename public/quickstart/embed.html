kolm.ai / quickstart / embed

Embed quickstart — ship the AI compiler in your product

Ship a signed .kolm artifact inside another product. Browser via SDK.js, Node via the SDK module, edge via Cloudflare Workers / Deno Deploy / Vercel Edge. One artifact, three runtimes, every request a signed receipt the user can verify offline.

target time: 5 minutes · runtimes: browser, node 18+, cloudflare workers, deno deploy, vercel edge · artifact format: .kolm (zip + manifest + receipt chain)

Pick a target

Same .kolm file, three host environments. Each one inherits the same verify-before-run contract.

browser

SDK.js + WebAssembly runtime

The artifact loads as a Blob, runs in a worker, and never hits the network after the first fetch. Good for in-app classifiers, redactors, and on-device summarizers.

node

local @kolm/kolm-sdk + native or GGUF

Server-side handler, agent runtime, batch worker. Picks the heavier native dispatch when the host has GPU access.

edge

local @kolm/kolm-sdk + WASM in workers

Cloudflare Workers, Deno Deploy, Vercel Edge. WASM-only artifacts run cold-start friendly. kolm compile --target=wasm-edge picks a quant that fits the per-worker memory budget.

Five-step flow

step 1Compile for the targetkolm compile --target=...

The target hint biases the compiler towards a runtime that the host can execute. Browser and edge default to WASM; node defaults to native or GGUF if a GPU is present.

$ kolm compile recipe.json --target=wasm-browser --tier=tiny
artifact_summarizer.kolm runtime=wasm size=38MB k_score=0.91
step 2Verify before bundlingkolm verify

Always verify before you bundle. A broken receipt chain is a wire-format bug, not a runtime bug, catching it pre-bundle stops it from ever shipping to a customer.

$ kolm verify artifact_summarizer.kolm
verify OK. 9/9 receipts signed, 0 drift breach, k_score 0.91 within tolerance.
step 3Mount in the browser/sdk.js + new Kolm(...)

Drop the artifact into your public/ dir and reference it from a <script type="module">. The SDK does the receipt-verify in the worker before the first run() call resolves.

<script type="module">
import Kolm from 'https://kolm.ai/sdk.js';
const kolm = await Kolm.load('/artifact_summarizer.kolm');
const out = await kolm.run({ prompt: 'Summarize this PR:...' });
render(out);
</script>
step 4Mount in a Cloudflare Workerlocal SDK package + env.KOLM_ARTIFACT

Push the artifact bytes as a Worker static asset; the SDK reads them off the binding. Verify on the first request and cache the verified handle on the global scope so warm starts skip the work. Until npm publication is verified, install the Node SDK from the repo checkout.

import Kolm from '@kolm/kolm-sdk';
let kolm;
export default {
 async fetch(req, env) {
 if (!kolm) kolm = await Kolm.load(env.KOLM_ARTIFACT, { verify: true });
 const body = await req.json();
 const r = await kolm.run({ prompt: body.prompt });
 return Response.json(r);
 },
};
step 5Re-verify on every releasekolm verify --strict

Wire kolm verify --strict into your CI before the artifact lands on your CDN. Strict mode fails on any drift snapshot outside tolerance, any missing signature, any unsigned recipe step. This is how a customer's audit-replay still works six months from now.

$ kolm verify --strict artifact_summarizer.kolm
verify OK (strict). 9/9 signed, drift within tolerance on all 6 axes, rekor inclusion proof valid.
