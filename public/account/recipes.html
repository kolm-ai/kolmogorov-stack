Recipes.

A recipe is a vetted compile spec you can run against your own namespace. Start from one of the 8 templates below, or run a saved recipe. Each template documents the base model, teacher mix, target format, and a known-good K-Score floor.

Automations

Bind a trigger (schedule on a cron cadence, or an event such as drift crossing its threshold) to a recipe so it re-runs without a manual step. Auto-recompile is opt-in and proposes by default. Backed by /v1/automations.

CLI mirror

Every action on this page has a CLI equivalent. Useful for scripting + CI.

# list available templates kolm recipe templates # start from a template against your namespace kolm recipe start support-bot --namespace mine # list your saved recipes kolm recipe list # inspect one kolm recipe show my-support-bot # run it (compiles using the saved spec) kolm recipe run my-support-bot # label corpus from the recipe kolm recipe label-corpus my-support-bot --since 7d

Recipes are reproducible: the spec, teacher list, target format, and K-Score floor are recorded with the compile so a downstream verifier can re-run an identical build. See docs/cookbook.
