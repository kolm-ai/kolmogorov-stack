Notification channels.

Slack, generic HTTP webhook, and email. Pick which events fire per channel, test the payload before you depend on it, and inspect the last 50 delivery attempts. Same source of truth as kolm notifications in the CLI.

Channels

Leave a URL blank to disable that channel. Slack webhooks must be on hooks.slack.com. HTTP webhooks must be https:// and resolve to a public hostname.

Incoming-webhook URL from your Slack app. Payload uses rich block layout with header, fields, and a dashboard action button.
JSON POST with {event, tenant, payload, ts}. Header x-kolm-event repeats the event name for routing.
Single recipient. Email sends through Resend; if Resend is not configured server-side, the row logs as skipped.

Events

Toggle which events dispatch to the configured channels. All events default on so a freshly-saved channel starts receiving traffic.

Test a payload

Send a synthetic event to all configured channels. Use to verify the Slack block layout, the HTTP body shape, or that the email recipient is correct.

Delivery log

The last 50 attempts across all channels. Status is the HTTP response code from the channel (200 = accepted). Email rows show 200 when Resend accepted the message.
