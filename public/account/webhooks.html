Webhooks.

Fire a message into Slack, a generic HTTP endpoint, or email whenever a chosen event lands — artifact_compiled, kscore_drift, capture_quarantined, and friends. Backed by /v1/notifications/settings.

Channels

Leave any field blank to disable that channel. Slack expects an incoming-webhook URL; HTTP fires a POST with a JSON envelope; email goes through the shared transactional sender.

CLI mirror

Every action on this page has a CLI equivalent. Useful for scripting + CI.

# show current webhook settings kolm webhooks show # set a Slack channel for compile + drift events kolm webhooks set --slack-url "https://hooks.slack.com/services/..." \ --events artifact_compiled,kscore_drift # add a generic HTTP receiver kolm webhooks set --http-url "https://example.com/kolm/webhook" # fire a sample payload to all configured channels kolm webhooks test --event artifact_compiled # tail the last 50 deliveries kolm webhooks log --limit 50

Payloads are signed with the tenant webhook secret (rotate from API keys). See docs/webhooks for the envelope schema + retry behavior.
