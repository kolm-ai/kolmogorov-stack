Audit log. read-only

Every account-level action on this tenant: who did what, from where, when. The log is append-only and immutable; admins cannot edit or delete entries. Use this for incident review, SOC 2 control evidence, and HIPAA accounting-of-disclosures. Retention is 365 days by default (configurable on Enterprise).

Filters

Combine filters to narrow the view. The export button below honors the current filter set. Backed by GET /v1/account/audit-log.

CLI mirror

Pipe-friendly export for SIEM ingestion or local archival.

# last 7 days, CSV
kolm audit --since 7d --format csv > audit.csv

# only key rotations, JSON Lines, last 90 days
kolm audit --since 90d --action key.rotated --format jsonl

# stream live events (SSE)
kolm audit --tail

# verify integrity (each entry signed; this re-checks the hash chain)
kolm audit --verify

What gets logged

  • Auth. login, logout, password reset, 2FA enrolled/disabled/challenged.
  • API keys. created, rotated, revoked; the prefix is recorded, never the secret.
  • Plan & billing. plan.changed, billing.event (invoice issued, payment failed, refund).
  • Team. member.added, member.removed, role.changed.
  • Compliance. baa.signed (with hash of the signed PDF), dpa.signed, byoc.enrolled.
  • Data control. tenant.deleted, dataset.exported, dataset.purged.

Each row is hash-chained to the previous one (Merkle-style). Tampering breaks the chain and surfaces on the next kolm audit --verify. See /security#audit for the chain spec.
