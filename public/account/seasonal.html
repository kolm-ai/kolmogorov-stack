Seasonal.

Seasonal capture tagging + per-season time-series + automatic variant recommendation for a namespace. Distill seasonal variants (winter, summer, black-friday) so the right model serves the right calendar window. Built by src/seasonal-capture.js. See /docs/seasonal for the envelope spec and Frontier AI on your own infrastructure.

N-hemisphere default: seasons follow meteorological convention (Dec-Feb winter, etc.). Events are US-retail anchored (black-friday, cyber-monday, tax-season-us). Per-tenant overrides land in a W748 follow-up — the envelope echoes hemisphere:'north' so callers can detect the bias.

Variant management

Register a seasonal-variant namespace (e.g. support/seasonal-black-friday) so the recommendation engine has a target to point at. Variant must be a known season or event key — free-form labels are rejected so typos cannot silently miss recommendations.
