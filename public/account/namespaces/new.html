Create a new namespace.

A namespace is a tenant-scoped bucket for captures, evals, and compiled artifacts. The optional fingerprint share lets a brand-new namespace in your vertical pick a warm-start checkpoint from a sibling, without your raw text ever leaving the tenant.

Namespace

Fingerprint sharing

Sharing your namespace fingerprint lets future new namespaces in the same vertical pick a warm-start checkpoint that already understands your domain's common patterns. The fingerprint is a one-way hash of bigram counts — no raw capture text leaves your tenant.

Default OFF. We never include raw capture text, prompts, outputs, user IDs, or API keys. The shared payload is: one sha256 token-bag hash, up to 32 sha256 bigram hashes, integer counts, and a vertical guess label.

Every share writes one audit row (op: fingerprint.share) so you can prove later exactly which sibling namespaces saw your fingerprint.

Cancel

CLI mirror

The CLI mirrors this form. The opt-in checkbox maps to the explicit --share-fingerprint flag — it is always OFF by default.

# create a namespace, no sharing (default)
kolm namespace new my-support-bot

# create + share fingerprint after first capture
kolm namespace new my-support-bot --share-fingerprint

# inspect what would be shared (read-only)
kolm namespace fingerprint --namespace my-support-bot --json

# discover warm-start siblings
kolm namespace warm-start-suggest --namespace my-support-bot
