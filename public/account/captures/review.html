How the four axes work.

Each capture is scored against a per-namespace baseline (last 200 promoted captures). An axis is flagged when its value sits more than 3σ from the baseline mean.

AxisWhat it catchesThreshold
output_lengthSuspicious length drift (truncation attack, prompt injection inflation)|value − mean| / σ > 3
vocab_entropyLow-entropy spam payloads ("aaaa..." or repeated tokens)|value − mean| / σ > 3
response_timeLatency anomalies (compromised upstream redirect, slowloris)|value − mean| / σ > 3
token_overlap_to_teacher_typicalUnique payloads that do not cluster with the baseline (potential poisoning)|value − mean| / σ > 3

First eight captures per namespace cannot trigger a flag (cold-start protection). After that, the baseline grows with each promoted row.
