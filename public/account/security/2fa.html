Two-factor authentication.

Add a second factor to every sign-in on kolm.ai. We support TOTP (any RFC 6238 authenticator app: 1Password, Authy, Bitwarden, Google Authenticator) and standard WebAuthn hardware keys (YubiKey, Titan, Touch ID, Windows Hello). No proprietary lock-in – you can take your enrolled keys to any other site using the WebAuthn standard.

Sign in to enroll a second factor.

2FA enrollment requires an authenticated session so we can bind the new factor to your tenant.

Type the code currently shown by your authenticator. Codes roll every 30 seconds; you have one window of grace.

Step 2 · Save your recovery codes

Use these one-time codes if you lose your phone or hardware key. Each code works exactly once. Print them, store them in a password manager, or both. Endpoint: POST /v1/account/2fa/recovery.

Regenerating immediately invalidates the prior code set. Audit entry: 2fa.recovery_regenerated.

Step 3 · Add a hardware key (optional)

A WebAuthn-compatible key (YubiKey, Titan, or any platform authenticator: Touch ID, Windows Hello, Android device) gives you phishing-resistant 2FA. Standard WebAuthn – no proprietary attestation required, no vendor lock-in.

How WebAuthn works

Browser support: Chrome 67+, Firefox 60+, Safari 13+, Edge 18+. On unsupported browsers the button stays disabled and 2FA via TOTP is sufficient.

Disable 2FA

Turn off the second factor for this account. We require your current authenticator code to confirm. Endpoint: DELETE /v1/account/2fa.

A current 6-digit code from your authenticator confirms you still control the second factor.

CLI mirror

Headless flows for CI workers and air-gap setups.

# start enrollment (prints otpauth:// URI and a QR ASCII)
kolm 2fa enroll

# verify the enrollment
kolm 2fa verify 123456

# show recovery codes (only after enrollment, only once)
kolm 2fa recovery

# disable
kolm 2fa disable

Recovery codes are stored as bcrypt hashes on the server. We cannot read them back – if you lose both your authenticator and your codes, email support@kolm.ai for an out-of-band identity check.
