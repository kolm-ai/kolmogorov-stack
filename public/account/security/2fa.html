Account MFA policy.

kolm keeps account recovery and second-factor enforcement out of the public browser API. Hosted accounts use identity-provider MFA and scoped API keys; enterprise tenants can enforce SSO MFA, key rotation, and audit review from their own IdP.

Sign in to enroll a second factor.

2FA enrollment requires an authenticated session so we can bind the new factor to your tenant.

Sign in to enroll   Create account

Current enforcement

MFA is enforced at the identity-provider layer for hosted and enterprise accounts. kolm does not expose browser routes that create TOTP secrets, recovery codes, or hardware-key credentials from a static page. No CLI 2FA verbs are published.

Sign-in MFA: idp enforced
API access: scoped keys
Audit trail: recorded

Solo developer accounts

Use the MFA controls on the identity provider that signs you into kolm. For API access, create narrow keys, rotate them regularly, and review account activity after every device change.

Manage API keys Review audit log

Enterprise accounts

Enterprise tenants enforce second factors through SSO policy. SAML/OIDC groups drive role assignment, IdP MFA gates the session, and kolm records key creation, rotation, revocation, and privileged account actions in the audit log.

SSO deployment guide Team access docs Trust center

Recovery path

Browser pages do not reveal API keys or accept password-reset tokens. Recover account access through your IdP administrator, then rotate API keys from the account console. For hosted solo accounts, support verifies ownership before changing recovery state.

Recovery options Contact support

CLI mirror

The CLI mirrors the account controls that belong in kolm: API-key listing, rotation, revocation, and audit review. MFA enrollment remains an identity-provider action.

# shipped account controls
kolm keys list
kolm keys rotate
kolm audit --format json

For account lockout or MFA policy questions, email support@kolm.ai. This page will switch to live enrollment only after matching account API and CLI routes exist.
