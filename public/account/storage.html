Storage truth.

No surprises. This page tells you exactly where each kind of data lives, what we keep, and what leaves your machine. Three panels: paths, mode, cloud sync.

1. Where your data lives right now

Local-first layout. Every kolm install owns this tree. If you uninstall, deleting the tree wipes everything.

PathWhat is in itSize on disk
~/.kolm/config.jsonAPI key, plan tier, default base model-
~/.kolm/events/events.sqliteValidated event log (metadata + redacted text). Drives every other panel here.-
~/.kolm/events/raw/Raw prompts and responses. Only written if storage mode is raw_local or raw_cloud_sync.-
~/.kolm/redactions/The vault. Original sensitive spans encrypted at rest, addressed by VAR_ token.-
~/.kolm/datasets/Train + holdout splits from your accepted labels.-
~/.kolm/artifacts/Built .kolm artifacts and their receipts.-
~/.kolm/models/Cached base model weights (shared across artifacts).-
Export = signed zip with config + events + datasets + artifacts. Purge = irreversible delete on this device.

2. Storage mode (changes what we keep locally)

Pick one. Applies to every capture going forward. Past events keep whatever mode they were captured under.

3. Cloud sync (changes what leaves your machine)

Pick one. This is the kill switch for uploads. Mode and sync are intentionally separate so you can keep raw locally without ever syncing.

Per-class override

Even with raw_enabled, you can carve out classes that must never leave the device. 17 privacy classes total.
