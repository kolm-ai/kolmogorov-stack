Continuous monitoring — SOC 2 Type II signals.

Fifteen Trust Services Criteria controls bound to live operational signals. Each row tracks one signal against a numeric threshold and emits a green / yellow / red / unknown pill. The values below are the static dashboard template stamped w767-v1; the authenticated GET /v1/security/continuous-monitoring/snapshot route returns the live snapshot for the calling tenant.

Snapshot totals

The four-column rollup shipped by every snapshot envelope (summary.green_count, summary.yellow_count, summary.red_count, summary.unknown_count). On a fresh tenant with no signal probes attached, every control begins as unknown — never green.

Green0

Red0
Unknown15

Unknown is never green. When the underlying signal source is offline, the snapshot returns status: 'unknown' with current_value: null for that control. A dashboard that fabricates a green pill to hide a probe outage is worse than no dashboard. The invariant is enforced inside src/continuous-monitoring.js _grade() — null in, unknown out, always.

Threshold grading rule

The dashboard grades each control with a deterministic rule (_grade() in src/continuous-monitoring.js):

  • threshold_direction: max — lower is better. value ≤ 60% of threshold → Green; value ≤ threshold → Yellow; value > threshold → Red.
  • threshold_direction: min — higher is better. value ≥ threshold → Green; value ≥ 95% of threshold → Yellow; otherwise Red.
  • value is null or non-finite — Unknown. Always.
