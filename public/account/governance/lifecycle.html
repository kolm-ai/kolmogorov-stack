Artifact lifecycle.

State browser across the full lifecycle: created → signed → deployed → monitored → superseded → revoked → archived. Each transition lands a row with actor, reason, optional evidence id, and optional successor pointer.

States

created signed deployed monitored superseded revoked archived

Pulls are blocked once an artifact transitions to revoked. superseded redirects to the successor id but does not block pulls of the older artifact (operators may pin to historical versions for reproducibility).
