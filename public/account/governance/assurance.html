Assurance case.

Generate a signed assurance envelope per artifact or per workspace. Claims and framework controls each carry evidence pointers and an explicit status taxonomy so that nothing reads as more proven than it is.

Status taxonomy

Every claim and control declares a status; the renderer never inflates an unmet control into a met one.

StatusWhat it means
implementedBuilt, signed, and covered by evidence in this artifact.
package-gatedBuilt but waiting on a downstream package release for end-to-end coverage.
certification-gatedBuilt and operationally enforced but waiting on a third-party certification body.
external-proof-neededBuilt but the proof requires evidence from outside this stack.

Frameworks covered

Control mappings ship for the frameworks below. Per-control implementation status is computed from the artifact + workspace evidence, never asserted by hand.

  • SOC 2 (CC1 - CC9)
  • HIPAA Security Rule (administrative / physical / technical)
  • GDPR (Art. 5, 25, 30, 32, 35)
  • NIST AI RMF (Govern / Map / Measure / Manage)
  • EU AI Act (high-risk system technical documentation)
  • ISO/IEC 42001 (AI management system)
