Frontier API keys.

Bring your own keys for the teacher council. kolm calls Claude, GPT, Gemini, DeepSeek, Mistral, and Cohere through your keys; the cost lands on your provider invoice. kolm never sees the raw model output beyond what it needs to produce labels.

Why bring-your-own-key

Frontier models charge per token. If kolm proxied them through one shared key, you would pay a markup and we would absorb your usage spikes. With BYOK: you keep your provider rate limits, your provider commit credits, and your provider's audit trail. kolm only ever sees the labels needed to train the student.

Add only the providers your selected recipe needs. The Trinity recipe uses Anthropic + OpenAI + DeepSeek + Google (judge).

Add a key via CLI

kolm provider add anthropic --key sk-ant-...
kolm provider add openai --key sk-proj-...
kolm provider list
kolm provider test anthropic   # one-token roundtrip, no spend on real labels

Keys are stored encrypted at rest (libsodium secretbox + per-tenant KEK). They are never logged, never echoed to the TUI, and never included in receipts.

Cost forecast

Before a distill run starts, kolm computes the forecast against current provider list prices and shows it in the wizard. If the forecast exceeds your spend ceiling (set in billing), the run is paused for confirmation.

Air-gapped deployments

If you cannot call public APIs, set the council to local-only teachers (a heavyweight base model running on your own GPU). The wizard will warn that label diversity drops; verify the resulting student against the holdout set before promoting.
