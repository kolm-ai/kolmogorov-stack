Team. Team tier

Shared workspace for Persona C teams: invite members, assign roles, manage shared artifacts, and let the K-Score gate fail CI on regressions. Roles are owner / admin / member / viewer; viewer is default. Each artifact respects the tenant-fenced event store, so a member can never see another team's data.

K-Score CI gate

Block PRs that regress quality. Configure the threshold + baseline once; every PR that touches a tracked artifact gets a status check.

# .github/workflows/kolm-gate.yml name: kolm K-Score gate on: [pull_request] jobs: gate: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: kolm-ai/kolm-gate-k-score@v1 with: artifact: artifacts/support-bot.kolm threshold: '0.85' baseline: artifacts/baseline/support-bot.kolm max-regression: '0.01' comment-on-pr: 'true' github-token: ${{ secrets.GITHUB_TOKEN }}

The action lives at /docs/github-actions. It calls kolm verify on the candidate artifact, compares against the baseline, and posts a PR comment with the diff when it fails.

CLI mirror

Every action on this page has a CLI equivalent. Useful for scripting onboarding, off-boarding, and role audits.

# list members kolm team members # list pending invites kolm team invites # invite a teammate kolm team invite alice@example.com --role member # change a role kolm team set-role alice@example.com admin # remove a member kolm team remove alice@example.com # export the member roster (for SOC 2 control evidence) kolm team members --format csv > members.csv

Need stronger controls (SSO, SCIM, custom roles, BAA, data residency)? Upgrade to Enterprise — or jump straight to the enterprise admin surface.
