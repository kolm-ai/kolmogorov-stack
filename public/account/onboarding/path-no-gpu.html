Path B · No GPU on this box

Route traffic now, compile in the cloud when ready.

Same install + capture flow as Path A, with a four-step finish that wires up RunPod or Modal for compile jobs. Free-tier Colab is linked below as a cost-free fallback for ≤7B models at int4.

Step 1 of 4

Install the CLI.

Same install everywhere. kolm doctor reports your environment as CPU-only and surfaces the right cloud path.

npm i -g github:kolm-ai/kolmogorov-stack
kolm login --key YOUR_KS_KEY
kolm doctor
Step 2 of 4

Route through the gateway.

Capture every call locally before any compile decision. Your SDK doesn't need to know.

kolm gateway start
export OPENAI_BASE_URL="http://127.0.0.1:8787/v1"
Step 3 of 4

Pick a cloud provider for compile.

One env var lets kolm compile --cloud route to a GPU you don't own. RunPod is pay-as-you-go; Modal is serverless. The free Colab notebook is enough for the first specialist.

export RUNPOD_API_KEY="rp_..."
kolm test cloud --provider runpod
kolm compile spec.toml --cloud runpod --gpu A100
export MODAL_TOKEN_ID="ak-..."
export MODAL_TOKEN_SECRET="as-..."
kolm test cloud --provider modal
kolm compile spec.toml --cloud modal

No card, no setup. Open the notebook, paste your API key, run all cells. The compiled .kolm downloads to your machine when done.

# Open in browser
https://kolm.ai/colab
Step 4 of 4

Send your first call.

Any OpenAI SDK works. Cloud compile fires later, from the dashboard, the moment you have enough captures.

curl http://127.0.0.1:8787/v1/chat/completions \
  -H "content-type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hello kolm"}]}'
