Path C · Route LLM traffic

API key, one SDK call, verify the receipt.

No install. The gateway lives at https://kolm.ai/v1. Point any OpenAI/Anthropic SDK at it with your ks_ key and every call comes back with a signed receipt. The verifier is the same on the CLI and on the website.

Step 1 of 3

Grab your API key.

The key was generated when you signed up. Keep it secret — treat it like a password. Rotate any time from /account.

If you don't see your key, open /account and click Show.

Step 2 of 3

Pick your SDK and fire a call.

Both tabs show a one-shot call against the hosted gateway. The response includes id, choices, and an X-Kolm-Receipt header you can hand to the verifier.

import OpenAI from "openai";

const openai = new OpenAI({
  baseURL: "https://kolm.ai/v1",
  apiKey:  process.env.KOLM_API_KEY,
});

const r = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "hello kolm" }],
});
console.log(r.choices[0].message.content);
from openai import OpenAI
import os

client = OpenAI(
    base_url="https://kolm.ai/v1",
    api_key=os.environ["KOLM_API_KEY"],
)

r = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "hello kolm"}],
)
print(r.choices[0].message.content)
curl https://kolm.ai/v1/chat/completions \
  -H "authorization: Bearer $KOLM_API_KEY" \
  -H "content-type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hello kolm"}]}'
Step 3 of 3

Verify the receipt.

Every response includes an X-Kolm-Receipt header. Hand it to kolm verify (CLI) or paste it into the verifier; both walk the same signature + hash chain.

# From the response header:
# X-Kolm-Receipt: rcpt_01...

kolm verify rcpt_01... --json

Or paste the receipt ID into the in-browser verifier at /verify.
