Drift detection — your traffic vs your training distribution.

Compare live embeddings against the training distribution via KL divergence; pair with fallback-rate lift; act when either crosses threshold. Auto-remediate is opt-in.

W813 honesty contract: drift_detected is never true without crossing a numeric threshold. Auto-remediate never fires unless auto_remediate_drift is explicitly enabled per namespace AND dry_run:false is sent on the trigger call. Stamped w813-v1.

Per-namespace configuration

Override the W813-2 defaults per namespace via POST /v1/drift/configure (requires confirm:true).

settingdefaultactivebounds
kl_threshold 0.10 0.10 (0, 10]
fallback_rate_lift 0.20 0.20 (0, 1]
auto_remediate_drift false false boolean (fail-safe off)

Auto-remediate triggers W720 self-improvement only when the namespace flag is true AND the operator sends dry_run:false. Either alone returns would_trigger:false or triggered:false. This is a P0 trust invariant.

CLI quickstart

# scan (requires --confirm because compute cost is non-trivial)
kolm drift scan --namespace prod --live live.json --training training.json --confirm

# read current status
kolm drift status --namespace prod

# tweak thresholds per-namespace
kolm drift configure --namespace prod --kl-threshold 0.08 --fallback-rate-lift 0.15 --confirm

# list recent drift alerts
kolm drift alerts --namespace prod --limit 10

# explicit live trigger (dry-run by default)
kolm drift auto-remediate --namespace prod --live
