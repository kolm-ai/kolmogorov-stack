kolm.ai / integrations / docker-compose

The kolm daemon, in docker-compose.

One docker-compose.yaml. The daemon comes up on localhost:7403 with an OpenAI-compatible endpoint, SQLite-backed captures, and signed receipts. Drop it into your existing stack – the rest of your services talk to kolm at http://kolm:7403/v1.

How it works in one paragraph

The kolm image bundles the Node 22 runtime, the CLI, and the embedded SQLite WAL driver. Boot it with KOLM_API_KEY + KOLM_BASE_URL pointing at your hosted kolm (or omit both for fully-local mode). The container exposes port 7403 for the OpenAI-compatible endpoint and writes captures + receipts under /data, which you mount as a volume so they survive container restarts. Health probes hit GET /health on the same port.

docker-compose.yaml

Copy this file. Run docker compose up -d. The daemon is reachable from peer services on the kolm network as http://kolm:7403/v1.

docker-compose.yaml
version: "3.9"

services:
 kolm:
 image: ghcr.io/sneaky-hippo/kolm:latest
 container_name: kolm
 restart: unless-stopped
 ports:
 - "127.0.0.1:7403:7403"
 environment:
 KOLM_API_KEY: "${KOLM_API_KEY}"
 KOLM_BASE_URL: "https://kolm.ai"
 KOLM_STORE_DRIVER: "sqlite"
 KOLM_DATA_DIR: "/data"
 KOLM_REDACTION_POLICY: "redact"
 volumes:
 - kolm-data:/data
 healthcheck:
 test: ["CMD", "wget", "-qO-", "http://127.0.0.1:7403/health"]
 interval: 30s
 timeout: 5s
 retries: 3

 app:
 image: your-app:latest
 depends_on:
 kolm:
 condition: service_healthy
 environment:
 OPENAI_BASE_URL: "http://kolm:7403/v1"
 OPENAI_API_KEY: "${KOLM_API_KEY}"

volumes:
 kolm-data: {}
