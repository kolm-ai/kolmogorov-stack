kolm + n8n

How do kolm and n8n work together?

Point any n8n OpenAI or HTTP node at the kolm gateway and your automation's LLM steps are captured, redacted, and signed automatically — so a no-code workflow quietly builds the dataset for an owned model. In short, n8n keeps doing what it does best while kolm turns the resulting traffic into an owned, signed model you run yourself with no per-token bill.

What this integration does

What does wiring n8n to kolm change?

n8n drives the workflow; kolm is the model endpoint behind your AI nodes. No custom code: change the credential base URL and capture begins. The integration is configuration-only: you change a base URL, keep your existing code, and the gateway handles capture, PII redaction, and the signed receipt transparently underneath your normal calls.

Environment

What configuration does n8n need?

The integration is driven entirely by two environment values: the kolm gateway base URL and your kolm API key. You set them once on the n8n side and every LLM call routes through the gateway, where capture and redaction happen before anything is stored in your tenant lake.

Setup steps

How do I set up the n8n integration?

Setup is five short steps: get a kolm key, point n8n at the gateway base URL, run your normal workload so pairs are captured and redacted, then distill and compile a specialist, and finally verify the signed build. No model code changes are required beyond the base URL swap.

  1. Create a kolm key. Sign up at /signup and copy your API key.
  2. Add an OpenAI credential in n8n. Create an OpenAI credential whose base URL points at the kolm gateway /v1 endpoint, using your kolm key.
  3. Use it in your nodes. Reference that credential from your OpenAI / AI Agent nodes. Each call is captured into your lake with PII redaction.
  4. Build a specialist. Run kolm distill then kolm compile on the captured pairs to produce a signed, quantized model.
  5. Verify the receipt. Confirm the build at /verify-prod or with kolm verify.

FAQ

Do I need to write code?
No. You only change the base URL on an n8n OpenAI credential. The capture, redaction, and signing happen in the gateway.
Can I route to multiple providers?
Yes. The gateway speaks OpenAI, Anthropic, Gemini, vLLM, SGLang, TGI, and TRT-LLM protocols and can route between them.
Where do captures live?
In your tenant lake, which is self-hostable. PII is redacted before storage.
