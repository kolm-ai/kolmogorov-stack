The Vercel AI SDK, with kolm as the provider.

The ai + @ai-sdk/openai combo already accepts a custom baseURL. Point it at kolm.ai and every generateText, streamText, and server action in your Next.js app keeps working. Captures, redaction, and signed receipts come along.

How it works in one paragraph

The Vercel AI SDK exposes createOpenAI({ baseURL, apiKey }) from @ai-sdk/openai. kolm.ai speaks the OpenAI Chat Completions wire format, so the provider works as a drop-in. Streaming responses (streamText), tool calling, structured outputs (generateObject), and the Next.js edge runtime all behave the same way. Every call lands in your captures inbox and is tagged with the namespace you pass through headers.

Next.js server action – one line

Create a single provider instance, pass it to generateText or streamText. Works in app/ route handlers, server actions, and edge runtime alike.

next.js – ai >= 4 · @ai-sdk/openai
// app/api/chat/route.ts
import { createOpenAI } from "@ai-sdk/openai";
import { streamText } from "ai";

const kolm = createOpenAI({
 baseURL: "https://kolm.ai/v1",
 apiKey: process.env.KOLM_API_KEY,
});

export async function POST(req: Request) {
 const { messages } = await req.json();
 const result = await streamText({
 model: kolm("qwen-3.5-27b"),
 messages,
 });
 return result.toDataStreamResponse();
}

Server-side – generateObject with kolm

Structured outputs work the same way. Pass a Zod schema, get a typed object back, see the call in your captures inbox.

typescript – ai/generateObject
import { createOpenAI } from "@ai-sdk/openai";
import { generateObject } from "ai";
import { z } from "zod";

const kolm = createOpenAI({
 baseURL: "https://kolm.ai/v1",
 apiKey: process.env.KOLM_API_KEY,
});

const { object } = await generateObject({
 model: kolm("qwen-3.5-27b"),
 schema: z.object({ severity: z.enum(["low","med","high"]), reason: z.string() }),
 prompt: "Classify this support ticket: ...",
});
