kolm + DSPy

How do kolm and DSPy work together?

Configure DSPy's LM with the kolm gateway base URL and every optimized program call is captured and signed — so your compiled DSPy prompts feed straight into a distilled, owned specialist model. In short, DSPy keeps doing what it does best while kolm turns the resulting traffic into an owned, signed model you run yourself with no per-token bill.

What this integration does

What does wiring DSPy to kolm change?

DSPy compiles prompt programs; kolm compiles the model. Wire DSPy's LM to the kolm gateway and the traces DSPy already produces become the training pairs for an owned model. The integration is configuration-only: you change a base URL, keep your existing code, and the gateway handles capture, PII redaction, and the signed receipt transparently underneath your normal calls.

Environment

What configuration does DSPy need?

The integration is driven entirely by two environment values: the kolm gateway base URL and your kolm API key. You set them once on the DSPy side and every LLM call routes through the gateway, where capture and redaction happen before anything is stored in your tenant lake.

Setup steps

How do I set up the DSPy integration?

Setup is five short steps: get a kolm key, point DSPy at the gateway base URL, run your normal workload so pairs are captured and redacted, then distill and compile a specialist, and finally verify the signed build. No model code changes are required beyond the base URL swap.

  1. Sign up for kolm. Create an account at /signup and copy your API key.
  2. Configure dspy.LM. Set dspy.LM to an OpenAI-compatible model with api_base pointed at the kolm gateway /v1 endpoint.
  3. Run your DSPy program. Compile and run your DSPy module. Each LM call is captured with inline PII redaction.
  4. Distill to a specialist. Run kolm distill and kolm compile to turn the captured pairs into a signed, quantized model.
  5. Verify. Replay the receipt at /verify-prod.

FAQ

Does kolm replace DSPy's optimizer?
No. DSPy optimizes the prompt program; kolm compiles the underlying model. They compose: DSPy traces become kolm training pairs.
Which DSPy LM backends work?
Any OpenAI-compatible backend, since the kolm gateway speaks the OpenAI protocol.
Is the build verifiable?
Yes. Every compile emits an Ed25519-signed receipt replayable at /verify-prod.
