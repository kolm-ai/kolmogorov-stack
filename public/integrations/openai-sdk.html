kolm.ai / integrations / openai sdk

The OpenAI SDK, pointed at kolm.

Change one line. Your existing OpenAI Python or JavaScript code keeps working – the calls now flow through kolm.ai, every prompt is captured, the redaction membrane runs in-line, and the response carries a signed receipt header. Same SDK. Same surface. New compiler underneath.

How it works in one paragraph

kolm.ai speaks OpenAI Chat Completions on the wire. The SDK sees a normal HTTPS endpoint at https://kolm.ai/v1 with Bearer auth. Set base_url (Python) or baseURL (JS), use your ks_... key as api_key, and every call is forwarded to the model you selected. The capture log writes durably before the response returns, the receipt header shows up as x-kolm-capture-durable: true, and the artifact ID gets emitted when a compile job promotes the namespace.

Python – one line

Pip-install the official OpenAI SDK. Pass base_url and api_key at client construction. Everything else – streaming, function calling, vision, structured outputs – behaves exactly the way the OpenAI SDK behaves.

python – openai >= 1.0
from openai import OpenAI

client = OpenAI(
 base_url="https://kolm.ai/v1",
 api_key="ks_live_...", # your kolm API key
)

resp = client.chat.completions.create(
 model="qwen-3.5-27b",
 messages=[{"role": "user", "content": "Summarize this PR: ..."}],
)

print(resp.choices[0].message.content)
print(resp.response.headers.get("x-kolm-capture-durable")) # "true"

JavaScript / TypeScript – one line

Same idea on Node. Same idea in the browser. The baseURL field is all you need to switch.

node – openai >= 4
import OpenAI from "openai";

const client = new OpenAI({
 baseURL: "https://kolm.ai/v1",
 apiKey: process.env.KOLM_API_KEY,
});

const resp = await client.chat.completions.create({
 model: "qwen-3.5-27b",
 messages: [{ role: "user", content: "Summarize this PR: ..." }],
});

console.log(resp.choices[0].message.content);
