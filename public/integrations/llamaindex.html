kolm + LlamaIndex

How do kolm and LlamaIndex work together?

Point LlamaIndex's LLM at the kolm gateway base URL and every retrieval-augmented call is captured as an (input,output) pair, PII-redacted on the way in, and signed on the way out — so your RAG traffic becomes the training set for an owned specialist. In short, LlamaIndex keeps doing what it does best while kolm turns the resulting traffic into an owned, signed model you run yourself with no per-token bill.

What this integration does

What does wiring LlamaIndex to kolm change?

LlamaIndex orchestrates retrieval and prompting; kolm sits underneath as the OpenAI-compatible gateway. Swap the base URL, keep your indexes and query engine, and start capturing the pairs that compile into a model you own. The integration is configuration-only: you change a base URL, keep your existing code, and the gateway handles capture, PII redaction, and the signed receipt transparently underneath your normal calls.

Environment

What configuration does LlamaIndex need?

The integration is driven entirely by two environment values: the kolm gateway base URL and your kolm API key. You set them once on the LlamaIndex side and every LLM call routes through the gateway, where capture and redaction happen before anything is stored in your tenant lake.

Setup steps

How do I set up the LlamaIndex integration?

Setup is five short steps: get a kolm key, point LlamaIndex at the gateway base URL, run your normal workload so pairs are captured and redacted, then distill and compile a specialist, and finally verify the signed build. No model code changes are required beyond the base URL swap.

  1. Install kolm and sign up. Sign up at /signup for an API key. Install the kolm gateway with npm i -g github:kolm-ai/kolm.
  2. Point LlamaIndex at the gateway. Set the OpenAI base URL in your LlamaIndex Settings.llm to the kolm gateway /v1 endpoint and use your kolm API key.
  3. Run your queries. Run your normal LlamaIndex query engine. Each LLM call is captured into your tenant lake with PII redacted on the capture path.
  4. Distill and compile. Once you have enough pairs, run kolm distill then kolm compile to produce a signed specialist model quantized to your hardware.
  5. Verify the build. Run kolm verify on the artifact or replay the receipt at /verify-prod to confirm the signature chain and K-score.

FAQ

Does kolm change my LlamaIndex indexes?
No. kolm only intercepts the LLM completion calls via the gateway base URL. Your vector store, retrievers, and query engine are unchanged.
Is my retrieved context redacted?
PII is redacted on the capture path before anything is stored in your lake. The redaction runs inline in the gateway.
Can I keep using OpenAI underneath?
Yes. The gateway can route to OpenAI, Anthropic, or a local model while it captures pairs for distillation.
