kolm.ai / integrations / cursor + claude code

Run kolm as the local proxy for Cursor and Claude Code.

This is the dogfood pattern: kolm boots on 127.0.0.1:7403, Cursor and Claude Code read their base URL from a project-local config, and every request lands in ~/.kolm/captures/ with PHI redaction applied. No app-side changes – the editor never knows.

How it works in one paragraph

kolm ships a local-proxy mode that listens on 127.0.0.1:7403 and forwards to the real upstream (api.anthropic.com, api.openai.com, your hosted kolm endpoint, etc). Claude Code reads ANTHROPIC_BASE_URL from .claude/settings.local.json, Cursor reads its proxy from ~/.cursor/proxy. Point both at 127.0.0.1:7403 and every request is mirrored to durable storage as it streams through. PHI redaction (emails, SSNs, MRNs, 21 PHI classes) runs in-line. This is the same pattern that powers the project's own dogfood – we shipped it because we use it.

Claude Code – project-local settings

Drop this file into your repo. Claude Code reads it at startup. The proxy is project-local so different repos can mirror to different namespaces.

.claude/settings.local.json
{
 "env": {
 "ANTHROPIC_BASE_URL": "http://127.0.0.1:7403"
 }
}

Start the proxy

Run the helper script. It pins the upstream, listens on the loopback, and tails durable captures to ~/.kolm/captures/claude-code.jsonl.

powershell · macos · linux
# start the proxy in the background
node scripts/dogfood-proxy.ps1 start

# check status
node scripts/dogfood-proxy.ps1 status

# tail captures as they land
node scripts/dogfood-proxy.ps1 tail

# stop when done
node scripts/dogfood-proxy.ps1 stop

Cursor – environment variable

Cursor reads OPENAI_BASE_URL + ANTHROPIC_BASE_URL from the user environment. Export both before launching Cursor.

shell · one-liner
export OPENAI_BASE_URL=http://127.0.0.1:7403
export ANTHROPIC_BASE_URL=http://127.0.0.1:7403
open -a Cursor # macOS
# or: cursor # Linux / Windows from a shell with the env exported
