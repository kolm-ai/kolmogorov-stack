kolm.ai / integrations / env vars

Environment variables, in full.

Every KOLM_* variable, what it does, what the default is, and whether it's read by the CLI, the daemon, or both. Set what you need. Skip what you don't. The defaults are conservative on purpose.

How it works in one paragraph

kolm reads its config from the process environment first, then from ~/.kolm/config.json, then from on-disk defaults. The CLI and daemon share the same variable names so a single export covers both. Variables with a "sensitive" mark must come from a secrets manager in production – never check them into git.

Reference table

VariableDefaultWhat it does
KOLM_BASE_URLhttps://kolm.aiHosted endpoint. Set to http://127.0.0.1:7403 for fully-local mode.
KOLM_API_KEY(none) · sensitiveBearer key. Format ks_live_* or ks_anon_*. Required for hosted; optional in local mode.
KOLM_STORE_DRIVERsqliteCapture store backend. One of sqlite, postgres, memory.
KOLM_DATA_DIR~/.kolmWhere captures, artifacts, receipts, and config live on disk.
KOLM_ALLOW_RAWfalseIf true, raw un-redacted text is allowed in the capture log. Default refuses non-redacted writes.
KOLM_REDACTION_POLICYredactHow the membrane handles PII / PHI. One of redact, hash, allow. Production should be redact.
KOLM_REDACTION_CLASSES(all 21)Comma-separated list of PHI classes to redact. Empty means all.
KOLM_NAMESPACEdefaultDefault namespace tag on capture writes. Override per-call with the namespace body field.
KOLM_TENANT_ID(from API key)Override the tenant binding the API key resolves to. Useful in multi-tenant tests.
KOLM_PORT7403Daemon listen port for the local OpenAI-compatible endpoint.
KOLM_HOST127.0.0.1Daemon listen address. Set to 0.0.0.0 only if you intend external access.
KOLM_LOG_LEVELinfoOne of debug, info, warn, error.
KOLM_LOG_FORMATtextOne of text or json. JSON for shipping to a log aggregator.
KOLM_DISABLE_TELEMETRYfalseIf true, the CLI never emits anonymous usage pings.
KOLM_RECEIPTS_KEY(generated) · sensitiveEd25519 signing key for receipts. Auto-generated on first use under $KOLM_DATA_DIR/keys/.
KOLM_UPSTREAM(none)When kolm runs as a local proxy, the real upstream to forward to. Defaults vary per provider.
KOLM_TIMEOUT_MS30000Default request timeout for upstream calls. CLI verbs accept per-call overrides.
KOLM_RETRY_MAX3Upstream retry budget. Set to 0 to disable.
KOLM_INSECURE_TLSfalseSkip TLS verification on upstream calls. Use only against self-signed dev endpoints.

Quick start – the four you actually need

In practice you only export four variables. Everything else has a sensible default.

shell · .envrc · docker-compose env
export KOLM_BASE_URL=https://kolm.ai
export KOLM_API_KEY=ks_live_...
export KOLM_REDACTION_POLICY=redact
export KOLM_NAMESPACE=my-app
