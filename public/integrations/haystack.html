kolm + Haystack

How do kolm and Haystack work together?

Configure Haystack's OpenAIGenerator with the kolm gateway base URL and your pipeline runs unchanged while kolm captures, redacts, and signs every generation — turning production RAG traffic into a compilable training set. In short, Haystack keeps doing what it does best while kolm turns the resulting traffic into an owned, signed model you run yourself with no per-token bill.

What this integration does

What does wiring Haystack to kolm change?

Haystack pipelines stay exactly as they are; kolm is the OpenAI-compatible endpoint behind the generator node. Capture flows automatically into your lake and becomes the basis for an owned specialist. The integration is configuration-only: you change a base URL, keep your existing code, and the gateway handles capture, PII redaction, and the signed receipt transparently underneath your normal calls.

Environment

What configuration does Haystack need?

The integration is driven entirely by two environment values: the kolm gateway base URL and your kolm API key. You set them once on the Haystack side and every LLM call routes through the gateway, where capture and redaction happen before anything is stored in your tenant lake.

Setup steps

How do I set up the Haystack integration?

Setup is five short steps: get a kolm key, point Haystack at the gateway base URL, run your normal workload so pairs are captured and redacted, then distill and compile a specialist, and finally verify the signed build. No model code changes are required beyond the base URL swap.

  1. Get a kolm key. Sign up at /signup and copy your API key.
  2. Set the generator endpoint. In your Haystack OpenAIGenerator, set api_base_url to the kolm gateway /v1 endpoint and api_key to your kolm key.
  3. Run the pipeline. Run your Haystack pipeline normally. Completions are captured into your tenant lake with inline PII redaction.
  4. Compile a specialist. Run kolm distill and kolm compile to build a quantized model trained on the captured pairs.
  5. Verify. Replay the signed receipt at /verify-prod or run kolm verify locally.

FAQ

Which Haystack version is supported?
Any version whose OpenAIGenerator accepts a custom api_base_url works, since kolm speaks the OpenAI wire protocol.
Does this add latency?
The gateway adds a thin capture + redaction + receipt step. The published wrapper benchmark records the end-to-end overhead; the receipt is signed asynchronously.
Can I self-host the gateway?
Yes, the gateway is Apache-2.0 and self-hostable, so traffic and captures stay in your environment.
