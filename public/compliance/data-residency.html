Data residency — EU stays EU.

Tag every capture with a region. Compile artifacts that name their region. Distill only from the region you target. Untagged data fails closed, not silent-included.

w769-v1

Region taxonomy

Nine canonical regions. Each carries a regulatory anchor and the ISO 3166-1 alpha-2 country codes used by inferRegionFromTenant. The taxonomy is frozen at the module boundary — downstream callers cannot mutate it. New regions land via a wave; we never split or rename an existing region in place because that would silently re-classify historical receipts.

Per-capture tagging

Every capture can carry an explicit region tag. The tag lives in the event store under provider=kolm_data_residency, model=capture-tag, so the same lake readers that back /account/billing and /account/bakeoff can attribute by region without learning a second storage schema.

Caveats floor. An unknown region returns {ok:false, error:'unknown_region'}. We never silently coerce to GLOBAL — silent coercion would let a caller think they had tagged EU when in fact the row was never persisted with the claimed region.

Confirm gate. tagCapture requires confirm:true. A residency tag is an externally-visible compliance claim and we refuse to write one without an acknowledged side-effect.

CLI

$ kolm residency tag --capture-id evt_8f2c... --region EU_WEST --confirm
$ kolm residency get --capture-id evt_8f2c...
$ kolm residency regions
$ kolm residency enforce --capture-id evt_8f2c... --target US_EAST

API

Per-namespace default region

Tagging every capture by hand is expensive. configureNamespaceRegion pins a default for a namespace so a connector that writes into support.eu can have every row come in already-tagged. The per-capture call still wins when it disagrees — explicit user action overrides namespace defaults.

$ kolm residency configure-namespace --namespace support.eu --region EU_WEST --confirm

Namespace defaults persist as model=namespace-default-region rows, separately keyed from per-capture tags so a tenant inspecting their audit trail can tell which decisions were operator-chosen and which were namespace-inherited.

Region-specific .kolm artifacts

Compile pinned a region into manifest.region. The field is conditional — absent / null / empty string all collapse to a byte-identical artifact_hash under byte-stability law, so introducing the field never breaks pre-existing receipts. Only a non-empty value mutates the hash via region_hash = sha256(region).

This shape mirrors the sparsity_profile_hash / kv_profile_hash / parent_cid conditional-slot pattern. Operators can re-compile a baseline artifact with a region tag at any time and the unregioned artifact remains valid — the receipt difference is fully attributable to the explicit region addition.

$ kolm compile spec.yaml --out support.kolm                    # region: absent -- baseline hash
$ kolm compile spec.yaml --out support-eu.kolm --region EU_WEST  # region: EU_WEST -- new hash, traceable

Region-aware distillation sampler

sampleForDistillation walks the tenant's captures, joins them against the residency tag index, and returns only the captures whose tag matches the target region. The sampler is the gate that keeps a US distillation run from accidentally inheriting EU-tagged training data — the most common GDPR violation we have seen in the wild.

Fail-closed. An untagged capture is excluded from any non-GLOBAL target. The only way to include untagged captures is to set target_region=GLOBAL, which is the explicit opt-out. A naive “assume GLOBAL when missing” rule would let an untagged EU capture flow into a US-only run.

Tenant fence. Every read passes tenant_id to listEvents AND re-filters per-row. A foreign-tenant tag can never surface.

POST /v1/distill/sample        (entry point — calls region-aware-sampler.js)

  "target_region": "EU_WEST",

Geo-fence cross-reference (-5)

covers where data lives. The complementary control — who can sign up to operate the workbench — is-5's EXPORT_CONTROL_DENYLIST in src/auth.js. The two are orthogonal threat models but both are required for a credible regulated-industry posture.

controlscopemodule
-5 geo-fencePerimeter: deny sign-up from a sanctioned country.src/auth.js EXPORT_CONTROL_DENYLIST · ISO codes CU IR KP SY RU BY (OFAC primary-sanction set).
residencyLocality: tag every capture, pin every namespace, gate every distillation by region.src/data-residency.js + src/region-aware-sampler.js. Persists into the event store; surfaced via /v1/residency/*.

If your compliance reviewer asks “what stops a Cuban operator with a US capture?” —-5 stops the sign-up. If they ask “what stops a US tenant from training on EU customer-support transcripts?” — fails the sampler closed.
