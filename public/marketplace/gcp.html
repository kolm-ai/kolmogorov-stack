Frontier AI on your own infrastructure. The GCP equivalent of the AWS stack: a Compute Engine instance running the kolm gateway, a Cloud Storage bucket for signed .kolm artifacts, Secret Manager for API keys, and Cloud Logging for audit plus operator telemetry. What you get (planned)

  • VPC-isolated GCE instance. A dedicated VPC and subnet in your chosen region with a firewall rule limiting SSH to your operator CIDR and 443 to your client CIDR. The kolm gateway runs as a systemd unit on a single n2-standard-8 instance by default.
  • Cloud Storage for artifacts. A private bucket with Uniform Bucket-Level Access enabled, customer-managed encryption keys optional via Cloud KMS, object versioning on, and a 90-day noncurrent expiration lifecycle rule.
  • Secret Manager for credentials. Your kolm API key and optional teacher key live as Secret Manager secrets. The instance authenticates via its attached service account; no secrets ever touch instance metadata.
  • Cloud Logging integration. Audit and operator log streams forward to Cloud Logging so existing log sinks (BigQuery, Pub/Sub, Cloud Storage) and alerting policies pick them up natively.
  • IAM service account scoping. The instance runs under a service account with only roles/storage.objectAdmin on the artifact bucket, roles/secretmanager.secretAccessor on the two named secrets, and roles/logging.logWriter. No project-wide grants.

Pricing on GCP (estimated)

Bring your own kolm Pro license. GCP bills you only for the GCE, Cloud Storage, Secret Manager, and Cloud Logging resources this template creates. Estimated cost for a typical workload sits between $130 and $300 per month on-demand:

  • n2-standard-8 in us-central1: ~$230/month
  • 100 GB pd-balanced root disk: ~$10/month
  • Cloud Storage standard (50 GB typical): ~$1/month
  • Cloud Logging ingest + retention: ~$5-15/month depending on traffic
  • Secret Manager: pennies/month

Committed Use Discounts cut the GCE line by 25-55% on a 1-year or 3-year commit. A2/G2 GPU instances land here for distill workloads at higher cost.

Deployment (planned)

Once the Deployment Manager template (or Terraform module) lands, deployment will look like this:

$ gcloud deployment-manager deployments create kolm-prod \
    --config infra/gcp-marketplace/kolm.yaml \
    --properties \
      kolmApiKey:ks_xxxxx,namespace:prod,region:us-central1

# Grab the gateway URL
$ gcloud deployment-manager deployments describe kolm-prod \
    --format='value(outputs[0].finalValue)'

The template will output gatewayUrl, artifactBucket, serviceAccountEmail, and logSinkName so downstream IaC can consume them.

For reference shape, see infra/aws-marketplace/cloudformation.yaml in the repository.
