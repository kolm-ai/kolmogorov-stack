kolm on AWS MarketplaceListing prep

Frontier AI on your own infrastructure. One CloudFormation stack brings up a VPC-isolated kolm gateway, an S3 bucket for signed .kolm artifacts, an IAM role scoped to that bucket, and CloudWatch log groups for audit plus operator telemetry. Bring your own kolm Pro license; AWS bills only for the compute and storage you actually use.

What you get

  • VPC-isolated kolm gateway. A dedicated VPC (10.42.0.0/16), one public subnet, an internet gateway, and a security group that lets only your operator CIDR reach port 22 and your client CIDR reach port 443. No shared tenancy.
  • S3 artifact storage. A private bucket with public-access fully blocked, AES-256 server-side encryption, versioning on, and a 90-day noncurrent-version expiration. Every .kolm the gateway publishes lands here with its ed25519 receipt chain intact.
  • IAM-scoped credentials. The gateway runs under an instance role whose only S3 permissions are Get/Put/Delete/List against your artifact bucket. CloudWatch and SSM Parameter Store access are likewise narrow.
  • CloudWatch-native audit. Two log groups (/kolm/<stack>/audit, retention 365 days; /kolm/<stack>/operator, retention 30 days) capture the same audit envelope the CLI emits locally.
  • Secrets in SSM. Your kolm API key and optional teacher key are stored as SSM parameters under /kolm/<stack>/. The instance re-fetches them on boot so rotation is a single AWS CLI call.

Pricing on AWS

Bring your own kolm Pro license. AWS bills you only for the EC2, S3, and CloudWatch resources this stack creates. Estimated cost for a typical workload sits between $120 and $300 per month on-demand:

  • t3.2xlarge on-demand in us-east-1: ~$240/month
  • 100 GB gp3 root volume: ~$8/month
  • S3 standard storage (50 GB typical): ~$1/month
  • CloudWatch Logs ingest + retention: ~$5-15/month depending on traffic

Reserved Instances cut the EC2 line by roughly 40%. Heavy distill workloads land on g5 instances and scale beyond this band; cost is linear in instance type and audit-log volume.

Deployment

One-click via CloudFormation. The template lives at infra/aws-marketplace/cloudformation.yaml in the repository and is reviewable in source before you ever launch it.

$ aws cloudformation create-stack \
    --stack-name kolm-prod \
    --template-body file://cloudformation.yaml \
    --capabilities CAPABILITY_NAMED_IAM \
    --parameters \
      ParameterKey=KolmAPIKey,ParameterValue=ks_xxxxx \
      ParameterKey=KeyName,ParameterValue=my-keypair \
      ParameterKey=Namespace,ParameterValue=prod

# Wait for CREATE_COMPLETE
$ aws cloudformation wait stack-create-complete --stack-name kolm-prod

# Grab the gateway URL
$ aws cloudformation describe-stacks --stack-name kolm-prod \
    --query 'Stacks[0].Outputs[?OutputKey==`GatewayURL`].OutputValue' \
    --output text
https://ec2-XX-XX-XX-XX.compute-1.amazonaws.com

The stack outputs GatewayURL, ArtifactBucket, InstanceRoleArn, AuditLogGroup, and OperatorLogGroup so your downstream IaC can consume them as !ImportValue exports.

See infra/aws-marketplace/cloudformation.yaml and infra/aws-marketplace/listing-spec.json in the repository for the full template and listing spec.

Adapted Terms of Service

The kolm Pro Terms of Service apply unchanged to the kolm software running on the EC2 instance this stack creates. Two AWS-specific clauses layer on top:

  • Data residency. All captured prompts, completions, signed artifacts, and audit logs remain inside the AWS region you select at stack creation. kolm never copies them out of region. Cross-region replication is opt-in and disabled by default.
  • AWS account isolation. The S3 bucket, IAM role, instance, and SSM parameters all live entirely inside your AWS account. kolm.ai has no cross-account access. If we ever need to assist with an incident, you grant temporary read-only access via your own STS AssumeRole policy and revoke at will.
