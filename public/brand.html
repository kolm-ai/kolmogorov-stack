Brand

Four names. One stack.

Kolmogorov is the company. kolm is the binary. .kolm is the file format. RS-1 is the open spec. We separate them on purpose: the same way git is the binary of Git/SCM, or gcc is the binary of GCC.

Kolmogorov
The company. Always capitalized. Named for Andrey Kolmogorov, who asked in 1965: what is the smallest program that produces a given output? That question is the AI compiler problem.
kolm
The CLI binary. Always lowercase. Short enough to type, long enough to be unambiguous. Same role as git or gcc.
.kolm
The file format. Always monospace, always lowercase, always with the leading dot. A deterministic zip with a fixed schema: model.gguf, lora.bin, recipes.json, index.sqlite-vec, verifier.json, manifest.json, signature.sig.
RS-1
The open spec. MIT-licensed. Defines the .kolm manifest, the receipt chain, and the K-score formula. Anyone can implement a kolm-compatible compiler or runtime against it.

Style rules

  • "Kolmogorov" with a capital K when referring to the company. Never "kolmogorov", never "KOLMOGOROV".
  • "kolm" always lowercase. Not "Kolm", not "KOLM". Even at the start of a sentence — same rule as git or npm.
  • ".kolm" always monospace, always with the leading dot. Same rule as .git or .tar.gz.
  • "RS-1" with the hyphen. The spec. Use the exact string when citing.

The prog-rock band

We know about Kolm, the Norwegian/Swedish prog-rock band with about five thousand monthly Spotify listeners. We don't share a category, but we share a name. They were there first by a few decades. We're using "Kolmogorov" as the company name and "kolm" as the binary partly because it sidesteps the collision and partly because the academic root is the part of this story that actually matters.

The KAN paper

The 2024 Kolmogorov-Arnold Networks paper is unrelated work. We're not building KANs. The shared root is just Andrey Kolmogorov, who has more than one beautiful idea named after him.
