Capture-loop provenance

Provenance is the chain of custody from a captured prompt to a signed artifact. Every stage of the kolm loop emits a content-addressed record, each record references the hash of its inputs, and the final receipt names every record in the chain so an auditor can re-walk it without trusting the runtime.

What it is

Provenance for an ML artifact is the sequence of operations and data that produced it, recorded in a way a third party can verify. The strongest formulation is the W3C PROV data model (Moreau et al., 2013): a directed acyclic graph of entities (datasets, models, configs), activities (training runs, evaluations), and agents (the human or machine that ran the activity), where every node is uniquely identified and every edge is signed. For ML supply chains the same idea appears as SLSA (Supply-chain Levels for Software Artifacts) and as recent academic frameworks like MLflow's tracking schema and the model cards proposal (Mitchell et al., 2019). Without provenance a buyer can verify what the artifact does today but not where its training data came from, whether the gate ran on the same split that was promised, or whether the published K-score was measured on the same holdout the receipt names.

How kolm uses it

Every captured prompt-response pair is content-addressed by SHA-256 and written to the capture log. The distillation stage references each pair by hash; the resulting corpus has a hash that is itself referenced by the trainer config. The trainer emits checkpoints that hash the corpus, the base model, the optimizer state, and the seed. The K-score gate references the checkpoint hash and the holdout hash. The final Ed25519 receipt-chain (/research/receipt-chain) seals all four hashes into a single signed root that the runtime serves at /v1/receipts/<id> and the CLI verifies offline. Stages that are not content-addressed in v0.2.6 (the seed-mining oracle, the per-tenant cloud sync metadata) ship content-addressed in v0.3; today they record an opaque source identifier and a timestamp.
