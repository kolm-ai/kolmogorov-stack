Teacher trace

A teacher trace is the full record of one teacher-model call: the rendered prompt, the response, any tool calls, sampling parameters, latency, cost, and the model identifier with version. It is the unit of provenance kolm records on capture and replays on demand. The receipt chain hashes the trace; the SBOM enumerates the teachers that produced them.

What it is

The distillation literature mostly treats a training example as a (prompt, response) tuple, which is the minimum required to compute cross-entropy. In production this is rarely enough. A trace records every input the teacher saw and every output it produced: the system prompt as rendered (including any few-shot block the framework injected), the user message with substituted variables, the assistant response, tool-call entries in order, sampling parameters (temperature, top-p, max-tokens, presence/frequency penalty, stop sequences), the model identifier including the exact version pin, the timestamp, the latency in microseconds, the input and output token counts, the cost in micro-dollars, and any provider-side metadata (cache-hit flag, safety-filter result, finish reason). This is the OpenTelemetry GenAI semantic-convention's gen_ai span schema, ratified in 2024 and adopted by LangSmith, Phoenix, Helicone, Langfuse, and OpenLLMetry. Recording at this level lets a buyer answer questions a (prompt, response) tuple cannot: which teacher version produced which row, what the cost was, whether a tool was called, whether the response is reproducible at temperature zero.

How kolm uses it

The kolm capture proxy records a full trace for every teacher call routed through it. The trace schema is OpenTelemetry GenAI v1 with kolm-specific extensions for receipt-chain id and corpus row content-address. Traces stream to durable storage (/research/capture-loop-provenance) and are listed in the inbox at /captures. The distillation pipeline derives training rows from traces, not from raw prompt-response pairs, so the corpus is reconstructible: given a trace id and the corpus recipe, the trainer regenerates the exact training row byte-for-byte. The replay verb (kolm replay <ns> <id>) re-runs each trace through the trained student and computes a per-row diff (k-score, latency, cost) against the original teacher response; this is how the buyer confirms the artifact actually replicates teacher behaviour on the captured distribution. The receipt's pre-image ring (ring 1; see /research/receipt-chain) is computed over the trace hash so any post-hoc edit invalidates the chain.
