Odds Ratio Preference Optimization (ORPO)

ORPO combines supervised fine-tuning and preference learning into a single loss without a reference model. The standard NLL loss on chosen responses is augmented with an odds-ratio penalty that suppresses the rejected response. No reference forward pass, no KL regularization, lower VRAM than DPO at the same model size.

What it is

Hong, Lee, and Thorne (2024) note that DPO and similar methods still require an SFT-warmed reference policy to define the KL anchor, and the reference forward pass doubles VRAM during preference training. ORPO replaces the reference with a direct odds-ratio penalty on the rejected response: L = L_SFT(yw) + lambda * L_OR(yw, yl), where L_OR = -log sigmoid(log(odds(yw|x) / odds(yl|x))). Because the loss is monolithic, ORPO can be run directly on the base model with no SFT warmup, which is the regime where it shows the strongest improvements in the paper. Hyperparameters reduce to a single lambda (typical: 0.1). The trade-off: ORPO does not converge to the KL-constrained optimum that DPO targets, so the resulting policies can drift further from the base in undesirable ways if lambda is set too high.

How kolm uses it

ORPO is the memory-efficient research-track preference optimizer for large buyer-supplied students where a reference forward pass is too expensive. It sits behind KTO in the preference stack because kolm's compile loop usually starts from a distilled checkpoint with known reference behavior, not from a from-scratch alignment run. The best-fit use case is a 14B student on a single A100 with no headroom for a separate reference model. The ORPO receipt records lambda and the per-row odds-ratio margin distribution on the holdout.
