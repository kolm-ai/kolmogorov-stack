Tool traces

A tool trace records the full sequence of tool calls a teacher made while answering a prompt: function names, arguments, results, and the ordering between them. Distilling a tool-using agent into a smaller student requires the trace, not just the final response: the student needs to learn which tool to call and in which order.

What it is

Modern teacher models do not always answer in a single forward pass. They call search, calculator, retrieval, code-execution, and domain-specific functions, then condition their next step on the results. The trace of these calls is the agent's working memory. ReAct (Yao et al. 2023) formalized the interleaving of thought, action, and observation; Toolformer (Schick et al. 2023) showed that a model can be fine-tuned to insert tool calls in-line; the OpenAI function-calling API and the Anthropic tool-use API ratified the structured-tool-call schema in 2024. For distillation, a response-only training row is insufficient when the teacher used tools: the student sees the answer but not the lookup chain that produced it, and at inference time will hallucinate the lookups. The standard fix is to record the full tool trace and train the student on a flattened representation: (prompt -> tool_call_1 -> tool_result_1 -> tool_call_2 -> ... -> final_response). Tool-trace distillation produces students that are weaker per-token (more conditional generation steps) but stronger end-to-end on agentic tasks. The Gorilla line (Patil et al. 2023) and the ToolLLM corpus (Qin et al. 2023) demonstrated the gain on API-calling benchmarks.

How kolm uses it

Tool-trace capture is wired into the kolm capture proxy when the teacher uses the OpenAI tools or Anthropic tool_use API. Each tool call and tool result becomes a span in the trace (/research/teacher-trace) with name, arguments, result, and ordering preserved. Tool-trace distillation as a training mode is partially shipped in v0.2.6: the trainer can target a flattened tool-call sequence when the recipe sets distill.with_tools: true and the student tokenizer covers the tool-call special tokens. The K-score gate scores tool-call exact-match (function name + canonicalized arguments) at intermediate steps and end-to-end answer correctness at the leaf; both must pass the recipe threshold to sign. Replay re-executes the tool calls (when the tools are deterministic and online) and diffs the student's call sequence against the teacher's. Streaming tool calls, parallel tool calls, and MCP tool transport are on the v0.3 roadmap.
