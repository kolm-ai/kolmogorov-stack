Cross-lingual distillation.

Most frontier teachers are English-heavy. Distilling on captures that mix English with long-tail languages produces students whose K-Score collapses on the under-represented languages. kolm's cross-lingual stack detects the imbalance, offers opt-in synthetic translation (always stamped synthetic_translation:true), trains on a multilingual mixture, and publishes per-language K-Score in the artifact manifest.

32 languages first-class Synthetic translation always stamped Non-listed languages fall back to English + truthfulness pill
Why

Teachers are English-heavy. Students should not be.

If 92 percent of your teacher traces are English and 8 percent split across 15 other languages, the resulting student will silently regress on the long tail. The K-Score on the holdout will look fine -- because the holdout inherits the same English skew. Per-language K-Score makes the regression visible.

Language distribution detector

Read the corpus before you train.

The detector tags each capture with a language code, then publishes the distribution and flags any language with below-threshold representation. The threshold is configurable per namespace.

{ "total_captures": 14200, "language_dist": { "en": 0.78, "es": 0.09, "fr": 0.04, "de": 0.03, "hi": 0.02, "ja": 0.02, "other": 0.02 }, "below_threshold": ["hi", "ja", "other"], "threshold": 0.05, "recommendation": "synthetic_translation_for_long_tail" }
Synthetic translation

Always stamped. Never silent.

If you opt in, the teacher generates translations for under-represented languages so the student sees enough samples to learn from. Every synthetic row is stamped synthetic_translation:true in the training-row manifest. We never silently insert translated rows.

{ "row_id": "row_4f8e2a", "source_language": "en", "target_language": "hi", "synthetic_translation": true, "translation_teacher_id": "anthropic-claude-sonnet-4-7", "translation_quality_score": 0.94, "original_capture_id": "cap_9b3c1f" }

Per-tenant opt-in setting cross_lingual.synthetic_translation defaults to false. Even when on, the per-language toggle lets you accept synthetic for some target languages and refuse for others.

Multilingual mixture training

Sample to a target language distribution.

The distill loop samples the training set against a target language distribution (uniform across listed languages by default; tunable per namespace). This rebalances English-heavy raw corpora without throwing away the dominant-language signal.

  • Default target

    uniform across covered languages

    The default target distribution is uniform across listed first-class languages, capped at the number of available rows per language so we never oversample beyond what the source corpus supports.

  • Custom targets

    per-namespace JSON config

    Tenants can override the target with a per-namespace JSON config, e.g. weight English heavily for an English-customer-service workload but still cover Spanish at a documented minimum.

  • Per-language eval split

    held-out, separate per language

    Eval splits are held out per language, never globally pooled. Per-language K-Score in the artifact manifest comes from these per-language splits, not from a globally averaged number.

Per-language K-Score in manifest

Long-tail regressions become visible.

A multilingual artifact ships per-language K-Score in the manifest. Viewers can render a heatmap. Tenants can refuse to deploy if any first-class language falls below their per-namespace minimum.

{ "artifact_id": "art_cs_multilingual_v3", "k_score": 0.89, "k_score_by_lang": { "en": 0.91, "es": 0.87, "fr": 0.86, "de": 0.84, "hi": 0.78, "ja": 0.82, "zh": 0.85, "pt": 0.86 }, "k_score_min_lang": ["hi", 0.78], "k_score_max_lang": ["en", 0.91] }
Languages currently covered

32 first-class. The rest fall back to English with a clear pill.

First-class languages have validated detection, supported synthetic-translation pipelines, and per-language eval splits. Languages outside this list fall back to English-only and the manifest carries an truthfulness pill so the operator never deploys blind.

enesfrde itptnlsv nodafipl csrohutr ruukarhe fahibnta uridvith jakozhsw
Outside this list: English-only fallback with manifest.cross_lingual.coverage = "english_only_fallback"
Plain contract. Synthetic translation rows are always stamped synthetic_translation:true. Per-language K-Score is computed from per-language held-out eval splits, never from a global average. Languages outside the listed 32 fall back to English-only with a manifest truthfulness flag and a user-visible pill -- the operator never deploys blind to a language we do not yet cover well.
