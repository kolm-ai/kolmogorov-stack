Trust surface

What kolm signs by default, where the receipts live, and the exact field that gates external attestation. No marketing claims that we cannot verify on disk right now.

Default receipt content

Every .kolm artifact ships an ed25519 receipt chain that covers the manifest, the recipe source, the seed corpus split, and the K-score eval block. kolm verify <artifact> walks the chain offline against the artifact bytes.

FieldStatusWhere
artifact_hashrequiredmanifest, computed sha256 of zipped payload
ed25519_signaturerequiredreceipt chain, key on GET /v1/keys/public
k_scorerequiredmanifest, source = real holdout split
seed_provenancerequiredmanifest, with train/holdout counts
signature_sigstoreoptional / dry-run bundlemanifest, populated when kolm sigstore-attest is wired

The signature_sigstore gate

kolm sigstore-attest attaches a sigstore bundle to the artifact and writes a signature_sigstore block into the manifest. In the current release this is a dry-run bundle — the signature flow runs end-to-end and the bundle shape is real, but the Rekor inclusion proof is not yet a hard production gate.

External callers that must refuse non-attested artifacts can set KOLM_REQUIRE_REKOR=1. With that flag, kolm verify escalates a missing or unverified signature_sigstore.rekor_log_entry to a non-zero exit code instead of a warning. Without the flag, the field stays informational so the rest of the receipt chain still verifies offline.

$ KOLM_REQUIRE_REKOR=1 kolm verify ./examples/claims-redactor/claims-redactor.kolm
artifact_hash ok
ed25519 ok
receipt chain ok
signature_sigstore: dry-run bundle (no Rekor proof) — FAIL (KOLM_REQUIRE_REKOR=1)
exit 2

The honest read: today signature_sigstore is best-effort and is not part of the default verification gate. Setting KOLM_REQUIRE_REKOR=1 is the explicit opt-in for callers who want to refuse anything that does not have a verifiable Rekor entry. When that becomes the default, this page will say so.

What we do not claim

  • No SLSA Level 3 attestation. The build provenance status lives at /slsa; the short version is "Level 3 is a target, not yet shipped."
  • No multi-anchor timestamping. Earlier copy mentioned external timestamp anchors; that surface was removed because it was not actually wired. The only chain we publish is the ed25519 receipt chain plus the optional sigstore bundle described above.
  • No registry-published cosign signatures. The artifact is signed locally before upload; kolm sigstore-attest is the only path to an external transparency log entry.
