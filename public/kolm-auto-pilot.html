kolm.ai › auto pilot

Open-source AI workbench

Frontier AI on your own infrastructure.

Auto pilot is the continuous-background-distill daemon. Opt in once; it polls W815 coverage gaps, gates on W813 drift, and silently re-distills when local quality can replace teacher calls. Every decision lands as a durable ledger row — the daemon never silently retrains on bad data.

W775 no-silent-pass contract: action:'redistilled' is never emitted unless W815 returns ok AND the W813 drift gate is green AND the top gap_score crosses the REDISTILL_THRESHOLD (0.25). When any gate trips, the daemon writes a holding_pattern row with the reason so the operator can see WHY the autopilot held. Stamped w775-v1.

Opt in to auto pilot

Opt-in is explicit and per-namespace. Toggle below to enable the daemon for the namespace you select. The toggle writes a durable autopilot:enabled row and returns a stable autopilot_id.

Requires a logged-in API key. The opt-in call is auth-gated — sign in or run kolm autopilot start --namespace default from the CLI.

How the tick decides


One tick = one pass through the gate sequence. The daemon writes EXACTLY one ledger row per tick (or zero if disabled).




1.
Disabled gate
If the latest disabled-row newer than the latest enabled-row, returns action:'disabled'. No event written.


2.
W815 readiness check
Calls getCoverageGapsForNamespace(namespace, {tenant_id}). If the helper returns error:'insufficient_captures_for_coverage', the daemon writes autopilot:holding with reason insufficient_captures.


3.
W813 drift gate
Reads listRecentAlerts({tenant_id, namespace, limit:1}). If the latest alert is drift_detected:true with severity in {moderate, severe}, the daemon holds with reason:'drift_red'.


4.
Gain threshold
If the top gap_score < REDISTILL_THRESHOLD (0.25), the daemon emits action:'no_op'. The cost + risk of swapping artifacts doesn't clear the bar.


5.
Orchestrate
Calls orchestrateImprovement({tenant_id, namespace, candidates, opts:{skip_spawn:true}}), writes autopilot:redistilled with the orchestrate envelope. Worker spawn is left to the W720 daemon — the tick never blocks on training.









Ledger query


Every tick decision is a durable row in the event-store. Query for one namespace via:


kolm autopilot status --namespace default
kolm autopilot status --namespace default --json | jq '.holding_pattern_reason'


HTTP equivalent (auth-required):


curl -H "Authorization: Bearer $KOLM_API_KEY" \
     "$KOLM_BASE_URL/v1/autopilot/status?namespace=default"







CLI


The CLI dispatcher is cmdW775Autopilot. Subverbs:




verbactionnotes



kolm autopilot start --namespace NSPOST /v1/autopilot/enablewrites opt-in row, returns autopilot_id


kolm autopilot stop --namespace NSPOST /v1/autopilot/disablewrites opt-out row; subsequent ticks no-op


kolm autopilot status --namespace NSGET /v1/autopilot/statusshows current state, last tick action, holding reason


kolm autopilot tick --namespace NSGET /v1/autopilot/tickforce one tick now (cron normally drives this)


kolm autopilot savings --namespace NSGET /v1/autopilot/savingscost saved + baseline, last 30 days


kolm autopilot disablealias for stopfull kill switch





Plain disclosure: system-tray integration (taskbar widget, OS-level start-on-boot) is on the roadmap but not yet shipped. The current daemon is cron-driven via /v1/autopilot/tick; the CLI tick verb is the manual-trigger equivalent.
