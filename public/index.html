How it works

Capture. Distill. Quantize. Run.

Use them as a pipeline to turn production traffic into a private model. Or use any one alone.

  [ app ] --> [ gateway ] --> [ teacher API ]           live traffic, today
              |
              +--> capture lake            signed, hash-chained rows
                       |
                       +--> distill          smaller open student
                                |
                                +--> quantize --> .kolm
                                              |
                                              +--> runtime  vLLM / llama.cpp / WebGPU
One unbroken pipeline. Each stage produces a signed artifact you can audit.

  1. Capture. One env var; your app keeps working.

    Point your OpenAI client at api.kolm.ai/v1. Every approved response is hashed, signed, and logged inside your tenant. Nothing leaves the gateway without a receipt.

    $ export OPENAI_BASE_URL=https://api.kolm.ai/v1

  2. Distill. Teach a smaller open model from your captures.

    Any OpenAI-compatible endpoint is a valid teacher. The compiler bake-off-scores the student against the teacher on your held-out rows before it ships.

    $ kolm distill --namespace=support --teacher=claude-sonnet-4-6 --student=qwen2.5-7b

  3. Quantize. Shrink 4x at INT4 with an accuracy gate.

    The compile step refuses to ship if K‑Score drops past your threshold. One default method covers 95% of teams; nine more are one flag away.

    $ kolm quantize ./adapter --out adapter-int4

  4. Run. One file, every backend, on your hardware.

    Runtime picks the backend per device — vLLM, llama.cpp, MLX, WebGPU. Every response carries a verifiable Ed25519 receipt that links back to a registry row.

    $ kolm run support.kolm '{"text":"..."}'

The artifact format is the contract.

A .kolm is one signed file: weights, the spec it was compiled from, the seed split, the frozen eval, every receipt that touched it, and an Ed25519 sidecar. The hash of the artifact lives in a public, hash-chained registry that anyone can mirror inside their VPC.

Three consequences. Verify any kolm in two seconds on any machine. Move the artifact between cloud and air-gap without renegotiating trust. And when a regulator asks how the model was built, the answer is one file long.

What's in the file

Look inside a .kolm.

One file. No registry calls, no proprietary loader. unzip it on any machine and the structure is self-describing.

  support.kolm  // 5.2 GB, one file, signed
  +-- spec.toml             // teacher, student, seed, quant method
  +-- weights/              // quantized tensors, runtime-neutral
  +-- eval.frozen.jsonl     // the eval rows that gated this build
  +-- bakeoff/              // K-Score vs the teacher, per row
  +-- receipts/             // every capture that fed the distill
  +-- sidecar.ed25519       // 64-byte signature over the SHA-256

Principles

What this compiler will not do.

Things we explicitly refuse to ship, so you can tell us apart from the next fine-tuning API. The rail scrolls; drag or arrow-key to see all eight.

No. 01
No fake K-Scores

If we cannot verify the eval set was untouched at training time, we report the result as unverified, not as a number.

No. 02
No silent fallbacks

When a redactor is missing, the route returns 503 with an install hint. Never a quiet passthrough that drops the guarantee.

No. 03
No lock-in artifact format

A .kolm is a zip with a sidecar. Open it with anything. Re-run it on any backend. Mirror the registry inside your VPC.

No. 04
No shipping past the gate

If the bake-off says the student dropped more than your threshold, compile refuses. Override is a flag you have to type.

No. 05
No tenant cross-talk

Every capture, every receipt, every artifact is fenced by tenant. The integration tests run the same path against a fresh tenant.

No. 06
No private endpoints

Every advertised route on this page is documented, public, and returns a real result or a 501 with the env vars to wire it up.

No. 07
No data exfil

Capture writes land inside your tenant. Redaction runs at the edge. Bring-your-own-key on every cloud row that touches a third party.

No. 08
No vaporware on the roadmap

If a feature ships, the route returns 200. If it does not, the doc says so plainly, with the open issue link.

FRONTIER METHODS

The research stack, wired into the build.

Six methods you would otherwise glue together by hand. Each is a flag on the forge.

FlashAttention

Tiled SRAM attention from Dao et al. Memory-linear, math-equivalent. kolm wires FA-2 on Ampere and FA-3 on Hopper or Blackwell automatically based on the target GPU bound at compile time.

attention

GPTQ plus AWQ

Two calibration-based weight quantizers: GPTQ's Hessian-driven rounding and AWQ's activation-aware scaling. The forge picks per-layer using your eval shard, not a single recipe across the whole net.

quantization

Dynamic 2.0 plus MTP

Per-layer learned bit budget (Dynamic 2.0) paired with multi-token-prediction heads. Best K-Score per byte across our seven test models. Greedy decode jumps 1.6x to 1.9x with MTP on.

speed

BitNet plus AngelSlim

Microsoft's 1.58-bit ternary training plus Tencent's AngelSlim group-wise structured pruning. Combined, they take a 7B teacher into a sub-1 GB student that still runs your eval over 80% K-Score.

low-bit

TurboQuant KV

Per-head KV-cache quantization with rotation. Drops the long-context cache footprint 4x at INT2 with under 0.3 perplexity delta. Long-document agents stop OOM-ing on the same GPU.

long-context

mergekit TIES and DARE

Task-vector merging in weight space. Combine your distilled specialists into one model, or graft a reasoning head onto a coding base. TIES dedupes conflicts; DARE sparsifies. Both shipped behind one flag.

merge

01 Is kolm a replacement for OpenAI? +
No. kolm sits in front of OpenAI (and Anthropic, vLLM, SGLang, TRT-LLM). You keep the SDK you already use. We route, log approvals, and let you compile those approved calls into a smaller model you own. After compile, you stop paying per token.
02 What is a .kolm artifact? +
One sealed file that contains the model weights, the spec it was compiled against, the seed split it trained on, the teacher receipts, and an Ed25519 signature binding them all. Verify any response a .kolm produces at /verify-prod.
03 Where does the data live? Can I self-host? +
Every capture, dataset, and artifact is tenant-isolated and tenant-fenced. Your data never crosses to another tenant. The whole stack runs on your own hardware (BYOC, VPC, on-prem, air-gapped) under Apache-2.0. See /self-host.
04 How real are the benchmarks on this page? +
Run them yourself. The DeepSeek-R1 32B to 17.9 GB INT4 in 125s number was measured on an RTX 5090 with bitsandbytes 0.49.2 (NF4 + double quant) and torch 2.11.0+cu128. Transcripts are linked from /benchmarks.
05 What about HIPAA, SOC 2, regulated workloads? +
SOC 2 Type I available today. HIPAA mapping (/hipaa-mapping) and a signed BAA (/baa) are Enterprise controls. Explicit doors for healthcare, finance, legal, defense, and EU/sovereign deployments. PII redaction (text, image, audio) on by default at the gateway.
06 Will my model drift after I deploy it? +
Every compile carries a K-Score. The runtime computes a fresh K-Score from production responses and alerts when it drops. /drift shows the recipe: snapshot at compile, detect on every batch, recompile from fresh captures when the verdict moves out of "within."
07 How is this different from "fine-tune a small model"? +
Fine-tuning is one step of compile. kolm wraps capture (where do the rows come from?), distill (which student? which teacher? what loss?), quantize (verified roundtrip), eval (frozen at compile), package (one file), sign (Ed25519), and verify (registry + receipt). The value is the wiring.
08 What if I just want to verify someone else's .kolm? +
Free, no signup. curl https://kolm.ai/v1/verify/<cid> returns the registry row. kolm verify file.kolm verifies an artifact locally. kolm verify --receipt <receipt.json> verifies a response. All three work without an API key.
09 How do I export my data if I leave kolm? +
kolm account export writes a JSONL of every capture, every artifact, every receipt, every registry row, and every audit-log entry. Apache-2.0 license + open artifact format means the data is yours forever; rebuild the same compile elsewhere from the export.
10 How do I cancel my plan? +
Dashboard: Settings → Plan → Cancel. CLI: kolm billing cancel. No phone tree, no retention specialist. Cancellation takes effect at the end of the current billing period; export your data first if you want it.
