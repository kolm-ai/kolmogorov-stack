In plain English

kolm is an open-source compiler for AI.

You point your existing LLM client at it; it captures the calls. It distills those captures into a small specialist model trained on your traffic. It quantizes that model to fit the hardware you already own. It signs the result into one .kolm file you can deploy anywhere - laptop, VPC, air-gap - with no per-token bill behind it.

Same compiler, same artifact format. Apache-2.0, self-hostable, OpenAI-compatible, no vendor lock-in.

wrapper

The gateway.

One URL that speaks the OpenAI, Anthropic, Gemini, vLLM, SGLang, TGI, and TRT-LLM protocols. Routes calls, captures (input, output) pairs into your tenant lake, scrubs PII on the way in, signs a receipt on the way out. kolm wrapper up.

 studio

The browser UI.

Same compiler, no CLI. Pick captures, pick a base model, watch the distill + quantize + sign loop run in your browser tab. Same .kolm file out the other side. kolm studio open.

01 / What kolm does
What kolm does

Route, distill, run. One .kolm file at the end.

Route traffic, capture what works, distill it down, and run it on your own hardware. Each step stands on its own; together they replace the gateway / fine-tune / inference / receipts stack you would otherwise glue from four vendors.

01

Route & Capture

One OpenAI-compatible URL, every frontier model behind it. Failover policies, per-namespace proxy keys, PII redaction at the edge. Every approved pair lands in a tenant-fenced capture log you can replay byte-for-byte.

  • Confidence routing. Frontier only when your model isn’t confident - cheap base for the easy 90%, premium teacher for the long tail.
  • Approval-only capture. Nothing trains on a response you didn’t green-light.
  • Multimodal too. Image, audio, video, and PDF captures land in the same lake - with voiceprint scrub and image-PII redaction before they ever touch training.
  • Same SDK in and out. If your code already calls OpenAI or Anthropic, you change one URL.
Route & Capture →
02

Distill & Compile

One command turns approved captures into a sealed .kolm artifact: a smaller model trained on your real traffic, quantized to your target, with a frozen eval and a bake-off receipt against the teacher attached.

  • Teacher Council. Blend multiple teachers into one specialist - GPT-4 for code, Claude for prose, Gemini for math, your domain expert for the rest. The student inherits all of them.
  • Five distillation strategies, ten quantization methods. The compiler picks the right combination for your target hardware.
  • Federated consortium. Pool training signal across orgs with differential privacy - your raw data never leaves, only the noised approval rows do.
  • Reproducible from seed. Same input, same artifact, bit-identical. Auditable in two seconds.
Distill & Compile →
03

Run & Govern

One signed .kolm runs on cloud GPU, laptop, phone, or air-gapped tin - no proprietary loader, no per-token bill. Every response carries a hash-pinned receipt so compliance can prove what answered what.

  • Runtime targets. vLLM, SGLang, llama.cpp, Core ML, WebGPU - same artifact, picked at deploy time.
  • Verifiable receipts. Ed25519-signed, frozen-eval gated, drift-alerted. Pull any response back to the exact weights that produced it.
  • Regulatory packets. One command emits an EU AI Act, GDPR, SOC 2 or HIPAA binder - the receipt chain is already the evidence.
  • Air-gap one binary. Self-host the whole stack on systemd, k8s, or a docker compose - Apache-2.0, no phone-home.

FAQ

Real questions, no marketing.

01 Is kolm a replacement for OpenAI? +
No. kolm sits in front of OpenAI (and Anthropic, vLLM, SGLang, TRT-LLM). You keep the SDK you already use. We route, log approvals, and let you compile those approved calls into a smaller model you own. After compile, you stop paying per token.
02 What is a .kolm artifact? +
One sealed file that contains the model weights, the spec it was compiled against, the seed split it trained on, the teacher receipts, and an Ed25519 signature binding them all. Verify any response a .kolm produces at /verify-prod.
03 Where does the data live? Can I self-host? +
Every capture, dataset, and artifact is tenant-isolated and tenant-fenced. Your data never crosses to another tenant. The whole stack runs on your own hardware (BYOC, VPC, on-prem, air-gapped) under Apache-2.0. See /self-host.
04 How real are the benchmarks on this page? +
Run them yourself. The 32B-class → 17.9 GB INT4 in 125 s number was measured on an RTX 5090 with bitsandbytes 0.49.2 (NF4 + double quant) and torch 2.11.0+cu128. Model identities and transcripts are at /benchmarks.
05 What about HIPAA, SOC 2, regulated workloads? +
SOC 2 Type I available today. HIPAA mapping (/hipaa-mapping) and a signed BAA (/baa) are Enterprise controls. Full healthcare tree at /healthcare (six clinical workflows + the BAA + the HIPAA mapping). Explicit doors for finance, legal, defense, and EU/sovereign deployments too. The loop

Eight steps from a request to a model you own.

Each step is a CLI verb, an HTTP route, and an account surface. Click any step to jump straight to the place where the work happens.

  1. Step 1

    Capture

    Every request through the OpenAI-compatible gateway lands in a hash-chained capture lake. Receipts bind input, output, route, and latency.

  2. Step 2

    Opportunity

    Cluster repeated prompts. The optimizer flags template families that would distill cleanly into a smaller specialist.

  3. Step 3

    Dataset

    Promote a cluster into a training set. Export as Parquet or HuggingFace Datasets so the same rows feed eval and SFT.

  4. Step 4

    Label

    Mix teacher labels from a council (Claude + GPT-4o + DeepSeek) with reviewer rubrics. Provenance per row.

  5. Step 5

    Bakeoff

    Pit candidate adapters against a held-out slice and a judge model from a different lineage. Cheapest configuration that meets the bar wins.

  6. Step 6

    Compile

    Run the LoRA, quantize (NF4 / INT4 / FP8 / GPTQ / AWQ / NVFP4 / HQQ / MLX), and emit a signed .kolm artifact with a passport.

  7. Step 7

    Verify

    Re-run the same K-Score pack on the artifact. Score, signature, and recipe receipt all reproducible offline.

  8. Step 8

    Run

    Serve the artifact on your own hardware (cloud, on-prem, edge, air-gapped). Per-token bill after this step is zero.
