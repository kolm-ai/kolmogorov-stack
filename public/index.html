Change one base URL. Walk the loop.

Point your OpenAI or Anthropic SDK at 127.0.0.1:7402. kolm captures every call locally, redacts secrets, finds what repeats, and turns the best calls into a signed artifact that runs on your hardware.

  1. 1

    Capture

    Every request and response lands in ~/.kolm/events as a canonical event row. Cost, latency, status, redacted prompt, redacted response, provider, model, namespace.

    See your captures →
  2. 2

    Spot opportunities

    kolm clusters repeated prompts, sums spend, scores trainability and risk. You see a ranked list: this workflow runs 5,000 times/month, costs $380, outputs are stable, safe to compile.

    Open the opportunities view →
  3. 3

    Build a dataset

    Promote any opportunity to a dataset. kolm splits train and hold-out with a deterministic seed and pins both hashes to the receipt chain.

    Datasets →
  4. 4

    Review and label

    Send rows to a review queue. Approve, reject, or rewrite. Every decision is identity-stamped and audited; nothing trains until a real human says yes.

    Review queue →
  5. 5

    Bake off

    Run the held-out set against frontier teachers, your current local artifact, and the candidate. See pass rate, cost, latency, and privacy side by side.

    Bake-offs →
  6. 6

    Compile

    Approved rows become a .kolm bundle: recipes, signed receipt, Ed25519 signature, train/hold-out hashes, source-seed count, runtime target. Stub identities are blocked at the gate.

    Builds →
  7. 7

    Verify

    Re-hash every bundled file, re-check the signature, refuse on train/hold-out leakage, missing binary, or synthetic-only seeds. One CLI command, six closed-enum failure codes.

    How verify works →
  8. 8

    Run

    JS rule-class runtime ships today. WASM, native, GGUF, and ONNX targets are declared in the manifest spec and gated on the weight-class roadmap (/roadmap). kolm run dispatches by runtime; kolm eval uses the same path so the bake-off and the production call run identical code.

    Runtimes →

Every step writes to the same canonical event store. Every artifact carries a receipt that re-verifies offline. Nothing leaves your machine unless you opt in.

Common questions.

The six things people ask before they try it.

What does kolm actually do?
It sits between your code and any AI model. It captures every call as structured telemetry, redacts secrets before they hit disk, and tells you which calls you repeat enough to be worth compiling into a local artifact you own. the value loop →
How do I plug it in?
Change one base URL. Point your OpenAI or Anthropic client at http://127.0.0.1:7402 and kolm proxies upstream while it captures locally. No SDK to install, no code to rewrite. quickstart →
Where does my data live?
Local by default. Captures land in ~/.kolm/events on your machine. Prompts are redacted before persistence; raw prompt storage is opt-in. Cloud sync and team sharing are explicit, not automatic. privacy posture →
What is inside a .kolm file?
A zip with your trained model or rule bundle, the approved train split, a disjoint holdout, a signed receipt chain (Ed25519) that proves how it was built, and a runtime manifest naming JS/WASM/native/GGUF/ONNX targets. spec walkthrough →
What if I want to leave kolm?
The artifact is yours. The runtime is MIT. The spec is open (RS-1). Run any .kolm forever without us, on any hardware. RS-1 open spec →
What does it cost?
CLI + runtime + local lake are free and MIT. Cloud compile and hosted registry start at $9 / mo. Team and Enterprise tiers add seats, BAA, SCCs, SOC 2 Type I (target Q3 2026), and SLAs. see pricing →
