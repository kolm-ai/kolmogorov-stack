The four engines, beneath one verb.

Every kolm compile drives four engines that compose at every token: Recall grounds in your corpus; Distill labels with verified inference; Decompose extracts deterministic patterns; Run packages and signs. You only ever see the compiler.

01 — RECALL

Multimodal substrate

qmd-indexed BM25 + vector + RRF + reranker. Every modality (text, image, audio, video, PDF) auto-becomes searchable Markdown sidecars. Per-tenant namespaces.

02 — DISTILL

Verified Inference

k-sample any frontier model, deterministically verify, accept the winner. Generator-Verifier asymmetry. P(correct) ≥ 1 − (1 − p·v)k.

03 — DECOMPOSE

Deterministic draft pack

Recipe registry slice covering this task's structured-token subset. Public registry compounds: every compile shrinks the next.

04 — RUN

Signed artifact

Single zip: manifest · model.gguf pointer · LoRA · recipes · sqlite-vec index · signature · evals. K-score visible. Inspectable. Portable.

Frontier agents call your skills automatically.

Run kolm serve --mcp once. Claude / Codex / Cursor / any MCP client now sees every compiled .kolm in ~/.kolm/artifacts/ as a tool. They reach for compiled skills before fresh inference — that's the cache.

# install $ npm i -g @kolmogorov/kolm # compile a skill $ kolm compile "triage support tickets" --data ./tickets ✔ ~/.kolm/artifacts/job_abc.kolm (2.3KB) K-score: composite 412.7 · accuracy 96.5% · coverage 92% · 1.4ms p50 # expose to frontier agents $ kolm serve --mcp ┌─ kolm MCP server ──────────── │ artifacts: ~/.kolm/artifacts │ tools: 1 │ • triage-tickets — k=412.7, signed └────────────────────────────── # Claude (or any MCP client) now sees it. # Next time it needs to triage tickets: it just calls the compiled skill.

K-score: smallest passing artifact wins.

Every .kolm ships with five visible axes — accuracy, coverage, latency, cost, size — and a single composite score. Sort artifacts by composite. Compare versions. Watch the cache compound: every compile makes the next compile cheaper because the public registry grows.

composite = (accuracy × coverage × 1000) / log₂(size_kb + 2). Bigger is better.
