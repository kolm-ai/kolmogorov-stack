Beneath one verb · four engines

Every kolm compile drives four engines
that compose at every token.

Recall grounds in your corpus. Distill labels with verified inference. Decompose extracts deterministic patterns into a draft pack. Run packages, signs, and serves it as an MCP tool. You only ever see the compiler.

01 — RECALL

Multimodal substrate

Drop a folder. We auto-detect modality (text · image · audio · video · PDF), tokenize each into Markdown sidecars, embed, and serve a hybrid BM25 + vector + RRF + reranker index per tenant. Every Distill k-sample is grounded in retrieved chunks.

→ explore Recall
02 — DISTILL

Verified inference

k-sample any frontier model in parallel, deterministically verify each candidate, accept the winner. Generator-Verifier asymmetry: compute scales, reliability climbs. P(correct) ≥ 1 − (1 − p·v)k.

→ formal model
03 — DECOMPOSE

Deterministic draft pack

Recipe Speculative Decoding observes the deterministic-token subset of frontier behavior on this task and packs it into a registry-indexed draft. Coverage compounds: every public compile shrinks the next.

→ how RSD works
04 — RUN

Signed artifact, MCP-served

Single signed zip: manifest · model.gguf pointer · LoRA · recipes · sqlite-vec index · evals · signature. Run with kolm run; expose to Claude / Cursor / Codex with kolm serve --mcp.

From terminal to phone

One command. .kolm out.
Then call it from anywhere.

The CLI ships first. Run npm i -g @kolmogorov/kolm, point it at a task and a folder, and you get back a single .kolm file with a visible K-score. Drop it in ~/.kolm/artifacts/ and every MCP-aware agent — Claude Desktop, Cursor, Codex — can call it. The cache is the moat.
