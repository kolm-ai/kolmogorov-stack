        02 / First 10 Minutes
        
The first run should prove the whole loop.

        
The best onboarding path does not ask teams to migrate everything. It proves one controlled route end to end.

      

      

        
Key
Issue a scoped credential
Create the tenant, keep the key, and move credential handling into account controls instead of scattered environment notes.

        
Route
Send one namespace
Use an OpenAI-compatible route first, then expand to SDK, CLI, webhook, stream, trace, log, warehouse, and browser sources.

        
Capture
Attach policy to behavior
Prompts, completions, tool calls, labels, failures, redaction, retention, and source identity stay together.

        
Compile
Ship an artifact, not a promise
The output path is a signed artifact with manifest, eval evidence, runtime target metadata, receipt, and export bundle.

      

    

  


  

    

      

        03 / Enterprise Controls
        
The control center is the product surface buyers inspect.

        
Enterprise-grade software exposes the objects administrators need to govern, not just the model call that developers need to run.

      

      

        
Identity
Roles and service accounts
Workspace, project, environment, credential, key scope, user role, and account lifecycle state.

        
Sources
Every credible data path
API routes, SDKs, CLI uploads, webhooks, streams, logs, traces, browser events, warehouse imports, labels, and runtime receipts.

        
Policy
Provider and data controls
Model allowlists, connector state, redaction, retention, rate limits, cost controls, and risky-action approvals.

        
Evidence
Portable exports
Receipts, manifests, eval reports, event streams, warehouse exports, webhook callbacks, runtime packages, and readiness ledgers.

      

    

  


  

    

      

        04 / Developer Contract
        
One route should map to every later product object.

        
The signup snippet starts with a single route, but the contract is bigger: every write should emit lifecycle state that an admin can inspect or export.

      

      
curl -s https://kolm.ai/v1/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"you@company.com"}'

curl -s https://kolm.ai/v1/route/chat/completions \
  -H "Authorization: Bearer $KOLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "namespace":"default",
    "model":"openai:gpt-4.1-mini",
    "messages":[{"role":"user","content":"hello"}]
  }'
