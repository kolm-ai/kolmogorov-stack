Enterprise controls

        
The control center is the surface buyers inspect.

        
Enterprise-grade software exposes the objects administrators need to govern, not just the model call developers need to run.

        
    
          
  • Identity · workspace, project, environment, credential, key scope, user role, account lifecycle
    • 
          
  • Sources · API routes, SDKs, CLI uploads, webhooks, streams, logs, traces, browser events, warehouse imports, labels, receipts
    • 
          
  • Policy · model allowlists, connector state, redaction, retention, rate limits, cost controls, risky-action approvals
    • 
          
  • Evidence · receipts, manifests, eval reports, event streams, warehouse exports, webhook callbacks, runtime packages, readiness ledgers
    • 
        

      

      

        

          
Developer contract

          
One route should map to every later product object. Every write emits lifecycle state an admin can inspect or export.

          
curl -s https://kolm.ai/v1/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"you@company.com"}'

curl -s https://kolm.ai/v1/route/chat/completions \
  -H "Authorization: Bearer $KOLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "namespace":"default",
    "model":"openai:gpt-4.1-mini",
    "messages":[{"role":"user","content":"hello"}]
  }'
