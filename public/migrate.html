Migrate to kolm without rewriting your fine-tune pipeline.

Move from LoRAX, Predibase, OpenPipe, DIY LoRA, or hyperscaler fine-tuning without throwing away your data, evals, or serving stack. kolm adds signed artifacts, K-score gates, receipt chains, and portable .kolm output.

From LoRAX

Add kolm to your LoRAX stack.

LoRAX serves adapters; kolm builds the signed adapter LoRAX serves. Complement, not replacement. Your serving cluster keeps running.

  2. kolm extract LoRA weights
  3. LoRAX serves as before

What gets preserved when you migrate.

Your data, recipes, evals, and serving choices keep their meaning. The lock moves to the signed build artifact.

Preserved survives 1:1

  • Training data. seeds.jsonl is the canonical kolm input format: one object per row, {"input","output","tags"}. Your existing prompt/completion logs convert into that shape; the data does not get reshaped by kolm. If you already export to JSONL, your existing exporter is the migration tool.
  • Recipe semantics. Any deterministic rule, regex, lookup table, or template you wrote becomes a kolm rule-class recipe with no behavior change. The rules run with bit-identical output. The eval that gates them is the same eval you were already running.
  • Eval set. kolm forwards your existing holdout as the frozen external holdout (Wave 164). It hashes the corpus, binds the hash into the artifact, and re-runs the same eval on every rebuild. The numerics do not move just because the harness changed.
  • Team workflow. The serving infrastructure you hardened stays. kolm builds the artifact, then hands it off, LoRAX, Predibase, BYO inference, hyperscaler endpoint, any of them. The handoff is a signed file, not a runtime takeover.

What kolm adds new on top

  • Receipt chain. The 14-step provenance ledger: spec → seeds → split → train → recipes → evals → external_holdout → tenant_shadow → auditor → supersession → drift_report → export → signatures → rekor. Every step's hash binds into artifact_hash; tampering at any step breaks every downstream signature.
  • K-score gate. A composite gate across A / T / R / F / E / Z axes blocks the compile step when eval regresses. The gate is not a vibe-check, the threshold is part of the recipe spec and the verifier re-derives it offline.
  • Signed artifact. Ed25519 signature (Wave 161) plus Sigstore + Rekor transparency log entry (Wave 162). A third party with only the public key can confirm the file matches the build the signer claimed.
  • Portable .kolm format. The artifact is a single file. It runs on a laptop, a phone, an edge box, or an air-gapped VPC. RS-1 v2.1 is Apache-2.0, if kolm.ai disappears, your .kolm still runs.

The CLI verbs you will use during migration.

These are the verbs used across the migration guides, ordered by the path from hosted model to owned receipt chain.

VerbWhat it doesStatus
kolm compileBuild a .kolm from a spec (offline) or from a task description (cloud). The terminal verb every migration ends with.shipped
kolm captureDrop-in proxy for OpenAI / Anthropic that logs (input, output) pairs into a namespaced seeds.jsonl. Replaces the SDK-wrap step in OpenPipe / hyperscaler flows.shipped
kolm distillAuto-distill a captured namespace into a local student artifact via the kolm trainer bridge. The build step for ML-class recipes.shipped
kolm verifyRun every offline check kolm makes about an artifact: 24 checks at last count, including signature, receipt chain, drift gate. The post-migration confidence step.shipped
kolm extractPull text out of PDFs / HTML / JSON / images. Also reused in the per-source guides to surface "extract the LoRA weights" or "extract the training config" via the artifact-as-archive layout.shipped
kolm driftSnapshot, report, and cron the M+3 drift detection lifecycle. Used after migration to lock in the same baseline you had on the source system.shipped
kolm proxyOpenAI-compatible HTTP proxy variant of kolm capture (no SDK install, base-URL swap). Per-source guides reference this as a shorter migration path from OpenPipe / hyperscaler flows.shipped
kolm importOne-shot importer for foreign capture stores (OpenPipe project export, Predibase dataset, etc.) into seeds.jsonl. Aliases kolm import-chat.shipped
kolm wrapPass-through that points a foreign trainer config (llama-factory, axolotl, unsloth, trl) at the kolm receipt-chain emitter. Used in the DIY guide.shipped

Every verb above ships in the current CLI. The per-source guides walk the exact invocation order; nothing blocks on a missing primitive.
