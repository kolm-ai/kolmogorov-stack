Merge · the W866 operator

Two artifacts. One command. Five honest methods.

Combine a base .kolm and a head .kolm into one. Same-lineage required for the gradient methods; the gate trips loudly when base models disagree. Dry-run prints the plan, the heuristic K-score delta, and the lineage check — before the merge writes a byte.

$ kolm merge base.kolm head.kolm --method ties --dry-run

01 · methods

Five methods, five honest deltas.

Each method has a published heuristic K-score range. Not theoretical — empirical, from the same dataset of merges that calibrates --dry-run. ties is the default because it is the highest-delta method that still respects sign agreement; pick another when the heuristic favors it.

  • Linear — safe, no surprises, smallest delta
  • SLERP — spherical interp, preserves norm
  • TIES — sign-resolved, default, largest stable delta
  • DARE — drop-and-rescale, sparser updates
  • Passthrough — layer routing, no weight merge

02 · lineage gate

Same base, or it fails loud.

Linear, SLERP, TIES and DARE all need same-shape weight tensors. The dry-run inspects both artifacts and checks base_model + params_b. If they disagree, you get a warn-pill in the UI and a non-zero exit in the CLI — before the merge starts and discovers a tensor-shape mismatch the painful way.

  • Reads base_model from each artifact manifest
  • Same lineage → merge proceeds
  • Lineage mismatch → refuses unless --force
  • Passthrough exempt — designed for cross-lineage
# lineage mismatch trips before any GPU work.
$ kolm merge \
    artifacts/qwen-7b-router.kolm \
    artifacts/llama3-8b-prose.kolm \
    --method ties --dry-run
× lineage mismatch
   base: Qwen/Qwen2.5-7B-Instruct  (7.0B)
   head: meta-llama/Llama-3-8B     (8.0B)
   ties needs identical base; use --method passthrough
   or rebase one artifact to match.

# same-lineage merge proceeds.
$ kolm merge \
    artifacts/qwen-7b-router.kolm \
    artifacts/qwen-7b-prose.kolm \
    --method ties --alpha 0.5 --dry-run
✓ same lineage: Qwen/Qwen2.5-7B-Instruct
✓ method: ties (alpha=0.5)
✓ K-score Δ: +0.02 to +0.05 (heuristic)

03 · alpha

One number, two extremes.

--alpha is the blend weight from 0.0 to 1.0. At 0.0 the merged artifact is the base; at 1.0 it is the head; at 0.5 it is the canonical average. The dry-run computes the heuristic delta at your alpha, not a default — so you can sweep before you spend.

  • 0.0 → identical to base, smallest delta
  • 0.5 → default, canonical average
  • 1.0 → identical to head, head’s K-score
  • Sweep with --alpha-grid 0.25,0.5,0.75
# sweep alpha to find the K-score peak.
$ kolm merge base.kolm head.kolm \
    --method ties \
    --alpha-grid 0.25,0.5,0.75 \
    --dry-run
✓ alpha=0.25 → K-score Δ +0.01 to +0.03
✓ alpha=0.50 → K-score Δ +0.02 to +0.05
✓ alpha=0.75 → K-score Δ +0.02 to +0.04
recommended: alpha=0.50 (largest stable upper bound)

# execute the winner.
$ kolm merge base.kolm head.kolm \
    --method ties --alpha 0.5 \
    --out merged.kolm
✓ wrote merged.kolm (14.2 GB)
✓ ed25519 signed by ks_4b7bc3b...
✓ K-score: 86.5 (was 84.1 · 84.7)

04 · surfaces

CLI, TUI, Account, route. Same merge.

The merge operator ships in every face of the product. The route POST /v1/merge is the single source of truth; the CLI verb, the TUI merge-jobs view, and the Account wizard at /account/merge/new all hit it. Same dry-run delta, same lineage gate, same signed output.

  • POST /v1/merge — the route
  • kolm merge — the CLI verb
  • TUI :merge-jobs — the live list
  • /account/merge/new — the wizard
