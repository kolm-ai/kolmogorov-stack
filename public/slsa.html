Security / provenance

Build provenance status.

.kolm artifacts already carry local receipts, Ed25519 signatures, K-score evidence, and an optional Sigstore/Rekor policy gate. A public SLSA Level 3 attestation is still a target until release artifacts, workflow identity, and transparency-log entries are published with verifiable evidence.

ImplementedManifest hash, receipt chain, Ed25519 signature, local verifier, optional Sigstore bundle.
Opt-inRekor pinning through KOLM_SIGSTORE_REKOR_URL and verifier enforcement through KOLM_REQUIRE_REKOR=1.
Not Yet Public-AttestedPublic SLSA Level 3 release attestation, Fulcio identity proof, and public release Rekor entries.

How to verify a .kolm artifact now

kolm verify ./artifact.kolm --json

# Require public-key signatures and a Rekor-pinned Sigstore bundle.
KOLM_REQUIRE_ED25519=1 KOLM_REQUIRE_REKOR=1 \
 kolm verify ./artifact.kolm --json

The second command fails if the artifact only has a dry-run Sigstore bundle. That is the correct result for procurement or production environments that require public transparency-log evidence.

How to publish a Rekor pin

kolm sigstore-attest ./artifact.kolm \
 --rekor-url https://rekor.example \
 --json

The command republishes the existing digest, Ed25519 signature, and public key as a Rekor hashedrekord entry, then rewrites the artifact with rekor_log_entry and dry_run:false. The private signing key is not required for this post-build upgrade.

SLSA target state

Kolm should not claim SLSA Level 3 until the release workflow publishes evidence that can be independently checked. The target bundle is:

  • Release artifact SHA-256 and .kolm manifest hash.
  • in-toto SLSA v1.0 provenance emitted by the release workflow.
  • Sigstore certificate identity bound to the exact repository and workflow.
  • Transparency-log entry or offline Rekor bundle for air-gapped verification.
  • SBOM digest tied to the same release artifact digest.
