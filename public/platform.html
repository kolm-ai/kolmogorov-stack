01 / The problem

A questionnaire used to clear it. An autonomous agent does not.

The one-week security review runs four to eight the moment a reviewer has to vet an agent that acts on its own. The shape of the system below exists to answer that review in evidence, not in promises.

The clock

One week becomes eight

A self-attested answer no longer clears an agent that can act on its own. The review reopens from the start, and a six-figure deal waits on it.

The burden

Proof lands on your buyer

Say-so moves the work onto the reviewer's desk. They want evidence they can check themselves, not a promise that an audit happened somewhere.

The half-life

It goes stale on deploy

Even a clean point-in-time answer expires on your next release. A permission granted in January can still fire in August.

03 / Architecture

Onramp, sign, log, verify.

Two ways in, one way out. Import your existing logs or run a sidecar proxy, and the same attestation core seals every finding into a report anyone you hand it to can check offline.

01 · Onramp Import or sidecar

Bring LiteLLM, Helicone or Portkey logs, or run a sidecar / MCP proxy for live capture. No rebuild of your agent and no new SDK to ship.

02 · Sign Canonical Ed25519 seal

The findings are canonicalized to key-sorted JSON and signed with Ed25519. The signature covers the exact bytes, so a downgraded finding or an inflated score breaks it.

03 · Log Merkle transparency log

Each issuance enters an append-only Merkle log (RFC 6962 style, Ed25519 and SHA-256, not a blockchain). A per-report inclusion proof shows it was logged and never replaced.

04 · Verify Offline, in the browser

Your buyer verifies the signature on WebCrypto in their own browser, against your public key. No account, no upload, no kolm server in the trust path.

The verifier, the CLI and the library all use the identical canonicalization, and the verifier is open source. The trust path has no black box.

04 / Always current

A report is a photograph. Your agent is a film.

A point-in-time report goes stale on the next deploy. Continuous re-attests weekly or on every release and exposes one live Trust link you hand any buyer. No human in the loop.

05 / Proof

They don't trust us. They check the math.

Security theater can't be falsified. This can. Two checks run in the buyer's own browser, against the key inside the report, with no kolm server in the path.

Tier 1 · signature

Edit one field, the seal breaks

The Ed25519 signature covers the canonical bytes. Change a finding or inflate the score and the match fails, in front of the reviewer.

Tier 2 · issuer

A rogue key signs, but doesn't match

The signing key is pinned to the keyring your buyer expects. Forge a fresh key and tier 1 clears, but tier 2 exposes it.

Offline · no server

kolm is never in the path

Verification runs on WebCrypto in the buyer's browser. No account, no upload, nothing for us to fake. Scope is part of the signed object.
