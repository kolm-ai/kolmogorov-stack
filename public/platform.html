01 / The problem

A questionnaire used to clear it. An autonomous agent does not.

The one-week review runs four to eight when a reviewer has to vet an agent that acts on its own. Everything below exists to answer that review in evidence.

The clock

One week becomes eight

A self-attested answer no longer clears an autonomous agent. The review reopens, and a six-figure deal waits on it.

The burden

Proof lands on your buyer

Say-so moves the work to the reviewer's desk. They want evidence they can check themselves, not a promise that an audit happened.

The half-life

It goes stale on deploy

Even a clean point-in-time answer expires on your next release. A permission granted in January can still fire in August.

03 / Architecture
03

Onramp, sign, log, verify.

Two ways in, one way out. The same attestation core seals every finding into an object anyone you hand it to can check offline.

The pipeline, end to end
onrampOpenTelemetry, Datadog, Langfuse, LangSmith, MCP server logs, OpenAI Agents SDK or gateway logs · or sidecar / MCP proxy
analyzepermissions · audit trail · egress · injection · provenance
signcanonical key-sorted JSON · ed25519 over the exact bytes
logappend-only Merkle log · per-report inclusion proof
verifyoffline · WebCrypto in the buyer's browser · no account

Onramp · no rebuild

Bring the logs you already have

Import OpenTelemetry, Datadog, Langfuse or LangSmith telemetry, MCP server logs, OpenAI Agents SDK traces, gateway logs from LiteLLM, Helicone or Portkey, or run a sidecar / MCP proxy for live capture. Your application ships unchanged.

Seal · exact bytes

Edit one field, the seal breaks

Findings are canonicalized and signed with Ed25519 over the exact bytes. A downgraded finding or an inflated score breaks the signature.

Log · append-only

Issued once, on the record

Every issuance enters an append-only Merkle transparency log (RFC 6962 style, Ed25519 and SHA-256, not a blockchain). The inclusion proof shows the report was logged and never replaced.

The verifier, the CLI and the library share one canonicalization, and the verifier is open source · inspect the verifier · transparency log · no black box in the trust path
