Eighteen HIPAA identifiers, none of them out.

A local .kolm that removes the 18 HIPAA Safe-Harbor identifiers from free text. The verifier rejects PHI reintroduction, and each run carries a signed receipt with K-score and input hash.

base modelqwen2.5-3b-instruct
gold pairs1,200 (840 train / 360 eval)
k-score floor0.90
artifact size42 MB
compile time~28 min
spec sourceredactor + verifier

What this recipe does

Anywhere free-text touches PHI, intake messages, EHR-adjacent notes, support tickets, voice transcripts, OCR output, you need a deterministic redactor before that text leaves the audited boundary. This recipe compiles one. The spec is generic by design: it does not assume a clinical structure (use hipaa-summarizer for SOAP) and does not assume a single document type.

The verifier holds a frozen regex bundle covering the 18 Safe-Harbor identifiers, plus a learned classifier head for the harder long-tail cases: surnames adjacent to titles, dates phrased as "two weeks before her birthday," and license plates inside narrative prose. Any output token that matches a banned pattern breaks the receipt chain and is rejected.

The spec

{
 "output_kind": "text",
 "verifier": {
 "banned_token_regex": "./hipaa-18.regex",
 "banned_classifier": "./hipaa-long-tail.cls",
 "max_edit_distance": 0.35,
 "preserve_semantics_floor": 0.92
 },
 "redact_token": "[REDACTED]"
}

Compile

kolm compile "strip 18 HIPAA identifiers, preserve clinical meaning" \
 --base qwen2.5-3b-instruct \
 --pairs ./phi-gold/*.jsonl \
 --verifier hipaa-18-regex,hipaa-long-tail-cls,semantics-92 \
 --k-floor 0.90 \
 --output phi-redactor.kolm

K-score gate

K-score 0.93 held-out 360 inputs · PHI-leak 0% · semantic-faithful 94% · over-redact 3%

PHI-leak is the load-bearing metric; it is exact (regex + classifier) and any non-zero value fails the build. Over-redact tracks when the model removes a non-PHI token by mistake (rare but tracked).

Deploy

# Belt-and-suspenders: even on already-de-identified data, run this
# as a final pass before anything crosses the audited boundary.
on_request = (msg) => {
 const safe = kolm.run('phi-redactor.kolm', msg.body);
 msg.body = safe.text;
 audit.log({
 artifact: 'phi-redactor.kolm',
 request_id: msg.id,
 k_score: safe.receipt.k_score,
 receipt_hash: safe.receipt.hash
 });
};

When to use this vs other redactors

  • phi-redactor (this recipe), generic free-text in, free-text out. Use when you do not know the document shape upfront.
  • hipaa-summarizer, clinical note in, structured SOAP out. PHI is redacted as part of the summarization; do not run both.
  • drug-name-redact, strips proprietary drug names while preserving generic equivalents. Different problem; not a PHI redactor.
  • soap-redactor, takes a SOAP-formatted note and redacts each section in place, preserving the SOAP structure.
