Shape of a row

What an agent distill row looks like.

A traditional distill row is a chat turn: prompt in, response out. An agent row carries the tool_calls array the teacher emitted, the arguments it passed, and the tool results the agent saw next. The student learns to pick the same tool with the same arguments, not just to imitate the prose. Both halves below are the same turn from the same trajectory — teacher on the left, student on the right after training.

The prose differs — "Looking up the order" vs "Pulling that order now" — and we do not score it. What matters is that the tool name matches, the argument shape matches, and the value of order_id is preserved exactly. The eval harness grades on that, not on prose similarity.

Council distillation

Three teachers, one council.

A single teacher gets a single agent right. Three teachers from different lineages catch each other’s edge cases. kolm runs Claude 4.7, GPT-4o, and Cerebras Llama-3.3-70B through the same trajectory and emits one canonical row per turn — when the teachers agree on the tool, the row passes; when they disagree, the council vote picks the majority and tags the row for human review.

Teacher A · Anthropic

Claude 4.7 Sonnet

Strongest on argument fidelity — long tool schemas, nested structured args, JSON shape under pressure. Drives the agent recipe’s ground-truth tool selection on hard turns.

Teacher B · OpenAI

GPT-4o (or 4-Turbo)

Strongest on tool sequencing — deciding whether to call a tool now or ask the user first, when to stop calling tools, how to chain three calls in order. Independent lineage from Claude.

Teacher C · Meta via Cerebras

Cerebras Llama-3.3-70B

Fastest of the three at thousands of tokens per second on Cerebras Cloud. Drops the council collect step from ~60 min to ~10 min on 2000-row corpora and contributes a Meta-lineage vote.

Open vs proprietary

Pioneer Agent Mode vs kolm agent distill.

Pioneer (and Fastino, and a growing list) ship hosted “Agent Mode” products that distill trajectories behind their console. We do the same job in the open, with portable artifacts and signed receipts you can hand to an auditor.

Eval harness

Does the distilled agent pick the same tools?

The eval harness runs the student over a held-out slice of trajectories and grades each turn against the teacher decision. Three numbers come out, and the receipt for each eval run is signed and re-verifiable offline.

Tool-call accuracy

Percent of turns where the student picked the same tool the teacher picked. Reported per tool, with confidence intervals. A turn where the teacher answered without a tool counts as tool=none.

Argument-shape match

Percent of correct-tool turns where the student’s arguments validate against the teacher’s JSON shape and exact-match on load-bearing fields (IDs, dates, enum values). Loose match on free text.

Latency

Median and p95 wall-clock per turn on the student runtime you compiled to. Reported alongside teacher latency so the cost-per-turn delta is legible at a glance.

Reproducible

Reproducible in 5 commands.

From a captured trajectory to a deployed student model. Each command is its own copyable line; defaults are sane for the agent recipe, overrides live in the recipe doc.

$ # agent distill, end to end
$kolm capture replay --trajectory ./traces/agent-runs.jsonl# 1. load trajectory
$kolm distill spec --mode=agent --teachers=claude-4.7,gpt-4o,cerebras-llama-3.3-70b# 2. write spec
$kolm distill collect# 3. council fan-out
$kolm distill train --base qwen2.5-7b-instruct# 4. LoRA train
$kolm deploy --target ollama# 5. ship
