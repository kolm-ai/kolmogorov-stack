Capture → compile → compose → deploy

From live calls to a model on your own hardware.

Four steps from your API traffic to a signed model you own. Capture the behavior you already use, compile it into one portable file, combine your specialists, and run it where it fits. No rewrite to get here, no lock-in once you have it.

01 · /v1/route

Capture

Point your existing calls at Kolm. It keeps the prompts, tool calls, outcomes and receipts that define the behavior you want to own.

02 · /v1/compile

Compile

Turn that behavior into a deployable .kolm file — model, recipe, evals and signature in one package you can move anywhere.

03 · /v1/compose

Compose

Combine your narrow specialists into one system, with routing that picks the right expert for each call.

04 · /v1/devices

Deploy

Run it on the smallest runtime that fits, then serve, install or export — on the hardware you already have.

For teams

Run it for the whole team, safely.

Scoped keys, your provider keys in one place, spend controls, capture retention, compile receipts and deployment state — every workspace stays its own, with one clear contract behind it.

  • Each workspace stays its own. Every capture, job, artifact and device belongs to the workspace that made it — nothing bleeds across teams.
  • Your keys, one place. Routing, provider health and receipts in a single view, so model calls stop scattering across tools.
  • Ship with proof. Every artifact carries its manifest, hash and a verify link — so anyone can check exactly what shipped.
  • Hand buyers the evidence. When a customer asks how it holds up, point them to the security report at audit.kolm.ai.
