The Kolm loop · capture → compile → compose → deploy

Bring back the original Kolm loop.

The core product is not an audit report. It is a repeatable loop for turning real API behavior into portable, signed, smaller model systems — capture, compile, compose, deploy.

01 · /v1/route

Capture

API collection wrapper. Keep the traces, prompts, tool calls, outcomes and receipts that describe the behavior you care about.

02 · /v1/compile

Compile

Artifact builder. Convert curated examples into a deployable .kolm package with manifests, evals and signatures.

03 · /v1/compose

Compose

Model composition. Train and route specialist behaviors so one system can combine narrow experts cleanly.

04 · /v1/devices

Deploy

Device shrinking. Choose the smallest viable target, then serve, install or export for the runtime.

Enterprise

Compiler controls for teams.

Kolm keeps tenant boundaries, scoped keys, provider vaulting, spend controls, capture retention, compile receipts and deployment state in one backend contract.

  • Tenant isolation. Every capture, job, artifact and device row is fenced to the authenticated workspace.
  • Provider control. Gateway routing, provider health, keys and receipts without scattering model calls across tools.
  • Signed outputs. Artifacts carry manifests, hashes and verification routes so runtime changes are auditable.
  • Audit module. Security readiness reports stay available at audit.kolm.ai when a buyer asks for evidence.
