SOC 2 Type II readiness, from gateway to artifact.

The kolm stack ships every Trust Services Criteria control by default. This page is the operator-facing readiness checklist: eighteen line-items across Security, Availability, Processing Integrity, Confidentiality, Privacy, plus Continuous Monitoring. Items we ship are marked YES with the kolm component that satisfies the criterion. Items the external auditor supplies are marked NO with a pointer to who provides them. Stamped w767-v1.

12-month audit retention statement

The kolm audit chain defaults to 365-day retention per DEFAULT_RETENTION_DAYS in src/audit-retention.js. The minimum permitted floor is 90 days (SOC 2 Type I), and the maximum is 2555 days (~7 years — HIPAA / GDPR right-to-erasure exception ceiling). Operators can set a per-tenant override via kolm cert audit-retention-set <days> --confirm; values outside the [MIN, MAX] window are rejected with an honest envelope rather than silently clamped.

Retention enforcement is dry-run by default. The enforceRetentionPolicy entry point ALWAYS counts what would be evicted; live eviction requires BOTH confirm:true AND dry_run:false. A bare confirm:true without explicit dry_run:false remains a dry run — two-key destruction so an accidental flag cannot trigger eviction.

Honesty floor. A retention policy that silently truncates audit data below the configured floor is worse than no policy — it produces a false-clean report. We log every retention-policy change through the same audit chain, so an auditor can reconstruct the operator's intent across the sample period.

Continuous monitoring dashboard

SOC 2 Type II audits look for continuous monitoring as evidence that controls operate effectively over time, not just at a point-in-time snapshot. The kolm continuous-monitoring dashboard maps each TSC criterion to an operational signal and rates it green / yellow / red / unknown.

Open the continuous monitoring dashboard →green / yellow / red / unknown

Unknown is never green. When the underlying signal source is offline, the dashboard returns status: 'unknown' with current_value: null. We never fabricate a green pill to hide a probe outage — a dashboard that lies is worse than no dashboard.
