From install to a signed .kolm artifact on your laptop.

You have an LLM API key, a few captures, and a terminal. By the end of this quickstart you have a signed specialist running locally with verifiable receipts. No mocks, no scratch repos, no waiting on procurement.

Prerequisites

You need three things. Each takes under two minutes if you do not already have it.

Node 20+ or Python 3.11+

Either runtime works. Node ships the CLI fastest; Python ships the richest data path.

An LLM API key

OpenAI, Anthropic, Mistral, Google, Groq, or any OpenAI-compatible endpoint.

A kolm key

Free tier on /signup. No card. 50k requests per month.

Air-gapped? Skip the kolm key. The whole flow runs locally with --offline. Read /enterprise for BYOC, VPC, and air-gap deployment patterns.
Step 1

Install the CLI

Pick the runtime that matches your stack. Both ship the same surface; the CLI is the binary you will reach for most.

$ npm i -g github:kolm-ai/kolm-stack
$ kolm --version
  kolm 1.0.0  node 20.11.0
$ pip install git+https://github.com/kolm-ai/kolm-stack@main#subdirectory=sdk/python  # GitHub source package; PyPI release pending
$ kolm --version
  kolm 1.0.0  python 3.11.6
$ brew install node          # runtime setup (no Kolm tap)
$ npm i -g github:kolm-ai/kolm-stack
$ kolm --version
  kolm 1.0.0
$ git clone https://github.com/kolm-ai/kolm-stack
$ cd kolm-stack && npm install && npm link
$ kolm --version
  kolm 1.0.0

Run kolm doctor to verify your environment. It prints a green checklist and writes a JSON receipt to ~/.kolm/doctor.json.

Step 2

Authenticate

You authenticate once. The CLI stores a short-lived token in your OS keychain; teams can use organization keys instead.

$ kolm login
   opened browser to https://kolm.ai/cli/auth
   logged in as you@example.com
   tenant tenant_4d3d8db6c820 (free plan)
   50,000 / month included gateway requests
$ kolm whoami
  email: you@example.com
  key:   ks_4b7bc3b...  scope: all
Headless / CI? Use export KOLM_API_KEY=ks_... and skip kolm login. The CLI prefers KOLM_API_KEY over the keychain when both are present.
Step 3

Route a call through the gateway

Kolm is OpenAI-compatible and Anthropic-compatible. Point OPENAI_BASE_URL (or the equivalent) at the gateway and the SDK works without changes.

$ curl https://kolm.ai/v1/chat/completions \
    -H "Authorization: Bearer $KOLM_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "model": "openai/gpt-5",
      "messages": [{"role":"user","content":"hello"}]
    }'
import OpenAI from "openai";
const client = new OpenAI({
  baseURL: "https://kolm.ai/v1",
  apiKey:  process.env.KOLM_API_KEY,
});
const r = await client.chat.completions.create({
  model: "openai/gpt-5",
  messages: [{ role: "user", content: "hello" }],
});
console.log(r.choices[0].message.content);
from openai import OpenAI
client = OpenAI(
    base_url="https://kolm.ai/v1",
    api_key=os.environ["KOLM_API_KEY"],
)
r = client.chat.completions.create(
    model="openai/gpt-5",
    messages=[{"role":"user","content":"hello"}],
)
print(r.choices[0].message.content)
import Anthropic from "@anthropic-ai/sdk";
const client = new Anthropic({
  baseURL: "https://kolm.ai/v1/capture/anthropic",
  apiKey:  process.env.KOLM_API_KEY,
});
const r = await client.messages.create({
  model: "anthropic/claude-sonnet-4-6",
  max_tokens: 256,
  messages: [{ role: "user", content: "hello" }],
});
console.log(r.content[0].text);

The gateway adds a x-kolm-trace header to every response. Save it somewhere; you will reference it in step 4.

Step 4

Capture approved traffic

You only distill what you trust. Mark a call as approved with kolm capture or by adding the x-kolm-approve: 1 header on the request. Approved captures land in a per-namespace store and become the training set for the next distillation.

$ kolm capture log --namespace support --trace trc_018f3a...
   appended capture cap_018f3b... to namespace "support"
$ kolm capture status --namespace support
  approved:  1,247
  pending:     312
  rejected:    104
  size:       4.7 MB
  threshold:  500   (distill is ready)
Threshold notification: when an approved namespace crosses your threshold, you get a WebPush ping with a deep link to /captures. Configure thresholds in kolm capture config --threshold 500.
Step 5

Distill a specialist

One command. The CLI picks a sensible teacher, holds out 10% for evaluation, runs the bake-off against your captures, and writes a real evaluation report.

$ kolm distill from-captures \
    --namespace support \
    --teacher claude-sonnet \
    --student qwen2.5-7b

   sampled 1,247 approved captures (held out 124 for eval)
   generated 3,741 teacher completions
   trained student on 12,341 (capture, teacher) pairs
   bake-off: student matches teacher on 91.4% of held-out captures
   wrote run_5d2a1.../report.json

The bake-off compares student answers against teacher answers on the held-out set. The Jaccard token-overlap score is the floor; if you have a stricter grader (regex, JSON schema, golden set), pass --grader path/to/grader.js.

Step 6

Quantize to INT4

Hardware shrinks the model. NF4 + double quantization keeps the K-score within 2 points of BF16 on every model we have shipped to date. Pick int4 for a 4x size reduction, int8 for 2x with near-zero quality loss.

$ kolm compile run_5d2a1... --quantize int4

   loaded student qwen2.5-7b   14.2 GB BF16
   NF4 + double quant         5.2 GB INT4   (29.2s)
   K-score drift            0.8 / 100   (within budget)
   wrote support-2026-05-22.kolm   (5.2 GB)
Drift is bounded: the compiler refuses to ship the artifact if quantization drops the K-score by more than your budget (default: 3 points / 100). Override with --max-drift 5 at your own risk.
Step 7

Sign and verify

Every .kolm artifact ships with an Ed25519 signature and a SHA-256 manifest. The compiler signs with your tenant key; kolm verify checks both the manifest and the signature.

$ kolm verify support-2026-05-22.kolm
   manifest: 23 files, 5.2 GB
   sha-256 match
   Ed25519 signed by tenant_4d3d8db6c820
   receipt cid: bafkreig...
  receipt-only verification: https://kolm.ai/verify/bafkreig...

The receipt CID is your audit hook. Send it to procurement, your SOC 2 auditor, or your customer; anyone can verify against the public ledger without your private key.

Step 8

Run it locally

The runtime ships in the CLI. No GPU? It runs on CPU with llama.cpp. GPU present? It picks vLLM, SGLang, TensorRT-LLM, or ONNX Runtime, in that order, depending on what is installed.

$ kolm run support-2026-05-22.kolm \
    "I want to cancel my plan; the export is broken."

  → I am sorry the export failed. Could you share the export ID
     so I can pull the trace? Cancellations need a confirmation
     from your admin; I have escalated this to billing@.

  11.5 tok/s   8.9 GB RAM   trace: trc_018f3c...

Run a single prompt, run a batch, run as a daemon on port 11434 (Ollama-compatible), or compile to WASM and embed it in a browser tab. The artifact is the same; the runtime is your call.

$ kolm run support-2026-05-22.kolm --serve --port 11434
   listening on http://127.0.0.1:11434 (Ollama-compatible)
   openai-compatible at http://127.0.0.1:11434/v1
   idle in 14ms; warm in 320ms
FRONTIER RECIPES

Or build a frontier recipe in one line.

Same compiler. Different starting point. These commands produce verified, signed, K-scored .kolm artifacts ready for the leaderboard.

  • Frontier reasoning, 8B

    REASONING · multi-teacher · NF4

    Opus-4.7 teacher distilled into a Qwen2.5-7B student with NF4 quantization and reasoning-trace supervision.

    $ kolm build reasoning --teacher=claude-opus-4-7 --student=qwen2-5-7b --bit=4.0 --mode=reasoning

  • Frontier coding, 13B + MTP

    CODING · multi-token prediction · aider bench

    Multi-token prediction head bolted onto a 14B coder student. Aider bench drives the bake-off grader.

    $ kolm build coding --student=qwen2-5-14b --bit=4.0 --mode=mpt --bench=aider

  • Low-bit generalist, 1.58-bit

    BITNET · CPU · <2 GB resident

    BitNet ternary weights, drift-bounded against a BF16 reference. Runs on CPU with under 2 GB resident.

    $ kolm build low-bit-generalist --bit=1.58 --quant=bitnet

  • Multi-teacher fused, 14B

    CLAUDE · GPT · DEEPSEEK

    Claude, GPT, and DeepSeek each generate the same prompts. The fuser keeps the answer that the K-score grader ranks highest.

    $ kolm build multi-teacher-fused --teachers=claude,gpt,deepseek

VERIFY

Verify any artifact.

Validates the Ed25519 signature, recomputes the JCS-canonical SHA-256, replays the K-score eval pack. Works offline.

$ kolm verify <artifact-cid>
Ledger receipts are public. Share a CID with your auditor; they verify against /verify-prod without your private key.

Troubleshooting

The dozen issues we see on real laptops, in the order we see them.

kolm: command not found after npm install
Your global npm bin is not on PATH. Run npm bin -g and add the output to PATH. On macOS with Homebrew node this is usually /opt/homebrew/bin; on Linux it is /usr/local/bin.
kolm login opens the browser but never returns
A corporate proxy is blocking the loopback callback. Run kolm login --device for a device-code flow that does not need the loopback.
SSL/TLS error on kolm distill
Your egress proxy is doing TLS inspection. Either trust the proxy CA system-wide, or set NODE_EXTRA_CA_CERTS=/path/to/ca.pem before running the CLI.
OOM during quantize on a 7B model
You need 16 GB free RAM for a 7B INT4 quantize. Free memory, or pass --quantize int8 (needs 24 GB), or pass --remote-quantize to run the heavy step in our hosted workers.
Distill says no approved captures
Captures are pending until you mark them approved. Run kolm capture review --namespace support for a TUI; press a to approve, r to reject. You can also auto-approve a fraction with --auto-approve 0.20.
Bake-off score is below threshold
Try a bigger student (--student qwen2.5-14b), a richer teacher (--teacher claude-opus), more captures (--min-captures 5000), or a stricter grader. The bake-off report shows the worst 50 examples; read those first.
Run is slow on Apple Silicon
The CLI uses Metal via llama.cpp on M-series chips. Make sure kolm doctor reports metal: ok. If not, package release pending: reinstall with pip install git+https://github.com/kolm-ai/kolmogorov-stack@main#subdirectory=sdk/python/tap/kolm when the channel lands (the npm build does not bundle Metal kernels).
Verify says signature mismatch
Someone modified the artifact (or your filesystem is doing transparent compression on a non-block-aligned write). Re-download or re-compile. Never run an artifact that fails verification.
WebPush threshold notification never arrives
Push subscriptions are per-browser. Re-enable notifications at /account/notifications. The threshold also fires a webhook if you have kolm webhooks set --on capture.threshold.
I want to start over
Run kolm reset --confirm. Removes ~/.kolm, revokes the local key, clears the keychain entry. Your captures on the server are preserved; pass --also-server to nuke them too.
