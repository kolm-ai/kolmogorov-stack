The 5 foundations

kolm slots into the agentic stack you already run. Tailscale handles addresses. Termius is the shell in your pocket. tmux survives disconnects. A private git repo is the memory layer. Scripts make every flow reproducible. kolm is the AI compiler that fits between these tools, not above them.

$ kolm compile spec.json && kolm sync push artifact.kolm --git ts://repo
$ kolm tail captures --namespace prod  # tmux + Termius + Tailscale

Why these

The agentic stack that ships has been around for a decade. Tailscale (2019), Termius (2013), tmux (2007), git (2005), shell scripts (1971). Unglamorous, load-bearing. What changed in 2026 is that agents do real work on the same infrastructure, and the seam between agent and tools is where the friction lives.

kolm fits the same row. A compiled AI is a file. The file moves over Tailscale. It runs in a tmux session. You version it in git. You invoke it from a script. You tail its captures from Termius. No new abstraction layer.

What kolm does not do

kolm replaces none of these tools. No bundled VPN, no forked multiplexer, no embedded git, no phone client, no shell abstraction. If you already use them, kolm is one more file on disk and a handful of CLI verbs.

The verbs that close the seam are kolm jobs, kolm watch, kolm tail captures, kolm sync, and kolm profile. Each is a thin wrapper over the foundation tool's native interface. Read the CLI reference for the full surface.

If you do not use these tools yet, kolm still works. Defaults run locally with no remote, sessions stay in your shell, profiles are optional, captures stream to stdout. The foundations setup is for teams running multi-box AI, not a prerequisite.

Recipes

One page per tool. Install snippet, exact kolm verbs that pair with it, copy-paste boilerplate. All recipes share the same shape so you can mix and match.

  • Tailscale recipe: tailnet-scoped git mirror, one-command artifact push to every device.
  • Termius recipe: mobile-runnable kolm tail captures + push-button distill from your phone.
  • tmux recipe: four-hour detached compile, reattach across reboots, live log follow.
  • Private git recipe: per-tenant artifact mirror, branch-per-tenant memory layer, receipt-chain in version control.
  • Script-everything recipe: kolmrc + per-machine profiles + a doctor that catches drift before it matters.
