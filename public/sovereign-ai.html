Sovereign AI · on-prem · air-gapped

AI that stays inside your borders.

Sovereign AI has to keep working when hyperscaler access is cut by policy, sanction, or incident. kolm compiles the task into a .kolm artifact that runs on your hardware, inside your perimeter, without a cloud control-plane callback.

02  Air-gapped compile

Air-gapped compile, signed receipts.

The kolm compile server installs as a single container with no internet egress requirement. You pull the base model weights once, mirror them inside your perimeter, and run every subsequent compile on local compute. The receipt chain is signed by your tenant key; verification works offline.

Install

Docker, Compose, or Helm.

One container, one shared secret, zero external dependencies. The kit ships with Dockerfile, docker-compose.yml, and a production Helm chart. No new npm dependencies were added to support it.

Self-hosted kit →
Egress

Zero outbound traffic.

Verifiable with tcpdump or your existing network telemetry. The container does not phone home, does not pull updates, does not stream telemetry. What you install is exactly what runs.

Air-gap recipe →
Receipts

Signed under your key.

Each compile emits a deterministic receipt chain signed by your tenant key. Drift detection, supersession, and K-score axes all work without any cloud round-trip.

03  On-prem from day one

On-prem from day one, not day three hundred.

kolm treats on-prem as the reference deployment. Cloud is an optional convenience on the same code path, not a reduced enterprise edition.

Single binaryServer, orchestrator, and CLI are one container. No external queue, no external KMS required for first install. Bring your own KMS when ready.
HardwareRuns on commodity GPUs (NVIDIA A100, H100, L40S, RTX 4090, AMD MI300X) and CPU-only fallbacks. Apple Silicon supported for developer machines.
Model registryPull the frontier open-weight registry once. Re-export to your local mirror. Compiles never reach back to fetch.
UpdatesManual, signed. You decide when to upgrade. Old artifacts remain reproducible against pinned versions.
Compute placementWherever your hardware lives. On-prem rack, sovereign cloud region, classified enclave, ship at sea, all use the same container.
No SaaS dependencyThe product does not require kolm.ai to be reachable. Licence verification is offline; the licence file is the licence.
04  Localization roadmap

Arabic, Hindi, Mandarin localization.

Sovereign buyers do not want translated documentation. They want models that perform in the working languages of their workforce and citizens. The kolm compile pipeline accepts seeds and evaluations in any language; the registry is being extended with frontier open-weight base models tuned for the sovereign-buyer languages first.

Arabic · AR
العربية

Modern Standard Arabic base head, plus Gulf and Levantine register tuning, for UAE and KSA workloads. Right-to-left and bidirectional input fully supported in the CLI.

Hindi · HI
हिन्दी

Devanagari Hindi head for India deployments, with Tamil, Telugu, Bengali, and Marathi extensions shipping alongside as the Indic registry matures.

Mandarin · ZH
普通話

Simplified Chinese head for Singapore Mandarin-language workloads. Cantonese and Traditional Chinese variants on the same registry track.

05  Procurement

Government procurement model.

Sovereign procurement needs a different path from commercial SaaS. Kolm provides a perpetual-license option, source-availability framework for design partners, and a published price book so buyers can complete first-pass evaluation without a sales cycle. Contracting is handled through an architecture review.

Licence

Perpetual on-prem licence.

Annual or perpetual. One licence per deployment site, no per-call billing, no usage telemetry. Air-gap-friendly licence file with offline verification.
