Components

Status is organized by source-to-proof stage.

The page avoids false live-health claims. It shows the public operational posture for static surfaces and the readiness gates that still limit what Kolm can market as fully shipped.

SurfaceStateDetailRoute
Public site Operational Primary compiler surface. Main pages, product docs, pricing, trust gates, and route-level API reference are available. /
API reference Operational Generated operating surface. Public route inventory is generated from backend route metadata and checked by tests. /docs/api
Control center Mounted Tenant control loop. Channel policy, opaque-event handling, compile state, readiness, and export posture are visible in the account UI contract. /account/api-control-center
Compiler artifacts Gated Artifact proof path. Manifests, examples, evaluators, hashes, target recipes, and receipts are represented; package-release and benchmark gates remain explicit. /platform
Governance exports Mounted Proof outside the UI. Export modes cover governance packets, SIEM/GRC handoff, catalogs, and verifier receipts without exposing secret values. /integrations
Audit evidence Secondary Signed audit evidence remains separate. The audit verifier and signed report workflow stay available on the audit module, but they are not the primary product status model. audit.kolm.ai

A green uptime check does not close public benchmark, certification, package-channel, standards, or external adoption gates. The closeout ledger remains the source of truth for production-final claims.

Readiness gates

Open gates are visible instead of being hidden behind status green.

A serious infrastructure page separates uptime from product maturity. Kolm still treats external partner proof, live certification, public benchmark evidence, and package release as open readiness work.

Implemented controls

Capture routes, product graph, account control-center UI contracts, API reference generation, claim-scope audit, and route inventory checks are represented in source and tests.

Readiness-gated claims

Public copy stays bounded until external partner, live certification, package release, and public benchmark evidence gates have proof in the readiness closeout.

Secret-safe status

Status and public API responses must expose posture, counts, routes, and gates without dumping credentials, provider payloads, or unknown vendor semantics.
