04 / Offline verification

Evidence reports survive any outage.

Every evidence report issued by kolm is signed with an Ed25519 key whose public counterpart is published and pinned. Verification runs on in-browser WebCrypto against that key, so our uptime is never in the trust path. A buyer's security team can confirm the signature, the scope, and the timestamp of any report even when this status page is unreachable.

Verify a report Trust center

Signature

Ed25519, published key

Reports carry a detached Ed25519 signature verifiable against a key pinned at /v1/pubkey. Infrastructure availability does not affect this check.

Log inclusion

Merkle inclusion proof

Each report ships a Merkle inclusion proof against the RFC 6962-style transparency log. The log root is published separately and can be checked offline.

Scope

Cryptographically bound

The agent name, version, capability scope, and audit date are embedded in the signed payload; they cannot be altered without invalidating the signature.

No account

No login required to verify

The public verifier at /verify runs entirely in the browser using the WebCrypto API. No credentials, no session, no server round-trip.
