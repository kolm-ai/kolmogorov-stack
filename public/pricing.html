02 / Billing Contract

The meter follows the lifecycle.

Pricing should make the product object obvious. Kolm charges around controlled API traffic, compile work, target/runtime activity, and enterprise controls because those are the objects that create durable value.

03 / Why This Packaging

The best category position replaces disconnected tools.

The plan structure follows the research: one product should own the path from real API behavior to governed, portable runtime artifacts.

Capture

Behavior before claims

Plans start with routed API traffic, traces, source policy, and controlled capture instead of abstract model access.

Build

Compile credits are explicit

Compile work is visible by tier, so teams know when they are paying for artifact generation instead of hidden infrastructure.

Control

Enterprise features map to objects

Seats, RBAC, namespaces, provider policy, exports, and readiness gates are product controls, not generic add-ons.

Export

Evidence leaves the UI

Artifacts, manifests, receipts, reports, event streams, and procurement packets should stay useful outside the dashboard.

Readiness-Gated Claims

Choose a tier, then prove one controlled loop.

Start with one namespace, capture the behavior, compile the artifact, inspect the receipt, and expand only when the readiness gates support it.
