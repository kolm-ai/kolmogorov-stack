Studio · export wizard

Export your .kolm to any runtime format.

Pick an artifact, pick the runtime your serving stack expects. GGUF for llama.cpp + Ollama. AWQ + EXL2 for vLLM + exllamav2. MLX for Apple silicon. ONNX / CoreML / TFLite / OpenVINO for embedded.

Which artifact?

Pass the artifact ID or path you want to export. Find IDs in /account/artifacts or with kolm artifacts list. Local paths must already exist on this machine.

No artifacts yet? Compile one first.

Which target format?

Pick the runtime you'll serve from. Multiple targets are fine — rerun the wizard for each, or use kolm export <artifact> --target gguf,awq,mlx.

Review the export plan, then confirm.

The planner reports estimated output size + the local command to run. Caveat: the actual export executes locally via kolm export (heavy compute, often GPU-bound). The wizard only writes the plan today.
