The evidence DAG: provenance as a graph.

Every .kolm artifact carries an evidence_dag: {nodes, edges} block. The DAG records the upstream provenance graph behind the artifact: which captures fed which evals, which evals seeded the teacher, which teacher seeded the student, which runtime probe produced the bound passport, which signature chain sealed the bytes. Cycle-free by construction. Spec version kolm-evidence-dag-1.

Schema

Two arrays. Nodes carry an id and a kind from a fixed alphabet; edges carry from, to, and a relationship from a four-verb alphabet.

{
  "spec": "kolm-evidence-dag-1",
  "nodes": [
    { "id": "cap_01", "kind": "capture",  "ts": "2026-05-20T10:00:00Z" },
    { "id": "eval_01","kind": "eval",     "score": 0.91 },
    { "id": "tch_01", "kind": "teacher",  "model": "claude-opus-4-7" },
    { "id": "stu_01", "kind": "student",  "artifact_id": "qwen3-8b-q4km" },
    { "id": "rt_01",  "kind": "runtime",  "target": "rtx-5090-vllm" },
    { "id": "sig_01", "kind": "signature","mode": "ed25519" },
    { "id": "pol_01", "kind": "policy",   "name": "rekor-required" },
    { "id": "rgt_01", "kind": "rights",   "covers": ["cap_01"] }
  ],
  "edges": [
    { "from": "eval_01", "to": "cap_01", "relationship": "validated_by" },
    { "from": "tch_01",  "to": "eval_01","relationship": "derived_from" },
    { "from": "stu_01",  "to": "tch_01", "relationship": "derived_from" },
    { "from": "stu_01",  "to": "rt_01",  "relationship": "validated_by" },
    { "from": "sig_01",  "to": "stu_01", "relationship": "validated_by" }
  ]
}

Node kinds

KindWhat it records
captureA captured input / output pair from the gateway.
evalAn evaluation case plus its judge verdict.
teacherA teacher model rollout that produced training signal.
studentA student model snapshot — the artifact's predecessor.
runtimeA runtime probe (passport row) on a target host.
signatureA signature event (HMAC chain, Ed25519, Sigstore).
policyA policy gate verdict (Ed25519-required, Rekor-required).
rightsA rights / license check (copyright filter, license grant).

Relationships

Direction is FROM dependent TO dependency. Read each edge as a sentence.

VerbReading
derived_fromstudent derived_from teacher — "the student was derived from the teacher".
validated_byeval validated_by capture — "the eval was validated by the capture".
invalidatesrevoke-event invalidates capture — the revoke fan-out.
supersedesB supersedes A — "B replaced A in the canonical position".

buildDag

The constructor validates and indexes the input. Returns a frozen object with hidden adjacency maps; throws with a specific reason on any validation failure rather than coercing the malformed input.

import { buildDag } from './src/evidence-dag.js';

const dag = buildDag({ nodes: [...], edges: [...] });
// Throws on:
//   - nodes / edges not arrays
//   - duplicate node id
//   - edge from/to id not in nodes
//   - self-loop (from === to)
//   - unknown kind or relationship
//   - cycle ('cycle detected (node "X" is part of a loop)')

Cycle detection uses Kahn's algorithm: peel zero-in-degree nodes, decrement neighbours, repeat. If any node is left un-peeled the graph has a cycle and the first stuck id is surfaced in the error message.

trace — walking ancestors

Walks every transitive dependency of a node. Used to answer "what evidence is upstream of this artifact?".

import { trace } from './src/evidence-dag.js';

const result = trace(dag, 'stu_01');
// {
//   node: { id: 'stu_01', kind: 'student', artifact_id: 'qwen3-8b-q4km' },
//   ancestors: [
//     { id: 'tch_01', kind: 'teacher', ... },
//     { id: 'rt_01',  kind: 'runtime', ... },
//     { id: 'eval_01',kind: 'eval', ... },
//     { id: 'cap_01', kind: 'capture', ... }
//   ],
//   edges: [...the edges that connected them...]
// }

descendants — walking dependents

Walks every node that points at the given node via reverse adjacency. The mirror of trace. Used by revoke to compute the needs_review fan-out.

import { descendants } from './src/evidence-dag.js';

const result = descendants(dag, 'cap_01');
// {
//   node: { id: 'cap_01', kind: 'capture', ... },
//   descendants: [
//     { id: 'eval_01', kind: 'eval', ... },
//     { id: 'tch_01',  kind: 'teacher', ... },
//     { id: 'stu_01',  kind: 'student', ... },
//     { id: 'sig_01',  kind: 'signature', ... }
//   ],
//   edges: [...]
// }

revoke — fan-out semantics

Marking a capture as revoked invalidates every eval that was validated_by it, which in turn casts doubt on every student derived_from those evals. revoke() returns a verdict object the caller persists; it never mutates the input DAG.

import { revoke } from './src/evidence-dag.js';

revoke(dag, 'cap_01');
// {
//   revoked: ['cap_01'],
//   needs_review: ['eval_01', 'tch_01', 'stu_01', 'sig_01']
// }

revoke(dag, 'unknown_id');
// { revoked: [], needs_review: [], error: 'unknown_node:unknown_id' }

Storage

The DAG rides INSIDE the artifact manifest but is NOT bound into artifact_hash_input. Same pattern as runtime passports: the provenance graph is an operational fingerprint that a re-prover can legitimately re-walk and re-emit (for example, adding a new validated_by edge after an eval re-run) without invalidating the receipt chain.

Persistence helpers live in src/evidence-store.js:

import { writeEvidenceDag, readEvidenceDag, findArtifactForNode } from './src/evidence-store.js';

writeEvidenceDag(artifactId, dag);
const dag = readEvidenceDag(artifactId);
// Reverse index: given a node id, which artifact carries it?
const artifactId = findArtifactForNode('cap_01');
Caveats. The DAG is data, not proof. A signature node only proves the bytes were signed at some point; it does not prove the signer was authorized to sign on behalf of the workspace. Pair the DAG with a Rekor inclusion proof when an external verifier requires public log presence.
