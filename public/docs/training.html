Training

Training in kolm.ai is plan-first. You hand the planner a frozen dataset, the planner detects the task kind from row shape, picks a path profile, picks a backbone, prints the GPU-minute budget and the expected score, and waits for your approval. Then it runs.

The plan command

kolm training plan ds_018e...
# task detected: classification (3 classes, balanced)
# path profile: distill-small
# backbone: qwen-0.5b-instruct
# budget: 22 GPU-min on a single 3090
# expected k: 0.94 +/- 0.02 on holdout
# expected cost: $0.31 amortized over 10k inferences
# ---
# approve with: kolm training run ds_018e... --plan plan_018f...

Task detection

The planner inspects the dataset rows to pick a task kind.

TaskDetected from
redactionVAR_* placeholders dominate the response; response is a redacted echo of input
classificationFinite output set, ≤ 100 distinct labels, short responses
extractionJSON responses conforming to a stable schema
generationFree-form responses, variable length, no detectable schema

Path profiles

A path profile bundles the recipe pipeline: data prep, augmentation, training loop, eval gate, export format. Profiles are matched to task kind + hardware tier.

ProfileUse whenOutput
recipe-only< 1000 rows, narrow taskRule + prompt template, no weights
distill-small1k-10k rows, classification or extractionLoRA on small backbone, < 1 GB
distill-medium10k-100k rows, generationLoRA on medium backbone
specialist100k+ rows, high-value taskFull fine-tune, quantized

Backbone selection

The planner picks the smallest backbone that hits the holdout score target. You can override.

kolm training plan ds_018e... --backbone gemma-3n-e2b
kolm training plan ds_018e... --tier 3090
kolm training plan ds_018e... --max-budget-min 60

Run

kolm training run ds_018e... --plan plan_018f...
# job_018g ... running
# epoch 1/3 step 142/420 loss=0.51 val_k=0.88
# epoch 2/3 step 280/420 loss=0.34 val_k=0.92
# epoch 3/3 step 420/420 loss=0.28 val_k=0.94
# holdout: k=0.943 latency_p50=22ms cost_per_1k=$0.03
# artifact ready: ~/.kolm/artifacts/summarizer-v1.kolm

Resume

Training jobs are checkpointed every epoch. Interrupted runs resume from the last checkpoint.

kolm training resume job_018g

Next steps

Hand the artifact to distillation for packaging, evals for the bakeoff, and runtime to swap live traffic onto it. Start from /docs/quickstart if you have not built a dataset yet.
