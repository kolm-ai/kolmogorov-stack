Prometheus and Grafana

Every kolm runtime exposes a /metrics endpoint in Prometheus text format. Six pre-registered metrics cover capture rate, load-queue depth, GPU memory by tier, kernel selection, speculative-decoding acceptance, and HTTP latency. Drop the bundled dashboards/kolm-runtime.json into Grafana and you have a working observability stack in two minutes.

Scrape config

Add a job to your prometheus.yml. The endpoint speaks text/plain; version=0.0.4 so Prometheus 2.x and 3.x both scrape it natively without exporters.

scrape_configs:
  - job_name: 'kolm'
    metrics_path: '/metrics'
    scrape_interval: 15s
    static_configs:
      - targets: ['kolm.example.com:443']
    scheme: https
    # Optional: if you set KOLM_METRICS_BEARER on the kolm side, send the
    # matching bearer here so 401s don't pollute your scrape errors.
    authorization:
      type: Bearer
      credentials: 'your-shared-bearer-token-here'

To gate the endpoint, set KOLM_METRICS_BEARER=<random-token> on the kolm process. The honest dev-default is no auth so local curl localhost:8080/metrics just works. On a missing or mismatched bearer the endpoint returns HTTP 401 with {ok:false, error:'invalid_metrics_bearer'} so a misconfigured scraper fails loud.

Available metrics

Six metrics are pre-registered at boot. They emit # HELP and # TYPE lines even when no samples have been written yet, so a fresh scrape never returns a blank page.

MetricTypeLabelsHelp
kolm_capture_total counter tenant, namespace Total capture events written to the event store.
kolm_load_queue_depth gauge — Current depth of the W729 load queue (artifacts waiting to be paged in).
kolm_runtime_gpu_memory_bytes gauge device, tier GPU memory usage in bytes by device and W724 tier (vram, ram, nvme, network).
kolm_runtime_kernel_selected_total counter profile_name Count of W726 kernel-selector profile selections (latency vs batching).
kolm_accelerate_acceptance_rate histogram task_class W727 speculative-decoding acceptance rate distribution by task class.
kolm_http_request_duration_seconds histogram method, route, status HTTP request duration in seconds.

You can also list the registered shape from the CLI:

kolm metrics list
# Prints JSON with every metric's name, type, label names, and bucket array.
kolm metrics show
# Pipes renderMetrics() to stdout. Useful for cron + log shipping.

Grafana dashboard import

The repo ships a ready-to-import Grafana dashboard at dashboards/kolm-runtime.json. It targets schema v38 (Grafana 10) and contains six panels — one per metric above.

  1. In Grafana, open Dashboards → New → Import.
  2. Upload dashboards/kolm-runtime.json (or paste the contents).
  3. Pick your Prometheus datasource from the dropdown. The dashboard uses ${DS_PROMETHEUS} as the datasource UID variable so any Prometheus instance works.
  4. Save. The dashboard refreshes every 30 seconds and the default time window is now-1h.

Panels

Panel 1

Capture rate (per minute)

rate(kolm_capture_total[1m]) sliced by namespace. Watch the green line trend up after you wire a new SDK; sustained drops to zero are an upstream outage signal.

Panel 2

Load queue depth

kolm_load_queue_depth as a gauge with thresholds at 5 (orange) and 20 (red). Sustained non-zero values indicate the runtime is GPU-bound or NVMe-bound and you should provision more capacity.

Panel 3

GPU memory by tier

kolm_runtime_gpu_memory_bytes grouped by device + tier. Stacked area chart so you can see VRAM fill before it spills to RAM, then to NVMe.

Panel 4

Kernel selection histogram

sum by (profile_name) (rate(kolm_runtime_kernel_selected_total[5m])). Bars per second showing how often the W726 selector chose latency vs batching. Drift toward batching means your traffic mix is shifting toward API-serving workloads.

Panel 5

Acceleration acceptance rate

histogram_quantile(0.50, rate(kolm_accelerate_acceptance_rate_bucket[5m])) by (task_class). Median speculative-decoding acceptance per task class. Values approaching 1.0 mean the speculator's drafts are nearly always accepted — the dream case.

Panel 6

HTTP p95 latency

histogram_quantile(0.95, rate(kolm_http_request_duration_seconds_bucket[5m])) by (route). Tail latency per route, for SLO conformance. Spikes localized to one route usually point to an upstream-provider regression.

Honesty contract

  • A registered metric with zero samples still emits HELP and TYPE lines — this is Prometheus-correct (the scraper reports absent() and the metric stays in the registry).
  • Histogram observations are non-negative finite numbers only. NaN, Infinity, and negatives raise at the callsite, not silently corrupt the wire format.
  • Label values are escaped for backslash, double-quote, and newline so a malicious or careless namespace name can never break a sample line.
  • The exporter version is pinned by PROMETHEUS_EXPORTER_VERSION (currently w730-v1) so a scrape can detect a schema migration.
  • The renderer is PURE — no I/O, no timers, no auto-scrape. Tests reset sample state via _resetForTests(); production never calls it.
