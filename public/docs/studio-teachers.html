Studio / Teachers

A council, not a single teacher.

Studio supports a council of teachers, not a single one. Mixing weighted responses from two or three frontier models gives the student a more robust prior than any one teacher can — and it makes the artifact less brittle if any single teacher drifts in a future version. Every teacher call goes through the same provider adapter the Wrapper uses, so credentials and rate limits are shared.

Why a council

A single teacher leaks two things into the student: its style and its blind spots. A council averages out style and dilutes blind spots. The W869 Trinity-500 ablation showed:

  • Claude-only student: 91.2% asks-first-question, but every response opened with "Of course! Let me help you with that.".
  • GPT-only student: 94.1% asks-first-question, but hallucinated order IDs.
  • 3-way council (Claude 0.45 + GPT 0.30 + DeepSeek 0.25): 96.5% asks-first-question, no signature phrasing, no hallucinated IDs.

How to pick a council

  1. One strong reasoner — pick whoever scores highest on the eval axes you care about. Currently Claude Opus 4.7 or GPT-5.
  2. One fast-and-cheap — Claude Haiku 4.5 or GPT-4o-mini. Provides the bulk of pairs.
  3. One different-lineage — DeepSeek V3 or Gemini 2.5 Pro. Keeps the student from inheriting only one lab's quirks.

Weights normally split 0.45 / 0.30 / 0.25 — biased toward the strongest reasoner but never below 0.20 for the diversity teacher.

Adding a custom provider

Custom providers go in ~/.kolm/providers/<name>.js. The adapter must export generate({prompt, system, max_tokens, temperature, stop}) returning {text, usage:{prompt_tokens, completion_tokens}, raw_response}.

// ~/.kolm/providers/my-internal-llm.js
module.exports = {
  id: 'my-internal-llm',
  models: ['internal-7b', 'internal-70b'],
  async generate({ model, prompt, system, max_tokens=1024, temperature=0.2, stop=[] }) {
    const res = await fetch(`https://internal.example/v1/chat/completions`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${process.env.INTERNAL_KEY}` },
      body: JSON.stringify({
        model, max_tokens, temperature, stop,
        messages: [{ role:'system', content:system }, { role:'user', content:prompt }],
      }),
    });
    const j = await res.json();
    return {
      text: j.choices[0].message.content,
      usage: { prompt_tokens: j.usage.prompt_tokens, completion_tokens: j.usage.completion_tokens },
      raw_response: j,
    };
  }
};

Once dropped in, the provider is auto-discovered — kolm distill --list-teachers will show my-internal-llm/internal-7b next to the built-ins. Reference it in spec.toml as { id = "my-internal-llm/internal-7b", weight = 0.5 }.

Caveats. Every teacher you list in spec.toml is named in the resulting artifact passport — the deployment target sees the full council. If your distill runs against a provider that contractually forbids redistribution of its outputs in trained weights, that's your responsibility to clear. See teacher source disclosure.

Rate limits & retries

The teacher adapter respects each provider's published rate limit and applies exponential backoff on 429. Concurrency defaults to 4 in-flight requests per teacher; raise with --teacher-concurrency 8 if your tier supports it. Failed pairs are queued; the distill driver will retry up to 3 times before dropping the pair and noting it in the receipt.
