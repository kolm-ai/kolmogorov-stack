VS Code extension.

Not Kolm therapeutics, Kolm band, or other unrelated organizations using similar names.

A passive monitor that lives in your editor. It watches which Copilot, Cursor, and Claude Code suggestions you actually accept, clusters them by intent, and routes the matching ones to a local distilled student once you compile. The boring suggestions stop costing you teacher tokens.

Privacy contract: captures stay on disk in ~/.kolm/captures/ and never leave the machine unless you flip an opt-in in the settings panel. There is no telemetry, no anonymous usage ping, and no auto-upload. Stamped w819-v1.

code --install-extension kolm.kolm-vscode

Passive monitor

The extension subscribes to the suggestion-acceptance events emitted by other code-completion extensions and writes them to the local kolm capture log:

SourceEvent hookWhat we capture
GitHub Copilotgithub.copilot.didAcceptCompletionItemprefix, accepted text, file language, cursor position
Cursorcompletion-acceptance webhook (Cursor's VS Code build)prefix, accepted text, file language
Claude Codeanthropic.claude-code.didAcceptSuggestionprefix, accepted text, file language, tool name (if applicable)
Inline tab-complete (built-in)vscode.commands.executeCommand("editor.action.inlineSuggest.commit")prefix and accepted text

What we deliberately do NOT capture: the surrounding file contents, the project name, the git remote, the API key the upstream tool used, or any environment-variable values. The capture is intentionally narrow.

Pattern detection

Once enough captures exist for a namespace, the extension runs a local clustering pass to surface repetition patterns. The dominant clusters today:

  • Boilerplate. Imports, license headers, the standard React component skeleton, the standard Express route signature.
  • Tests. Mocha / Jest / pytest / Go-test scaffolds, parametrize blocks, mock setup.
  • Docstrings. JSDoc, Python docstring, Rustdoc paragraphs. Same shape over and over per project.
  • Error handling. Try / catch wrappers, error-result types, defer-cleanup blocks.
  • Logging. Structured log calls with a recurring schema.

Clusters are visible in /account/repeated-workflows. Once a cluster crosses the distill-ready threshold, the extension highlights it in the status bar with a one-click "Distill this pattern" action.

Status bar

A single chip in the VS Code status bar shows:

LabelMeaning
kolm 0Extension active, no captures yet in this workspace.
kolm 4747 captures in this workspace's namespace.
kolm 47 ↑ distillA cluster has crossed the distill-ready threshold; click to launch.
kolm · routingA compiled artifact is serving suggestions for this language; teacher calls are being avoided.
kolm · offThe extension is paused via the command palette. No captures, no routing.

Click the chip to open the extension's panel; right-click to pause, change namespace, or open the local kolm dashboard.

Post-distill routing

Once a cluster is compiled into a .kolm artifact, the extension routes the matching completions to the local student via an in-process runtime. The teacher is bypassed for that pattern; the upstream completion tool sees the student's response as a "system-suggested" completion item.

The routing decision is logged like any other kolm routing event: which artifact handled it, the K-Score for that artifact, the model-class budget, and the cost saved. View the rollup at /account/routing.

If the student fails to produce a response within the latency budget (default 80ms), the extension falls through to the teacher silently. The user sees no degradation; the rollup shows a route_kind:'fallback' row so the operator knows the local artifact missed.

Settings panel

The extension exposes five settings under Cmd/Ctrl + , › "kolm":

SettingDefaultEffect
kolm.captureThreshold20Minimum cluster size before the status bar prompts to distill.
kolm.teacherPreferenceautoOne of auto, anthropic, openai, local. auto picks the cheapest qualifying teacher.
kolm.namespacevscode-defaultNamespace under which this workspace's captures land. Per-workspace override possible via .kolm-namespace file at the project root.
kolm.routeLatencyBudgetMs80How long the local student has to respond before the extension falls back to the teacher.
kolm.shareCapturesWithKolmfalseOpt-in to anonymised cluster sharing for the kolm benchmark. Off by default. Toggling on logs a durable audit row.

Privacy note

Captures are stored in ~/.kolm/captures/<namespace>/<date>.jsonl. They include the accepted text and a short prefix snippet for clustering. They do NOT leave the machine unless kolm.shareCapturesWithKolm is set to true. The default is false.
  • The extension does not run a background network connection of its own. The only network traffic from VS Code through this extension is the explicit, user-triggered kolm compile call when you click the "Distill this pattern" action.
  • Audit-log every routing decision via /account/audit-log. If anything routes that you did not expect, the log shows the artifact, the namespace, and the timestamp.
  • To wipe captures: run kolm capture purge --namespace vscode-default from the terminal. The extension prompts to reconfirm before any irreversible delete.
