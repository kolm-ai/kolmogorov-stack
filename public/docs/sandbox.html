Docs / Sandbox

kolm.ai · sandbox spec

The constrained runtime for .kolm artifacts.

Recipe code inside a .kolm bundle runs inside a sandbox: no process, no require, no eval, no outbound sockets. The verifier rejects any artifact whose recipe code references a forbidden identifier.

Forbidden identifiers

  • process, process.env, process.exit.
  • require, import(), dynamic module resolution.
  • eval(, Function(, new Function.
  • Direct file-system access: fs, node:fs.
  • Outbound network: fetch, http, https, net, dgram.
  • Child processes: child_process, node:child_process.

What the sandbox allows

  • Pure deterministic transforms on prompt and response strings.
  • The kolm.* host API: kolm.score(), kolm.replay(), kolm.embed().
  • Use of the bundled model, tokenizer, and frozen-eval set.

Verify before bundling

Run kolm verify on any recipe before shipping. The verifier scans the recipe AST against the forbidden-identifier list and rejects anything that would not pass at runtime.
