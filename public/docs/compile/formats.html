Compile

Export formats.

A signed .kolm ships the BF16 weights plus a manifest. kolm export projects those weights into the quantization format your serving runtime expects. The choice is dictated by accelerator (CUDA / Metal / CPU) and runtime (vllm / llama-cpp / mlx / trt-llm), not by personal preference.

Usage

kolm export art_a1b2c3d4 --format gguf-q4_k_m --output ./out/
kolm export art_a1b2c3d4 --format awq-int4   --output ./out/
kolm export art_a1b2c3d4 --format mlx-int4   --output ./out/

# Multi-format build (parallel where the calibration set is shared):
kolm export art_a1b2c3d4 --format gguf-q4_k_m,gguf-q5_k_m,gguf-q8_0,awq-int4 --output ./out/

Calibration

Activation-aware formats (AWQ, GPTQ, EXL2) need a calibration set. By default kolm export samples 256 captures from the same namespace the artifact was distilled from, which keeps the quantization aligned with production traffic. Override with --calibration <file> for offline or air-gapped builds.

Verifying quality after quantization

kolm eval art_a1b2c3d4 --format awq-int4 --rubric trinity
# k-score: 0.871 BF16
# k-score: 0.864 AWQ-INT4  (delta -0.7%, within floor)
# OK to ship

If the post-quant k-score drops below the artifact's floor, the export is marked quality_regressed and is not auto-published. Re-export at a higher bit width or pick a different format.
