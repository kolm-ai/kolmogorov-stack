kolm.ai / docs / copyright-scan

Open-source AI workbench

Frontier AI on your own infrastructure.

Copyright scan is a heuristic copyright detector for captured prompts and responses. It runs a small pattern pack — well-known Disney character names, Top-100 song-title n-grams, code copyright headers, and SPDX license lines — and stamps a structured copyright_heuristic:<categories> reason on any staged capture that crosses the risk threshold. Same envelope is returned by POST /v1/copyright/scan, GET /v1/copyright/queue/:namespace, and the kolm copyright-scan CLI path. Version stamp: w750-followup-v1.

Not legal advice. The heuristic flags content that looks like it may contain copyrighted material. A hit is a signal for human review, not a confirmed violation. The pattern packs are public knowledge (character names, song titles), not the protected works themselves. We do not bundle lyrics, prose, or licensed code. Use the result as one input to your own clearance process.

Local-only. Scans are 100% in-process. No text is sent to any external service, no third-party content classifier, no telemetry. The pattern packs ship as frozen arrays in src/copyright-detector.js — nothing leaves your machine.

Why a heuristic instead of an ML classifier

A full content classifier (BERT-style copyright detector, fingerprint database lookup, dataset-membership inference) is a real product on its own and lives in a worker package, not the proxy hot path. The heuristic here is the lightweight in-process scanner that runs on every capture without adding network or model load. It catches the cases that scream "this might be copyrighted" — a model output that begins with Copyright (c) 2024 Microsoft Corporation, a chat transcript that references Mickey Mouse, a code snippet that carries SPDX-License-Identifier: GPL-3.0-only. These are decision-quality signals that justify human review; they are not the end of the analysis.

The complementary scanner is W708-4 capture-copyright-filter in src/capture-copyright-filter.js: paywall boilerplate (Subscribe to read more, All rights reserved), [Verse 1]-style lyrics-site scrapes, and high-comma-density prose. W708-4 targets ingester content shape; the W750-followup detector here targets fingerprint-shaped content. Both run before the row reaches the canonical observations table, and a row can be flagged by either, both, or neither.

Pattern catalogs

Disney character names (DISNEY_NAMES)

~30 well-known names matched case-insensitively as literal substrings:

mickey mouseminnie mousedonald duckdaisy duckgoofyplutoelsaannaolafsvensimbamufasatimonpumbaaarielsebastianflounderaladdinjasminegeniesnow whitecinderellaaurorabellebeasttianarapunzelflynn ridermeridamoanamaui

Disney character names are public knowledge but their appearance in a captured model input or output may signal user-generated derivative content. A hit is informational on its own; a hit combined with a song-title hit OR a code-copyright hit gets the row above the default 0.5 risk threshold and into the quarantine queue.

Lyric fingerprints (LYRIC_FINGERPRINTS)

~30 recognizable Top-100 song titles and short n-grams. The list is heuristic content — we ship the titles, not the lyrics themselves:

Era / genreExamples
Classic rock / pophey jude, let it be, imagine all the people, bohemian rhapsody, sweet caroline, born to run, livin on a prayer, dont stop believin
Pop / R&B canoni will always love you, thriller michael jackson, billie jean, beat it, baby one more time
2010s poprolling in the deep, someone like you, shape of you, blank space, shake it off, old town road
2020s popflowers miley, as it was harry, watermelon sugar, levitating dua, dynamite bts, butter bts
Grunge / altsmells like teen spirit, toxic britney
Other recentdespacito luis, closer chainsmokers, sorry justin bieber

The pattern is title or short title-n-gram, not lyric reproduction. The presence of "Hey Jude" in a capture is the signal — we don't need (and don't include) the rest of the song.

Code copyright headers (CODE_COPYRIGHT_REGEX)

Matches Copyright (c) 2023 Acme, Copyright (C) 2023-2025 Foo, copyright © 2024 Bar, Copyright &copy; 2025 Baz. Requires a 4-digit year followed by at least one letter so we don't flag bare "copyright 2024" boilerplate that appears in a sentence about a model's training cutoff.

// Matches: "Copyright (c) 2023 Acme Corp"
//          "Copyright (C) 2023-2025 Foo"
//          "copyright © 2024 Bar"
//          "Copyright &copy; 2025 Baz"
const CODE_COPYRIGHT_REGEX = /\bcopyright\s*(?:\([cC]\)|©|&copy;)?\s*\d{4}(?:-\d{4})?\s*[a-zA-Z]/;

SPDX license identifier (SPDX_REGEX)

Matches the canonical SPDX-License-Identifier: <name> comment line. Code that carries an SPDX identifier almost always has a real license attached that the model may not be allowed to reproduce verbatim:

const SPDX_REGEX = /SPDX-License-Identifier:\s*[A-Za-z0-9.+-]+/;

Risk score

Each hit contributes 0.25 to the risk score, capped at 1.0. Four hits saturate the score; one hit gives 0.25, two give 0.5, three give 0.75. The default quarantine threshold is 0.5 — two hits are required before the staged row gets a copyright_heuristic: reason. Tune via opts.threshold in code or via the POST body in the API.

OutcomeRisk scoreHitsUI signal
Clean0.000clean
One hit (informational)0.251informational
Two hits (default flag)0.502review
Three+ hits (high signal)0.75+3+quarantine

W808 staged_captures integration

W750 (copyright filter + capture quarantine) was MERGED into W808 in the 2026-05-24 dup-cleanup. The capture-quarantine half shipped under W808: the staged_captures table, /account/captures/review.html manual-review inbox, and cmdCapturesReview --release/--quarantine. This W750-followup is the remaining slice — the copyright-classifier — promoted to a single follow-up item that hooks into the same pipeline as a post-quarantine classifier.

The wiring lives in src/capture-store.js insertCapture, immediately after the existing W708-4 attachCopyrightFlag call. When classifyForQuarantine returns should_quarantine:true, the insert path:

  1. Appends copyright_heuristic:<categories> to row.flag_reason (semicolon-joined if a prior reason existed)
  2. Stamps row.copyright_heuristic_flagged = true
  3. Stamps row.copyright_heuristic_risk = <risk_score>
  4. Stamps row.copyright_heuristic_hits with the matched kinds + values

The W808 staged_captures table picks up all four fields when the staged copy is taken (because we mutate the row in place before insertStagedCapture reads it). The manual-review UI surfaces them next to the existing anomaly_flagged + 3σ metrics so an operator can release, block, or accept the row with both signals visible.

Gating. The detector is enabled by default. Set KOLM_W750_COPYRIGHT_DETECTOR=off to skip it entirely — the W808 happy-path becomes byte-stable, no flag_reason mutations, no sidecar fields. Useful when an operator wants the heuristic disabled without redeploying.

Envelope shape

Same envelope is returned by POST /v1/copyright/scan and the kolm copyright-scan text CLI path. Field stability is part of the contract.

{
  "ok": true,
  "hits": [
    { "kind": "disney_character", "matched": "mickey mouse", "index": 12, "side": "input" },
    { "kind": "spdx",             "matched": "SPDX-License-Identifier: MIT", "index": 0, "side": "output" }
  ],
  "risk_score": 0.5,
  "scanned_chars": 1024,
  "version": "w750-followup-v1"
}

The GET /v1/copyright/queue/:namespace envelope returns the captures that carry a copyright_heuristic: tag in their staged_captures.flag_reason, newest first:

      "staged_capture_id": "stg_abc123",

      "quarantine_state": "pending",
      "flag_reason": "copyright_heuristic:disney_character,spdx",

API + CLI surface

SurfaceInvocation
CLI (literal text)kolm copyright-scan text "<text>"
CLI (persisted capture)kolm copyright-scan capture <capture_id>
CLI (queue listing)kolm copyright-scan queue <namespace>
API (text or capture_id)POST /v1/copyright/scan body {text} or {capture_id}
API (queue)GET /v1/copyright/queue/:namespace
Manual review inbox/account/captures/review

CLI exit codes are honest: 0 when the scan finds no hits, 3 when hits are found (informational — the operator should review), 2 on real errors (auth missing, bad arguments, network failure). The hits-found path is intentionally not exit-1 so CI can keep "review needed" separate from "build failed".

Quickstart

# Local scan with no auth (text is never sent off your machine)
kolm copyright-scan text "Copyright (c) 2024 Acme. SPDX-License-Identifier: MIT"
# → {"ok":true,"hits":[{"kind":"code_copyright",...},{"kind":"spdx",...}],
#    "risk_score":0.5, ...}
# exits 3 (hits found, informational)

# Look up the verdict for a staged capture
kolm copyright-scan capture stg_abc123

# List the copyright-flagged staged captures for a namespace
kolm copyright-scan queue support

Pair this with drift-alert for production-vs-training distribution drift, and with the W708-4 paywall-shape detector that runs in the same insert path. See quickstart for the broader capture pipeline.
