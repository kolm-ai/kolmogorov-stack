Deploy to Cerebras.

Cerebras Cloud Inference runs llama / qwen / deepseek-distill on wafer-scale CS-3 silicon at ~10-20× the per-token latency of typical GPU clouds. kolm treats it as a managed-inference deploy target: you bind a namespace to a pre-loaded Cerebras model id, and kolm gateway dispatch routes through api.cerebras.ai instead of vLLM, Ollama, or a local kolm runtime.

What "deploy" means here. Cerebras pre-loads a fixed model catalog — you cannot upload custom weights or LoRA adapters (no public adapter-upload API as of 2026-05). A kolm artifact is "deployed to Cerebras" by binding its namespace to the closest matching pre-loaded model id and routing inference traffic to /v1/chat/completions. Training, compile, and quantization stay on your own GPU or on a training-class cloud (RunPod, Modal, Together).

Prerequisites

  • An account at cloud.cerebras.ai (free tier is sufficient for verification).
  • An API key minted at cloud.cerebras.ai/platform/credentials.
  • kolm CLI ≥ v1.0.0.
export CEREBRAS_API_KEY=csk-...
# (or) export KOLM_CEREBRAS_TOKEN=csk-...
kolm test cloud --provider cerebras   # one-shot reachability check

Bind a namespace

# 1. probe the catalog
kolm cloud cerebras list-models

# 2. bind your namespace to a model
kolm cloud cerebras bind \
  --namespace support \
  --model llama3.1-8b \
  --artifact support-2026-05.kolm \
  --max-tokens 2048 \
  --temperature 0.7

# 3. verify the route
kolm cloud cerebras chat --namespace support --prompt "ping"

The binding is persisted to ~/.kolm/cerebras-bindings/<namespace>.json. From that point on, kolm gateway dispatch --namespace support routes to Cerebras instead of the local kolm runtime.

Or use kolm deploy --device

If you've registered a Cerebras device in your fleet (kolm devices add prod-cerebras --type cerebras), kolm deploy uses the same device-adapter contract as RunPod, Modal, or SSH hosts:

kolm devices add prod-cerebras --type cerebras \
  --config '{"cerebras_model":"llama3.1-8b","namespace":"support"}'

kolm deploy artifact.kolm --device prod-cerebras
# emits ok:true + deployment_id + endpoint + next_actions

  --output-tokens 500000
# estimated_cost_usd: $0.20

The CLI returns a quote; the authoritative invoice is at cloud.cerebras.ai/platform/usage. kolm savings --namespace support --period 30d reads the local capture ledger and computes per-call token spend independent of Cerebras invoicing.

Caveats

  • Inference only. No training, no compile, no LoRA upload. If you need to deploy a custom-trained student, distill on RunPod/Modal/Together first and serve the GGUF locally or via vLLM.
  • Fixed catalog. The catalog changes; list-models queries live before every bind. We never treat the bundled snapshot as authoritative.
  • Streaming. Cerebras supports SSE on /v1/chat/completions; the kolm gateway streams through transparently via the OpenAI-compatible adapter (src/compute/backends/cerebras.js).
  • No public usage endpoint. Cerebras does not expose a per-tenant usage API as of 2026-05. kolm cloud cerebras chat surfaces the server-reported time_info on every completion for per-call tok/s telemetry; aggregate spend tracking lives in the local cost-tracking ledger (src/cost-tracking.js).
