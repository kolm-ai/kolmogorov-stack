Receipt format

The kolm-audit-1 schema, field by field.

The full reference for the 19-field kolm-audit-1 receipt schema plus the Ed25519 signature block. Every field is required unless marked optional; every type is documented; every constraint is enforced at signing time.

Complete example

{
  "schema": "kolm-audit-1",
  "receipt_id": "bafy2bzaceabcdefghijklmnopqrstuvwxyz1234567890",
  "timestamp": "2026-05-20T14:32:01.847Z",
  "namespace_id": "customer-support",
  "route_decision": "teacher",
  "provider": "openai",
  "model": "gpt-4o",
  "artifact_id": null,
  "confidence": 0.82,
  "fallback_reason": null,
  "input_hash": "sha256:9c8b7a6f5e4d3c2b1a09f8e7d6c5b4a39281706f5e4d3c2b1a09f8e7d6c5b4a3",
  "output_hash": "sha256:7d6c5b4a39281706f5e4d3c2b1a09f8e7d6c5b4a39281706f5e4d3c2b1a09f8e",
  "capture_eligible": true,
  "capture_id": "cap_01HXYZABCDEFGHJKMNPQRSTVWX",
  "redaction_applied": "redact_captures",
  "input_tokens": 142,
  "output_tokens": 488,
  "cost_usd": 0.00834,
  "signing_key_id": "tenant_acme_v3",
  "verify_url": "https://verify.kolm.ai/?cid=bafy2bzaceabc...",
  "signature_ed25519": {
    "alg": "Ed25519",
    "key_id": "tenant_acme_v3",
    "value": "base64url:MEUCIQDx7s...kQ"
  }
}

Field reference

FieldTypeRequiredNotes
schemastringYesAlways "kolm-audit-1" for this version.
receipt_idstring (CID)YesContent-addressed id; multibase-base32 of SHA-256 over the unsigned body.
timestampRFC 3339 UTCYesPipeline completion time; millisecond precision.
namespace_idstringYesRouting scope that handled the call.
route_decisionenumYespinned / fallback / student / teacher.
providerstringYesProvider slug that handled the call.
modelstringYesModel identifier returned by the provider.
artifact_idstring | nullYesLocal .kolm artifact id if applicable; null for hosted providers.
confidencefloat [0,1] | nullYesStudent first-token softmax confidence; null when not confidence-routed.
fallback_reasonstring | nullYesWhy the fallback chain advanced; null when primary succeeded.
input_hashstringYesSHA-256 of canonical input body, prefixed sha256:.
output_hashstringYesSHA-256 of canonical output body, prefixed sha256:.
capture_eligibleboolYesWhether the namespace policy allows this call to be captured.
capture_idstring | nullYesULID of the capture row; null when capture_eligible is false.
redaction_appliedenumYesnone / detect_only / redact_captures / redact_all.
input_tokensint ≥ 0YesInput token count, provider-reported when available.
output_tokensint ≥ 0YesOutput token count, provider-reported when available.
cost_usdfloat ≥ 0YesComputed from the published rate card at signing time.
signing_key_idstringYesKey id used for the Ed25519 signature; resolves at /v1/keys/<id>.
verify_urlstring (URL)YesPublic verifier URL for this receipt CID.

The signature block

The signature is computed over the canonical serialization of every field above (signature itself excluded). The block sits as the last top-level key.

SubfieldTypeNotes
algstringAlways "Ed25519" for kolm-audit-1.
key_idstringSame as signing_key_id; duplicated for verifier convenience.
valuestringbase64url-encoded 64-byte signature, prefixed base64url:.

Canonical serialization

Before signing, the receipt body is canonicalized: keys sorted lexicographically at every level, no whitespace, UTF-8 NFC, integers as base-10 without leading zeros, floats as RFC 8259 grisu3. The verifier re-canonicalizes when validating so the on-disk representation can be pretty-printed without breaking the signature.

Constraints enforced at signing

  • receipt_id matches the CID of the unsigned body.
  • capture_id is non-null if and only if capture_eligible is true.
  • confidence is non-null if and only if route_decision is student or teacher (i.e. confidence routing was active).
  • fallback_reason is non-null if and only if route_decision is fallback.
  • cost_usd reconciles with (input_tokens × input_rate) + (output_tokens × output_rate) against the rate card at timestamp.

Field omission and forward compatibility

New fields added in future schema versions are written as additional top-level keys. Verifiers that target kolm-audit-1 ignore unknown fields; verifiers that target a newer schema reject older receipts only when explicitly configured to do so. The schema name is the contract.

Caveats. The receipt body is <1 KB on average. The signature block adds ~96 bytes. Receipts are stored uncompressed in the capture lake; compression happens at the storage-backend layer (Postgres TOAST, S3 gzip on object PUT). The verify URL points at the hosted verifier; for air-gapped environments, run a private verifier and override verify_url via [signing] verify_url_template in gateway.toml.
