Runtime

Runtime is what happens at request time. The proxy walks a deterministic decision chain on every call: privacy gate, cache lookup, local artifact lookup, cheaper-model swap, frontier fallback. You pick the policy that matches your risk tolerance. The runtime records every decision so you can audit and revert.

The 4 runtime policies

PolicyBehavior
local_firstTry local artifact, fall back to cheaper model, fall back to requested frontier model. Default.
frontier_firstAlways call the requested model. Local artifacts are scored offline but never serve traffic.
cost_optimizedTry cache, try local artifact, try cheaper model, fall back to requested frontier. Aggressive.
privacy_onlyOnly call local artifacts. Block anything that would require a network call. Strict.

The decision chain

Every request walks this chain in order. The first stage that produces a confident response wins. Every decision is recorded in privacy_actions_json and artifact_id on the lake row.

  1. Privacy. Run the 17 detectors. Apply allow / redact / block / override from the policy. If the result is block, return 451 immediately.
  2. Cache. Hash the redacted prompt. If a cache row exists in the window, return it.
  3. Local artifact. Look up artifacts registered for this namespace. Score the prompt against the artifact's seeds. If above the confidence threshold, run locally and return.
  4. Cheaper model. If a cheaper-model routing rule fires for this template, rewrite the upstream model id.
  5. Frontier. Forward to the originally requested upstream model. Capture as usual.

Register an artifact

kolm runtime register ./summarizer.kolm --namespace summarizer --policy local_first
# artifact summarizer-v1.kolm bound to namespace=summarizer
# next call to /summarize will try local first

Set the policy

# Default policy for this machine
kolm runtime policy set local_first

# Per-namespace override
kolm runtime policy set privacy_only --namespace medical-claims

# Inspect
kolm runtime policy show
# default local_first
# namespaces:
# medical-claims privacy_only

Replacement stats

How often did each stage win? The runtime maintains a rolling counter.

kolm runtime stats --window 24h
# total requests: 18,442
# blocked (privacy): 312 1.7%
# cache hits: 3,118 16.9%
# local artifact: 9,420 51.1%
# cheaper model: 1,840 10.0%
# frontier: 3,752 20.3%
# avg savings vs frontier-first: $0.083 per call

Rollback

Unregister an artifact or switch policy back. Live traffic switches on the next call. No restart.

kolm runtime unregister summarizer-v1
kolm runtime policy set frontier_first --namespace summarizer
