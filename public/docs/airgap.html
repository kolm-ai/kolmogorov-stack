Air-gapped mode

Run kolm on a machine that has never seen the public internet. Compile against a local model cache, run recipes against local artifacts, verify receipts against a local secret. Sneakernet a signed .kolm across the gap on USB. Capture from a local-Ollama or kolm-local-teacher endpoint. Audit your own network for leaks. Stamped w779-v1.

What stays local

Zero outbound network required. The four pillars:

  • Compile — pre-staged base models in ~/.kolm/models. No HF Hub, no S3.
  • Run — kolm run <artifact.kolm> executes recipes in a local sandbox. Self-contained zip.
  • Verify — kolm inspect <artifact> --receipts walks the Ed25519 chain against an embedded or local key. No registry lookup.
  • Store — concepts, versions, audit log, RAG indices land in data/*.json or local SQLite. Nothing reaches kolm.ai.

Enable air-gap mode

Flip the kill-switch, inspect what is set, and confirm the environment refuses outbound:

kolm airgap enable     # writes ~/.kolm/airgap.env
kolm airgap status     # inspect runtime env vars
kolm airgap test       # network-leak audit (W779; shape-only by default)
kolm airgap disable    # remove ~/.kolm/airgap.env

kolm airgap enable writes ~/.kolm/airgap.env with the four offline switches: KOLM_AIRGAP=1, TRANSFORMERS_OFFLINE=1, HF_DATASETS_OFFLINE=1, HF_HUB_OFFLINE=1. Source that file from your shell rc, your systemd unit, or your container entrypoint to pin a process tree to local-only mode.

Sneakernet deployment

A sneakernet bundle ships a .kolm artifact across an air-gap on removable media (USB, optical disc, paper QR sheets). The bundle is a single tar archive containing:

  • artifact.kolm — byte-identical to the source artifact
  • signature.sig — HMAC-SHA256 over (artifact_bytes + canonical manifest)
  • manifest.json — {artifact_id, sha256_artifact, sha256_signature, created_at, version, transport:"sneakernet"}
  • README.txt — one-page operator note: how to verify + unpack

Pack on the source host:

kolm pack --sneakernet ~/.kolm/artifacts/support-bot.kolm --out ./support-bot.sneakernet.tar

Unpack on the target host (verifies signature against artifact bytes):

kolm unpack --sneakernet ./support-bot.sneakernet.tar
# or extract into a directory:
kolm unpack --sneakernet ./support-bot.sneakernet.tar --dest-dir ~/inbox
No-silent-pass contract. unpack sets verified:false when the embedded signature does not match the actual artifact bytes. The unpack helper REFUSES to write the artifact to --dest-dir on a verify failure — a corrupted USB cannot accidentally drop bytes into your filesystem. Always read verified and sha_matches before treating the artifact as trustworthy.

The tar archive is uncompressed and uses a deterministic file order so the SHA-256 over the bundle is reproducible — same inputs, same archive bytes. Use that property to compare across couriers without trusting either one.

Local-Ollama capture

When the gap is total, capture is still possible: point KOLM_LOCAL_TEACHER_URL at a local Ollama daemon (or any HTTP endpoint that speaks Ollama's /api/generate shape), and kolm will treat it as the teacher for distillation pairs. 

# Start Ollama on the air-gapped machine:
ollama serve            # default port 11434

# Tell kolm where the local teacher lives:
export KOLM_LOCAL_TEACHER_URL=http://127.0.0.1:11434
export KOLM_LOCAL_TEACHER_MODEL=llama3      # optional; defaults to llama3

# Capture pairs from the local teacher (no internet involved):
kolm capture --prompt "summarize this email" --teacher local-ollama

The capture path uses the same wrapFetch seam that the network-leak audit uses, so even with KOLM_AIRGAP=1 the request to KOLM_LOCAL_TEACHER_URL is explicitly allowed (same-host + listed-URL match). Every other URL still throws airgap_blocks_network.

No-silent-pass contract. When KOLM_LOCAL_TEACHER_URL is unset and you ask for local-Ollama capture, kolm returns {ok:false, error:"local_teacher_unconfigured"} with a one-line fix hint. It never silently falls back to a public model, and never silently passes through to the public internet.

Network-leak audit

kolm airgap test reports the active air-gap envelope without making real network calls. Pass --probe to also attempt a real probe and confirm the gateway actually blocks non-loopback URLs. 

kolm airgap test
# {
#   "ok": true,
#   "leaked": false,
#   "hits": [],
#   "probed_urls": [],
#   "airgap_active": true,
#   "local_teacher_url": "http://127.0.0.1:11434",
#   "version": "w779-v1"
# }

kolm airgap test --probe
# Attempts a HEAD against a synthetic remote URL. Should throw
# airgap_blocks_network; "leaked":false confirms the gateway held.

The audit accepts the same hostnames the runtime allows: localhost, 127.0.0.1, 0.0.0.0, ::1, plus whatever KOLM_LOCAL_TEACHER_URL resolves to. Any other URL throws the loud-fail envelope:

{
  "ok": false,
  "error": "airgap_blocks_network",
  "blocked_url": "https://api.openai.com/v1/...",
  "hint": "Set KOLM_AIRGAP=0 or capture from local-Ollama at $KOLM_LOCAL_TEACHER_URL",
  "version": "w779-v1"
}

Environment switches

variableeffect
KOLM_AIRGAP=1Master switch. CLI refuses to hit kolm.ai for publish, leaderboard, registry, or telemetry. Compiler skips opportunistic remote checks.
KOLM_LOCAL_TEACHER_URLLocal-Ollama or kolm-local-teacher endpoint. Allowed through wrapFetch even when KOLM_AIRGAP=1.
KOLM_LOCAL_TEACHER_MODELDefault model name for local-Ollama capture. Defaults to llama3.
TRANSFORMERS_OFFLINE=1HuggingFace transformers will not call out. Missing models fail loudly instead of silently fetching.
HF_DATASETS_OFFLINE=1Datasets library will not contact the Hub. Eval sets must be on disk.
HF_HUB_OFFLINE=1huggingface_hub blocks every network op. Combined with the two above, HF stack runs read-only.
RECIPE_RECEIPT_SECRETOptional. When set, signs sneakernet bundles + receipt chains. Without it kolm uses a fixed-fixture secret (fine for offline verification, not for cross-org trust).

API surface

Four auth-gated routes mirror the CLI:

  • GET /v1/airgap/status — {enabled, mode, local_teacher_url, env, version}
  • POST /v1/airgap/test — body {actually_probe?, probe_urls?}; returns the leak envelope.
  • POST /v1/sneakernet/pack — body {artifact_id, artifact_path, dest_path}; returns {ok, path, sha256, manifest, ...}.
  • POST /v1/sneakernet/unpack — body {src_path, dest_dir?}; returns {ok, artifact_id, verified, manifest, ...}.
