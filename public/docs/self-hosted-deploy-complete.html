Wrapper / Gateway

Self-hosted deploy reference.

Every environment variable, every secret, every port, every volume mount that the self-hosted gateway accepts. Copy the table, run the docker-compose or Helm install, and the gateway boots with no managed-only knobs hidden behind a paywall.

Required environment

VariablePurposeExample
KOLM_DATA_DIRWhere the receipt log, capture lake, and artifact store live. Must persist across restarts./var/lib/kolm
KOLM_LISTENBind address for the gateway.0.0.0.0:8080
KOLM_SIGNING_KEYEd25519 private key used to sign receipts and artifacts. Rotate via kolm key rotate.file /run/secrets/kolm-signing-key
KOLM_DATABASE_URLPostgres or SQLite for tenant + key metadata.postgres://kolm@db:5432/kolm

Optional environment

VariableDefaultNotes
KOLM_LOG_LEVELinfodebug, info, warn, error
KOLM_CAPTURE_PII_MODEredactoff | redact | hash | vault
KOLM_RECEIPT_ANCHORnonenone | s3 | gcs | cosmos | filecoin
KOLM_RATE_LIMIT_RPS100Per tenant, per second. Burst is 4×.
KOLM_TEACHER_ROUTINGdirectdirect, proxy, airgap
KOLM_TELEMETRYoffSelf-hosted defaults to off. Opt-in sends version + counter metrics, never payload bytes.

Secrets (provider keys, optional)

Only set the providers you actually route to.

ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-proj-...
GOOGLE_API_KEY=AIza...
DEEPSEEK_API_KEY=sk-...
COHERE_API_KEY=...
MISTRAL_API_KEY=...

Ports and volumes

PortPurpose
8080Gateway HTTP (set via KOLM_LISTEN)
8090Prometheus /metrics (opt-in via KOLM_METRICS_LISTEN)
8091Admin TUI socket (Unix domain by default)

Volume mounts: /var/lib/kolm (data dir), /run/secrets (signing key + provider keys), /etc/kolm/spec.toml (compiled spec, read-only).

Install

# docker-compose
# download from https://github.com/kolm-ai/kolmogorov-stack/releases/compose.yml > compose.yml
docker compose up -d

# Helm
helm repo add kolm https://kolm.ai/charts
helm install kolm-gateway kolm/gateway \
  --set signingKey=$(cat signing.key) \
  --set database.url=postgres://...
