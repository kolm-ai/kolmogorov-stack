kolm.ai · receipts spec v2.1

RS-1: every step has a signed receipt.

RS-1 is the on-disk format for receipts emitted by capture, compile, distill, eval, and replay. The full spec lives at /spec/rs-1; this page summarizes the public-facing shape and the verification path.

What a receipt records

  • The verb that ran (capture, compile, distill, eval, replay, bundle).
  • Inputs: artifact hashes, recipe hash, dataset rows, model id.
  • Outputs: K-score, latency, cost, durability flag.
  • Provenance: actor (key fingerprint), tenant, timestamp, prior receipt hash.
  • Signature: Ed25519 over the canonical JSON.

Chain rules

  1. Every receipt embeds the previous receipt hash for the same artifact.
  2. The bundle of receipts ships inside the .kolm artifact as receipts.jsonl.
  3. Verification is offline: the artifact carries the public key it was signed with.
  4. The x-kolm-capture-durable: true response header is the live counterpart at capture time.

Verify locally

kolm verify ./phi-redactor.kolm
kolm verify --strict ./phi-redactor.kolm # also re-runs the frozen eval set
