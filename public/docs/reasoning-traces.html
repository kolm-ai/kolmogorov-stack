Open-source AI workbench

Reasoning-trace distillation.

Four major providers expose a reasoning chain alongside the answer (Anthropic thinking, OpenAI reasoning_tokens, DeepSeek reasoning_content, Gemini thinking). The capture layer auto-detects the format, normalizes the chain, and stores it next to the response so the student can train on the trace, not just the final token. Compile-then-verify guarantees the replay matches.

Why capture the trace

Distilling the final answer teaches the student WHAT to say. Distilling the trace teaches the student HOW to get there. On multi-step problems (proofs, debugging, tool sequences) the trace is where the teaching signal actually lives.

Sequence-level supervision on a 200-token answer that hinges on a 12-token chain of intermediate reasoning under-trains the chain. The student learns to mimic the surface form of the answer without learning the steps that produced it. Trace distillation makes the chain visible to the loss.

The layer is provider-agnostic by design. The same captured.kolm row works regardless of which teacher produced the trace; the auto-detection writes a normalized reasoning field with the chain in canonical form, and the trainer reads from that field uniformly.

Auto-detection across four providers

The detector reads the response shape and identifies the provider's reasoning convention. No configuration is required; the format string is stamped on the capture row so a later reader can re-derive the source.

Supported formats

format stringproviderwhere the chain lives in the response
anthropic_thinkingAnthropiccontent[] blocks of type thinking
openai_reasoning_tokensOpenAIchoices[0].message.reasoning_tokens
deepseek_reasoningDeepSeekchoices[0].message.reasoning_content
gemini_thinkingGoogle Geminicandidates[0].content.parts with thought:true

The detector lives at src/capture.js as autoDetectReasoningCapability(response). Calling it returns {format, has_reasoning, location} or null when the response carries no detectable chain. A companion autoExtractReasoningTrace(response, format) returns the normalized chain text.

Captured row shape

A reasoning-trace capture is a regular captured.kolm row with two extra fields: reasoning (the normalized chain text) and reasoning_format (the format string from the detector).

{
  "cid": "cap_8a3f...",
  "namespace": "solver",
  "input": "Prove that the sum of the first n odd integers is n^2.",
  "output": "By induction. Base: 1 = 1^2. Inductive step: ...",
  "reasoning": "We can try a few small cases first: 1 = 1, 1+3 = 4, 1+3+5 = 9. These match 1, 4, 9 = 1^2, 2^2, 3^2. That suggests induction on n with the inductive step adding 2k+1 to k^2.",
  "reasoning_format": "anthropic_thinking",
  "teacher": "anthropic/claude-opus-4-7",
  "latency_ms": 4127
}

The trace is stored as plain text after normalization (provider-specific block structure is flattened). The original raw response is also available via the multimodal capture lake if the operator needs to re-derive the chain with a different normalization rule.

Compile and verify the trace

A captured trace becomes a compiled replay artifact via POST /v1/trace/compile. The compiled artifact replays the (input -> trace -> output) tuple deterministically; POST /v1/trace/verify re-runs the replay against the trainer's outputs and reports coverage and mismatches.

The compile path takes a trace_id (returned by the capture layer) and emits an IR that pins both the input and the final output, with the reasoning chain as a witness. On a cache hit the replay returns the original output without re-running the teacher.

The verify path returns {ok, total, matches, mismatches, coverage}. A mismatch row contains the diff between the captured chain and the student's reproduced chain, so you can see at a glance whether the student is matching the surface form of the answer while skipping a step.

Compile then verify

kolm trace compile cap_8a3f0c1d2e
# -> { ok:true, trace_id, ir_path, artifact_id }

kolm trace verify cap_8a3f0c1d2e
# -> { ok:true, total:3, matches:3, mismatches:[], coverage:1.0 }

Distill jobs collection

The bare collection endpoint GET /v1/trace/distill returns an plain pending envelope until the distill queue is mounted. The per-trace endpoints POST /v1/trace/compile and POST /v1/trace/verify are live today.

The pending envelope is intentional: tenants can poll the collection endpoint to know whether the queue has been wired without parsing source. Once mounted, the envelope will list completed and in-flight distill jobs keyed by namespace.

curl -H "Authorization: Bearer $KOLM_API_KEY" \
  https://kolm.ai/v1/trace/distill
# -> { "ok":true, "view":"reasoning-distill", "status":"pending", "items":[],
#      "note":"per-trace endpoints already shipped: POST /v1/trace/compile + /v1/trace/verify" }

TUI and CLI

The TUI binds reasoning-trace view to hotkey U (and command aliases :trace / :reasoning). The CLI exposes kolm trace compile and kolm trace verify.

From the TUI you can browse captures, filter by reasoning_format, and trigger compile against the selected row. The view shows the per-format counts (how many anthropic_thinking vs openai_reasoning_tokens captures live in the namespace) so an operator can spot when a provider is silently dropping the chain.

kolm tui
# inside the TUI:
#   U        switch to reasoning view
#   /        filter by reasoning_format
#   :        command palette - :trace, :reasoning, :compile cap_...

kolm trace compile cap_8a3f0c1d2e
kolm trace verify cap_8a3f0c1d2e
