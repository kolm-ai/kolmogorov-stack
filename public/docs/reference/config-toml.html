Reference

config.toml.

The user-level config file at ~/.kolm/config.toml. Every CLI verb reads it, every section is optional, and every key has a deterministic default. Use kolm config get / kolm config set to edit programmatically; hand-editing is supported and round-trips.

Configuration hierarchy

kolm resolves a configuration key in this order, first match wins:

  1. Flag. --gateway-port 9090 on the command line.
  2. Env var. KOLM_GATEWAY_PORT=9090. Bindings listed per-key below.
  3. Project file. ./kolm.toml or ./.kolm/config.toml if present at the cwd or an ancestor.
  4. User file. ~/.kolm/config.toml.
  5. Built-in default. The value in the Default column below.

Inspect the resolved value with kolm config get gateway.port --explain — it prints which layer supplied the answer.

Example file

# ~/.kolm/config.toml

[account]
api_key = "ks_4b7bc3b9...e8a1"
tenant_id = "tenant_f33c240034c5"
base_url = "https://kolm.ai"

[gateway]
port = 8787
allow_cors = ["http://localhost:3000"]
upstream_timeout_ms = 60000

[compile]
default_quant = "Q4_K_M"
keep_intermediates = false

[serve]
runtime = "auto"
port = 8080

[cloud]
provider = "runpod"           # runpod | modal | none
runpod_api_key = "rp_..."
modal_token = "ak-..."

[storage]
type = "local"                # local | s3 | postgres
events_path = "~/.kolm/events/events.sqlite"

[devices]
default_namespace = "default"
deploy_user = "kolm"
deploy_path = "/var/lib/kolm"

[telemetry]
enabled = false
endpoint = "https://kolm.ai/v1/telemetry"

Editing safely

# Show one key with explain
kolm config get gateway.port --explain
# → 8787 (default; layer=builtin)

# Set one key
kolm config set gateway.port 9090
# → wrote ~/.kolm/config.toml [gateway].port = 9090

# Print everything as JSON (masks secrets)
kolm config dump --json

# Validate the file syntax
kolm config check
# → ok (8 sections, 27 keys)
Caveats. Secrets are written in cleartext to a chmod-600 file. Use a system keychain (kolm config set account.api_key --keychain) if the file at rest must be encrypted. Project-level kolm.toml is checked into git by some teams; never put account.api_key there.
