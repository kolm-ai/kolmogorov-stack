Reference

Doctor checks.

kolm doctor runs the full check matrix in < 2 seconds. Every probe has a stable id (the rows below), a severity (err blocks, warn degrades, info is informational), and a one-line fix. Use kolm doctor --json for machine output and kolm doctor --fix to attempt the safe automatic remediations.

One-line example

kolm doctor
# → 26 checks: 24 ok / 1 warn / 1 err
# err  network.kolm-reachable     cannot reach kolm.ai (proxy?)
# warn devices.ssh-key-present    ~/.ssh/id_ed25519 not found

kolm doctor --json | jq '.checks[] | select(.status!="ok")'

Severity

  • err — CLI exits with code 3. Something the user needs to fix before kolm can do its job.
  • warn — CLI exits 0 (or 1 if any err is also present). The feature works, but degraded.
  • info — Reported but never affects exit code. Versioning, capabilities.

  "exit_code": 3,
  "summary": { "total": 26, "ok": 24, "warn": 1, "err": 1 },
  "checks": [
    {
      "id": "network.kolm-reachable",
      "status": "err",
      "severity": "err",
      "detail": "fetch failed: ETIMEDOUT https://kolm.ai/health",
      "fix": ["kolm config set account.base_url ...", "check VPN/proxy"]
    }
  ]
}

--fix flag

kolm doctor --fix attempts the safe-to-automate remediations only. Specifically it will:

  • Create ~/.kolm and subdirectories if missing.
  • Generate ~/.ssh/id_ed25519 if no key exists (with empty passphrase warning).
  • Install llama.cpp via brew / apt when the package manager is detected.
  • Pip-install bitsandbytes when a venv is active and Python ≥ 3.10.

It will never: install a Node or Python runtime, modify network config, push SSH keys to remote hosts, or write credentials.

Limitations. Doctor cannot probe what it cannot reach. Air-gapped installs should run kolm doctor --offline to skip the network checks. Devices behind jump-hosts may report green on the local box but fail at deploy time — pair with kolm devices health --all.
