Docs / Webhooks

kolm.ai · webhooks

Push notifications for compile, distill, and capture events.

Subscribe a URL to receive signed POSTs when something interesting happens: a compile finishes, a frozen-eval gate trips, a capture namespace crosses a threshold, an artifact is signed.

Event types

  • compile.completed: a .kolm artifact finished baking.
  • compile.failed: compilation aborted or gate failed.
  • distill.completed: a distill job emitted a recipe.
  • capture.threshold: namespace hit 100, 500, or 1000 pairs.
  • replay.diff_ready: a side-by-side replay report is available.
  • account.api_key_rotated: a key was created, revoked, or rotated.

Signing

Every POST carries X-Kolm-Signature: t=<unix-time>,v1=<hmac-sha256-hex>. Compute the HMAC over t.body with your webhook secret and compare in constant time. Reject any request older than five minutes.

const hmac = crypto.createHmac('sha256', secret);
hmac.update(`${t}.${rawBody}`);
const expected = hmac.digest('hex');
if (!crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(v1))) reject();

Retry policy

We retry on any non-2xx response with exponential backoff (1m, 5m, 15m, 1h, 6h, 24h). After the final attempt the event is moved to a 14-day dead-letter queue accessible from /account/connectors.
