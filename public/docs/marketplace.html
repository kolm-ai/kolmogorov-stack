Artifact marketplace.

A storefront for signed .kolm artifacts. Publishers upload with an Ed25519 signature over the manifest; the route layer verifies on every upload. Free listings download to any tenant; paid listings gate on plan or explicit entitlement. Ratings carry an anti-gaming gate to keep the leaderboard honest. Publisher payouts settle at a 70/30 split. Stamped w825-mvp-v1.

Data layer

The canonical store is ~/.kolm/marketplace/listings.jsonl. One JSON line per upsert; readers fold the trailing entry per id so writes never need to rewrite the file.

The append-only JSONL shape was chosen so concurrent publishers can write without a global lock. The reader walks the file once, keeps the last revision per id (later-line wins), and returns the rolled-up listing array. A corrupt or partial line is skipped; the reader never throws on missing or malformed input - listing reads return [] in the worst case, never an exception.

The listing shape:

{
  "id": "phi-redactor-v1",
  "publisher_tenant_id": "tenant_abc123",
  "title": "PHI redactor (healthcare)",
  "vertical": "medical",
  "task_type": "extraction",
  "k_score": 0.91,
  "hardware_targets": ["single-gpu consumer", "rtx", "h100"],
  "teacher_model": "claude-opus-4-7",
  "artifact_uri": "/var/kolm/uploads/phi-redactor-v1.kolm",
  "manifest_sha256": "8f3a...e2d0",
  "signature_b64": "MEUCIQ...",
  "rating_avg": 4.6,
  "rating_count": 28,
  "downloads": 412,
  "paid": false,
  "price_micro_usd": 0,
  "created_at": "2026-05-22T14:08:12Z"
}

The data-layer module sits at src/marketplace-w825.js and exposes three pure-read helpers - listListings(), getListing(id), upsertListing(listing) - and the frozen facet enums (W825_VERTICALS, W825_TASK_TYPES, W825_HARDWARE_TARGETS, W825_SORT_MODES) that the route + UI share.

Signed upload

Every upload carries an Ed25519 signature over the manifest_sha256. The route layer verifies the signature against the publisher's registered public key before accepting the row.

Without a registered public key, the upload returns 400 with error: "missing_public_key". With a bad signature, the upload returns 400 with error: "signature_invalid". A passing verification writes the row to listings.jsonl and emits an audit row through tryAppendAudit with op marketplace.uploaded.

The signature material is preserved in the persisted listing (manifest_sha256 + signature_b64 + the publisher's registered key in the registry) so any downstream verifier - a downloader, an auditor, the public bytes-of-listings tool - can re-verify offline without contacting kolm.ai.

Paid listings, 402, and entitlement

Free listings download to any authenticated tenant. Paid listings (paid: true, price_micro_usd > 0) return HTTP 402 when the requesting tenant lacks entitlement.

Entitlement is OR-gated across three conditions: the tenant is the publisher (self-download is always allowed), the tenant's plan is anything other than free or anon (paid plans get all paid listings), or the tenant has an explicit entry in tenant.entitlements[listing.id] === true. Any of those three opens the download; otherwise the route returns 402 with the listing's price.

The 402 is honest - it carries the price in micro-USD, the listing id, and a hint pointing at /pricing. We never silently pass the bytes through on a paid listing.

Ratings and anti-gaming

A rating is 1-5 stars plus optional review text. The anti-gaming rate gate at src/marketplace-ratings.js returns 403 unless the rater meets two conditions: account age above the floor, and a prior download of the listing.

The account-age floor is MIN_ACCOUNT_AGE_DAYS (default seven days). The download record is the row written by the recordDownloadEvent helper on a successful GET /v1/marketplace/download/:id; without that row, the rating route refuses the submission with error: "rate_requires_prior_download".

The aggregate is recomputed on every rating write so the listing row's rating_avg and rating_count stay in sync with the underlying rating events. Raw rating rows are returned by GET /v1/marketplace/ratings/:id so a reviewer can drill into the distribution rather than trusting the headline number.

Payouts (70/30 split)

Publishers earn 70% of paid listing revenue; the platform retains 30%. The payout cycle aggregates the revenue ledger and emits payout rows for each publisher.

The payout module sits at src/marketplace-payouts.js; the two exported helpers are recordRevenue({listing_id, tenant_id, paid_micro_usd, cycle}) (called on a successful paid download) and payoutCycle({cycle}) (called by the periodic settlement worker). calcPayout(paid_micro_usd) returns {publisher_micro_usd, platform_micro_usd, rate: 0.7} - the rate is exported as a constant so downstream tools can show the split alongside the dollar amount.

The payout cycle emits one row per (cycle, publisher_tenant_id) pair via POST /v1/marketplace/payout-cycle. The result envelope carries the per-publisher amount, the platform retention, and a transaction ID for downstream settlement - we never actually move funds in this module; that's the deploy pipeline's job.

Fine-tune queue

Any marketplace artifact can be the starting point for a transfer-learning fine-tune. POST /v1/marketplace/finetune queues a distill run against the artifact's bytes using your tenant's capture namespace.

The endpoint copies the artifact into ~/.kolm/artifacts/<artifact_id>.kolm for the distill pipeline to pick up by stable filename, then writes a queued event with provider: 'kolm_marketplace_finetune_queued'. The response carries status: 'queued' (never 'running' or 'completed' - the real distill runs in a follow-up worker). The W381 distill pipeline scans for these rows and picks them up on its next cycle.

Honesty contract: when artifact_uri is missing on disk or is a remote URL we cannot stage locally, the response still carries ok: true + copied_to: null + copy_skipped_reason so the operator can see why the bytes did not stage. The distill row is still queued; the worker can retry the copy when the bytes show up.

HTTP API

Eight routes under /v1/marketplace/*. Reads are public; writes are auth-required + tenant-fenced.

method + pathpurpose
GET /v1/marketplace/listingsBrowse listings. Filters: vertical, task_type, k_score floor, hardware target, sort mode. Pagination via page + limit.
GET /v1/marketplace/facetsThe frozen facet enums (verticals, task types, hardware targets, sort modes) for the sidebar UI.
POST /v1/marketplace/uploadPublish a new listing. Requires manifest signature; 400 on missing/invalid signature.
GET /v1/marketplace/download/:idStream artifact bytes. Records the download. 402 if paid + tenant unentitled.
POST /v1/marketplace/finetuneQueue a transfer-learning fine-tune from a marketplace artifact_id.
POST /v1/marketplace/rateSubmit 1-5 star rating + optional review_text. 403 unless account age + prior download.
GET /v1/marketplace/ratings/:idAggregate ratings + raw rating rows for one listing.
POST /v1/marketplace/payout-cycleAggregate the revenue ledger and emit payout rows. Auth-required, called by the settlement worker.

CLI verbs

The kolm marketplace verb covers the browse + install + publish loop. kolm marketplace install verifies the SHA-256 against the catalog row and runs the productionReady() gate before writing the bytes to disk.

kolm marketplace search                                  # full catalog
kolm marketplace search --category compliance --verified
kolm marketplace search --min-k 0.9 --json | jq .artifacts
kolm marketplace install phi-redactor
kolm marketplace show qwen-distill-classifier --json
kolm marketplace publish ./my-artifact.kolm

Without --force, a failing productionReady() gate exits 4 and writes nothing to ~/.kolm/artifacts/. With --force, the install still runs the SHA-256 check (a tampered artifact fails install even in --force mode) and writes the bytes alongside a warning row in the install log. The publish path submits to the manual-review queue and returns a queue_id for tracking; the reviewing team verifies bytes + signature + K-Score + lineage before the artifact appears in search.
