Team

Once more than one person works on a kolm.ai project, you want shared namespaces, role-based access, and an approval queue for the operations that matter (publishing an artifact, changing a privacy policy, enabling raw cloud sync). The team layer is opt-in and works on top of the local-first lake.

Shared namespaces

A namespace is a tag on lake events. A shared namespace is a namespace whose synced rows are visible to other members of the same team. Members see each other's captures, datasets, and artifacts in real time.

kolm team namespace share summarizer --team my-team
# summarizer is now shared with my-team (4 members)

RBAC roles

RoleCan do
adminEverything. Invite, remove, change policy, approve, delete.
reviewerApprove queue items. Read everything. Cannot change policy.
contributorCapture, build datasets, train, publish artifacts. Cannot change policy.
viewerRead-only. Browse captures, datasets, artifacts.

Invite

kolm team invite teammate@example.com --role contributor
# invite_018h ... sent. accept link valid 7 days.

Approval queue

Sensitive operations require an approval from a second member. The queue lives in the synced lake so anyone with reviewer or admin can act on it.

OperationNeeds approval?
Publish artifact to shared namespaceyes (any reviewer)
Change privacy policy on shared namespaceyes (admin only)
Enable raw_enabled cloud syncyes (admin only)
Delete a shared datasetyes (admin only)
Capture into a shared namespaceno
Train against a shared datasetno
 
kolm team approvals list
# req_018i publish summarizer-v2.kolm proposed by alice age=4m
# req_018j policy change on /billing proposed by bob age=22m

kolm team approvals approve req_018i
kolm team approvals reject req_018j --reason "lowered ssn from block to redact"

Team audit log

Every team operation is mirrored into ~/.kolm/runtime/team-audit.log on every member's machine. The audit is content-addressed so members can compare hashes and detect divergence.

kolm team audit --window 7d
# 2026-05-18T09:14 alice publish artifact summarizer-v2 (req_018i)
# 2026-05-17T16:02 bob invite carol@ as contributor

Rotate signing keys

When a member leaves, their public key is removed from the team trust file and any future sync from their device is rejected.

kolm team member remove carol@example.com
# carol@example.com revoked. trust file updated. next sync from her device will 401.

Next steps

Once a team is wired, read enterprise for on-prem deployment, SSO, and SLA, and cloud sync for the sync states. Start from /docs/quickstart if you have not yet captured anything.
