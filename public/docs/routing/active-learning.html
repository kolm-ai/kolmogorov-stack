Active learning

Every fallback is a training signal.

A teacher-handled call (because the student was uncertain) is the most valuable training example you can collect — it is exactly where the student needs to improve. The gateway tags those captures and feeds them straight into the next distill iteration.

The loop

Routing produces two kinds of receipts: student-handled (the student was confident; the answer was cheap) and teacher-handled (the student was uncertain; the answer was expensive). The teacher-handled receipts are the gold seed set for the next round of distillation, because they are the exact prompts the current student does not yet cover.

      [gateway]
         |
   ------+-------
   |             |
student      teacher
 confident   uncertain (high nats)
   |             |
   |             v
   |   [tagged: active_learning_seed=true]
   |             |
   |             v
   |   [accumulated in capture lake]
   |             |
   |             v
   |   kolm distill --seed-tag active_learning_seed
   |             |
   |             v
   |   new .kolm artifact (covers what previous missed)
   |             |
   |             v
   +-----[deployed back to local-kolm provider]

Tagging and surfacing seeds

High-entropy teacher captures are auto-tagged with active_learning_seed: true and surfaced in the /account/active-learning dashboard. The dashboard groups them by prompt-template similarity so you can see which weak spots are repeated.

# List candidate seeds for the next distill
kolm seeds list --namespace customer-support --min-entropy 0.85 --limit 200

# Inspect a cluster of similar seeds
kolm seeds cluster --namespace customer-support --threshold 0.9 --json | jq .clusters[0]

Triggering a distill iteration

Feed the tagged seeds straight into kolm distill. The compiler produces a new artifact; you redeploy by updating the local-kolm provider base_url to the new artifact path and reloading the gateway.

kolm distill \
  --namespace customer-support \
  --seed-tag active_learning_seed \
  --teacher openai/gpt-4o \
  --base-student Qwen2.5-7B-Instruct \
  --eval-from captures:last-500 \
  --output support-v4.kolm

# Verify before deploying
kolm verify support-v4.kolm
# K-Score: 0.91 (was 0.87) on 500-row eval
# Student share at threshold 0.7: 84% (was 74%)

# Redeploy
kolm gateway provider update local-kolm-support \
  --base-url file:///var/kolm/artifacts/support-v4.kolm
kolm gateway reload

Convergence behavior

Each iteration shrinks the teacher share. Typical trajectory on a 50k-capture / month workflow: v1 student handles 60%, v2 handles 75%, v3 handles 85%, then asymptote near 90% with the residual being genuinely novel prompts that need the frontier model.

Avoiding loop pathologies

  • Drift detection — if a new prompt-template appears in the active-learning queue without prior history, the dashboard flags it; do not auto-distill blindly.
  • Poison filter — the capture lake runs the five poison signals (output_length_anomaly, embedding_anomaly, injection_pattern, inconsistent_labels, synthetic_spam) on every seed; flagged rows are excluded from the next iteration by default.
  • Held-out eval — a frozen 500-row eval set is never used as a seed; the K-Score is measured on it across all iterations to detect quality regression.
Caveats. Active learning amplifies whatever bias is in the teacher. If the teacher gets a class of queries wrong, the student will learn the same mistake. Pair with a periodic teacher-bakeoff (different teacher on the same seeds) to catch teacher bias before it ossifies into the student.
