Cost attribution

Every dollar attributed to a namespace, model, and decision.

Every receipt carries cost_usd, tokens_in, tokens_out, and the route decision that picked the model. The dashboard rolls them into per-namespace, per-model, and per-decision views; the CLI prints the same data.

What the receipt stores

Pricing is computed at receipt time using the published rate card for each provider + model + tier. The rate card ships in the gateway image and is refreshed on each release; you can override per provider for negotiated rates.

FieldTypeSource
cost_usdfloattokens × rate card.
tokens_inintFrom provider response, or local count if not returned.
tokens_outintFrom provider response, or local count if not returned.
providerstringProvider slug that handled the call.
modelstringModel identifier.
route_decisionstringpinned / fallback / student / teacher.
cost_saved_usdfloat(teacher rate - student rate) × tokens, when student handled it.

Dashboard breakdown

The Account dashboard at /account/billing rolls receipts up four ways: by tenant, by namespace, by model, by route decision. Pivot on any combination; export to CSV or chargeback report.

# Pull the same data from the CLI
kolm cost --by namespace --since 30d
kolm cost --by model --since 30d --json
kolm cost --by decision --namespace customer-support --since 7d

The cost-saved metric

When the student handles a call, the receipt records cost_saved_usd as the difference between what the teacher would have cost and what the student actually cost. Summed across the namespace it is the dollar saving from confidence routing. The dashboard headline number is this sum over the trailing 30 days.

# Example: a confidence-routed namespace over 30 days
# 142,000 calls, 78% student-handled
#
# kolm cost --by decision --namespace customer-support --since 30d
#
# decision     calls    cost_usd   cost_saved_usd
# student      110,760     22.15        842.30
# teacher       31,240    194.45          0.00
# ----------------------------------------------
# total        142,000    216.60        842.30
#
# Effective rate: $1.53 per 1k calls (vs. $7.45 always-teacher)
# Saving: 79.5% of teacher-only baseline

Chargeback to internal teams

Map namespaces to internal cost centers and run kolm chargeback to produce a per-cost-center bill. The output is a CSV ready to feed your finance system.

# namespace -> cost center mapping in ~/.kolm/chargeback.toml
[mapping]
"support-bot"     = "CC-1001-customer-success"
"rag-pipeline"    = "CC-1042-product-eng"
"dev-sandbox"     = "CC-9000-engineering"

# generate the bill
kolm chargeback --month 2026-05 --out chargeback-2026-05.csv

Negotiated provider rates

If your enterprise OpenAI / Anthropic contract has discounted rates, override per provider so cost attribution reflects reality.

[[provider]]
name        = "openai"
base_url    = "https://api.openai.com/v1"
auth_scheme = "bearer_passthrough"

# per-model overrides (USD per 1M tokens)
[provider.rate_override."gpt-4o"]
input_per_1m  = 2.00       # default 2.50
output_per_1m = 8.00       # default 10.00
Caveats. Cost attribution assumes the request reached the upstream provider. Requests that the gateway rejected pre-forward (PII block, quota, auth) carry cost_usd = 0. The metered usage and the provider invoice should reconcile within tokens but may differ on edge cases like cached prompts; reconcile monthly against the provider invoice.
