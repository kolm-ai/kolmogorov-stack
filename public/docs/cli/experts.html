Docs / CLI / experts
CLI contract $ kolm experts --help $ kolm experts mixtral-8x7b.kolm --bars

Probe the MoE router. Per-expert activation %, prune candidates below the threshold, estimated size reduction and K-Score impact. Same handler powers POST /v1/experts and /account/experts.

Operational proof

Pure read of cached router decisions in receipts/router.jsonl. No live inference, no GPU touch. The walkout line is a ready-to-paste kolm compile --prune-experts command.

kolm experts

What experts are doing the work? For a Mixture-of-Experts artifact, report per-expert activation, flag cold experts, and estimate what pruning would cost in K-Score and save in size.

Usage


kolm experts <artifact.kolm>                          # compact table (top-5 + bottom-5)
kolm experts <artifact.kolm> --bars                   # full activation bar chart
kolm experts <artifact.kolm> --threshold 0.005        # tighter prune threshold (0.5%)
kolm experts <artifact.kolm> --json                   # full envelope for scripts

Flags

FlagDefaultDescription
<artifact.kolm>requiredpath to a .kolm file or an unpacked artifact directory
--threshold <f>0.01activation fraction below which an expert is a prune candidate (e.g. 0.005 = 0.5%)
--barsoffrender the full activation bar chart instead of the top-5 / bottom-5 compact table
--jsonoffemit the full analysis envelope as JSON

Response shape


{
  "is_moe":                 true,
  "num_experts":            8,
  "num_experts_per_tok":    2,
  "total_decisions":        4096,
  "source":                 "cached_router_decisions",
  "expert_activations":     [{ "expert_id": 0, "count": 1812, "pct": 22.12 }, ... ],
  "prune_candidates":       [{ "expert_id": 6, "pct": 0.41 }, ... ],
  "pruned_size_pct_reduction":  12.5,
  "estimated_kscore_impact":    0.013
}

For dense (non-MoE) artifacts, is_moe is false and the response carries a reason + hint field instead.

Examples


# compact view: top-5 most-used and bottom-5 cold experts
kolm experts mixtral-8x7b.kolm

# full bar chart for every expert
kolm experts mixtral-8x7b.kolm --bars

# tighter prune threshold (anything used < 0.5% of decisions is a candidate)
kolm experts mixtral-8x7b.kolm --threshold 0.005

# script: pull just the prune candidate IDs
kolm experts mixtral-8x7b.kolm --json | jq -r '.prune_candidates[].expert_id'

# walk into the compile step with the suggested prune list
kolm experts mixtral-8x7b.kolm --json \
  | jq -r '.prune_candidates | map(.expert_id) | join(",")' \
  | xargs -I {} kolm compile --spec mixtral.spec.json --prune-experts {}

Notes

kolm experts is a passive read of receipts/router.jsonl inside the artifact — one JSON record per routing decision, written during the eval-set pass that gates K-Score. If no cached router log exists, the response carries reason: "no_cached_router_decisions" and a hint to re-run kolm verify or recompile with router-trace enabled. No live inference is performed.

The estimated_kscore_impact is conservative: it assumes pruned experts contribute proportionally to their activation share, capped at -0.05 (5 K-Score points). Real impact depends on whether the cold experts were specialists on rare-but-important inputs — always re-evaluate with kolm bench after pruning.

For dense (non-MoE) artifacts the command exits 0 with a one-line message and no analysis — there are no experts to analyze. Use kolm inspect to confirm architecture.
