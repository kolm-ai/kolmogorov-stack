Estimate weight + KV + total memory footprint of a model on a given VRAM. Same handler powers POST /v1/fit and the wizard at /account/quantize.

Operational proof

Pure compute, no GPU touch. Use --pick-best to get the highest-quality target that fits with comfortable headroom — the same answer the bench wizard highlights.

kolm fit

Will it fit? Estimate the weight + KV-cache + total VRAM footprint of a model at a given quant target. Pure compute, no GPU touch — the direct answer before you queue the job.

Usage


kolm fit --params-b <N> --quant <target> --vram-gb <N> [--context <N>] [--batch <N>]
kolm fit --params-b <N> --pick-best --vram-gb <N>          # auto-pick the best target
kolm fit --artifact <path.kolm> --quant <target> --vram-gb <N>  # infer params from artifact

Flags

FlagDefaultDescription
--params-b <N>requiredmodel size in billions of params (e.g. 7 for 7B)
--quant <target>required*one of the 19 quant targets (see kolm hardware)
--vram-gb <N>requiredVRAM cap for the fit calculation (auto-fills from /v1/hardware in the wizard)
--context <N>8192context length, for the KV cache size
--batch <N>1batch size, for the KV cache size
--pick-bestoffauto-pick the highest-quality target that fits with comfortable headroom (skip --quant)
--artifact <path>noneinfer --params-b from a compiled .kolm instead of typing it
--supported <list>autorestrict the target picker to a comma-separated list (used with --pick-best)
--jsonoffemit the response envelope as JSON

* --quant is required unless you pass --pick-best.

Response shape


{
  "ok":              true,
  "quant":           "gguf-q4km",
  "bytes_per_param": 4.0,
  "est_weights_gb":  5.2,
  "est_kv_gb":       3.4,
  "est_total_gb":    8.6,
  "headroom_gb":     +23.4,
  "fits":            true,
  "tight":           false
}

Examples


# will a 7B fit at GGUF Q4_K_M on a 24 GB card?
kolm fit --params-b 7 --quant gguf-q4km --vram-gb 24

# auto-pick the best target for 7B on a 24 GB card.
kolm fit --params-b 7 --vram-gb 24 --pick-best

# infer params from an artifact, then pick the best target.
kolm fit --artifact qwen-7b.kolm --vram-gb 24 --pick-best

# longer context blows up the KV cache.
kolm fit --params-b 7 --quant gguf-q4km --vram-gb 24 --context 32768

# JSON output for CI.
kolm fit --params-b 7 --quant gguf-q4km --vram-gb 24 --json | jq '.fits'

Notes

kolm fit is pure compute; it never touches a GPU. The numbers are estimates: real-world memory depends on framework overhead (vLLM vs llama.cpp vs TRT-LLM), activation memory, and CUDA kernel choice. The tight flag flips to true when headroom drops below 2 GB — you might fit on paper but you’ll OOM the first time someone sends a long prompt.

--pick-best walks the canonical 19-target quality order (fp16 → gguf-iq2xs) and returns the first target that fits with comfortable headroom. The Account bench wizard at /account/bench shows the full matrix and highlights the same row.
