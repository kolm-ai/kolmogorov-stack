Every reference page is generated from the command source, keeps usage and examples close to the product, and links into the same API, account, and artifact workflow.

Operational proof

Use this verb inside the capture, dataset, distill, compile, verify, or runtime loop. Scripts can pin JSON output, humans get the same flow in the account console and TUI.

kolm logs

Tail the local run-history log at ~/.kolm/logs/runs.jsonl. Append-only. No cloud egress. No PII in the body.

Usage


kolm logs [--limit n] [--artifact <name|path>] [--since 7d|24h|10m] [--json]

Flags

FlagDefaultDescription
--limit <n>50maximum rows to print
`--artifact <namepath>`nonefilter by artifact
--since <window>noneonly rows newer than the window. 7d, 24h, 10m, etc.
--jsonoffmachine-readable JSONL

Examples


kolm logs --limit 20
kolm logs --artifact redactor.kolm --since 24h
kolm logs --json | jq '.[] | select(.k_composite < 0.85)'

Notes

Each row records: timestamp, command (run|bench|mcp), artifact, recipe, latency, K-score composite, success. The log is append-only at ~/.kolm/logs/runs.jsonl; no cloud egress, no PII in the body.
