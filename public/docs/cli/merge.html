Operational proof

Lineage gate refuses cross-base merges before any GPU work. Heuristic K-score delta is published per method — the dry-run prints it before you spend.

kolm merge

Combine two compiled .kolm artifacts into one. Five methods. Same-lineage gate. Dry-run prints the plan, the heuristic K-score delta, and the lineage check before the merge writes a byte.

Usage


kolm merge <base.kolm> <head.kolm> [flags]            # write merged.kolm
kolm merge <base.kolm> <head.kolm> --dry-run         # plan only, no write
kolm merge <base.kolm> <head.kolm> --alpha-grid 0.25,0.5,0.75 --dry-run  # sweep alpha

Flags

FlagDefaultDescription
--method <name>tiesone of linear, slerp, ties, dare, passthrough
--alpha <n>0.5blend weight from 0.0 (=base) to 1.0 (=head)
--alpha-grid <list>nonesweep multiple alphas in one dry-run, e.g. 0.25,0.5,0.75
--out <file.kolm>autowhere the merged artifact lands (default: ~/.kolm/artifacts/merged-<cid>.kolm)
--dry-runoffprint plan + heuristic K-score delta; do not write
--forceoffproceed despite a lineage mismatch (only meaningful for passthrough)
--jsonoffemit the response envelope as JSON instead of human text

K-score delta per method (heuristic)

MethodDelta rangeWhen to pick
linear+0.00 to +0.02Averaging two checkpoints from the same run. Lowest risk; lowest upside.
slerp+0.01 to +0.03Two specialists from the same base — preserves the weight-norm geometry.
ties (default)+0.02 to +0.05Resolves sign conflicts; best on multi-domain specialists.
dare+0.01 to +0.04Drops the smallest-magnitude updates and rescales. Strong on instruction-following.
passthrough+0 (router gain)Layer-level routing, not weight merge. Use for frankenmerges across sizes.

Examples


# inspect what the merge will do, without writing.
kolm merge router.kolm prose.kolm --method ties --alpha 0.5 --dry-run

# same-lineage TIES merge, default alpha.
kolm merge router.kolm prose.kolm --method ties --out merged.kolm

# sweep alpha to find the K-score peak.
kolm merge router.kolm prose.kolm --method ties --alpha-grid 0.25,0.5,0.75 --dry-run

# cross-lineage passthrough (router only, no weight merge).
kolm merge qwen-7b.kolm llama3-8b.kolm --method passthrough --force --out frankenmerge.kolm

# JSON output for CI.
kolm merge router.kolm prose.kolm --method ties --dry-run --json | jq '.heuristic_kscore_delta'

Lineage gate

Linear, SLERP, TIES and DARE all require same-shape weight tensors. The dry-run inspects both artifacts and checks base_model and params_b. If they disagree, you get a non-zero exit and a clear message before the merge starts and discovers a tensor-shape mismatch the painful way. Passthrough is exempt — it’s designed for cross-lineage routing.

Notes

The merge calls into src/forge-experts.js · mergeArtifacts(). The same code path serves the route at POST /v1/merge and the Account wizard at /account/merge/new. The heuristic K-score delta is calibrated from a public corpus of merges and refreshed on every release.

The output artifact inherits the union of both inputs’ tenant attestations and re-signs with your tenant key. Verification (kolm verify merged.kolm) walks both lineage chains.
