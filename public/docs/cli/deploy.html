kolm deploy -- CLI reference.

Deploy a signed .kolm artifact to a docker, kubernetes, or systemd target. Runs a pre-flight signature verify and a memory probe before any host changes.

Usage

kolm deploy <artifact> [--target docker|kubernetes|systemd] [--region us-east-1]
                          [--namespace prod] [--dry-run] [--json]

Arguments

  • <artifact> -- path to a local .kolm file, or a marketplace artifact id (e.g. kolm-finance-7b).
  • --target -- one of docker, kubernetes, systemd. Default: detected from environment (presence of kubectl or docker on PATH).
  • --region -- region label stamped on the deployment receipt. Required for multi-region setups; defaults to local otherwise.
  • --namespace -- kubernetes namespace, or systemd unit prefix, or docker network name. Default: prod.
  • --dry-run -- run pre-flight checks and print the deploy plan; make no host changes.
  • --json -- machine-readable envelope output.

Examples

# Deploy locally with docker
kolm deploy ~/.kolm/artifacts/kolm-finance-7b.kolm --target docker

# Deploy to a k8s cluster in us-west-2
kolm deploy kolm-finance-7b --target kubernetes --region us-west-2 --namespace prod

# Deploy as a systemd unit on a bare-metal host
kolm deploy ./my-distill.kolm --target systemd --namespace inference

# Pre-flight only
kolm deploy ./my-distill.kolm --dry-run --json

Pre-flight checks

Before any host change kolm deploy runs two checks and aborts on failure:

  • Signature verify. Reads the ed25519 signature from the .kolm manifest and verifies it against the trust store at ~/.kolm/trust/. A failed verify exits 65 (signature_invalid).
  • Memory probe. Reads the artifact's resource_requirements.memory_mb field and checks available host memory. If host memory is below the threshold deploy exits 66 (insufficient_memory) with the gap printed in the envelope.

Loud-fail envelope

Unknown target. If --target is set to a string outside the known set the CLI exits 64 with {ok:false, error:'unknown_target', hint:'expected one of: docker, kubernetes, systemd'}. We never silently fall back to a default target -- silent fall-back has historically caused deploys to land in the wrong cluster.

Exit codes

  • 0 -- deploy succeeded.
  • 3 -- auth required (kolm login).
  • 64 -- unknown --target value.
  • 65 -- artifact signature failed verify.
  • 66 -- insufficient memory on target host.
  • 67 -- target backend not installed (docker / kubectl / systemctl not on PATH).
