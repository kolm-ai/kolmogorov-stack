Hardware compatibility.

What kolm runs on. Recommended GPUs, minimum specs, the memory hierarchy, CPU-only fallback, air-gap notes, and the hardware partner program.

Recommended GPUs

Minimum spec for INT4 7B

  • GPU VRAM: 8 GB (5.2 GB for the weights + activation headroom).
  • System RAM: 16 GB.
  • Disk: 16 GB free for the artifact, manifest, and receipt chain.

Below 8 GB VRAM the runtime falls back to the memory hierarchy described below.

Memory hierarchy

kolm runtime probes for available memory in order: VRAM -> system RAM -> NVMe-mmap. When VRAM is insufficient the runtime falls back through:

  1. VRAM. Fastest path; preferred.
  2. System RAM with weight offload. Layers swap on demand. Throughput drops by roughly 5-10x.
  3. NVMe-mmap. The .kolm compile is mmap'd from disk; weight pages page in via the OS page cache. Throughput drops by another 5-10x.

Each fallback step is logged in the runtime receipt so the throughput number on your inference page is explainable after the fact.

CPU-only fallback

The runtime can execute purely on CPU when no GPU is present. The receipt stamps {accelerator:'cpu', estimated_tok_s:N} with the measured CPU throughput. For interactive workloads we recommend at least one of the GPUs above; CPU-only is for batch inference, evaluation, air-gapped review, and small-model on-device deployment.

Throughput on CPU is bound by memory bandwidth, not compute. Apple Silicon punches above its core count because UMA bandwidth (M4 Max: ~546 GB/s) approaches mid-range GPU territory. Intel/AMD desktop CPUs are bandwidth-starved on dual-channel DDR5 (~80 GB/s); server CPUs with 8-12 memory channels (~400-700 GB/s) close most of the gap.

Compile path on no-GPU laptops

Quantization itself benefits from a GPU, but is not strictly required. The CPU fallback paths:

  • Q4_K_M / Q5_K_M / Q8_0 (llama.cpp imatrix quants) — compile entirely on CPU. A 7B model takes ~6-12 min on Apple M2 Max; a 32B model takes ~40-90 min. RAM required: roughly 2x the FP16 footprint (28 GB for 7B, 128 GB for 32B).
  • NF4 / INT4 (bitsandbytes) — requires CUDA. Use the cloud path (kolm compile --cloud modal) or Colab notebook (examples/colab-compile.ipynb).
  • GPTQ / AWQ — requires CUDA for calibration. Pre-compiled artifacts can run on CPU via llama.cpp's GPTQ-to-GGUF converter.

For Persona B (no local GPU, prefers desktop workflow), the recommended path is: capture locally → kolm compile --cloud modal → download the .kolm artifact → kolm export gguf-q4km on the laptop → kolm run from llama.cpp/ollama. See /docs/colab-compile for the zero-cost free-tier path.

Air-gap notes

  • The kolm runtime is fully air-gap compatible. The signature trust store at ~/.kolm/trust/ is the only external dependency at run time, and you can pre-populate it on a connected host and rsync into the air-gapped host.
  • The CLI kolm verify and kolm run commands work offline by default; no telemetry, no phone-home.
  • The optional kolm phone-home opt-in beacon is described in /security; it ships disabled.

Hardware partner program

If you ship a GPU, accelerator, or edge device and want a "Powered by kolm" badge plus a documented runtime target, see /partners/hardware. Co-marketing terms, dedicated runtime support, and the buy-a-partner-GPU-get-one-year-kolm-Pro template are described there.

Constraints. Throughput numbers above are reproducible on the listed GPUs with bitsandbytes 0.49.2 and torch 2.11.0+cu128, but they will vary with CPU host, PCIe generation, BIOS settings, and competing GPU workloads. Always run kolm bench on your own hardware before quoting a number externally.
