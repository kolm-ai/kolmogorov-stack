The assurance case envelope.

An assurance case is the trust packet a buyer's security and risk team attaches to a third-line review. It folds three signals into one envelope: outcome-shaped claims about the artifact, an explicit controls mapping across SOC 2, HIPAA, the EU AI Act, and ISO 27001, and a small meta block (generated_at, artifact pointer, workspace pointer, spec version). Spec kolm-assurance-case-1.

Invocation

import { buildAssuranceCase, validateAssuranceCase } from './src/assurance-case.js';

const envelope = buildAssuranceCase({
  artifact: /* { id, manifest:{...}, receipt:{...} } */,
  workspace: { id, drift_namespaces: {...}, procurement_vault: {...} },
});

// Validate before render
const v = validateAssuranceCase(envelope);
if (!v.ok) throw new Error('shape: ' + v.reasons.join(', '));

The function NEVER throws. Missing inputs downgrade individual claims or controls to package-gated or external-proof-needed with a descriptive limitations string. Silence is never treated as success.

Status taxonomy

StatusMeaning
implementedThe control IS in production now. Evidence points at a receipt or vault row that proves it.
package-gatedThe technical control ships but the buyer must run a specific install or config to enable it (for example, on-prem self-hosted air-gap mode).
certification-gatedControl exists, third-party certification in progress (for example SOC 2 Type II window not yet closed). The gating cert is named in the vault row.
external-proof-neededControl depends on a signal the platform cannot attest to itself (for example a separately-enabled namespace drift detector). The reviewer must verify externally.

Claim generators

Five claim generators run by default. Each emits {claim, status, evidence_ids, limitations}.

GeneratorWhat it assertsDowngrade trigger
_claimDataProvenanceAll training captures are tied to a verified rights-holder.No capture nodes on the DAG, or some captures are not covered by a rights node.
_claimModelIntegrityArtifact bytes are signed and carry a replayable kolm-audit-1 receipt chain.Missing Ed25519 signature; or signature present but no kolm-audit-1 receipts attached.
_claimDeploymentIntegrityRuntime capabilities are documented per-(runtime, target) with measured or estimated capacity.No runtime_passports attached; all rows are estimates rather than tested.
_claimDriftMonitoringProduction drift is monitored against the training distribution with threshold alerts.Drift detector not enabled for the artifact's namespace.
_claimJurisdictionVendor maintains a documented risk-management program reviewed annually.Procurement vault row SIG.A.1.2 not present in the workspace.

Control rows (minimum 8 across 4 frameworks)

FrameworkControl IDLabel
SOC 2CC6.1Logical and physical access controls.
SOC 2CC7.2System monitoring of anomalies and events.
SOC 2A1.2Capacity, availability, and resilience.
HIPAA164.308Administrative safeguards (workforce, BAA).
HIPAA164.312Technical safeguards (encryption, integrity).
EU AI ActArt-13Transparency and provision of information to users.
EU AI ActArt-14Human oversight.
ISO/IEC 27001A.5.1Policies for information security.

Each row is mapped to a CAIQ or SIG-Lite vault evidence id when the corresponding vault answer exists in the workspace. When no matching vault row is present, the row drops to certification-gated with evidence_id: null.

PDF render

src/assurance-case-pdf.js renders the envelope to a PDF via pdfkit. Lazy import so callers that only need the JSON do not pay the dependency load.

import { renderAssuranceCasePdf } from './src/assurance-case-pdf.js';

const pdfBuf = await renderAssuranceCasePdf(envelope);
fs.writeFileSync('assurance-case.pdf', pdfBuf);

validateAssuranceCase

Enforces shape invariants before render. Returns {ok, reasons:[]}.

const v = validateAssuranceCase(envelope);
// { ok: false, reasons: ['claim_bad_status:tentative', 'control_missing_id'] }

Checks: spec matches, claims and controls are arrays, every claim has a non-empty claim string, status is one of the four canonical values, evidence_ids is an array, limitations is a string, controls carry both framework and control_id.

Caveats. The assurance case does not by itself constitute certification. A SOC2 CC6.1: implemented row means the workspace has the control wired up and has a vault answer pointing at the supporting evidence; it does not mean a SOC 2 auditor has signed off. The certification-gated status is the explicit label for "cert audit in progress".
