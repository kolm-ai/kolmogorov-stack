Capture export

Three formats, one verb, deterministic output.

kolm captures export dumps the lake (or a filtered slice of it) as JSONL, Parquet, or a HuggingFace datasets directory. Same row schema across formats; byte-identical between runs for the same filter.

The three formats

FormatUse caseToolingCompression
jsonlStreaming pipelines, jq, simple inspection.Any.gzip on the fly with --gzip.
parquetColumnar analytics, DuckDB, Spark.pyarrow, duckdb.Snappy by default.
hf-datasetDirect feed into HuggingFace datasets.datasets, transformers.Arrow + dataset_info.json.

Filters

Filters compose. The exporter walks the lake in capture_id order, applies filters in the storage backend where possible (Postgres / SQLite push down WHERE clauses; JSONL / S3 stream-filter), and writes the matching rows.

FlagEffect
--namespace <name>Limit to one namespace; repeat for multiple.
--state <state>Filter by state (default: approved).
--since <duration>Captures newer than the duration (e.g. 7d, 30d).
--until <timestamp>Captures before the timestamp (RFC 3339).
--provider <slug>Only captures handled by a specific provider.
--model <name>Only captures for a specific model.
--route <decision>Filter by route decision (pinned / fallback / student / teacher).
--limit <n>Cap at N rows; useful for sampling.

Examples

# JSONL export of all approved support captures, last 30 days
kolm captures export \
  --format jsonl \
  --namespace customer-support \
  --state approved \
  --since 30d \
  --out ./support-2026-04.jsonl

# Parquet, multiple namespaces, gzip not needed (Snappy default)
kolm captures export \
  --format parquet \
  --namespace customer-support \
  --namespace billing-bot \
  --since 90d \
  --out ./multi-2026-Q2.parquet

# HuggingFace dataset directory, ready for datasets.load_from_disk
kolm captures export \
  --format hf-dataset \
  --namespace customer-support \
  --since 30d \
  --route teacher \
  --out ./hf-support-teacher-2026-04/

# Streaming JSONL to stdout for piping into jq
kolm captures export --format jsonl --namespace customer-support --since 1d --out - | jq '.cost_usd' | datamash sum 1

HuggingFace dataset layout

The hf-dataset output is a directory with an Arrow table and a dataset_info.json describing the schema. Load directly with datasets.load_from_disk; push to the hub with dataset.push_to_hub.

# Directory layout
./hf-support-teacher-2026-04/
  dataset_info.json
  state.json
  data-00000-of-00003.arrow
  data-00001-of-00003.arrow
  data-00002-of-00003.arrow

# Load in Python
from datasets import load_from_disk
ds = load_from_disk("./hf-support-teacher-2026-04/")
print(ds)

What the exporter omits

The exporter never writes the raw signing key, the upstream provider credentials, or the chain hashes (those are lake-internal). The receipt CID is included so any consumer can re-fetch the signed receipt from /v1/receipts/<cid> if needed.

Determinism

For the same filter set against the same lake state, the exporter produces byte-identical JSONL and Arrow output. Row order is capture_id ascending; field order in JSONL is fixed. Parquet bytes can vary across pyarrow versions; pin the version in CI for reproducibility.

Caveats. Export reflects lake state at the moment the export runs. If new captures land mid-export, they are not included even if they fall within the filter window; rerun for the latest. For large exports (>10M rows) prefer --format parquet; JSONL is fine for under that, but the file size grows linearly with row size.
