Capture poisoning

Five signals, one quarantine threshold.

Five orthogonal detectors score every capture for poisoning risk. The aggregate score crosses a per-namespace threshold and the row is auto-quarantined. Manual override is recorded on the chain.

Why this matters

A capture lake feeds distillation. A poisoned capture — whether by accident (a model hallucination) or by intent (a prompt injection that survived the input scan) — trains the next student on the wrong answer. Detection is cheap; quarantine is reversible; the cost of letting one through is amplified by every later iteration.

The five signals

SignalWhat it measuresCost
output_length_anomalyResponse length is >3 sigma from the per-prompt-template mean.Cheap (statistics).
embedding_anomalyResponse embedding is >3 sigma from the per-prompt-template cluster centroid.Medium (one embedding call).
injection_patternResponse contains canary phrases ("ignore previous", "system prompt was").Cheap (regex + classifier).
inconsistent_labelsResponse classification disagrees with the user-attached label (when present).Cheap (string compare).
synthetic_spamResponse shows signatures of being LLM-generated rather than human-written (when the application expects human).Medium (classifier).

Score aggregation

Each signal returns a score in [0, 1]. The aggregate poison score is the weighted maximum: score = max(weight_i × signal_i). The weights default to 1.0; configure per-namespace to emphasize a specific risk.

# Defaults
[capture.poisoning]
auto_quarantine_threshold = 0.7
weights.output_length_anomaly = 1.0
weights.embedding_anomaly     = 1.0
weights.injection_pattern     = 1.0
weights.inconsistent_labels   = 1.0
weights.synthetic_spam        = 1.0

# Per-namespace override (e.g. RAG namespace cares most about injection)
[[namespace.poisoning]]
namespace = "rag-pipeline"
weights.injection_pattern = 1.5
weights.embedding_anomaly = 1.2
auto_quarantine_threshold = 0.6

Auto-quarantine threshold

When score >= auto_quarantine_threshold, the capture transitions from pending to quarantined and the operator dashboard surfaces it. The default threshold of 0.7 is tuned for low false-quarantine rate on the W411 reference dataset; lower to be stricter, raise to be more permissive.

What a quarantined row looks like

# Quarantine annotation on a capture row
{
  "state": "quarantined",
  "poison": {
    "score": 0.91,
    "signals": {
      "output_length_anomaly": 0.42,
      "embedding_anomaly": 0.18,
      "injection_pattern": 0.91,
      "inconsistent_labels": 0.0,
      "synthetic_spam": 0.12
    },
    "trigger": "injection_pattern",
    "matched_canary": "ignore the previous instructions"
  }
}

Manual override

A reviewer can release a quarantined row back to pending (or directly to approved). The override writes a new chain row recording the operator id, the reason, and the original poison score for the audit trail.

kolm captures release cap_01HXYZ... --reason "false positive: canary in legitimate docs"
kolm captures release cap_01HXYZ... --approve --reason "reviewed, OK to train on"

# List quarantined rows for a namespace
kolm captures list --namespace customer-support --state quarantined --limit 50

# Stats: what fired this week
kolm captures poison-stats --namespace customer-support --since 7d
# signal               fires   share  median_score
# injection_pattern      142   62%       0.88
# output_length_anomaly   58   25%       0.74
# embedding_anomaly       21    9%       0.81
# inconsistent_labels      8    3%       0.92
# synthetic_spam           1   <1%       0.71

Configuring the canary list

The injection_pattern signal uses a default canary list shipped with the gateway. Add or remove patterns per namespace; the matched phrase is recorded on the row so post-hoc analysis is straightforward.

[capture.poisoning.canaries]
phrases = [
  "ignore the previous",
  "ignore previous instructions",
  "system prompt was",
  "you are now",
  "disregard the above",
  "new instructions:",
  "[INST]",
  "###",
]
case_sensitive = false
window_chars   = 200          # match only in first/last 200 chars of response
Caveats. Poison signals are heuristic, not proof. The aggregate score has a documented false-positive rate of ~3% at threshold 0.7 on the W411 reference dataset; on your traffic it may be higher or lower. Review the quarantine queue weekly; promote false positives back so you do not lose training data. The signals do not prove an attack happened, only that the row looks anomalous.
