Studio / FAQ

Studio FAQ.

Questions people actually ask before their first kolm distill or kolm compile. If yours isn't here, open a GitHub discussion or ping the team in chat — we add answers as patterns emerge.

1. What hardware do I actually need?

For a 7B distill, an RTX 4090 / 5090 with 24–32 GB VRAM finishes a 410-pair Trinity-style run in 80–120 seconds. A 13B fits on the same hardware with bf16 + gradient checkpointing + bs=1. A 32B distill wants 48 GB+ (RTX 6000 Ada, A6000, H100) or Modal/RunPod. For compile-only (no fine-tune), CPU works for any size that fits in RAM — it's just slow. A Mac with 64 GB unified memory can do up to 30B distill on the MLX backend.

Run kolm hardware probe to get a list of recipes your machine will actually finish.

2. How much does a Trinity-style distill cost in teacher API fees?

The Trinity-500 reference run (410 pairs, three teachers, weighted council) cost $0.31 per 1000 tokens through hosted Claude Haiku + GPT-4o-mini + DeepSeek V3 — roughly $14–$22 for a complete 410-pair pull, depending on average prompt length. Self-hosted teachers (DeepSeek-R1 via a local stdlib server) cost zero in API fees and trade about 2× the wall-clock time.

Pass --budget 5 on kolm distill to hard-cap spend; the job fails before exceeding it.

3. Which quant format should I pick?

Defaults that almost never go wrong: GGUF Q4_K_M for general use on consumer GPUs, GGUF Q5_K_M when you have an extra 1–2 GB to spend and want a quality hedge, GGUF Q8_0 for evaluation baselines. For vLLM/SGLang serving: AWQ on Ampere/Ada, FP8 on Hopper, NVFP4 on Blackwell. For Apple Silicon: MLX 4-bit. The export formats decision table walks through the trade-offs.

4. Can I distill a model whose license forbids derivative works?

You can run the bytes through Studio — we don't lock that down — but Studio writes the source license into the artifact passport and the auto-generated model card. If the source license forbids derivative works (some "research-only" licenses do), the published artifact inherits that restriction. Studio flags this at kolm compile time with a warning. Pass --license-override only if your legal team has cleared it; the override is recorded in the receipt.

5. How does Studio handle MoE models like Mixtral or DeepSeek-V3?

Native. The compiler enumerates experts per layer, runs activation tracing on a calibration set, and lets you set per-expert quant policy (typically router-high + experts-aggressive) and prune low-traffic experts. See MoE compilation. The 671B DeepSeek-V3 has been compiled and exported to GGUF on a single H200 node; on consumer hardware you'll need 200 GB+ unified memory or a multi-GPU split.

6. Is a run reproducible bit-for-bit?

Yes, if you pin everything. Studio records the random seed, base model SHA, teacher model versions, dataset hash, CUDA/cuDNN version, and the kolm version into the receipt. Re-running with the same --seed on the same hardware reproduces the artifact's SHA-256 to the byte. Different hardware (e.g., 5090 vs H100) can produce different floating-point rounding; we surface this as a "reproducibility class" in the passport: same-host-same-bytes, same-arch-same-bytes, or cross-arch-numerical.

7. What happens if my distill crashes at step 800 of 1000?

Studio checkpoints every --ckpt-every steps (default 100) to ~/.kolm/runs/<job_id>/checkpoints/. Re-run with kolm distill --resume <job_id> to pick up from the last checkpoint. The teacher cache is also persisted — you don't pay twice for the same teacher completions on resume.

8. Can I use my own teacher instead of Claude/GPT/DeepSeek?

Any OpenAI-compatible HTTP endpoint works as a teacher — including your own vLLM, SGLang, llama.cpp, or kolm-served model. Register it with kolm teachers add --name my-teacher --url http://127.0.0.1:8765 --weight 1.0. See Teacher providers. Self-hosted teachers are how the team distilled the Trinity-7B reference model: a local INT4 DeepSeek-R1 32B served by Python http.server on a 5090, no provider fees.

9. Why is the judge a different model lineage from the teachers?

If Claude grades a student trained on Claude, the grade is biased high — same prior, same blind spots. We use Gemini as the default judge because it's a fully separate lineage from Claude/GPT/DeepSeek. The judge sees pairs blind (no labels of which model produced which response) and scores against an explicit rubric. See Judges. You can swap the judge with --judge model-name; the receipt records whichever judge was used.

10. Where does my data go during a distill run?

Local-first by default. Prompts and teacher completions are written only to ~/.kolm/runs/<job_id>/. Nothing is uploaded to kolm.ai unless you explicitly pass --telemetry on or publish the artifact. The teacher HTTP calls themselves go to whichever provider you configured (Anthropic, OpenAI, etc.) under your API key. Self-hosted teachers keep everything on your machine end to end. The Enterprise tier adds a BYOC option where even the orchestration runs in your VPC.

Caveats. Cost numbers above reflect 2026-Q2 provider pricing for the Trinity reference recipe and will drift as providers reprice. Always check kolm distill --dry-run --json for a current estimate before running.
