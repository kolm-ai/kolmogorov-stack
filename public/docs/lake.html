Lake

Everything that goes through the kolm.ai proxy lands in a local event lake. One row per request, 35 fields, indexed for time-range, namespace, and model queries. The lake is plain SQLite at ~/.kolm/events/events.sqlite and is the single source of truth for cost, latency, drift, and dataset building.

Storage layout

Per-tenant, per-machine. The SQLite file uses WAL mode and is safe for concurrent readers + one writer (the proxy).

~/.kolm/
 events/
 events.sqlite # main row store
 events.sqlite-wal # write-ahead log
 events.sqlite-shm # shared memory index
 runtime/
 policy.json
 audit.log
 artifacts/ # compiled .kolm files
 datasets/ # train/val/holdout exports

idTEXT PKULID, monotonic
ts_msINTEGERUnix ms at proxy receive
namespaceTEXTTag from x-kolm-namespace header
tenant_idTEXTLocal tenant (single-tenant by default)
upstream_providerTEXTopenai, anthropic, gemini, openrouter, local
upstream_modelTEXTFinal resolved model id
requested_modelTEXTModel the caller asked for
surfaceTEXTchat, messages, embeddings, completions
modalityTEXTtext, multimodal, audio, tool
messages_jsonBLOBFull request body (after redaction)
response_jsonBLOBFull upstream response (after redaction)
messages_raw_jsonBLOBPre-redaction body, gated by privacy mode
response_raw_jsonBLOBPre-redaction response, gated by privacy mode
template_hashTEXTStructural hash for clustering
prompt_shaTEXTsha256 of canonical prompt
response_shaTEXTsha256 of canonical response
in_tokensINTEGERReported input tokens
out_tokensINTEGERReported output tokens
cached_tokensINTEGERProvider-reported cache hits
cost_usdREALPer-call USD spend
cost_sourceTEXTheader, table, estimate
latency_msINTEGEREnd-to-end at the proxy
ttft_msINTEGERTime-to-first-token, streaming only

Querying

The CLI wraps the most common rollups. For anything else, talk to SQLite directly.

kolm lake stats --window 24h --by namespace
kolm lake stats --window 7d --by upstream_model
kolm lake list --namespace summarizer --limit 10 --json
sqlite3 ~/.kolm/events/events.sqlite "SELECT upstream_model, AVG(latency_ms) FROM events WHERE ts_ms > strftime('%s','now','-1 day')*1000 GROUP BY 1;"

Retention

By default the lake keeps everything forever. Set retention_days in ~/.kolm/config.json to vacuum older rows on proxy start.

{
 "retention_days": 90,
 "retention_keep_artifact_rows": true
}
