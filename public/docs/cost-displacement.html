Per-receipt cost displacement.

computeDisplacement from src/cost-displacement.js walks the receipt log for a tenant and namespace, and for each row asks "if this had ALWAYS gone to frontier, what would it have cost?" versus "what did it actually cost?". The delta is the value local routing displaced. No fee math — this is operator-facing "what did kolm save me this month" without the pricing model baked in.

Distinct from savings-tracker

ModuleQuestion
savings-tracker.js (W835)"Baseline period vs post-kolm period" with a fee-on-savings model. Used for the kolm.ai pricing fee — the 12.5% of saved spend.
cost-displacement.js (R-8)Per-namespace, per-receipt reasoning using ConfidenceRouter's route_decision tag. Used for operator-facing dashboards. No pricing model baked in.

Invocation

import { computeDisplacement } from './src/cost-displacement.js';

const r = computeDisplacement({
  tenant_id: 'tenant_4d3d8db6c820',
  namespace: 'support',
  period_days: 30,
  artifact_id: 'qwen3-8b-support-q4km',  // for compile_cost lookup
});

// {
//   ok: true, version: 'r8-v1',
//   baseline_cost_usd: 1840.55,        // every routed call at frontier prices
//   actual_cost_usd: 162.30,           // what was actually billed
//   savings_usd: 1678.25,
//   cumulative_savings_usd: 6204.10,   // since deployed_at
//   payback_period_months: 0.18,       // compile_cost / monthly_savings_rate
//   period: {...}, ok_status: 'computed'
// }

Route-decision branching

Every receipt carries a cost_usd (the actual billed cost) and a route_decision tag from the ConfidenceRouter. The baseline cost depends on which branch ran:

Route decisionActual costBaseline cost
local$0 (or near-zero compute)Frontier-equivalent cost re-priced from the rate card for the configured frontier model.
frontierReal frontier costSame as actual — no displacement (would have gone to frontier anyway).
frontier_fallbackReal frontier costSame as actual — we already paid full frontier price.
missingWhatever cost_usd saysSame as actual. No savings claim — older rows without route_decision get a pass-through.

The rate-card counterfactual

For local rows we re-price the call as if it had hit the configured frontier model. The price table is the frozen PROVIDER_RATE_CARD from src/savings-tracker.js — same source of authority for both modules.

// Per-million-token rates for the namespace's frontier model
{ input_per_million_usd: 15.0, output_per_million_usd: 60.0 }

cost = (input_tokens * input_rate + output_tokens * output_rate) / 1_000_000

When no rate card entry is found for the configured frontier model, we fall through to the receipt's own provider and model. When neither matches, the row is excluded and the envelope's ok_status reports partial:N_unrepriced. We never invent numbers.

Parameters

FieldTypeDefaultNotes
tenant_idstringrequiredCost displacement is tenant-scoped.
namespacestring'default'Use '*' for tenant-wide.
period_daysnumber30The scoring window (1 to 3650).
artifact_idstringoptionalUsed by the passport reader to look up compile_cost_usd and deployed_at.
compile_cost_usdnumber0Explicit override for the compile cost (skips passport lookup).
deployed_at_msnumbernullExplicit override for the deployment epoch (used by cumulative math).
frontier_provider / frontier_modelstringnullCounterfactual model for re-pricing local calls.

Payback period

The artifact's compile cost (training plus quantize plus GPU time) divided by the monthly savings rate, in months. Three special values:

ValueMeaning
'instant'Compile cost is zero (the artifact was free to compile, for example a pre-existing weight file).
nullMonthly savings rate is non-positive — the artifact has not yet displaced enough spend to define a payback window.
numberCompile cost divided by (savings / period_days) * 30.

ok_status values

ValueTrigger
computedEverything resolved, every local row had a rate-card entry.
partial:N_unrepricedN local rows lacked a rate-card entry; those rows contributed $0 to savings rather than an invented number.
no_receiptsThe window contained zero receipts. All numbers are zero, payback_period is 'instant'.
no_route_decisionsThe window had receipts but none carried route_decision. Savings is zero.
Caveats. Cumulative savings counts from the artifact's deployed_at (or, if missing, from the earliest receipt in the namespace). It re-walks all rows since that point on every call — cheap at thousands of receipts, expensive at hundreds of thousands. For very large workspaces consider a daily roll-up table; the current implementation is sufficient through ~250k rows per namespace before noticeable latency.
Limitations. The "local" cost is treated as zero by default because the artifact runs on hardware you already own. If you want to factor in amortized GPU lease cost per call, pre-stamp it as cost_usd on each receipt before the cost-displacement read — this module trusts cost_usd as authoritative and does not re-compute it.
