Distillation

Distillation in kolm.ai turns lake rows into a portable artifact. The artifact is a single signed file: .kolm. It carries the manifest, the seed examples, the executable recipe, optional weights, and a chain of receipts proving how it was made. You can hand it to anyone with the runtime and they get the same outputs you do.

Distill vs compile

Both produce a .kolm. The difference is the input.

  • Distill reads lake rows and learns from them. The teacher is the prompts-and-responses you already captured. Use this when you have traffic.
  • Compile reads a hand-written spec (rules, prompts, schemas) and produces an artifact without any captured calls. Use this when you are bootstrapping.

Recipe vs specialist

Output kindBytes on diskWhat it is
recipe~5 KB to ~50 KBPure JS module + JSON schemas, no weights. Runs anywhere a JS runtime runs.
specialist~100 MB to ~4 GBRecipe + quantized weights for the backbone. Runs on the runtime targets in devices.

The .kolm artifact format

A .kolm is a deterministic zip. Open one with any zip tool. The layout is fixed.

artifact.kolm/
 manifest.json # name, version, kind, sha256s, runtimes, modalities
 seeds.jsonl # the N representative examples
 recipe.bundle.mjs # the executable. ES module, no deps.
 schema.json # input + output schemas, JSON Schema 2020-12
 weights/ # specialists only
 weights.gguf
 weights.gguf.sig
 receipts/
 spec.json # the source spec
 seeds.json # the seed selection trace
 split.json # train/val/holdout assignments
 train.json # training metrics if any
 recipes.json # recipe compile log
 evals.json # bakeoff results
 export.json # how the bundle was assembled
 signatures.json # ed25519 over every other receipt
 rekor.json # sigstore transparency log entry

Distill from captures

kolm distill --from-captures summarizer --kind recipe
# selecting seeds ... 240 of 4204 rows
# compiling recipe ... ./out/summarizer.kolm (38 KB, 9 receipts signed)

Compile from spec

kolm compile ./summarizer.spec.json --kind specialist --tier 3090
# parsing spec ... 1 input schema, 1 output schema, 3 rules
# distilling ... 22 GPU-min on 3090
# quantizing ... int4 weights, 412 MB
# signing ... 9/9 receipts
# ./out/summarizer.kolm

Verify

Verify replays the receipt chain offline. Never trust an artifact you cannot verify. kolm verify exits non-zero if any signature is invalid, any sha256 is wrong, or any receipt is missing.

kolm verify ./summarizer.kolm
# verify OK. 9/9 receipts signed, sha matches, recipe ESM parses clean.

Run

kolm run ./summarizer.kolm --prompt "Summarize this PR diff."
# This PR refactors the auth middleware to fail-closed on token expiry...

Modes deep dive

Seven distill modes are first-class in the compiler. Each one is plain about its research lineage; every receipt records the mode in spec.json so an auditor can replay the exact pipeline. The seven map one-to-one with the cards on /distill.

REASONING

Reasoning / CoT-trace distillation

Capture full step-by-step traces from the teacher. The student learns the reasoning shape, not just the final answer. Lineage: Hinton, Vinyals, Dean 2015 on soft targets; the DeepSeek-R1-Distill family 2025 is the canonical open reference at 1.5B-70B scale.

$ kolm distill --from-captures --namespace support --base-model Qwen/Qwen2.5-7B-Instruct --reasoning-trace-loss-weight 0.5
# student inherits the trace, not just the label
ON-POLICY

On-policy / self-distillation

Student generates rollouts. A grader (the teacher or a separate judge) scores them. Only the positives flow into the SFT batch. Lineage: the RLHF reward-model family and the more recent self-distill papers that close the loop without a full reinforcement-learning stage.

$ kolm distill onpolicy --namespace support --grader claude-opus-4-7 --base-model Qwen/Qwen2.5-7B-Instruct
# student rolls out, grader scores, only positives flow back
MULTI-TEACHER

Multi-teacher fusion

Per-token best-of across multiple teacher logits. Pair a reasoning teacher, a code teacher, and a generalist into one student. The runner records which teacher won each token; auditable end to end. Lineage: the multi-teacher distillation line; Fukuda et al. 2017 is an early reference.

$ kolm distill --teachers claude-opus-4-7,gpt-4o,deepseek-v4-pro --weights auto --namespace support --base-model Qwen/Qwen2.5-14B-Instruct
# per-token best-of across teachers, attribution preserved
AGENTIC

Agentic workflow distillation

Replay captured multi-step LangGraph or CrewAI runs. Compile the orchestrator plus the tool dialogues into one forward pass on a single student. Reduces inference cost on workflows that repeat. Lineage: the 2025-2026 agent-distill literature.

$ kolm distill --from-captures --namespace workflows --base-model Qwen/Qwen2.5-7B-Instruct
# capture LangGraph/CrewAI rollouts under ns=workflows, then distill the orchestrator + tool calls into one forward
MERGE-FIRST

Merge-first

Run mergekit TIES (Yadav et al. 2023), DARE (Yu et al. 2024), or SLERP on multiple bases or teachers, then distill the merged result. Lineage: mergekit by Charles Goddard; community recipes popularized by mlabonne. The merge config and source SHAs are pinned in the receipt.

$ kolm distill --auto-arch --namespace support --base-model Qwen/Qwen2.5-14B-Instruct
# TAAS searches student-arch + merge recipe; result + source SHAs land in spec.json. Pin a specific merge with --merge-recipe ties|dare|slerp.
MTP

MTP (multi-token prediction)

Architectural option in the student head: predict N tokens per forward and accept the longest consistent prefix. Native speculative decoding without an external draft model. Lineage: the 2025-2026 Alibaba and DeepSeek MTP lines. The runner refuses MTP for student families that have not validated the head.

$ kolm distill --auto-arch --mtp 4 --namespace support --base-model Qwen/Qwen2.5-14B-Instruct
# N=4 default; student head predicts ahead. Runner refuses MTP for families that haven't validated the head.
LOW-BIT

Low-bit specialist

Aggressive quantization with a K-score safety floor. The runner walks 4.0 to 2.0 to 1.58-bit and stops the moment K-score drops below the floor you set. Lineage: BitNet b1.58 (Microsoft Research 2024) and the SpQR family for outlier-aware low-bit. The receipt records the final bit-width and the floor.

$ kolm distill --from-captures --namespace support --base-model Qwen/Qwen2.5-7B-Instruct
$ kolm quantize --local-worker --method=bitnet158 --in ./out/student --out ./out/q158 --kscore-floor 0.85
# distill, then quantize walks 4.0 → 1.58-bit and aborts if K-score breaches floor (BitNet b1.58 + SpQR outlier-aware path)

For the full technique inventory across all four pillars (quantization, distillation, compilation, merges), see /frontier-stack.

Next steps

Once you have an artifact, plug it into runtime so live calls hit it instead of the upstream provider, and run evals to confirm it wins the bakeoff. Start from /docs/quickstart if you have not built a dataset yet.

Training →The training run feeds into the specialist artifact. Runtime →Serve the .kolm to live traffic. Evals →Bakeoff the .kolm against contestants.
