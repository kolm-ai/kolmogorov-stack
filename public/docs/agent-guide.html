Agent guide

kolm.ai exposes natural-language verbs so coding agents can drive the product without a custom integration. This page is what every Claude / Cursor / Devin / Aider session should read before its first edit in a kolm workspace.

Natural-language verbs

Five top-level verbs cover the agentic surface. Each one is intent-classified by src/intent.js and dispatches into the right code path. None of them require an API key for read-only use.

kolm do

Plain-English requests that map to one or more verbs. kolm do "compile a privacy redactor and verify it" resolves into compile + verify.

kolm what

State summary. Lists artifacts, jobs, captures, and pending receipts. kolm what is broken filters to failures only.

kolm next

Suggests the next concrete step against the current workspace. Reads kolm.yaml, the seeds dir, and the last receipt to decide.

kolm explain

Reads an artifact, a receipt, or a failed gate and returns plain English plus the file:line pointer.

kolm fix

Applies a one-shot remediation. Idempotent. Prints the exact diff before touching disk so the calling agent can review.

kolm agent guide

Writes AGENT_GUIDE.md at the project root. Pin this in the agent context window so it sees the local rules every turn.

The AGENT_GUIDE.md generator

Run once per workspace, then commit:

kolm agent guide # writes ./AGENT_GUIDE.md
kolm agent guide --out docs/AI.md # custom path
kolm agent guide --check # exit 1 if guide is stale or absent

The generator inspects kolm.yaml, the recipe set, the privacy membrane configuration, and any frozen-eval pointers, then emits a focused 1-2 page brief. Agents that read it before editing avoid the most common failure modes (re-running kolm compile on every save, editing inside .kolm/, writing receipts by hand).

Rules every agent should follow

Never edit inside .kolm/ or .kolm-state/. These directories are write-once + signed. Tampering invalidates every downstream receipt. Use kolm checkpoint if you need a new revision.
Run kolm verify before claiming an artifact is shipped. The 7-check audit catches manifest-mismatch, broken Ed25519 receipt chains, missing receipts, and K-score gate regressions in under a second.
Do not invent CLI flags. Every flag is enumerated in cli/kolm.js under HELP.<verb>. Read it before writing a shell call.
Treat seeds as the only training input. kolm compile reads exactly seeds.jsonl + the recipe file. Editing the artifact directly is undefined behavior.
Never paste a real API key into the chat. Use kolm login (writes ~/.kolm/config.json) or set KOLM_KEY in the shell. The CLI accepts neither stdin keys nor argv keys.
Capture before optimizing. The capture loop is the moat. Compiling without at least 50 captured pairs in the namespace usually fails the K-score gate.

Suggested workflow for autonomous agents

  1. Read AGENT_GUIDE.md if it exists. If not, run kolm agent guide and commit it.
  2. Run kolm what to learn the workspace state.
  3. Run kolm next to get a concrete suggestion. If the user gave you a goal, run kolm do "<goal>" instead.
  4. Make the smallest reversible change. Avoid editing .kolm/ or kolm.yaml unless the user asked.
  5. Run kolm verify on any touched artifact. Report the exit code verbatim. Do not paraphrase.
  6. If you compiled a new artifact, run kolm replay <namespace> <artifact> --limit 25 to spot regressions against recent captures.

Environment for agents

Set these so the CLI behaves predictably in non-interactive sessions:

export KOLM_NO_REST_HINT=1 # suppress the "REST equivalent" suggestion
export KOLM_DEBUG= # leave empty unless debugging
export KOLM_KEY=<your token> # CI safe-fail: empty string exits 3
export KOLM_PROFILE=default # pick a saved profile if the user has many
