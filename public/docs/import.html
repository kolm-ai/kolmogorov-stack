kolm.ai / docs / import

Open-source AI workbench

Frontier AI on your own infrastructure.

W740 is the symmetric counterpart to kolm export: it takes a runtime-native binary you already own (GGUF from llama.cpp, safetensors from HuggingFace, ONNX from any toolchain) and wraps it as a partial manifest tagged not_kolm_compiled:true. The model file is never modified, never silently re-tagged as a kolm-compiled artifact, and never gains a K-Score until it actually passes through kolm distill again.

Three formats supported

Each format is parsed by a small stdlib-only Python script in apps/import/. Heavy parsers (the official onnx pip package, full gguf 1.0 tensor walking) are deliberately not added to the kolm dependency surface; the import path reads enough of the header to produce honest provenance and nothing more.

FormatMagic / signatureWhat we parse
ggufASCII "GGUF" at offset 0, then u32 LE version, u64 LE tensor_count, u64 LE kv_countmagic, version, tensor_count, kv_count; metadata KV keys (best-effort); general.file_type -> quant label; general.parameter_count -> params_b
safetensorsu64 LE header_size at offset 0, then header_size bytes of UTF-8 JSON, then raw tensor bytesheader_size; tensor names (sorted, capped at 256); shape-product -> total parameter count; dominant dtype -> quant label; __metadata__ map
onnxprotobuf ModelProto; first byte is a wire-type tag (0x08 / 0x12 / 0x3a) because there is no fixed magicbest-effort sniff of the first 1 KiB; ir_version, producer_name, producer_version, domain; marked partial:true with hint onnx_metadata_partial when stdlib protobuf walk cannot reach ir_version

In every case we compute the sha256 of the whole file (streaming, so a 60 GiB model doesn't hold the whole binary in memory), record the size in bytes, and surface the raw metadata keys we recovered. That is enough provenance for the W740-2 wrap; no tensor data is ever read.

Honesty contract: not_kolm_compiled

Every wrapped manifest carries a single load-bearing field:

{
  "ok": true,
  "manifest_version": "w740-v1",
  "manifest_id": "",
  "not_kolm_compiled": true,
  "k_score": null,
  "holdout": null,
  "source_format": "gguf",
  "source_sha256": "<64-hex>",
  "source_path": "/abs/path/to/model.gguf",
  "source_size_bytes": 1939525632,
  "source_params_b": 3.21,
  "source_quant": "q4_k_m",
  "source_metadata_keys": ["general.architecture", "general.file_type", "..."],
  "imported_at": "2026-05-24T12:34:56.789Z",
  "version": "w740-v1"
}

The not_kolm_compiled:true flag is the W740-2 honesty lock. Downstream tools (catalog, inspect, signature, K-Score gates) MUST treat a manifest with this flag set as a non-kolm-compiled artifact: no production-ready verdict, no automatic deploy, no aggregated K-Score. To claim the model is kolm-compiled you have to run it through kolm distill again on real seeds and let it earn a K-Score on a holdout the same way a fresh distill would.

Round-trip: import -> distill -> export

The symmetric pair is intentional. A buyer who already has a GGUF model can land in kolm without losing the file, get the W740-2 honest manifest for the catalog, re-distill on their own seeds, and emit a new GGUF binary that is the kolm-compiled version of the same network.

# 1. Import the existing GGUF  -  emits not_kolm_compiled manifest
kolm import wrap ./llama-3.2-3b-q4_k_m.gguf --out imported.manifest.json

# 2. Re-distill against your captured traffic (this is where the K-Score is earned)
kolm distill --seeds ./seeds.jsonl --base ./llama-3.2-3b-q4_k_m.gguf --out new.kolm

# 3. Export back to GGUF for llama.cpp / Ollama / LM Studio
kolm export new.kolm --backend gguf --quant q4_k_m --out ./exports

Fidelity is teacher-quality-bound, NOT bit-exact. The round-trip is a re-distillation, not a copy. The output GGUF will have different tensor weights from the input GGUF because kolm distill retrains against the teacher and the captured traffic; it does not preserve the exact numbers in the source binary. If you need bit-exact preservation, do not use kolm - copy the file. If you need a model that has actually passed through the kolm K-Score gate, this is the path.

Honest error envelopes

Every failure mode is a named error code with a one-line hint. No silent passthrough, no fake parses, no zero-byte manifests.

ErrorCauseExit code
python3_missingNo python3 (or python) on PATH3 (MISSING_PREREQ) / HTTP 503
file_not_foundPath does not exist5 (NOT_FOUND)
unknown_formatFirst 16 bytes match no known signature1 (BAD_ARGS)
gguf_parse_failedMagic mismatch, truncated header, or implausible kv_count2 (GATE_FAIL) / HTTP 200 (envelope)
safetensors_parse_failedHeader size 0 or beyond EOF; non-JSON header; root not an object2 (GATE_FAIL) / HTTP 200 (envelope)
onnx_parse_failedFirst byte is not a valid ModelProto wire-type tag2 (GATE_FAIL) / HTTP 200 (envelope)
onnx_metadata_partialONNX file looks valid but stdlib walk could not reach ir_version - install the onnx pip package for a full walk0 (envelope still wraps with partial:true)
