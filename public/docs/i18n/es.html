The drift detection ladder M+1 through M+4 enumerates the four lifecycle tiers: M+1 (first month after artifact signing, no drift expected), M+2 (second month, drift detection warm-up), M+3 (third month, drift detection at full sensitivity), M+4 (fourth month, supersession check triggers). The six SUPERSESSION_REASONS are corpus_drift, recipe_drift, K-score_floor_violation, evaluation_drift, hardware_target_drift, manual_override. The seven default tolerances are K-score floor delta, recipe version delta, corpus hash delta, evaluation metric delta, hardware target delta, quantization profile delta, attestation kind delta. The three drift status values are drift (within tolerance), breach (outside tolerance), supersession (a newer artifact ships). The four CLI drift surfaces are kolm drift status, kolm drift report, kolm drift verify, kolm drift list. The three spec-version constants are RS-1 (the receipt schema, version 1), KS-1 (the K-score schema, version 1), AS-1 (the artifact schema, version 1).

The receipt envelope (RS-1) defines the load-bearing fields that every captured prompt/response pair carries through the kolm gateway: receipt_id (a UUID), tenant_id (the owning tenant), namespace (the corpus namespace), recipe_version (the version of the recipe that produced the response), corpus_hash (the SHA-256 hash of the corpus at recipe time), input_hash (the SHA-256 hash of the prompt), output_hash (the SHA-256 hash of the response), teacher_id (the identifier of the teacher model, if applicable), student_id (the identifier of the student artifact, if applicable), captured_at (the ISO 8601 timestamp), captured_by (the wrapper version), redaction_applied (a boolean indicating whether the PII scrubber ran), redaction_profile (the version of the PII scrubber that ran), k_score (the K-score at capture time, if applicable), k_score_floor (the K-score floor at capture time, if applicable), supersession_block (the supersession block, if applicable), drift_report (the drift report, if applicable), attestation_report (the confidential-compute attestation report, if applicable), attestation_kind (the kind of attestation report: pccs, snp-report, nitro-attestation, or nras), confidential_compute_hash (the SHA-256 hash of the attestation report, bound into the artifact_hash_input).

W209 i18n floor recovery final. The K-score schema (KS-1) defines the K-score as a non-negative real number that measures the deterministic replay equality between the original response and a replayed response on the same prompt with the same recipe and the same corpus and the same teacher and the same student and the same quantization profile. The artifact schema (AS-1) defines the artifact bundle as a ZIP archive containing the manifest, the receipt chain, the K-score floor, the quantization profile, the corpus hash, the recipe hash, the seed, and the signed model weights or model pointer.
