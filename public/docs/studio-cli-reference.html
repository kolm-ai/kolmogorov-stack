Studio / CLI reference

Every Studio verb. Every flag.

Studio is fully driven from the CLI — the web UI at /studio calls the same code paths. This page is the canonical reference for the eight Studio verbs: distill, compile, export, publish, bench, experts, inspect, hardware. Global flags listed once at the top apply to every verb.

Global flags

FlagMeaning
--jsonEmit machine-readable envelope; no human prose.
--plainNo colour, no Unicode glyphs; ASCII only.
--no-color / --no-unicodeNarrower forms of --plain.
--dry-runValidate + print plan, no side effects, no provider calls.
--resumePick up an interrupted run from the last receipt.
--verbose / -vPrint per-step timings + memory snapshots.
--workdir <dir>Override the scratch dir (default ~/.kolm/runs/).

kolm distill

kolm distill [--recipe <name> | --recipe <path.toml>] [options]

  --namespace <ns>            Pull captures from this namespace
  --since-last-distill        Only new captures since the previous distill
  --teacher <provider/model>  Add or override teacher (repeatable, weight via @0.4)
  --judge <provider/model>    Override judge model
  --student <base>            Override base in [student]
  --method sft|seq-kd|distillm-2|dpo|token-dpo
  --output <file.kolm>        Output artifact path
  --max-pairs N               Cap training pairs (after judge filter)
  --teacher-concurrency N     Default 4
  --list-recipes              Print catalogue (use with --json for machine output)
  --list-teachers             Print supported teacher catalogue
  --list-rubrics              Print judge rubric catalogue
  --dump                      Emit the resolved recipe as TOML to stdout
  --cloud modal|runpod        Run on hosted GPU under your account
  --seed N                    Reproducibility (default 42)

kolm compile

kolm compile (<artifact.kolm> | --recipe <name>) [options]

  --target <target>           kolm | gguf-q4km | exl2-4.0bpw | gptq-4bit | awq-4bit |
                              fp8 | nvfp4 | hqq-4bit | mlx-4bit (also q2k..q8, iq2s, iq4nl, etc.)
  --output <file>             Output path
  --adapter <dir>             LoRA adapter to merge (if not from a .kolm)
  --base <hf-or-path>         Base model id (if not from a .kolm)
  --spec <file.toml>          spec.toml (if not from a .kolm)
  --imatrix <file>            Use existing imatrix (IQ-quants only)
  --imatrix-calibration <f>   Generate fresh imatrix from this calibration set
  --moe-policy <p>            uniform | router-high | router-very-high (MoE only)
  --split-shard-size <sz>     Force GGUF split-file at this size (e.g. 24G)
  --low-ram-merge             Merge per-shard at ~2x wall time

kolm export

kolm export <artifact.kolm> --format <list> [options]

  --format f1,f2,...          Comma list (see compile --target values)
  --output <file>             Single output (one format only)
  --output-dir <dir>          Multi-format output destination
  --ollama-modelfile          Also emit Ollama Modelfile
  --hf-model-card             Also emit HF README.md

kolm publish

kolm publish hf <artifact.kolm> --repo <user-or-org/slug> [options]

  --formats f1,f2,...         Default: gguf-q4km,gguf-q5km,gguf-q8,fp16-kolm
  --private                   HF repo private
  --gated                     HF gated repo
  --card-extra <file.md>      Append to auto-generated card body
  --also-kolm-registry        Dual-publish to kolm registry
  --ignore-teacher-license-warnings   You've cleared distribution with each provider

kolm publish registry <artifact.kolm> [options]
  --visibility public|unlisted|private
  --tags t1,t2,...

kolm bench

kolm bench --models m1,m2,... --eval <file.jsonl> [options]
kolm bench --compare <a.kolm> <b.kolm> --eval <file.jsonl>

  --judge <provider/model>    Default: google/gemini-2.5-pro
  --rubric <name>             Default: derived from artifact passport
  --n N                       Prompt count (default: all in eval)
  --judges N                  Per-prompt judge calls (default 1; 3 for high-confidence)
  --output <file.json>        Machine output
  --markdown <file.md>        Human-readable comparison table
  --cost-per-kwh <rate>       Electricity rate for local-model cost estimate

kolm experts

kolm experts <artifact.kolm> [options]

  --eval <file.jsonl>         Run profiling on this eval set
  --prune-below <frac>        Prune experts firing on less than this fraction
  --output <artifact.kolm>    Pruned artifact destination
  --activations-only          Skip pruning, just print activations
  --json                      Machine output

kolm inspect

kolm inspect <hf-id | path.kolm | path.gguf> [options]

  --json                      Machine output
  --estimate-vram <quant>     Forecast VRAM at this quant level
  --layers                    Print per-layer parameter counts

kolm hardware

kolm hardware detect              Print local GPU/CPU/RAM
kolm hardware fit <model> --vram N   What quant fits in N GB?
kolm hardware throughput <artifact>  Live tok/s on local hw

Exit codes

CodeMeaning
0Success
1Generic error
2Bad arguments
3Missing prerequisite (env var, file, GPU)
4Validation failure (schema, signature, hash)
5Network / provider error
6OOM (CPU or VRAM)
Caveats. Flags marked optional are stable; flags marked with (beta) in kolm <verb> --help are subject to change before v2. Always pin a CLI version in CI: kolm --version in the run log.
