C SDK quickstart.

The kolm C SDK is a single-header stb-style library. You drop kolm.h into your project, define KOLM_IMPLEMENTATION in exactly one translation unit, link libcurl, and the rest is a thin function-call API. No bundled JSON parser — pair with cJSON or your own. This page takes you from header drop to your first captured request in five minutes.

Install

Download the single header. The file lives at sdk/c/kolm.h in the kolmogorov-stack repo.

curl -fsSL https://raw.githubusercontent.com/sneaky-hippo/kolmogorov-stack/main/sdk/c/kolm.h \
  -o kolm.h

You also need libcurl. On Debian/Ubuntu: apt install libcurl4-openssl-dev. On macOS with Homebrew: brew install curl. On Windows: vcpkg install curl.

Requires C99 or newer. The implementation guard pattern means KOLM_IMPLEMENTATION must be defined in exactly one .c file across your project.

Authenticate

The library reads KOLM_API_KEY from the environment by default. Get a key from the CLI or from the dashboard.

# option 1: log in with the CLI
kolm login

# option 2: set the env var directly
export KOLM_API_KEY=ks_your_key_here
export KOLM_BASE_URL=https://kolm.ai   # optional, defaults to https://kolm.ai

Sign up for a free key at kolm.ai/signup if you do not have one yet.

First capture

Create first_capture.c. Define KOLM_IMPLEMENTATION exactly once. Build with libcurl linked.

#define KOLM_IMPLEMENTATION
#include "kolm.h"
#include <stdio.h>

int main(void) {
    kolm_t k = kolm_from_env();   /* reads KOLM_API_KEY + KOLM_BASE_URL */

    kolm_response_t who = kolm_whoami(&k);
    printf("authed: status=%d body=%s\n", who.status, who.body);
    kolm_response_free(&who);

    const char *body =
        "{\"namespace\":\"quickstart\","
         "\"input\":{\"prompt\":\"What is the capital of France?\"},"
         "\"output\":{\"reply\":\"Paris\"},"
         "\"meta\":{\"source\":\"c-quickstart\"}}";

    kolm_response_t cap = kolm_capture_log(&k, body);
    printf("captured: status=%d body=%s\n", cap.status, cap.body);
    kolm_response_free(&cap);

    kolm_free(&k);
    return 0;
}

Build and run:

cc -std=c99 first_capture.c -lcurl -o first_capture
./first_capture
# authed: status=200 body={"ok":true,"tenant_id":"tenant_f33c240034c5",...}
# captured: status=200 body={"ok":true,"id":"cap_a1b2c3d4..."}

Every response returns { int status; char *body; }. Always pair the call with kolm_response_free — the body is heap-allocated.

First deploy (outline)

The compile-and-deploy endpoints are HTTP-shaped; the library exposes kolm_post for any path. From C:

const char *job_body =
    "{\"namespace\":\"quickstart\","
     "\"target\":\"qwen2.5-3b-instruct-int4\","
     "\"epochs\":3}";

kolm_response_t job = kolm_post(&k, "/v1/compile", job_body);
printf("compile job: %s\n", job.body);
kolm_response_free(&job);

/* poll /v1/compile/<job_id>/status until done,
 * or call: kolm compile wait $JOB_ID from a shell.
 */

const char *dep_body =
    "{\"artifact_cid\":\"<cid-from-compile>\","
     "\"runtime\":\"vllm\","
     "\"endpoint\":\"https://gpu.internal:8000\"}";

kolm_response_t dep = kolm_post(&k, "/v1/deploy", dep_body);
printf("deployed: %s\n", dep.body);
kolm_response_free(&dep);
