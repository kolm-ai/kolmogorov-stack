Docs / Quickstart / SDKs

Open-source AI workbench

Pick your SDK. Ship a capture in five minutes.

Each quickstart installs the SDK, authenticates against kolm.ai, runs your first capture, and outlines the compile-and-deploy path. The SDKs all wrap the same HTTP API, so picking one is just about matching your stack. All quickstart code compiles and runs as-is.

Pick your SDK

Node SDK

TypeScript-first SDK for Node 18+. Pairs with Express, Hono, Next.js, or any OpenAI-compatible client.

npm i @kolmogorov/sdk

Python SDK

Sync and async client for Python 3.10+. Drops into FastAPI, LangChain, or a notebook.

pip install git+https://github.com/sneaky-hippo/kolmogorov-stack@main#subdirectory=sdk/python

Rust SDK

Sync ureq-backed crate for Rust 1.75+. No async runtime required. Honest envelope.

cargo add kolm

C SDK

Single-header stb-style library for C99+. libcurl backend. Bring your own JSON parser.

#define KOLM_IMPLEMENTATION

MCP SDK

Model Context Protocol server that exposes kolm.ai capture, compile, and deploy as MCP tools.

npx @kolmogorov/mcp

CLI-first quickstart

If you would rather start at the terminal without an SDK in your code, the five-minute CLI quickstart walks the same loop using kolm capture log and kolm compile directly.

What to do after the quickstart

  • Cookbook recipes — full capture-distill-compile-deploy walkthroughs for the three workloads most teams distill first.
  • HTTP API reference — the surface every SDK wraps.
  • CLI reference — every flag, every command.
  • K-Score methodology — the score that decides ship-or-no-ship.
