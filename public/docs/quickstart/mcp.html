MCP SDK quickstart.

The kolm MCP SDK exposes the kolm.ai surface as a Model Context Protocol server. Once registered with an MCP-aware client (Claude Desktop, Cursor, Continue, or any custom agent), the assistant can call kolm_capture, kolm_compile, kolm_deploy, and kolm_verify as native tools. This page takes you from npx install to the first agent-driven capture in five minutes.

Install

The MCP server is published on npm as @kolmogorov/mcp. The easiest path is to run it with npx so it stays current.

npx @kolmogorov/mcp --version
# 0.1.x

# or install globally
npm install -g @kolmogorov/mcp
kolm-mcp --version

Requires Node 18 or newer. The server speaks MCP over stdio by default; pass --http for HTTP mode.

Authenticate

The MCP server reads KOLM_API_KEY from the environment by default. Set the key once at the OS level (or in the client's MCP config) and every tool call inherits it.

# option 1: log in with the CLI (writes ~/.kolm/auth.json)
kolm login

# option 2: set the env var directly
export KOLM_API_KEY=ks_your_key_here
export KOLM_BASE_URL=https://kolm.ai   # optional, defaults to https://kolm.ai

Sign up for a free key at kolm.ai/signup if you do not have one yet.

Register with your MCP client

Most MCP clients read a JSON config that maps server name to command. Drop this block into your client's MCP config file.

{
  "mcpServers": {
    "kolm": {
      "command": "npx",
      "args": ["-y", "@kolmogorov/mcp"],
      "env": {
        "KOLM_API_KEY": "ks_your_key_here"
      }
    }
  }
}

Restart the client. The kolm tools (kolm_whoami, kolm_capture, kolm_compile, kolm_deploy, kolm_verify) will show up in the tool palette.

First capture (agent-driven)

Once registered, prompt the agent to capture a request. Example prompts:

You: "Use the kolm tools to log a capture in the 'quickstart' namespace
      with input { prompt: 'capital of France' } and output { reply: 'Paris' }."

Agent: [calls kolm_capture]
       Captured to namespace 'quickstart' (id: cap_a1b2c3d4...).

For testing without an agent, you can call the MCP server directly from a script using the MCP TypeScript client:

import { Client } from "@modelcontextprotocol/sdk/client";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio";

const client = new Client({ name: "test", version: "0.0.1" }, {});
await client.connect(new StdioClientTransport({
  command: "npx",
  args: ["-y", "@kolmogorov/mcp"],
}));

const result = await client.callTool({
  name: "kolm_capture",
  arguments: {
    namespace: "quickstart",
    input:  { prompt: "What is the capital of France?" },
    output: { reply:  "Paris" },
  },
});
console.log(result);

First deploy (outline)

The agent can drive the compile-and-deploy path end-to-end. The tools are kolm_compile, kolm_compile_wait, and kolm_deploy. A typical agent transcript:

You: "Compile the 'quickstart' namespace to qwen2.5-3b-instruct-int4
      with 3 epochs and deploy to https://gpu.internal:8000 on vllm."

Agent: [calls kolm_compile] -> job_a1b2c3...
       [calls kolm_compile_wait] -> artifact cid bafy..., k_score 0.84
       [calls kolm_verify]       -> signed, attestation ok
       [calls kolm_deploy]       -> endpoint live
       Done. Artifact bafy... is serving at https://gpu.internal:8000.

Compile is the long step — the agent will sit on kolm_compile_wait until the job completes.
