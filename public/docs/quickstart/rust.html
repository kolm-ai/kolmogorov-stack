Rust SDK quickstart.

The kolm Rust crate is a synchronous client built on ureq. It does not bring an async runtime, does not hide non-2xx responses behind a transport error, and pairs cleanly with serde_json for response parsing. This page takes you from cargo add to your first captured request in five minutes.

Install

Add the crate to your project with cargo.

cargo add kolm
cargo add serde_json   # for parsing responses

Requires Rust 1.75 or newer. The crate is sync; pair with tokio::task::spawn_blocking if you need to call it from an async context.

Authenticate

The crate reads KOLM_API_KEY from the environment by default. Get a key from the CLI or from the dashboard.

# option 1: log in with the CLI
kolm login

# option 2: set the env var directly
export KOLM_API_KEY=ks_your_key_here
export KOLM_BASE_URL=https://kolm.ai   # optional, defaults to https://kolm.ai

Sign up for a free key at kolm.ai/signup if you do not have one yet.

First capture

Create a new binary with cargo new first-capture, replace src/main.rs, and run.

use kolm::Kolm;
use serde_json::json;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let kolm = Kolm::from_env()?;   // reads KOLM_API_KEY + KOLM_BASE_URL

    let whoami = kolm.whoami()?;
    println!("authed: status={} body={}", whoami.status, whoami.body);

    let body = json!({
        "namespace": "quickstart",
        "input":  { "prompt": "What is the capital of France?" },
        "output": { "reply":  "Paris" },
        "meta":   { "source": "rust-quickstart" }
    });

    let captured = kolm.capture_log(&body.to_string())?;
    println!("captured: status={} body={}", captured.status, captured.body);
    Ok(())
}

Run it:

cargo run
# authed: status=200 body={"ok":true,"tenant_id":"tenant_f33c240034c5",...}
# captured: status=200 body={"ok":true,"id":"cap_a1b2c3d4..."}

Note the honest envelope: every response returns { status, body }. A 4xx is not a transport error, it is a response. Parse with serde_json on the caller side.

First deploy (outline)

The compile-and-deploy path is HTTP-shaped; the SDK exposes thin wrappers. From a Rust script:

let job = kolm.compile_create(&json!({
    "namespace": "quickstart",
    "target":    "qwen2.5-3b-instruct-int4",
    "epochs":    3
}).to_string())?;
println!("compile job: {}", job.body);

// poll kolm.compile_status(job_id) until done, or call the CLI:
//   kolm compile wait $JOB_ID

let deploy = kolm.deploy_create(&json!({
    "artifact_cid": "<cid-from-compile>",
    "runtime":      "vllm",
    "endpoint":     "https://gpu.internal:8000"
}).to_string())?;
println!("deployed: {}", deploy.body);

The crate is intentionally thin around the HTTP surface so you can drop down to the raw API when needed.
