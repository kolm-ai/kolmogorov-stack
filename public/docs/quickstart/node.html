Node SDK quickstart.

The kolm Node SDK is a TypeScript-first wrapper around the kolm.ai HTTP API. It runs on Node 18+, ships with full type definitions, and is drop-in compatible with Express, Hono, Next.js route handlers, and any code that already uses an OpenAI-compatible client. This page takes you from npm install to your first captured request in five minutes.

Install

Install the SDK from npm. The package is published as @kolmogorov/sdk.

npm install @kolmogorov/sdk
# or
pnpm add @kolmogorov/sdk
# or
yarn add @kolmogorov/sdk

Requires Node 18 or newer for native fetch. Works in both ESM and CommonJS projects.

Authenticate

The SDK reads KOLM_API_KEY from the environment by default. Get a key from the CLI or from the dashboard.

# option 1: log in with the CLI
kolm login

# option 2: set the env var directly
export KOLM_API_KEY=ks_your_key_here
export KOLM_BASE_URL=https://kolm.ai   # optional, defaults to https://kolm.ai

Sign up for a free key at kolm.ai/signup if you do not have one yet.

First capture

Create first-capture.mjs and run it. The snippet captures a single request into the quickstart namespace, then prints the captured row's id.

import { Kolm } from "@kolmogorov/sdk";

const kolm = new Kolm();   // reads KOLM_API_KEY + KOLM_BASE_URL

const me = await kolm.whoami();
console.log("authed as", me.tenant_id);

const captured = await kolm.capture.log({
  namespace: "quickstart",
  input:  { prompt: "What is the capital of France?" },
  output: { reply:  "Paris" },
  meta:   { source: "node-quickstart" },
});

console.log("captured id:", captured.id);

Run it:

node first-capture.mjs
# authed as tenant_f33c240034c5
# captured id: cap_a1b2c3d4...

First deploy (outline)

Once you have a few thousand captures in the namespace, the deploy path looks like this from a Node script. The same flow runs from the CLI; this is the SDK shape.

const job = await kolm.compile.create({
  namespace: "quickstart",
  target:    "qwen2.5-3b-instruct-int4",
  epochs:    3,
});
console.log("compile job:", job.id);

// poll until done, or use kolm.compile.wait(job.id)
const artifact = await kolm.compile.wait(job.id);
console.log("artifact cid:", artifact.cid, "k_score:", artifact.k_score);

const deploy = await kolm.deploy.create({
  artifact_cid: artifact.cid,
  runtime:      "vllm",
  endpoint:     "https://gpu.internal:8000",
});
console.log("deployed:", deploy.endpoint);

Compile is the long step. The SDK exposes kolm.compile.wait(id) as a convenience for polling.
