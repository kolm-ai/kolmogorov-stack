Run & Govern

Fleet.

A fleet is every registered device viewed at once. kolm fleet verbs select a subset by tag or namespace and operate on all of them in parallel — deploy, monitor, rollback, stop — with the same per-device guarantees as a single kolm deploy.

One-line example

kolm fleet deploy art_a1b2c3d4 --select tag=prod --rolling

Picks every device tagged prod, then runs the rolling deploy pipeline on each in parallel. Output is a per-device summary plus a fleet-wide status line.

Verbs

VerbWhat it does
kolm fleet statusTabular roll-up: device, artifact, namespace, health, p95, tok/s, last-seen. --json for machine output.
kolm fleet deploy <artifact>Push an artifact to all selected devices. Honors --rolling and --canary.
kolm fleet monitorStreams a live table; updates on each device's check-in interval (default 30s).
kolm fleet rollbackPer-device rollback to the previous artifact for that namespace.
kolm fleet stopStops the pod on every selected device. Use --all to skip the selector and stop everything.
kolm fleet updatesLists pending OTA updates per device (see updates).

Selectors

Every kolm fleet verb accepts the same selector syntax. Selectors AND together. Use --all to skip selection entirely.

SelectorMatches
--select tag=<label>Any device with the tag. Repeat for AND.
--select namespace=<name>Devices registered to the namespace.
--select type=<ssh|k8s|docker|ollama|local>By device type.
--select hardware.gpu=nvidiaBy auto-detected hardware field.
--exclude tag=<label>Negative selector. Repeat as needed.
--allEvery device in the registry.

fleet status

kolm fleet status --select tag=prod

prod-gpu-1      art_a1b2c3d4      prod  ok      318   41.2   12s ago
prod-gpu-2      art_a1b2c3d4      prod  ok      302   43.8   8s ago
prod-gpu-3      art_b2c3d4e5      prod  warn    612   24.1   31s ago
prod-k8s        art_a1b2c3d4      prod  ok      284   46.0   4s ago

Batch deploy

Per-device steps still run in serial within a device (preflight → upload → runtime → start → smoke → record), but devices run in parallel up to --parallel (default 4). On any per-device failure the fleet logs the error and continues; pass --halt-on-error to abort the whole batch.

kolm fleet deploy art_b2c3d4e5 \
  --select tag=prod --select hardware.gpu=nvidia \
  --rolling --parallel 8

# per-device output streams as it lands; final summary:
# 7/8 ok / 1 warn (prod-gpu-3: smoke p95 612ms over budget)
# deploy receipts: dpl_018f2a... dpl_018f2b... ...

Fleet-wide canary

A fleet canary deploys to a fraction of devices first, monitors, then expands. This is distinct from the per-device canary in deploy (which splits traffic on one device).

kolm fleet deploy art_b2c3d4e5 \
  --select tag=prod \
  --fleet-canary 20 --monitor 30m

# step 1: deploy to 20% of selected devices (rounded up)
# step 2: monitor 30m -- gateway compares K-Score / errors / p95 vs prior version
# step 3: if green, expand to 100%; if red, rollback the canary cohort

Fleet rollback and stop

kolm fleet rollback --select tag=prod
# per-device rollback to the previous artifact in each namespace

kolm fleet stop --select tag=prod
# graceful shutdown of every selected pod

kolm fleet stop --all --emergency
# ungraceful: SIGKILL after 5s if SIGTERM not honored
Constraints. Selectors operate on the local device registry — a fleet verb cannot deploy to a device that kolm devices add has not registered. K8s device entries select pods within the configured context+namespace, not the whole cluster. Fleet-canary needs at least one device matching the selector after the canary cohort is removed; on smaller fleets prefer per-device canary.
