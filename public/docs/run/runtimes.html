Run & Govern

Runtime install.

A .kolm artifact declares which runtime it expects (llama.cpp, vLLM, Ollama, mlx-lm, TGI, TRT-LLM). On deploy, kolm checks the device for the matching runtime + version; if missing, it installs from the per-OS command below; if mismatched, it warns. Air-gapped flows skip the install step and require manual provisioning.

One-line example

kolm devices runtime install vllm --device prod-gpu-1
# verifies cuda >= 12.1, pip-installs vllm in /opt/kolm/venv, smoke-tests with a tiny model

Per-OS install commands

llama.cpp

# Ubuntu / Debian (CPU)
apt-get install -y build-essential cmake git
git clone --depth=1 https://github.com/ggerganov/llama.cpp /opt/llama.cpp
make -C /opt/llama.cpp -j$(nproc)

# Ubuntu (CUDA)
LLAMA_CUDA=1 make -C /opt/llama.cpp -j$(nproc)

# macOS (Apple Silicon)
brew install cmake
git clone --depth=1 https://github.com/ggerganov/llama.cpp /opt/llama.cpp
make -C /opt/llama.cpp -j$(sysctl -n hw.ncpu)  # Metal auto-enabled

vLLM

# Ubuntu + NVIDIA
python3 -m venv /opt/kolm/venv
/opt/kolm/venv/bin/pip install --upgrade pip
/opt/kolm/venv/bin/pip install 'vllm==0.6.3'

# Verify
/opt/kolm/venv/bin/python -c 'import vllm; print(vllm.__version__)'

Ollama

# Linux: install via ollama.com (curl install script published by Ollama)
# See https://ollama.com/download for the current command

# macOS
brew install ollama
brew services start ollama

# Verify
ollama --version

mlx-lm (Apple Silicon)

python3 -m venv /opt/kolm/venv
/opt/kolm/venv/bin/pip install --upgrade pip
/opt/kolm/venv/bin/pip install 'mlx-lm>=0.21'

/opt/kolm/venv/bin/python -c 'import mlx_lm; print("ok")'

TGI (Text Generation Inference)

# Docker
docker pull ghcr.io/huggingface/text-generation-inference:2.4.0

# Verify
docker run --rm ghcr.io/huggingface/text-generation-inference:2.4.0 text-generation-launcher --version

TensorRT-LLM

# Ubuntu + NVIDIA Hopper/Ada
python3 -m venv /opt/kolm/venv
/opt/kolm/venv/bin/pip install --upgrade pip
/opt/kolm/venv/bin/pip install 'tensorrt_llm==0.16.0' --extra-index-url https://pypi.nvidia.com

Version check

Every deploy runs runtime ensure (step 3 of the pipeline) which calls --version on the runtime binary, parses, and compares to the artifact's passport minimum. If the installed version is lower, kolm either upgrades (default) or aborts (with --no-install).

# Force a re-check without deploying
kolm devices runtime check --device prod-gpu-1

DEVICE       RUNTIME    INSTALLED  REQUIRED   STATE
prod-gpu-1   vllm       0.6.3      0.6.3      ok
prod-gpu-1   llama.cpp  b3210      b3000+     ok
prod-gpu-1   trt-llm    -          0.16+      missing

Manual install fallback

For air-gapped or operator-controlled hosts, run the per-OS commands above by hand, then tell kolm to skip the install step:

# Once installed manually, register the path
kolm devices runtime register vllm --device prod-gpu-1 \
  --binary /opt/kolm/venv/bin/python --module vllm \
  --version 0.6.3

# Then deploy with --no-install
kolm deploy art_a1b2c3d4 --device prod-gpu-1 --no-install

Upgrade and rollback

kolm devices runtime upgrade vllm --device prod-gpu-1 --to 0.6.4
kolm devices runtime rollback vllm --device prod-gpu-1
# rollback restores the previously recorded version from /opt/kolm/runtime-history.json
Limitations. Auto-install requires outbound network from the device to the upstream package mirror (pip, apt, GitHub, Hugging Face). On hostile / closed networks set up an internal mirror and point at it via kolm config set runtime.pip-index / runtime.apt-mirror / runtime.git-mirror, or pre-install manually and use --no-install. Runtime upgrades on a live device do not auto-restart pods — redeploy or use kolm fleet deploy --redeploy to pick up the new version.
