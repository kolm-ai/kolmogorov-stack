Run & Govern

kolm serve.

A single command spawns an OpenAI-compatible pod from a signed .kolm artifact. The runtime is selected automatically based on what the host has installed; override per-pod for split-brain hosts. Every request writes a receipt; every receipt is verifiable.

Quickstart

kolm serve art_a1b2c3d4 --port 8000

# Smoke from another shell:
curl http://localhost:8000/v1/chat/completions \
  -H 'content-type: application/json' \
  -d '{"model":"art_a1b2c3d4","messages":[{"role":"user","content":"ping"}]}'

Flags

FlagDefaultWhat it does
--port8000Bind port. Multiple pods on one host should use distinct ports.
--runtimeautoOne of auto|vllm|sglang|tgi|llama-cpp|trt-llm. See hardware.
--namespacedefaultLogical routing target. The gateway routes to a namespace; the namespace points to a pod.
--max-concurrency32In-flight request cap. Excess returns HTTP 429 with a retry-after.
--reportonReports health + latency to your tenant. Set --report off for air-gapped pods.
--passportverifyVerifies the runtime passport against the artifact manifest before binding the port.
--optimizeoffHardware-aware preset: turns on speculative auto, Shard KV (if available), prompt cache, and a VRAM-keyed batch size. Any explicit flag wins. See Speed optimization.
--speculative auto|<model>offSpeculative decoding via a small draft model. auto picks a tokenizer-matching draft. 1.5–2.4× tok/s on accepting workloads. Details.
--kv-cache shard|standardstandardUse the Shard transformers.Cache subclass for ~10× KV compression. HF runtime only; vLLM/SGLang/TGI/TRT-LLM fall back to standard with a stderr warning. Details.
--prompt-cache auto|on|offautovLLM enable_prefix_caching. Repeated system prompts and few-shot prefixes hit cache. Details.
--max-batch N8vLLM max_num_seqs — the continuous-batching depth. Details.

Endpoints

Every pod exposes the same surface:

  • POST /v1/chat/completions - OpenAI Chat Completions, streaming + non-streaming.
  • POST /v1/completions - legacy completions (text only).
  • POST /v1/embeddings - only if the artifact ships an embedding head.
  • GET /health - liveness + readiness, returns artifact id + runtime build.
  • GET /metrics - Prometheus exposition.
  • GET /v1/verify/<rcpt> - receipt verification, served locally so air-gapped pods can answer too.

Multi-pod and namespaces

A pod is one process; a namespace is the routing handle. To run two pods of the same artifact for A/B, use distinct namespaces and let the gateway split traffic:

kolm serve art_a1b2c3d4 --port 8000 --namespace prod-a &
kolm serve art_e5f6g7h8 --port 8001 --namespace prod-b &

# Gateway weighted split (see /docs/gateway-confidence-router):
kolm gateway route set prod --split prod-a=0.9,prod-b=0.1
