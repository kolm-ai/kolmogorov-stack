Run & Govern / Reference

CLI reference.

Every verb under the Run & Govern surface. All commands accept --json for machine-readable output, --namespace when the operation is namespace-scoped, and exit with status 0 on success or a non-zero code that the FAQ documents. The CLI binary is kolm (installed via npm i -g @kolm/cli or via the global installer in the Studio surface).

kolm serve

Auto-detect the runtime, spawn it bound to the artifact, and optionally start the Prometheus sidecar.

kolm serve <artifact-id-or-path> [options]

Options:
  --runtime <name>       Force runtime (llama.cpp | vllm | mlx | ollama).
                         Default: auto-detect from passport + host probe.
  --port <n>             Bind port (default 8000).
  --gpu-count <n>        Override detected GPU count (vLLM tensor parallel).
  --metrics-port <n>     Start sidecar on this port (default off).
  --json                 Emit invocation envelope as JSON and exit (no spawn).

kolm passport

Read, estimate, or record a runtime passport row on an artifact.

kolm passport show <artifact-id>
kolm passport estimate <artifact-id> --runtime <name> --target <id>
kolm passport record  <artifact-id> --runtime <name> --target <id> \
                       --tokens-per-second <n> --p50-latency-ms <n> \
                       --p99-latency-ms <n> --gpu-mem-gb <n> \
                       --concurrent-requests <n> --measured-at <iso>

estimate stamps status: estimated and never overwrites an existing tested row. record stamps status: tested with the measurement metadata.

kolm lifecycle

Drive the seven-state machine for an artifact.

kolm lifecycle show       <artifact-id>
kolm lifecycle transition <artifact-id> --to <state> [--reason <text>]
kolm lifecycle history    <artifact-id>

States: created | signed | deployed | monitored | superseded | revoked | archived

Transitions validate against VALID_TRANSITIONS. An illegal move exits with code 4 and a JSON-shaped error: {ok: false, error: 'invalid_transition', from, to, valid_next: [...]}.

kolm evidence

Inspect and operate on the evidence DAG.

kolm evidence show     <artifact-id>
kolm evidence trace    <artifact-id> --node <node-id>
kolm evidence descend  <artifact-id> --node <node-id>
kolm evidence revoke   <artifact-id> --node <node-id> [--reason <text>]
kolm evidence add      <artifact-id> --node-json '{...}' --edge-json '{...}'

revoke emits {revoked: [id], needs_review: [...descendants]} and persists the revocation event as a new invalidates edge so the DAG retains an audit trail.

kolm assurance

Build the assurance case envelope and optionally render the PDF.

kolm assurance build    <artifact-id> [--workspace <id>] [--output <path>]
kolm assurance render   <artifact-id> --format pdf --output <path>
kolm assurance validate <envelope.json>

build writes a kolm-assurance-case-1 JSON envelope. render chains build plus the PDF renderer. validate runs the shape invariants and exits 0 if the envelope is well-formed.

kolm drift

Compute the three drift signals against a baseline window.

kolm drift compute  --namespace <ns> [--lookback-days <n>] [--baseline-days <n>]
kolm drift enable   --namespace <ns>
kolm drift disable  --namespace <ns>
kolm drift status   --namespace <ns>

Output includes fallback_rate_delta, distribution_distance, volume_ratio, a status of ok / warn / alert, and on warn/alert a literal recommendation command (typically kolm distill --priority-captures fallback_eligible --limit 200).

kolm cost

Per-namespace, per-receipt cost displacement.

kolm cost displacement --namespace <ns> [--period-days <n>] [--artifact-id <id>]
kolm cost summary      --namespace <ns>  # rollup last 30 days

Returns baseline_cost_usd (frontier counterfactual), actual_cost_usd, savings_usd, cumulative_savings_usd since deploy, and payback_period_months.

kolm deploy

Emit deployment configs without spawning a runtime.

kolm deploy --target compose  --artifact <id> [--runtime <name>] [--port <n>]
kolm deploy --target k8s      --artifact <id> [--namespace <ns>] [--gpu-count <n>]
kolm deploy --target vllm     --artifact <id> [--tensor-parallel <n>]
kolm deploy --target airgap   --artifact-path <path> --output-dir <dir>

Compose, k8s, and vllm write to stdout by default (pipe to kubectl apply -f - or to a file). Air-gap writes a real tar.gz to --output-dir.

Global flags

FlagEffect
--jsonEmit a JSON envelope on stdout. Default is human-readable.
--namespace <ns>Override the namespace for namespace-scoped verbs.
--tenant <id>Override the tenant (admin-only; rejected for normal API keys).
--no-colorStrip ANSI color from human output.
--no-unicodeUse ASCII-only glyphs in human output.
-h, --helpPer-verb help.

Exit codes

CodeMeaning
0Success.
1Generic failure (read the JSON envelope for error).
2Bad arguments (missing required flag, invalid value).
3Auth missing or rejected. Run kolm login or set KOLM_API_KEY.
4Validation failure (invalid lifecycle transition, DAG cycle, bad envelope).
5Resource not found (artifact id, evidence node, namespace).
JSON contract. Every --json envelope carries {ok: true|false, version, ...payload}. On ok: false there is always an error string and usually a hint string with the literal command to run next. Pipe through jq or pin specific fields with --filter.
