kolm.ai / docs / gateway-mode

Open-source AI workbench

Frontier AI on your own infrastructure.

KOLM_GATEWAY_MODE switches the /v1/chat/completions hot-path between four backends: the cloud teacher, a local Ollama instance, a local vLLM server, or a deterministic mock. Mock mode is free and never bills hosted_inference. local-ollama and local-vllm never send a byte of your prompt outside your network — the requests resolve to localhost (or your private GPU box) and the kolm cloud is not in the path.

Why a gateway-mode switch

The default cloud-teacher path is great for getting started, but it has three costs: dollars (per upstream call), latency (a round-trip to the provider), and exposure (your prompts cross the public internet). The gateway-mode switch lets you flip those costs off when you don't need them — for offline distillation against your own GPU, for testing CI loops without burning real API credits, or for an on-prem deployment where prompts must not leave the building.

The switch is intentionally a single env var (KOLM_GATEWAY_MODE) rather than a header or a per-tenant flag. That makes it auditable: an operator sees the deployment's mode in one place, and a typo (KOLM_GATEWAY_MODE=local-olama) fails loud at the very first request rather than silently leaking to the cloud.

The four modes

ModeWhen to useEnv varsBilling
cloud Default. Production traffic where you want the hosted teacher and accept the per-call cost. KOLM_GATEWAY_MODE unset or cloud. Provider keys (OPENAI_API_KEY, ANTHROPIC_API_KEY) configured upstream. Billed via hosted_inference meter. Soft + hard caps per plan.
local-ollama Distillation against a local Ollama instance, on-prem capture, or offline dev. Captures land in your local lake exactly as if cloud were running. KOLM_GATEWAY_MODE=local-ollama. Optional: KOLM_OLLAMA_URL (default http://localhost:11434), KOLM_GATEWAY_LOCAL_MODEL. No hosted_inference billing. Costs are your GPU + electricity.
local-vllm Same as local-ollama but pointing at a vLLM OpenAI-compatible server. Higher throughput on multi-GPU boxes. KOLM_GATEWAY_MODE=local-vllm. Optional: KOLM_VLLM_URL (default http://localhost:8000), KOLM_GATEWAY_LOCAL_MODEL. No hosted_inference billing. Costs are your GPU + electricity.
mock CI loops, deterministic regression tests, smoke checks that must run without an API key. The mock never makes a network call. KOLM_GATEWAY_MODE=mock. Optional: KOLM_MOCK_KIND (echo | reverse | fixed), KOLM_MOCK_RESPONSE for the fixed kind. Free. No billing of any kind. kolm_w742.billed:false is stamped on every response.
Privacy note. local-ollama, local-vllm, and mock never send your prompt outside your network. The dispatcher resolves localhost (or the URL you configured) and the kolm cloud is not in the path. Captures still land in your local lake so the rest of the workbench (datasets, evals, K-Score) works identically — you just keep the data.

CLI + API surface

Every surface is auth-gated except the public /health. The CLI mirrors the API exactly so a deployment audit lands in one envelope.

SurfaceInvocation
CLI statuskolm gateway status
CLI set (prints snippet)kolm gateway set local-ollama
CLI test-callkolm gateway test-call --message "hello"
API statusGET /v1/gateway/mode (auth-gated; returns {mode, ollama_reachable, vllm_reachable, allowed, version})
API dispatchPOST /v1/chat/completions (auth-gated; reads KOLM_GATEWAY_MODE server-side and dispatches accordingly)

Plain limits

  • Mock is deterministic on purpose. The token counts are Math.floor(chars/4) so the same input always produces the same usage. Do not use mock for any code path that needs a real model — it returns the echoed / reversed / fixed string only.
  • Local backends are unauthenticated. The dispatcher assumes localhost (or your private network). If you expose Ollama / vLLM on a public address, that's on you — the kolm cloud will not act as a reverse proxy or auth gate for them.
  • Unknown modes fail loud. A typo like KOLM_GATEWAY_MODE=local-olama returns HTTP 400 with unknown_gateway_mode on the very first request rather than silently falling back to cloud. The privacy promise depends on this.
  • No retry. If Ollama is not running, you get ollama_not_reachable with the start ollama with 'ollama serve' hint — no silent retry, no silent cloud fallback. The operator decides what to do.
  • Mock skips billing. The mock path stamps kolm_w742.billed:false and never calls usageIncrementMeter('hosted_inference', ...). That is the-3 lock — pinned in a test so it cannot regress.
  • Version stamp. Every gateway-mode response is stamped version:"w742-v1" so a downstream audit can pin the exact dispatcher revision.
