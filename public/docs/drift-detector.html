Drift detector.

A live watch on your capture stream. When the embedding distribution moves or the teacher-fallback rate lifts, the detector fires an alert and suggests an action.

What it watches

  • Embedding distribution. Per-namespace rolling sketch of capture embeddings, compared baseline-vs-current via KL divergence. Default baseline window: 7 days. Default current window: 24 hours.
  • Fallback rate. Per-namespace ratio of W709 routing decisions that escalated from student to teacher. A sudden lift in this number is usually the earliest signal that the distilled student has stopped covering production traffic.

Configurable thresholds

ThresholdDefaultEffect
kl_threshold0.10Fires alert when baseline-vs-current KL divergence exceeds this value.
fallback_rate_lift0.20Fires alert when current-window fallback rate exceeds baseline by this absolute lift.
min_baseline_captures200Below this floor the detector emits an honest {insufficient_baseline:true} envelope and does not fire.

Per-namespace overrides

Each threshold can be set per-namespace. Override at /account/drift or via the CLI:

kolm drift set --namespace prod-coding --kl-threshold 0.06 --fallback-rate-lift 0.15
kolm drift get --namespace prod-coding --json

Alert delivery

Alerts deliver via the W215 webhook + email transport that handles capture-threshold notifications. Each alert carries a suggested-action string the detector composes from the firing signals (for example: "fallback rate moved from 4% to 23%; consider re-running kolm distill --resume-from-active-queue on namespace prod-coding").

Auto-remediation (opt-in)

When the namespace setting auto_remediate_drift is set to true the detector forwards the firing event to the W720 self-improvement loop, which pulls the underperforming captures into the active-learning queue. This is opt-in -- the default behavior is alert-only, never silent auto-distill.

kolm namespace settings set prod-coding auto_remediate_drift true

Honest constraints

  • The detector uses the same KL/JSD math as W747 drift-alert but answers a different question. W813 compares baseline-vs-current windows within the live capture stream. W747 compares the training sketch against the live production sketch.
  • Below min_baseline_captures the detector returns insufficient_baseline:true and stays quiet. We never fire alerts on small-sample noise.
