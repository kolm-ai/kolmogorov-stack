Docs / State and sync

State and sync

A kolm workspace travels. From a laptop to a CI run, from one agent session to another, from a development machine to an air-gapped build server. State + sync is the layer that keeps it reproducible without forcing you into a hosted service.

The .kolm-state directory

Every kolm project keeps its volatile workspace under .kolm-state/. It is checked into git intentionally. The directory is small, deterministic, and load-bearing for reproducibility: a clean checkout plus .kolm-state/ reproduces the same compile result.

PathPurpose
.kolm-state/manifest.jsonPointer to the active recipe, gate, and frozen-eval set. Updated by kolm compile.
.kolm-state/checkpoints/Short-hashed revisions. Each is a deterministic snapshot of kolm.yaml + seeds + recipe.
.kolm-state/chats/Imported chat logs (Claude / ChatGPT exports / JSONL). Used as seed material.
.kolm-state/jobs.jsonlJSONL job ledger. Append-only. Pruned by kolm jobs --prune-older 30d.
.kolm-state/sync.lockMutex held by kolm sync. Prevents concurrent pushes to the same remote.

State verbs

Each verb is idempotent. Running it twice with the same input produces the same output and the same exit code.

kolm checkpoint

Snapshot the current kolm.yaml + seeds + recipe into .kolm-state/checkpoints/<hash>. Use this before risky edits. Roll back with --restore <hash>.

kolm import-chat

Convert a Claude or ChatGPT export, or any JSONL, into seeds.jsonl. Supports --strategy head_wins and --strategy evaluator_decides for resolving duplicates.

kolm merge

Merge two checkpoints. Conflicts are resolved per the chosen strategy. Output goes to a new checkpoint, not back to disk in place.

kolm sync

Push artifact + manifest + receipt chain to a private git mirror. Use this for cross-agent memory. Deterministic per-URL workdir so two laptops can push to the same mirror without colliding.

kolm jobs

List long-running compiles, distills, evals, and replays. Each row carries kind, status, started-at, duration, and exit code. Backed by .kolm-state/jobs.jsonl.

kolm watch

Live tail a single job's log. Survives disconnect. Pair with kolm compile --detach in tmux so a 20-minute compile keeps running after you close the SSH session.

Typical workflow

# 1. Snapshot before changing seeds.
kolm checkpoint --label "before-redaction-tune"

# 2. Pull chat history into seeds.
kolm import-chat ~/Downloads/claude-export.json --strategy head_wins

# 3. Compile with the new seeds.
kolm compile "redact PHI from notes" --detach
kolm jobs # find the job id
kolm watch <job-id> # live tail

# 4. If results regress, roll back.
kolm checkpoint --restore <hash>

# 5. When you're happy, sync to your private mirror.
kolm sync --remote git@github.com:acme/kolm-state.git

Reproducibility guarantees

Deterministic short-hash. Two checkouts of the same .kolm-state on different machines produce the same checkpoint hashes. No machine-specific timestamps or paths leak in.
Append-only ledger. jobs.jsonl is append-only by contract. kolm jobs --prune-older 30d rewrites the file but never deletes a row that the receipt chain still references.
Signed sync. kolm sync signs the manifest with the actor key before pushing. The remote refuses pushes whose signature does not match the configured tenant fingerprint.

Cross-agent memory

The intended pattern for multi-agent workflows: every agent reads from and writes to the same .kolm-state/ tree, synced through a private git repo. This makes the kolm workspace the source of truth across Claude, Cursor, Devin, and any custom orchestrator. No centralized API to break; no proprietary memory format to migrate off.

See /foundations for the broader integration story (Tailscale + tmux + private git + script-everything), and /docs/agent-guide for the rules every agent should follow before editing a kolm workspace.
