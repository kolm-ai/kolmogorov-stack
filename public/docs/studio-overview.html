Studio / Hub

Distill. Compile. Export. Publish.

Studio is the Forge half of kolm. You bring captured traffic (or a hand-built recipe), pick a teacher council, distill into a smaller student, compile the student to your hardware, export it to any runtime — GGUF, EXL2, GPTQ, AWQ, FP8, NVFP4, MLX — and publish the artifact. Every step emits a signed receipt and writes a row to the artifact passport.

The four Studio stages

Studio is a pipeline. Each stage produces an artifact the next stage consumes. None of them require leaving your machine.

  1. Distill — teacher generates labelled responses, student learns via LoRA / QLoRA / DistiLLM-2 contrastive loss. See studio-distill.
  2. Compile — bake the student + tokenizer + chat template + spec.toml into a single .kolm file. See studio-compile.
  3. Export — emit the artifact in the runtime your deployment target speaks. See studio-export-formats.
  4. Publish — push to HuggingFace, the kolm registry, or a private S3/R2 bucket. See studio-publish-hf.

How Studio plugs into Wrapper

The product loop closes when Studio consumes the captures Wrapper produced. The confidence router tags every fallback capture_eligible:true; you review and approve them in /account/captured; a new kolm distill --namespace <ns> --since-last-distill reads the approved rows and trains a new student. Deploy the new .kolm, and the next batch of traffic answers locally — the fallback rate falls.

Caveats. Studio is local-first. The CLI runs distill jobs on your GPU; the cloud option (cloud-sync) routes through Modal or RunPod under your account, never ours. No model weights leave the box unless you call kolm publish.

Start here

# 1. Make sure you have a kolm key + a GPU
kolm doctor

# 2. Inspect a model you want to distill
kolm inspect deepseek-ai/DeepSeek-R1-Distill-Qwen-32B --json

# 3. Run the canonical Trinity recipe end-to-end
kolm distill --recipe support-trinity-500 --teacher claude-haiku-4-5 --output trinity-support.kolm

# 4. Compile + export to GGUF Q4_K_M for a 5090
kolm compile trinity-support.kolm --target gguf-q4km --output trinity-support-q4km.gguf

# 5. Benchmark
kolm bench --models trinity-support-q4km.gguf,qwen-7b-base,claude-haiku-4-5 --eval holdout.jsonl
