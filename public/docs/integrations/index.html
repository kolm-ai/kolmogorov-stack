Open-source AI workbench

Cloud integrations

Frontier AI on your own infrastructure. Distill a teacher model into a compact .kolm artifact in your workbench, then deploy that artifact behind a managed inference endpoint in whichever cloud already runs your application. Your data, your VPC, your IAM.

Why integrate

kolm produces a portable .kolm compile: weights plus a receipt that names every input that shaped them. That bundle is a self-contained model artifact, not a service. The cloud hosting layer (SageMaker, Azure ML, Vertex AI) takes that artifact and wraps it in a load balancer, autoscaler, IAM perimeter, and metrics pipeline. Splitting the responsibilities lets you keep distillation reproducible and on-prem if you want to, while leveraging the cloud surface you already pay for.

  • Distillation lives in kolm. Capture, dataset construction, training, quantization, receipts, k-score.
  • Hosting lives in your cloud. Endpoints, autoscaling, observability, identity, governance.
  • The bundle is the contract. One .kolm file deploys to any of the three with a format-specific export.

Other clouds

Bring your own runtime

A .kolm artifact is a portable folder. It deploys anywhere with a Python or Node runtime and a GPU: Oracle Cloud Infrastructure, IBM Cloud, Alibaba Cloud PAI, OpenShift AI, Modal, RunPod, Replicate, Fly.io GPU machines, your own Kubernetes cluster, even a bare-metal box in a closet.

The compiled bundle exposes a single entrypoint that the kolm runtime calls; wire that to whatever ingress your platform supports. There is no cloud-specific code in the model itself.

Email integrations@kolm.ai for partnership inquiries (a co-authored guide, a managed bridge, a marketplace listing). We are also actively interested in feedback on what would make the cloud-specific exporters more useful.
