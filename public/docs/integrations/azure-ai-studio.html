Open-source AI workbench

kolm + Azure AI Studio

Frontier AI on your own infrastructure. Compile a teacher model into a compact .kolm student, register it as a custom model in Azure Machine Learning Studio, and serve it through a managed online endpoint inside your Azure subscription, VNet, and tenant.

What this integration does

You build a .kolm artifact in your workbench, export an Azure ML-shaped package (an MLflow-compatible folder with a custom conda.yaml and a kolm scoring script), upload the bundle to Azure Blob Storage, register the model in an Azure Machine Learning workspace, and deploy it as a managed online endpoint behind Azure AD authentication. Azure runs the inference container, autoscaling, and Application Insights telemetry. The weights stay inside your subscription once uploaded.

Prerequisites

  • A kolm API key. Get one from /signup or via kolm signup --email you@company.com.
  • A compiled .kolm artifact. The walk-through uses the artifact hash art_8f3a...; substitute your own.
  • An Azure subscription with the Machine Learning extension enabled and an existing Azure ML workspace.
  • Compute capacity: at minimum one Standard_NC6s_v3 (16 GB V100) or Standard_NC4as_T4_v3 for a 7B INT4 student. Larger students need Standard_NC24ads_A100_v4.
  • The Azure CLI v2 with the ml extension (az extension add -n ml) and credentials for the target subscription (az login).

Step-by-step deployment

Step 1

Export the artifact in Azure ML shape

kolm export --artifact art_8f3a... --format azureml --out ./kolm-azureml/

The exporter writes an Azure ML model folder: model/ (the student weights), code/score.py (the scoring script Azure invokes per request), conda.yaml (the runtime dependency manifest), and manifest.json (kolm's compile receipt).

Step 2

Upload to Azure Blob Storage

az storage blob upload-batch \
  --account-name mykolmstorage \
  --destination kolm-artifacts \
  --source ./kolm-azureml/ \
  --destination-path art_8f3a/

Use the same region as the target Azure ML workspace to avoid cross-region transfer costs. The default storage account that Azure ML provisions per workspace is fine.

Step 3

Register the custom model in Azure ML Studio

az ml model create \
  --name kolm-art-8f3a \
  --version 1 \
  --type custom_model \
  --path azureml://datastores/workspaceblobstore/paths/art_8f3a/ \
  --workspace-name my-workspace \
  --resource-group my-rg

The model now appears under Models in Azure AI Studio (and Azure Machine Learning Studio) and is ready to attach to an endpoint.

Step 4

Deploy as a managed online endpoint

az ml online-endpoint create --name kolm-art-8f3a --auth-mode aad_token
az ml online-deployment create \
  --name v1 \
  --endpoint kolm-art-8f3a \
  --model azureml:kolm-art-8f3a:1 \
  --instance-type Standard_NC4as_T4_v3 \
  --instance-count 1 \
  --environment azureml:huggingface-transformers-gpu:latest

Provisioning takes 6-12 minutes. Track with az ml online-endpoint show -n kolm-art-8f3a; the state moves Creating to Succeeded. The endpoint URL appears under scoring_uri.

Step 5

Test inference

from kolm import Client
import azure.identity
token = azure.identity.DefaultAzureCredential().get_token("https://ml.azure.com/.default").token
c = Client(base_url="https://kolm-art-8f3a.eastus.inference.ml.azure.com/score", bearer=token)
r = c.invoke({"input": "Summarize this ticket in one sentence: ..."})
print(r["output"], r["k_score"])

The Azure managed endpoint accepts a bearer token from Azure AD. The kolm SDK forwards the JSON envelope unchanged, so the response shape (output, k_score, artifact_hash, latency_ms) is identical to the REST API.

What kolm contributes

The workbench: capture from a live application, build a student dataset from your real traffic, distill against a frontier teacher, quantize to INT4, and write a receipt that names every input that shaped the student. The output is a portable .kolm compile with verifiable lineage. None of this depends on Azure.

What Azure AI Studio contributes

Hosting and governance: the managed online endpoint, autoscaling policy, blue/green deployment slots, Azure AD token authentication, VNet isolation, private endpoints, Application Insights metrics, and integration with the wider Azure AI Studio observability surface (drift, content safety, prompt flow). You do not maintain any of the operational surface.

Cost model

Plain example for a 7B INT4 student serving ~100 requests per minute in East US:

  • Distillation (one-time): runs on your local RTX 5090 in roughly 30 minutes, or on a single Azure Standard_NC4as_T4_v3 for about $0.90/hr until done.
  • Hosting (continuous): one Standard_NC4as_T4_v3 at $0.526/hr managed online = ~$379/month. Standard_NC6s_v3 (V100) is $3.06/hr = ~$2,203/month if you need V100-class throughput.
  • Storage: a 5 GB artifact in workspace blob storage is roughly $0.10/month.
  • Egress: free within the same Azure region; check the data-transfer tab if your callers live outside Azure.

The teacher cost stays on whatever frontier provider you used during capture. The hosted student replaces that ongoing spend with a flat compute charge.

Known limitations

  • The Azure ML exporter currently writes a portable .kolm compile with a generic Hugging Face scoring script. A first-party kolm inference environment (with built-in receipt verification and k-score logging) is a roadmap item.
  • Azure AI Studio's model catalog UI does not list arbitrary custom models in the prompt-flow picker; you reach the endpoint via the scoring URI or the Azure ML SDK, not the AI Studio chat UI.
  • Managed online endpoints require a minimum of one instance; there is no scale-to-zero. Use batch endpoints if your workload is bursty.
Format adapter status. kolm export --format azureml currently writes a portable.kolm compile that the corresponding cloud container can load. Cloud-specific packaging is a roadmap item (+ artifact composition).
Status

Documented integration. Not currently a managed offering. Bring your own Azure subscription and ML workspace. Email integrations@kolm.ai if you want kolm to operate a turnkey Azure AI Studio bridge for your team.
