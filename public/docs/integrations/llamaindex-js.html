LlamaIndex JS

LlamaIndex.TS wraps the official openai and anthropic SDKs under the hood. Pass additionalSessionOptions on the LLM constructor to override the base URL and your chat engines, query engines, and agents all flow through the local kolm proxy. Captures land in the named namespace; thresholds gate kolm distill.

Install

npm i llamaindex
npm i -g github:sneaky-hippo/kolm-stack

Start the proxy

kolm capture --provider openai --as docs-rag --namespace docs-rag
# proxy listening on http://127.0.0.1:8787

OpenAI LLM

import { OpenAI, Settings } from "llamaindex";

Settings.llm = new OpenAI({
 model: "gpt-4o-mini",
 apiKey: process.env.OPENAI_API_KEY,
 additionalSessionOptions: {
 baseURL: "http://127.0.0.1:8787/v1",
 defaultHeaders: {
 "x-upstream-api-key": process.env.OPENAI_API_KEY,
 "x-kolm-namespace": "docs-rag",
 },
 },
});

const response = await Settings.llm.complete({ prompt: "summarize: ..." });
console.log(response.text);

Anthropic LLM

import { Anthropic, Settings } from "llamaindex";

Settings.llm = new Anthropic({
 model: "claude-3-5-sonnet-latest",
 apiKey: process.env.ANTHROPIC_API_KEY,
 additionalSessionOptions: {
 baseURL: "http://127.0.0.1:8787",
 defaultHeaders: {
 "x-upstream-api-key": process.env.ANTHROPIC_API_KEY,
 "x-kolm-namespace": "docs-rag",
 },
 },
});

RAG and agents

The same Settings.llm override applies to every higher-level abstraction: VectorStoreIndex.fromDocuments(...), asChatEngine(), asQueryEngine(), and the agent runtime. Embeddings are a separate setting; if you want those captured too, override Settings.embedModel with the same base URL pattern.

Env-var shortcut

// before any import:
process.env.OPENAI_BASE_URL = "http://127.0.0.1:8787/v1";
process.env.ANTHROPIC_BASE_URL = "http://127.0.0.1:8787";

Verify the proxy sees traffic

kolm capture status --namespace docs-rag
# pairs captured: 412 / 1000
# pairs until distill: 588
