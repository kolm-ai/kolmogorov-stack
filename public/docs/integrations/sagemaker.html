Open-source AI workbench

kolm + Amazon SageMaker

Frontier AI on your own infrastructure. Distill a teacher model into a compact .kolm artifact on your local GPU, push the bundle to S3, and serve it through SageMaker's managed inference endpoints. Your data, your VPC, your IAM.

What this integration does

You compile a .kolm artifact in your workbench (the distilled student weights plus the kolm runtime bundle), export a SageMaker-shaped package, upload the bundle to S3, register a SageMaker Model resource with a kolm-compatible inference container, and deploy it as a real-time endpoint. SageMaker handles the load balancer, autoscaling group, and CloudWatch metrics. The model itself is yours - the weights never leave your AWS account once they are uploaded.

Prerequisites

  • A kolm API key. Get one from /signup or via kolm signup --email you@company.com.
  • A compiled .kolm artifact. The walk-through assumes the artifact hash art_8f3a...; substitute your own.
  • An AWS account with permission to create S3 buckets, SageMaker Models, Endpoint Configurations, and Endpoints.
  • Compute capacity: at minimum one ml.g5.xlarge (24 GB GPU) for a 7B INT4 student. Larger students need ml.g5.2xlarge or above.
  • The AWS CLI v2 configured (aws configure) with credentials that match the target account.

What kolm contributes

The distillation pipeline: capture in production, build a student dataset, train a smaller open-weights model against the teacher, quantize to INT4, write a receipt that names every input that shaped the artifact. The output is a portable .kolm compile with verifiable lineage. None of that needs SageMaker.

What SageMaker contributes

Hosting and operations: the inference container, the load balancer in front of it, the autoscaling group, CloudWatch metrics for latency and 5xx, IAM-gated invocation, VPC isolation, and the ability to roll a new model version without dropping traffic. None of that needs to be written in kolm.

Cost model

Plain example for a 7B INT4 student serving ~100 requests per minute:

  • Distillation (one-time): runs on your local RTX 5090 in roughly 30 minutes, or on a single ml.g5.2xlarge for about $1.50.
  • Hosting (continuous): one ml.g5.xlarge at $1.21/hr on-demand US East = $872/month. Reserved 1-year drops that to ~$520/month.
  • Storage: a 5 GB artifact bundle in S3 is $0.12/month.
  • Egress: free within the same AWS region; check the data-transfer tab if your caller lives outside AWS.

The teacher-side cost stays on whatever frontier provider you used during capture (Anthropic, OpenAI, your own vLLM). The hosted student replaces that ongoing spend with a flat instance cost.

Known limitations

  • The SageMaker exporter currently packages a portable .kolm compile with a generic Hugging Face handler. Cloud-specific packaging (custom kolm inference container with built-in receipt verification) is a roadmap item.
  • SageMaker Serverless Inference is supported only for artifacts that fit in 6 GB of RAM, which excludes most quantized 7B and up. Use real-time endpoints for those.
  • Streaming responses require SageMaker's InvokeEndpointWithResponseStream API; the kolm SDK exposes that as c.stream(...) but only on Python at the time of writing.
Format adapter status. kolm export --format sagemaker currently writes a portable .kolm compile that the corresponding cloud container can load. Cloud-specific packaging is a roadmap item (W736+ artifact composition).
Status

Documented integration. Not currently a managed offering. Bring your own AWS account and IAM role. Email integrations@kolm.ai if you want kolm to operate a turnkey SageMaker bridge for your team.
