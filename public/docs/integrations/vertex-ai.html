kolm.ai / docs / integrations / vertex-ai

Open-source AI workbench

kolm + Google Vertex AI

Frontier AI on your own infrastructure. Distill a teacher model into a compact .kolm student, register it as a Vertex AI Model with a custom container image, and serve it through a Vertex AI Endpoint inside your GCP project, VPC, and IAM perimeter.

What this integration does

You compile a .kolm artifact in your workbench, export a Vertex AI-shaped package, push it to a Cloud Storage bucket, create a Vertex AI Model resource that points at a custom container image (Artifact Registry), and deploy the model to a Vertex AI Endpoint. Vertex AI runs the prediction container, autoscales node pools, and exposes Cloud Logging and Cloud Monitoring metrics. The weights live in your GCP project once uploaded.

Prerequisites

  • A kolm API key. Get one from /signup or via kolm signup --email you@company.com.
  • A compiled .kolm artifact. The walk-through uses the artifact hash art_8f3a...; substitute your own.
  • A GCP project with the Vertex AI, Cloud Storage, and Artifact Registry APIs enabled (gcloud services enable aiplatform.googleapis.com storage.googleapis.com artifactregistry.googleapis.com).
  • Compute capacity: at minimum one n1-standard-4 with a single NVIDIA_TESLA_T4 for a 7B INT4 student, or a g2-standard-4 with NVIDIA_L4. Larger students require a2-highgpu-1g with NVIDIA_TESLA_A100.
  • The gcloud CLI installed and authenticated for the target project (gcloud auth login && gcloud config set project my-project).

Step-by-step deployment

Step 1

Export the artifact in Vertex AI shape

kolm export --artifact art_8f3a... --format vertexai --out ./kolm-vertex/

The exporter writes a Vertex AI predictor folder: model/ (the student weights), predictor/predictor.py (a class that implements predict(), the contract Vertex AI's custom-prediction-routine expects), requirements.txt (the runtime deps), and manifest.json (kolm's compile receipt).

Step 2

Upload to a GCS bucket

gsutil cp -r ./kolm-vertex/ gs://my-kolm-bucket/artifacts/art_8f3a/

The bucket must be in the same region as the planned Vertex AI Endpoint (for example, us-central1) or you pay cross-region transfer at request time.

Step 3

Create the Vertex AI Model resource with a custom container

gcloud ai models upload \
  --region us-central1 \
  --display-name kolm-art-8f3a \
  --container-image-uri us-docker.pkg.dev/vertex-ai/prediction/pytorch-gpu.2-1:latest \
  --artifact-uri gs://my-kolm-bucket/artifacts/art_8f3a/ \
  --container-predict-route /predict \
  --container-health-route /health \
  --container-ports 8080

The image is the Vertex AI-managed PyTorch GPU prediction container; it includes transformers and accelerate, which covers the kolm INT4 path. Swap to a custom image in your Artifact Registry if your artifact needs vLLM or TGI.

Step 4

Deploy to a Vertex AI Endpoint

gcloud ai endpoints create --region us-central1 --display-name kolm-art-8f3a

ENDPOINT_ID=$(gcloud ai endpoints list --region us-central1 \
  --filter='display_name=kolm-art-8f3a' --format='value(name)' | awk -F/ '{print $NF}')

MODEL_ID=$(gcloud ai models list --region us-central1 \
  --filter='display_name=kolm-art-8f3a' --format='value(name)' | awk -F/ '{print $NF}')

gcloud ai endpoints deploy-model $ENDPOINT_ID \
  --region us-central1 \
  --model $MODEL_ID \
  --display-name v1 \
  --machine-type n1-standard-4 \
  --accelerator type=nvidia-tesla-t4,count=1 \
  --min-replica-count 1 \
  --max-replica-count 3

Provisioning takes 8-15 minutes. Track with gcloud ai endpoints describe $ENDPOINT_ID --region us-central1. The endpoint becomes invokable once the deployment row reaches READY.

Step 5

Test inference

from kolm import Client
import google.auth, google.auth.transport.requests
creds, _ = google.auth.default(scopes=["https://www.googleapis.com/auth/cloud-platform"])
creds.refresh(google.auth.transport.requests.Request())
c = Client(base_url=f"https://us-central1-aiplatform.googleapis.com/v1/projects/my-project/locations/us-central1/endpoints/{ENDPOINT_ID}:predict", bearer=creds.token)
r = c.invoke({"instances": [{"input": "Summarize this PR description: ..."}]})
print(r["predictions"][0]["output"], r["predictions"][0]["k_score"])

Vertex AI wraps responses in a predictions array; the inner object preserves the kolm envelope shape (output, k_score, artifact_hash, latency_ms).

What kolm contributes

The workbench: capture in production, build a student dataset from your real traffic, distill against a frontier teacher, quantize to INT4, and write a receipt that names every input that shaped the student. The output is a portable .kolm compile with verifiable lineage. None of this depends on Vertex AI.

What Vertex AI contributes

Hosting and operations: the prediction container, autoscaling node pool, IAM-gated invocation, VPC Service Controls, Cloud Logging and Cloud Monitoring metrics, model versioning, traffic splitting between deployed model revisions, and optional Model Monitoring for drift. You do not run a serving stack yourself.

Cost model

Honest example for a 7B INT4 student serving ~100 requests per minute in us-central1:

  • Distillation (one-time): runs on your local RTX 5090 in roughly 30 minutes, or on a Vertex AI custom-job n1-standard-4 + T4 for about $0.95/hr until done.
  • Hosting (continuous): one n1-standard-4 + NVIDIA_TESLA_T4 is roughly $0.55/hr = ~$400/month. A g2-standard-4 with NVIDIA_L4 is roughly $0.70/hr = ~$504/month and noticeably faster for INT4.
  • Storage: a 5 GB artifact in standard-class GCS is $0.10/month.
  • Egress: free within the same region; check the data-transfer tab if your callers live outside GCP.

The teacher cost stays on whatever frontier provider you used during capture. The hosted student replaces that ongoing spend with a flat compute charge.

Known limitations

  • The Vertex AI exporter currently writes a portable .kolm compile plus a generic predictor shim. A first-party kolm prediction container (with built-in receipt verification and k-score logging) is a roadmap item.
  • Vertex AI Endpoints have no scale-to-zero; the minimum replica count is one. Use Vertex AI Batch Prediction if your workload is bursty.
  • Streaming responses require the Vertex AI raw-predict API and a custom container that flushes per token; the default kolm exporter ships a unary predict shape only.
Format adapter status. kolm export --format vertexai currently writes a portable .kolm compile that the corresponding cloud container can load. Cloud-specific packaging is a roadmap item (W736+ artifact composition).
Status

Documented integration. Not currently a managed offering. Bring your own GCP project and service account. Email integrations@kolm.ai if you want kolm to operate a turnkey Vertex AI bridge for your team.
