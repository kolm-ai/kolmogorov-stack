Studio / HTTP API

Studio HTTP API reference.

The CLI is a thin client over a stable HTTP API. Anything you can do with kolm distill, kolm compile, kolm export, or kolm publish you can do with an HTTP POST. All routes return the standard kolm envelope: {ok, surface, route, value, receipt}. Long-running jobs return a job_id you poll until state:done.

Conventions

Base URL: https://kolm.ai (prod) or http://127.0.0.1:8787 (local dev). Authenticate with a bearer token: Authorization: Bearer ks_…. Get one from kolm login or /signup. JSON in, JSON out. Content-Type: application/json on every POST. Idempotency: pass Idempotency-Key header on retried POSTs to avoid double-charging compute.

Envelope shape

{
  "ok": true,
  "surface": "studio",
  "route": "/v1/distill/start",
  "value": { /* per-route payload */ },
  "receipt": {
    "cid": "rcpt_01KYC1ZVTGDCW3FX06JQSC",
    "ts": "2026-05-26T14:23:01.412Z",
    "tenant": "tenant_f33c240034c5",
    "verify_url": "https://kolm.ai/v1/verify/rcpt_01KYC1ZVTGDCW3FX06JQSC"
  }
}

On error, ok:false and a error:{code,message,hint} field replaces value. The HTTP status mirrors the envelope: 2xx for ok:true, 4xx for caller error, 5xx for server error. Receipts are always issued, even on failure, so you can verify what was attempted.

Distillation routes

POST/v1/distill/start

Start a distillation run from a spec. Returns a job_id immediately.

POST /v1/distill/start
{
  "spec": {
    "base": "qwen2.5-7b-instruct",
    "teachers": [
      {"name": "claude-haiku-4-5", "weight": 0.5},
      {"name": "gpt-4o-mini",      "weight": 0.3},
      {"name": "deepseek-v3",      "weight": 0.2}
    ],
    "loss": "distillm2-srkl",
    "rank": 16,
    "epochs": 2,
    "judge": "gemini-2.5-pro",
    "dataset": "council-pairs-410"
  },
  "hardware": "rtx-5090"
}

200 OK
{"ok":true,"value":{"job_id":"job_d01...","state":"queued","eta_s":76},"receipt":{...}}
GET/v1/distill/jobs/:job_id

Poll for status. state is one of queued | running | done | failed. When done, value.artifact is a path you can pass to /v1/compile/start.

GET/v1/distill/jobs/:job_id/logs

Stream NDJSON logs from the run (one JSON object per line). Closes on state:done|failed. Each line has {ts, step, loss, lr, gpu_mem_gb}.

POST/v1/distill/jobs/:job_id/cancel

Cancel a running job. Returns 204 on success. Partial checkpoints are preserved at ~/.kolm/runs/<job_id>/checkpoints/ and you can resume with --resume <job_id>.

GET/v1/distill/recipes

List built-in recipes (see Distill recipes). Returns {recipes:[{name,desc,defaults}]}. POST to /v1/distill/recipes/:name/start to launch one with default overrides.

Compile routes

POST/v1/compile/start

Compile a model checkpoint into the .kolm artifact format. Accepts a raw HF repo, a path on disk, or a distill_job_id.

POST /v1/compile/start
{
  "input": {
    "kind": "hf",
    "repo": "Qwen/Qwen2.5-7B-Instruct"
  },
  "spec": {
    "chat_template": "chatml",
    "stop_tokens": ["<|im_end|>"],
    "context_len": 32768
  },
  "passport": {
    "vendor": "kolm",
    "license": "apache-2.0",
    "training_data_summary": "Distilled from teacher council; council-pairs-410 dataset"
  }
}

200 OK { "value": { "artifact_id": "art_qwen_7b_chatml_01" } }
GET/v1/compile/artifacts/:artifact_id

Returns the spec, passport, sha256 of the weights blob, and a list of available export formats. Does not stream the binary — use the GET /download endpoint for that.

GET/v1/compile/artifacts/:artifact_id/download

Stream the raw .kolm binary. Set Range: header for resumable downloads. Verifies the Ed25519 signature in the trailer before serving.

Export routes

POST/v1/export/start

Export an artifact to a target format. See Export formats for the full list.

POST /v1/export/start
{
  "artifact_id": "art_qwen_7b_chatml_01",
  "format": "gguf",
  "quant": "Q4_K_M",
  "imatrix": "data/wiki-en-1k.txt"
}

200 OK { "value": { "job_id": "job_exp_07b...", "eta_s": 110 } }
GET/v1/export/jobs/:job_id

Poll. On done, value.output_url is a signed URL good for 24 hours. Pass ?ttl=7d for week-long URLs on Pro+ tiers.

GET/v1/export/formats

List every supported export format with default flags, target runtimes, and minimum hardware. Used by kolm export --list.

Publish routes

POST/v1/publish/hf

Push an artifact (and one or more exports) to a Hugging Face repo. See Publish to HF for the full handshake.

POST /v1/publish/hf
{
  "artifact_id": "art_qwen_7b_chatml_01",
  "repo": "your-org/qwen-2.5-7b-instruct-kolm",
  "exports": ["gguf:Q4_K_M", "gguf:Q5_K_M", "safetensors"],
  "private": false,
  "license": "apache-2.0",
  "model_card": "auto"
}

200 OK { "value": { "url": "https://huggingface.co/your-org/…" } }
GET/v1/publish/hf/:job_id

Status of an in-flight publish job. value.stages shows progress per file.

POST/v1/publish/ollama

Push to an Ollama-compatible registry. Requires a GGUF export and an Ollama Modelfile.

Bench routes

POST/v1/bench/start

Run a multi-model benchmark. See Bench harness.

POST /v1/bench/start
{
  "suite": "trinity-500-council",
  "candidates": [
    "kolm:art_qwen_7b_chatml_01",
    "openai:gpt-4o-mini",
    "anthropic:claude-haiku-4-5",
    "hf:Qwen/Qwen2.5-7B-Instruct"
  ],
  "judge": "gemini-2.5-pro",
  "n": 57,
  "axes": ["asks-1Q","judge-clarify","judge-on-policy","latency","tokens"]
}
GET/v1/bench/jobs/:job_id/results

Get the full per-axis matrix. value.k_score is the weighted composite. Receipt embeds the raw judge transcripts for audit.

Experts routes (MoE)

GET/v1/experts/:artifact_id

List per-expert usage stats from the most recent benchmark or imatrix calibration. Returns expert activations per layer. See MoE compilation.

POST/v1/experts/:artifact_id/prune

Prune low-traffic experts. Body specifies threshold (e.g., {"min_traffic_pct": 0.5}) or an explicit list. Returns a new artifact_id.

POST/v1/experts/:artifact_id/policy

Set the per-expert quant policy (e.g., router stays Q6_K, experts go Q3_K_M). Returns a new artifact_id with the policy baked in.

Inspect

GET/v1/inspect/:artifact_id

Dump the spec.toml, passport, receipt chain, and a SHA-256 manifest of every byte in the artifact. The data behind kolm inspect.

Hardware

GET/v1/hardware/probe

Returns the detected GPU(s), driver, CUDA/ROCm version, available VRAM, and the list of quant formats that will fit. Used by the Studio web UI to gray out impossible options.

GET/v1/hardware/recipes

Returns a list of hardware-tuned recipes (e.g., rtx-5090-7b-distill, m3-max-128gb-13b-distill) you can pass to /v1/distill/start as a hardware hint.

Rate limits

Free tier: 5 concurrent jobs across distill+compile+export, 100 requests/min on GETs. Pro: 50 concurrent jobs, 1000 req/min. Enterprise: negotiated. Every response carries X-RateLimit-Remaining + X-RateLimit-Reset-At headers.

Versioning

The /v1/ prefix is stable. We add new fields without bumping; we never remove or repurpose existing ones. When a route fundamentally changes shape we mount a /v2/ sibling and deprecate /v1/ with a 12-month sunset. Deprecation shows up in the X-Kolm-Deprecation header.

Caveats. All Studio routes count against your monthly compute quota. --dry-run on the CLI maps to ?dry=1 on the HTTP API and is free. Bench runs that hit hosted teachers (Claude, GPT, Gemini) pass through provider fees at cost — see the cost_breakdown field in the receipt.
