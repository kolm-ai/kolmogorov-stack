Run & Govern / Docker Compose

One compose file, two runtime choices, one health endpoint.

generateDockerCompose in src/deploy-generators.js emits a deterministic docker-compose.yml for a compiled .kolm artifact. Pick vLLM or llama.cpp; the generator wires a healthcheck, mounts the model read-only, reserves an NVIDIA GPU, and ships a commented-out NGINX TLS terminator block you can flip on with two lines.

CLI

# Emit to stdout.
kolm deploy compose ./artifact.kolm --runtime vllm

# Write to a file and the model bind directory.
kolm deploy compose ./artifact.kolm --runtime llama.cpp --out ./deploy/

# Custom port.
kolm deploy compose ./artifact.kolm --runtime vllm --port 18080

vLLM output (excerpt)

# kolm deploy — docker-compose (vllm)
# artifact: support-7b-q4km
# generator: r4-v1
#
# Place your model files under ./models/support-7b-q4km before `docker compose up`.
# Container expects to find the model at /models/support-7b-q4km.

services:
  kolm-runtime:
    image: vllm/vllm-openai:latest
    container_name: kolm-support-7b-q4km
    restart: unless-stopped
    runtime: nvidia                 # remove if you do not have NVIDIA Container Toolkit
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    environment:
      - HUGGING_FACE_HUB_TOKEN=${HUGGING_FACE_HUB_TOKEN:-}
      - VLLM_NO_USAGE_STATS=1
    command:
      - --model
      - /models/support-7b-q4km
      - --port
      - "8000"
      - --host
      - "0.0.0.0"
      - --served-model-name
      - "support-7b-q4km"
    ports:
      - "8000:8000"
    volumes:
      - ./models/support-7b-q4km:/models/support-7b-q4km:ro
    healthcheck:
      test: ["CMD-SHELL", "curl -fsS http://localhost:8000/health || exit 1"]
      interval: 30s
      timeout: 5s
      retries: 5
      start_period: 60s

Adding TLS

The generator ships a fully-commented NGINX block at the bottom of the compose file. Uncomment, fill in cert paths, and drop an nginx.conf that proxies to http://kolm-runtime:<port>:

  nginx-tls:
    image: nginx:1.27-alpine
    container_name: kolm-tls
    restart: unless-stopped
    depends_on:
      kolm-runtime:
        condition: service_healthy
    ports:
      - "443:443"
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - /etc/ssl/kolm.pem:/etc/ssl/kolm.pem:ro
      - /etc/ssl/kolm.key:/etc/ssl/kolm.key:ro
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://localhost/health >/dev/null || exit 1"]
      interval: 30s

Defaults

FieldDefaultNotes
runtimevllmOne of vllm or llama.cpp.
port8000The compose file maps host:container 1:1.
model_path/models/<safe_artifact>Inside-container mount path.
host bind./models/<safe_artifact>Relative to the compose-up cwd.
healthcheck30s interval / 5s timeout / 5 retriesvLLM gets a longer 60s start period.
GPU1 NVIDIA device reservedRemove the runtime: nvidia line if you have no NVIDIA Container Toolkit.

Programmatic use

import { generateDockerCompose } from 'kolmogorov-stack/src/deploy-generators.js';

const yaml = generateDockerCompose({
  artifact:   'support-7b-q4km',
  runtime:    'vllm',
  port:       8000,
});
fs.writeFileSync('./docker-compose.yml', yaml);
Limitations. The compose file does not pull the model from a registry. Place the artifact bytes under ./models/<safe_artifact> before docker compose up. For an automated pull-on-boot init pattern, use the Kubernetes manifests (they ship a Job that calls your registry).
