Open-source AI workbench

Audit export — CSV, CEF, LEEF, JSONL.

Stream the tenant-fenced audit ledger to your SIEM in the format it speaks natively — ArcSight CEF, QRadar LEEF, Splunk JSONL, or RFC 4180 CSV for spreadsheet review.

Why four formats

Audit events live in one source: audit_events, append-only and HMAC-chained per tenant. Operators consume them in four different shapes. Rather than force every operator to write a converter, ships four canonical emitters that share one tenant-fence and one schema mapping. Bad format returns an loud-fail envelope. Cross-tenant reads return zero rows under defense-in-depth.

Format support

csv

RFC 4180 CSV

Eleven-column header row, CRLF line terminator, double-quote wrap on embedded commas / quotes / newlines, doubled quotes inside wrapped cells. Loads cleanly into Excel, Google Sheets, pandas, and any RFC 4180 parser.

Content-Type: text/csv; charset=utf-8

ts_iso,tenant_id,operator_id,op,target_kind,target_id,outcome,...
2026-05-24T12:00:00Z,t_abc,u_42,compile.completed,artifact,a_99,success,...
cef

ArcSight CEF v0

One header per event: CEF:0|kolm.ai|kolm|<version>|<sigid>|<name>|<severity>|<extension>. Extension is space-joined key=value with backslash-escape on =. Severity is 5 for normal ops, 8 for auth/admin/security/sso/scim/key prefixes, 10 for critical-suffix ops.

Content-Type: text/plain; charset=utf-8

CEF:0|kolm.ai|kolm|w770-v1|compile.completed|compile.completed|5|rt=2026-05-24T12:00:00Z cs1=t_abc cs1Label=tenant_id suser=u_42
leef

QRadar LEEF v2.0

Header: LEEF:2.0|kolm.ai|kolm|<version>|<eventId>|^|<extension>. Caret (^) is the literal delimiter slot AND the extension separator. Pair encoding is key=value with backslash-escape on ^ inside values.

Content-Type: text/plain; charset=utf-8

LEEF:2.0|kolm.ai|kolm|w770-v1|compile.completed|^|devTime=2026-05-24T12:00:00Z^tenant_id=t_abc^usrName=u_42^cat=compile.completed
json

JSONL (NDJSON)

One JSON object per line, newline-terminated. Splunk + Loki + Vector + Fluent Bit + Promtail ingest this natively without a parser plugin. Field ordering matches the CSV column ordering.

Content-Type: application/x-ndjson; charset=utf-8

{"ts_iso":"2026-05-24T12:00:00Z","tenant_id":"t_abc","operator_id":"u_42","op":"compile.completed","target_kind":"artifact","target_id":"a_99","outcome":"success","source_ip":null,"user_agent":null,"request_id":null,"meta_hash":"abcd..."}

SIEM integration

arcsight

CEF v0

The CEF emitter follows the ArcSight CEF Implementation Standard exactly. SmartConnectors auto-detect the CEF:0| header. Severity ladder (5/8/10) maps onto ArcSight's 1-10 scale without further translation.

qradar

LEEF v2.0

QRadar's LEEF DSM accepts the LEEF:2.0| header with a single-character delimiter slot. We emit caret (^) so SIEM pipelines can split on it without confusing it with literal equals in payload values.

splunk

JSONL / HEC

The JSONL format ingests directly via Splunk's HTTP Event Collector. Drop the body into the HEC raw endpoint; Splunk indexes each line as a discrete event with native field extraction.

CLI

The kolm audit-export verb (alias: ae, two-word: kolm audit export) thinly wraps the auth-gated routes. --out writes the body to disk; without it, the body streams to stdout.

kolm audit export csv --from 2026-01-01 --out audit.csv
kolm audit export cef --from 2026-05-01 --to 2026-05-31 --out may.cef
kolm audit-export export leef --max 5000 --out audit.leef
kolm audit-export preview json --from 2026-05-20
kolm ae formats                       # list supported formats + CSV columns
kolm ae export json --out events.jsonl

API

EndpointWhat it does
GET /v1/audit/exportStreams the full export body. Query: format, from, to, max_rows. Returns body with per-format Content-Type plus X-Kolm-Audit-Export-* headers.
GET /v1/audit/export/formatsLists supported formats + CSV column ordering + mime-type table.
GET /v1/audit/export/previewFirst ten rows + total_would_export so the UI can show "10 of N" before committing to a full export.
 
curl -H "Authorization: Bearer $KOLM_API_KEY" \
     "https://kolm.ai/v1/audit/export?format=cef&from=2026-05-01&to=2026-05-31" \
     -o may.cef

curl -H "Authorization: Bearer $KOLM_API_KEY" \
     "https://kolm.ai/v1/audit/export/formats"

Tenant fence

Every export reads audit_events via store.all() and then applies a per-row r.tenant_id !== tenant_id filter before encoding. The naive shortcut — findByTenant('audit_events', tenant_id) — would silently return zero rows because the index is keyed on the tenant field and audit_events rows carry tenant_id. The explicit per-row filter is the defense.

Cross-tenant audit-row leakage is a P0 security incident. The export module refuses to run without an explicit tenant_id argument, returning the loud-fail envelope {ok:false, error:'tenant_id_required'}.

Caps and previews

Export caps at max_rows (default 10000, hard ceiling 100000). Preview always caps at ten rows and surfaces total_would_export so the UI can render "showing 10 of N" without offering a full-dump backdoor. The truncation flag X-Kolm-Audit-Export-Truncated rides on the response header when the cap fires — we never silent-pass a partial export.

Schema

Eleven columns, RFC 4180 ordering. Column names match the JSONL field names match the CEF extension labels match the LEEF pair keys so a downstream operator can switch formats without rewriting field-mapping rules.
