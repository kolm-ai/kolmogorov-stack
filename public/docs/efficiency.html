Open-source AI workbench

Frontier AI on your own infrastructure.

Three compute-efficiency knobs for kolm distill: mixed-precision training (FP16/BF16), gradient checkpointing for large students on limited VRAM, and early stopping when the K-Score plateaus. Each is opt-in, recorded in the distill receipt, and exposed in the CLI without forking the trainer.

When to use what

KnobUse whenSkip when
--precision bf16 Ampere GPU or newer (RTX 30/40/50, A100, H100, H200). Same dynamic range as fp32, half the memory. Pre-Volta GPU (no bf16 hardware). Use fp16 or fp32.
--precision mixed-bf16 Production training where stability matters: autocast forward in bf16, fp32 master copy of weights. Quick experiments where 5-10% extra VRAM is not worth the wall-clock win.
--precision fp16 Volta/Turing GPU (V100, T4, RTX 20-series). Half memory but smaller numerical range than bf16. You see NaN-loss in training; promote to mixed-fp16 or bf16.
--gradient-checkpointing Student model is >7B parameters OR VRAM is <24 GB. Trades ~30% wall-clock for ~50% VRAM during back-prop. You have plenty of VRAM headroom; the slowdown is pure overhead.
--early-stop-patience N Long runs where the K-Score plateaus before --max-steps. Default plateau window is 3 eval rounds with <0.005 K-Score delta. Short runs (<50 steps); early-stop will never trigger before min_steps.

Mixed-precision training W787-2

The trainer accepts five precision modes via --precision:

ModeMemoryBest onNotes
fp321.0xCPU, debug, NaN recoverySafest baseline.
fp160.5xVolta/TuringHalf-precision; watch for overflow on long contexts.
bf160.5xAmpere+ (sm_80+)fp32 dynamic range in half the memory. Preferred where available.
mixed-fp16~0.6xVolta/Turing productionfp16 autocast + fp32 master copy.
mixed-bf16~0.6xAmpere+ productionbf16 autocast + fp32 master copy. Recommended default for new RTX GPUs.

Gradient checkpointing W787-3

Gradient checkpointing re-computes activations on the backward pass instead of caching them on the forward pass. The trade is ~30% slower wall-clock for ~50% lower peak VRAM during back-prop. Pays off when:

  • Student model is ≥ 7B parameters and you are training on a single 24 GB or 16 GB GPU.
  • You hit CUDA out of memory on the forward pass with no room to drop batch size further.
  • You want to enable mixed-precision on a smaller GPU than the model would otherwise fit on.

Skip it when VRAM is not the bottleneck; the recompute cost is pure overhead.

Early stopping when K-Score plateaus W787-1

The distill pipeline tracks K-Score per training step. When the last patience+1 samples vary by less than delta_kscore, the run signals plateau and stops.

Defaults: patience=3, delta_kscore=0.005, min_steps=50. Override the patience window with --early-stop-patience N.

The Python trainer wires transformers.EarlyStoppingCallback when KOLM_EARLY_STOP=1 is set on the worker env. The Node-side shouldStopEarly() helper is also available for post-hoc analysis of a completed run's progress.jsonl.

CLI

# Set precision and turn on gradient checkpointing for a large student
kolm distill --namespace tickets --precision mixed-bf16 --gradient-checkpointing

# Same on the local-worker path (skips the auto-distill API)
kolm distill --local-worker \
  --spec ./recipe.spec.json --seeds ./seeds.jsonl --out ./dist/ \
  --precision bf16 --gradient-checkpointing --early-stop-patience 5

# Doctor: ask what to use on THIS host
kolm distill efficiency doctor

# List supported precision modes (forward-compat reads the same enum)
kolm distill efficiency modes

Where it lands in the receipt

Every distill run records the effective efficiency block in run-meta.json at ~/.kolm/distill-runs/<run_id>/run-meta.json:

{
  "efficiency": {
    "precision_mode": "mixed-bf16",
    "gradient_checkpointing": true,
    "early_stop": {
      "enabled": true,
      "patience": 3,
      "delta_kscore": 0.005,
      "min_steps": 50
    },
    "version": "w787-v1",
    "surface_only": false
  }
}

The Python trainer also writes the effective precision + grad-checkpoint flag into training-summary.json next to the student weights, so the receipt chain documents which knobs were honoured at fine-tune time.
