Recipes.

Pre-baked workflows for the things kolm teams ship: distill a small replier from your captures, migrate off OpenAI fine-tuning, deploy to a Pi, or publish your own recipe to the registry. Each recipe lists the inputs you need, the exact steps, the artifact you end up with, caveats worth knowing before you start, and how to verify the result on your own hardware.

The recipe structure.

Every recipe in this library is shaped the same way so you can scan in seconds and pick up where another team left off.

  • Inputs. What you need before you start: dataset shape and size, API key, base model, GPU floor.
  • Steps. Copy-paste commands in order. Each step states what it does and where the artifact lands.
  • Artifact. The exact file you end up with. Format, size, where it sits on disk, what the signed manifest contains.
  • Caveats. The pitfalls we hit in real runs: data skew, label drift, teacher version pinning, anything that bites if you skip it.
  • How to verify. The offline check that proves the artifact is what the manifest says it is. Usually kolm verify <artifact> with the K-Score gate and the receipt chain.
