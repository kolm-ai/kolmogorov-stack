Wrapper quickstart

From zero to a signed capture in 60 seconds.

Install the CLI, set one environment variable, fire a normal OpenAI call. The response is identical; a signed receipt lands in your capture lake.

1. Install

The CLI ships as a Python package and a Node package. Either is fine; both speak the same config file.

# Python
pip install git+https://github.com/kolm-ai/kolm@main#subdirectory=sdk/python

# Node
npm install -g @kolm/cli

# Verify
kolm --version
kolm whoami   # prints "logged_in: false" until you sign up

2. Sign up for a free API key

The free tier includes 50,000 captures per month, all 11 providers, signed receipts, and one namespace. No card required.

kolm signup --email you@example.com
# or visit https://kolm.ai/signup

# Save the returned key
export KOLM_API_KEY=ks_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

3. Point your SDK at the wrap endpoint

The wrap endpoint is a drop-in replacement for the provider native URL. The path after /v1/wrap/<provider> is forwarded unchanged. Your OPENAI_API_KEY stays untouched — it travels through to OpenAI in the Authorization header. The KOLM_API_KEY authenticates you to the gateway via the X-Kolm-Key header.

# OpenAI
export OPENAI_BASE_URL=https://kolm.ai/v1/wrap/openai
export OPENAI_API_KEY=sk-...           # your existing OpenAI key

# Anthropic
export ANTHROPIC_BASE_URL=https://kolm.ai/v1/wrap/anthropic
export ANTHROPIC_API_KEY=sk-ant-...

# Header for kolm itself
export KOLM_API_KEY=ks_...

4. Fire a request

Use whatever SDK you already have. The wrap endpoint accepts the provider native request shape and returns the provider native response shape. The signed receipt is attached as response header X-Kolm-Receipt-CID and stored in your capture lake.

curl -s https://kolm.ai/v1/wrap/openai/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "X-Kolm-Key: $KOLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role":"user","content":"Say hi."}]
  }' | jq .

# In the response headers you will see:
#   X-Kolm-Receipt-CID: bafy2bzace...
#   X-Kolm-Route-Decision: pinned
#   X-Kolm-Cost-USD: 0.000041

5. Verify the capture

List recent captures, fetch the receipt, replay the signature.

kolm captures list --limit 5
kolm captures get <receipt_cid>
kolm verify <receipt_cid>   # recomputes Ed25519 signature, prints OK / FAIL

What you have now

  • Every OpenAI / Anthropic / Gemini / DeepSeek call is captured with a signed kolm-audit-1 receipt.
  • You can attribute cost per namespace, per model, per route decision.
  • You can replay any capture into a different provider for A/B testing (kolm bakeoff).
  • You can compile a local .kolm specialist from those captures once you have a few hundred (kolm distill).
Caveats. Free-tier captures are stored on kolm.ai infrastructure. If your captures contain regulated data, switch to PII redaction mode redact_captures (see configuration) or self-host the gateway. Keys ship in plaintext to upstream providers exactly as they do in your current setup — the gateway never logs them.
