Wrapper configuration

gateway.toml &mdash; the single source of truth.

One file declares the gateway’s bind address, signing key path, PII mode, capture backend, telemetry exporter, every upstream provider, and per-namespace routing rules. Loaded once at boot; SIGHUP reloads it without dropping connections.

File location

The CLI looks for gateway.toml in three places, first match wins: $KOLM_GATEWAY_CONFIG, ./gateway.toml in the working directory, then ~/.kolm/gateway.toml. The hosted kolm.ai endpoint ships with a managed default; this file matters when you self-host.

Complete example

Every section is documented below. Required sections: [server], [signing], at least one [[provider]], and at least one [[namespace]].

# gateway.toml - production self-host example

[server]
bind             = "0.0.0.0:8080"
public_url       = "https://gateway.internal.example.com"
request_timeout  = "30s"
shutdown_grace   = "15s"
max_body_bytes   = 4194304        # 4 MiB

[signing]
key_path         = "/etc/kolm/signing.ed25519"
key_id           = "tenant_acme_v3"
algorithm        = "Ed25519"
receipt_schema   = "kolm-audit-1"
key_rotation_overlap_days = 30    # NIST SP 800-57 aligned

[redaction]
mode             = "redact_captures"   # detect_only | redact_captures | redact_all | block
patterns         = ["email", "phone", "ssn", "credit_card", "api_token"]
replace_with     = "[REDACTED:{kind}]"

[capture]
backend          = "postgres"          # jsonl | sqlite | postgres | s3
connection_url   = "postgres://kolm:secret@db.internal:5432/kolm"
hash_chain       = "hmac_sha256"
batch_size       = 64
flush_interval   = "500ms"
async            = true                # release response before durable

[telemetry]
exporter         = "otlp"
endpoint         = "http://otel-collector:4317"
service_name     = "kolm-gateway"
sample_rate      = 1.0

# --- providers ---------------------------------------------------------

[[provider]]
name             = "openai"
base_url         = "https://api.openai.com/v1"
auth_scheme      = "bearer_passthrough"
timeout          = "60s"
retry_count      = 1

[[provider]]
name             = "anthropic"
base_url         = "https://api.anthropic.com"
auth_scheme      = "x_api_key_passthrough"
timeout          = "60s"

[[provider]]
name             = "local-vllm"
base_url         = "http://vllm-cluster:8000/v1"
auth_scheme      = "none"
timeout          = "120s"

# --- namespaces --------------------------------------------------------

[[namespace]]
name             = "default"
primary          = "openai/gpt-4o-mini"
fallback         = ["anthropic/claude-haiku-4-5", "local-vllm/qwen-3.6-7b"]
confidence_route = false

[[namespace]]
name             = "support-bot"
primary          = "local-vllm/kolm-support-v3.kolm"
fallback         = ["openai/gpt-4o-mini"]
confidence_route = true
confidence_profile = "balanced"          # aggressive | balanced | conservative
quota_captures_per_day = 50000

[signing]

Ed25519 keypair, stored on disk as a 32-byte seed. Use kolm key generate to create one. The key_id travels in the receipt header so verifiers can fetch the matching public key from /v1/keys/<key_id>. Rotation overlap defaults to 30 days — both old and new public keys remain valid during the overlap.

[redaction] &mdash; four PII modes

ModeForwarded to upstreamStored in captureUse case
detect_onlyRaw promptRaw prompt + detection annotationsInvestigate exposure before enforcing.
redact_capturesRaw promptRedacted copyDefault for regulated workloads.
redact_allRedacted promptRedacted copyMaximum: model never sees PII either.
block——Reject the request with HTTP 422; surface a privacy event.

[capture] &mdash; four storage backends

All four backends share the same hash-chained append-only schema. The hash_chain field is HMAC-SHA256 over (previous_hash, receipt_bytes). Choose the backend that matches the volume.

BackendBest forThroughputCompaction
jsonlDev, single-host, <100 req/sHigh (append-only file)Manual rotate
sqliteSingle-node prod, <500 req/sHighVACUUM nightly
postgresMulti-node, queryableMediumTimescaleDB recommended
s3Long-term retention, audit exportStreamingS3 lifecycle to Glacier

[telemetry]

Emits OpenTelemetry spans for every pipeline stage. The receipt CID is attached as a span attribute so you can correlate a capture row with the trace. Sample rate of 1.0 captures all; drop to 0.05 for high-volume production with sampling done at the collector.

[[provider]] &mdash; one block per upstream

See the providers page for the full table of base URLs and auth schemes. The name is the slug used in /v1/wrap/<provider> and in namespace primary / fallback entries.

[[namespace]] &mdash; routing + quota

A namespace is a routing scope. Requests pick a namespace via the X-Kolm-Namespace header, a ?ns= query parameter, or the default. Each namespace declares its primary model, a fallback chain, whether confidence routing is on, and an optional daily quota.

Reload without dropping connections

Send SIGHUP to the gateway process. The config is reparsed; in-flight requests finish with the old config, new requests use the new one. kolm gateway reload wraps the signal.

Caveats. Changing [server] bind or [signing] key_path requires a full restart, not a reload. The signing key swap walks the 30-day overlap window; do not delete the old key file until the overlap expires.
