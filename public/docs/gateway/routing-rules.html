Wrapper routing rules

Per-namespace routing in one TOML block.

A namespace declares a primary model and an ordered fallback chain. Optionally enable confidence routing so cheap local handles the easy queries and the expensive teacher only gets the hard ones.

The minimum routable namespace

A namespace needs a name and a primary. Fallback is optional but recommended.

[[namespace]]
name    = "default"
primary = "openai/gpt-4o-mini"

Primary + fallback chain

The fallback chain is tried in order on retryable failures (429, 502, 503, 504, network timeouts). Each entry is a <provider>/<model> pair. A retry budget caps the total attempts across the chain.

[[namespace]]
name           = "rag-pipeline"
primary        = "openai/gpt-4o"
fallback       = [
  "anthropic/claude-sonnet-4-5",
  "deepseek/deepseek-v3.1",
  "local-vllm/qwen-3.6-27b"
]
retry_budget   = 4              # total attempts across chain
retry_backoff  = "exponential"  # exponential | linear | none

Confidence routing (optional)

Set confidence_route = true to enable entropy-based routing. The gateway first probes the local student model for the next-token distribution. If the Shannon entropy of the first token is below the threshold, the student handles the call. If above, the request escalates to the primary teacher.

[[namespace]]
name               = "customer-support"
primary            = "openai/gpt-4o"          # teacher (escalation)
fallback           = ["anthropic/claude-sonnet-4-5"]
confidence_route   = true
confidence_student = "local-kolm/support-v3.kolm"
confidence_profile = "balanced"               # threshold = 0.7 nats
# or set raw threshold:
# confidence_threshold = 0.65
ProfileThreshold (nats)Student share (typical)Behavior
aggressive0.8590–95%Send almost everything local; escalate only on high uncertainty.
balanced0.7070–80%Default. Even cost vs. quality split.
conservative0.5540–55%Send only the easy half local; escalate on any wobble.

Pinning a request

A caller can override routing for one request via headers. Useful when the application already knows the right tier.

HeaderEffect
X-Kolm-Pin-ProviderForce a specific provider slug; bypass route decision.
X-Kolm-Pin-ModelForce a specific model; primary if pinned alone.
X-Kolm-No-FallbackDisable fallback; fail fast if primary is down.
X-Kolm-NamespacePick a namespace other than default.

Per-namespace quota

Cap the daily captures per namespace. When the cap is reached, requests get HTTP 429 with Retry-After set to seconds until midnight UTC. Useful for keeping a dev namespace from burning the team budget.

[[namespace]]
name                   = "dev-sandbox"
primary                = "openai/gpt-4o-mini"
quota_captures_per_day = 500
quota_action           = "throttle"   # throttle | log | block

Inspecting decisions

Every response includes the route headers X-Kolm-Route-Decision, X-Kolm-Confidence-Nats, and X-Kolm-Fallback-Reason. The same fields are stored on the receipt for offline analysis.

kolm routes list --namespace customer-support --limit 50
kolm routes stats --namespace customer-support --since 7d
# returns: student_share, escalation_share, mean_entropy_nats, USD saved
Caveats. Confidence routing requires the student model to be loaded and responsive. If the student is cold or fails its probe, the gateway logs fallback_reason = "student_probe_failed" and routes to the teacher. Warm the student at boot via preload = true on the local-kolm provider.
