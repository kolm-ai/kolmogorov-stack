Glossary.

Adapter#
A small set of weights, usually a LoRA, that overlays a base model to specialize it for one task. Adapters are smaller than a full fine-tune and cheap to ship as part of a specialist .kolm. The base model stays frozen; only the adapter changes.
AI compiler#
A toolchain that turns captured prompts, evals, and a base model into a self-contained artifact you can run on your own hardware. The way a C compiler turns source into a binary, kolm turns captures plus a recipe into a .kolm. Background reading at /what-is-an-ai-compiler.
Artifact (.kolm)#
A signed zip containing the manifest, the recipe, the eval suite, weights or adapter, tokenizer, and a receipt chain. The shippable output of kolm compile. Verifiable at /verify-prod.
Bundle#
An offline distribution package combining one or more artifacts with a runtime binary and verifier. Built by the air-gap export verb (see the CLI reference) for offline install. Same provenance guarantees as the individual artifact.
Capture#
A recorded request and response pair from a teacher model, plus the prompt template hash, timing, and cost. Captures are the raw material that distillation learns from. Live tail at /captures.
Capture namespace#
A label that groups captures by task, for example claims-router or phi-redactor. The unit of work for kolm distill --namespace and the slot that thresholds and gates live under.
Compile#
The end-to-end step that turns a recipe plus captures into an .kolm. Runs evals, gates on K-score, signs, and emits a receipt chain. CLI verb: kolm compile.
Distillation#
Training a smaller student model to imitate a larger teacher. kolm supports soft-label KD, top-K KD, and rejection sampling, picked automatically by --mode auto. See /docs/distill.
Drift#
Behavior change between a teacher model snapshot and a later snapshot, measured by re-running the same fixed eval set. Reported per provider on /drift. The number you cite when a frontier vendor silently updates a model.
Frozen-set gate#
An eval-pass gate that blocks compile from shipping if the held-out frozen set falls below a configured K-score. The frozen set is sealed at recipe-creation time so it cannot be optimized against.
Tenant-local HMAC chain#
A shared-secret receipt ledger for one-tenant deployments or contractually bound pairs. Public distribution and third-party verification use Ed25519 receipt signatures so the verifier does not need a tenant secret.
K-score#
kolm's composite quality score, scaled 0 to 1, computed on a held-out K-sample. Six axes: Accuracy, Stability, Latency, Cost, Variance, Teacher. Full methodology at /k-score.
K-sample#
The fixed sample set used to compute a comparable K-score across runs. Same prompts, same seeds, different artifact. K-samples are shareable so two teams can compare apples to apples.
Recipe#
A JSON document that names the task, the base model, the eval suite, the frozen-set gate, and the threshold values. Input to compile. Reusable across runs; you change one field and re-compile.
Receipt#
A signed JSON record for one build step. Includes input hashes, output hashes, K-score, timestamp, and a sigstore Rekor entry id. Format documented at /spec/rs-1.
Receipt chain#
The ordered list of receipts inside an .kolm. Verifies end-to-end build provenance: tokenize, distill, compile, sign, witness. Re-verified by kolm verify any time.
Sandbox#
The constrained runtime where a .kolm executes user code. Blocks process, require, eval(, Function(, and outbound sockets. Forbidden-identifier list at /docs/sandbox.
Specialist#
A .kolm that targets one narrow task such as PHI redaction or claims routing. Usually well under 5 GB and a fraction of teacher cost per call. Built from at least 1000 captures in one namespace.
Verifier#
The tool that re-checks an artifact: signature, receipt chain, frozen-set K-score. Run via kolm verify locally or hosted at /verify-prod.
Witness#
An external attester that countersigns receipts. kolm uses sigstore Rekor as the default public transparency witness. Self-hosted witness optional for air-gapped installs.
