GitHub Actions integration.

Drop a .github/workflows/kolm.yml in your repo. Every push runs the K-Score gate against your latest artifact, auto re-distills if quality drops below the threshold you set, and publishes the new .kolm to GitHub Releases for downstream consumers to pin.

No-silent-pass contract: the action only runs when the KOLM_API_KEY secret is configured. A fork without the secret sees the workflow skip cleanly; no red X on every commit. Stamped w820-v1.

Template

A reference-only template ships in the kolm repo at .github/workflows/kolm.yml. It is intentionally guarded with if: false on the distill job so the file does not run unintended CI minutes. Flip the guard and configure the secret to enable it.

Copy the template into your repo and commit. The template is self-documented with comments explaining each section.

kolm.yaml schema

The workflow expects a kolm.yaml at the repo root. Minimum shape:

KeyTypeRequiredNotes
artifactstringyesThe .kolm artifact name or hash this workflow gates against.
kscore_thresholdfloat 0..1yesBelow this, the gate fails and re-distill triggers.
teacher_idstringyese.g. claude-sonnet-4-7, gpt-4o, local:qwen-2.5-72b.
namespacestringyesThe kolm namespace whose captures feed re-distill.
pathslist of stringsnoGlob patterns; the workflow only triggers when files matching these change. Defaults to **.
publish_releasebooleannoIf true, the post-distill .kolm lands in GitHub Releases for this commit SHA. Default false.
quality_gatesmapnoPer-namespace overrides for min_kscore, max_latency_ms, etc.

The action: kolm/distill-action@v1

One Action, two phases. On every push:

  1. Reads kolm.yaml, replays recent captures through the active artifact, scores them.
  2. If the K-Score falls below kscore_threshold, runs kolm distill --from-yaml to produce a new artifact and gates the merge until it passes.

The action sets two output variables:

OutputTypeMeaning
kscorefloatThe measured K-Score for this run.
kscore_passedbooleantrue if K-Score >= kscore_threshold.
artifact_pathstringLocal path to the produced .kolm, present only when a re-distill ran.
artifact_hashstringsha256 of the produced .kolm.

Downstream steps can branch on kscore_passed to gate the merge: a simple if: steps.kolm.outputs.kscore_passed == 'true' on a "Mark mergeable" step is enough.

Auto re-distill on K-Score regression

When the gate fails, the action runs kolm distill --from-yaml --out ./dist. The new artifact replaces the previous one if and only if:

  • It passes the same kscore_threshold on the gate dataset.
  • It does not regress on any of the per-namespace gates declared in quality_gates.
  • Its drift gate is green (kolm autopilot rules apply if the workflow was launched by autopilot).

If any of those conditions fail, the new artifact lands in ./dist but is NOT promoted; the action emits a distill_failed annotation on the commit and the operator decides what to do. Silent promotion of a bad artifact is the failure mode the workflow refuses to allow.

Publish .kolm to GitHub Releases

With publish_release: true, the action attaches the produced .kolm to a release tagged kolm-distill-<commit-sha>. Downstream consumers can pin to a specific commit:

kolm pull github://kolm-ai/kolm@abc1234

The release body includes the K-Score, the teacher used, the namespace, and the commit range that changed since the last release. The artifact is also pushed to the kolm hosted registry so the same kolm://sha256-... URI resolves regardless of GitHub's availability.

kolm diff between versions

Once two artifacts exist, compare their quality on the same eval pack:

kolm diff v1.2.kolm v1.3.kolm

Output reports the K-Score delta, latency delta, size delta, and a per-namespace breakdown. Use this in a release-note generator to document the quality change without hand-editing.

For the CI version that posts the diff as a PR comment:

npx kolm diff dist/old.kolm dist/new.kolm --comment-on-pr

The PR comment is idempotent (each run rewrites the same comment) so the PR thread doesn't accumulate stale numbers.

Example minimal workflow

The smallest workflow that does something useful:

name: kolm
on:
  push:
    branches: [main]
  pull_request:

jobs:
  kolm-gate:
    runs-on: ubuntu-latest
    if: ${{ secrets.KOLM_API_KEY != '' }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
      - uses: kolm/distill-action@v1
        id: kolm
        with:
          kscore_threshold: 0.85
        env:
          KOLM_API_KEY: ${{ secrets.KOLM_API_KEY }}
      - name: Fail merge on K-Score regression
        if: steps.kolm.outputs.kscore_passed != 'true'
        run: exit 1
