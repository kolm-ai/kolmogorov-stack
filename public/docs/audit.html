Audit log.

Every privileged action on a kolm tenant lands in a hash-chained jsonl log. The chain is verified on every read; a break raises and refuses to return rows. Retention is 90 days hot for fast query, 7 years cold for compliance, with CSV, JSON, and CEF export.

What we log

Every operation in the AUDIT_OPS enum writes a line. The set covers tenant lifecycle, capture flow, model lifecycle, deployment, and trust-fabric actions:

OpTriggered by
TENANT_SIGNUPPOST /v1/signup
API_KEY_ISSUE / API_KEY_REVOKEPOST /v1/keys
CAPTURE_LOG / CAPTURE_QUARANTINE / CAPTURE_REVIEW_DECISIONcapture pipeline + reviewer action
DISTILL_RUN / DISTILL_ROLLBACKkolm distill
QUANTIZE_RUNkolm quantize
ARTIFACT_BUILD / ARTIFACT_SIGN / ARTIFACT_DEPLOYkolm compile / sign / deploy
FEDERATED_OPTIN / FEDERATED_OPTOUT / FEDERATED_SHAREW461 federated approvals
ATTESTATION_EMBEDW460 confidential-compute artifact
SNEAKERNET_PACK / SNEAKERNET_IMPORTW409w airgap transfer
SSO_LOGIN / SCIM_UPDATEW560 SSO bridges

Format

The log is append-only newline-delimited JSON. Each row carries a prev_hash that chains to the SHA-256 of the previous line; the first line of a chain has prev_hash: null.

{"ts":"2026-05-24T19:14:02Z","op":"DISTILL_RUN","tenant":"tenant_4d3d8db6c820",
 "actor":"ks_4b7bc3b1...","namespace":"prod","artifact":"art_a1b2c3d4",
 "detail":{"corpus_count":1832,"k_score":0.871,"teacher":"frontier-base-v3"},
 "prev_hash":"e9b7...c421","row_hash":"7f3a...20be"}

Retention

  • Hot. 90 days at full fidelity, indexed for query by tenant, op, namespace, actor, and time window.
  • Cold. 7 years in object storage, gzipped per day, separately encrypted, with the daily chain root committed to the tenant audit_anchors table.
  • Tenant deletion. On tenant termination the actor's PII is redacted from cold rows; the row_hash chain remains intact so prior verifications stay valid.

Export

kolm audit export --since 2026-01-01 --until 2026-04-01 --format csv  > audit.csv
kolm audit export --since 2026-Q1 --format json                       > audit.json
kolm audit export --since 2026-Q1 --format cef                        > audit.cef

CEF (Common Event Format) export is provided for SIEM ingestion (Splunk, QRadar, Sentinel, Sumo). The CEF schema documentation ships in the export archive.

Tamper detection

Every read of the hot log re-verifies the prev_hash chain over the returned window. If any row's row_hash does not equal sha256(canonical_json_minus_row_hash), or if any row's prev_hash does not equal the row_hash of its immediate predecessor, the read fails closed:

kolm audit verify --since 2026-04-01
# chain verified: 12,481 rows
# chain root: 7f3a...20be

kolm audit verify --since 2026-04-01
# ERROR audit_chain_break at row 8722: prev_hash mismatch
# exit code: 7

Cold-archive chain roots are anchored daily to the tenant's audit_anchors table so off-box tampering with cold gzip files is detectable on retrieval.
