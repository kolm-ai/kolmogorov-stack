Studio / Publish / HuggingFace

One command. HF repo. Auto-generated card.

Publishing a kolm artifact to HuggingFace is a single CLI call. The publisher pushes weights (in your chosen formats), generates a canonical model card from the artifact passport, attaches receipts as repo assets, and tags the release with the K-Score scores so search filters find it. Card generation is handled by src/hf-modelcard.js.

Prerequisites

  1. A HuggingFace account + access token with write scope. Set HF_TOKEN=<token> or run huggingface-cli login.
  2. A compiled .kolm artifact — or at least one exported format (e.g., my-artifact-q4km.gguf).
  3. Decide the repo id: <org-or-user>/<slug> (e.g., kolm-ai/trinity-support-7b).

The kolm publish hf verb

# Publish a single artifact in default formats (Q4_K_M + Q5_K_M + Q8_0 + fp16 .kolm)
kolm publish hf trinity-support.kolm --repo kolm-ai/trinity-support-7b

# Publish specific formats only
kolm publish hf trinity-support.kolm \
  --repo kolm-ai/trinity-support-7b \
  --formats gguf-q4km,gguf-q5km,gguf-q8,awq-4bit

# Private repo
kolm publish hf trinity-support.kolm --repo myorg/internal-support-7b --private

# Dry-run — build the card and stage uploads, don't push
kolm publish hf trinity-support.kolm --repo kolm-ai/trinity-support-7b --dry-run

# Resume a partial upload
kolm publish hf trinity-support.kolm --repo kolm-ai/trinity-support-7b --resume

What goes in the auto-generated card

src/hf-modelcard.js renders README.md from the artifact passport. The card includes:

  • Front matter — HF tags (base_model, library_name: transformers, license, language, pipeline_tag: text-generation, quantized_by: kolm-forge, tags: [kolm, distilled, <recipe-name>]).
  • Model card body — name, base, distill method, teacher council with weights, judge id + rubric, K-Score per axis with absolute number, training cost (USD), training time + hardware, eval set provenance.
  • Quick-start snippets — transformers, vLLM, llama.cpp, Ollama, LM Studio, and the kolm verify command.
  • Provenance & verification — how to verify the signature locally (one paste-in command) + a link back to https://kolm.ai/v1/verify/<artifact_hash>.
  • Citation — BibTeX block.
  • License — from spec.toml, with a footer naming any teacher whose ToS requires attribution.

Customising the card

Drop a Markdown file at publish/card-override.md in your run directory and the publisher will append it after the auto-generated body but before the citation. Useful for "Why we built this", custom examples, or vertical-specific guidance:

kolm publish hf trinity-support.kolm \
  --repo kolm-ai/trinity-support-7b \
  --card-extra publish/card-override.md

Repo visibility & gating

  • --private — repo is private to your account/org.
  • --gated — HF gated repo (users must accept terms before download).
  • --with-extras receipt-chain — attach the full receipt chain (default true; set false to slim the repo).
Caveats. Some teacher providers' ToS forbid redistribution of their outputs as model weights. The publisher reads passport.json and refuses to push if any teacher in the council has allow_distill_publish=false in data/teacher-license-map.json. Override with --ignore-teacher-license-warnings if you've cleared it directly with the provider; the override is logged in the publish receipt.

Also publishing to the kolm registry

If you want the artifact searchable on kolm.ai/registry, add --also-kolm-registry — same push, dual destination, single command.

kolm publish hf trinity-support.kolm \
  --repo kolm-ai/trinity-support-7b \
  --also-kolm-registry
