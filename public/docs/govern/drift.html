Run & Govern

Drift signals.

A distilled model that wins on its eval set today can quietly drift away from the population it serves next quarter. kolm watches three independent layers and raises a signal the moment any one of them slips. The signal is visible in Governance → Drift and exportable via kolm drift report.

The three layers

LayerStatisticWhat it catches
Input distributionPopulation stability index over embedding centroidsUsers start asking about new topics the artifact was not trained on.
Output distributionKL divergence of token-1 distribution vs baselineThe model has started answering differently even when inputs look the same.
k-scoreRolling 7-day k-score on the held-out replay setReal-world quality (judge-graded) has slipped past the per-artifact floor.

Signal shape

{
  "artifact_id": "art_a1b2c3d4",
  "namespace": "prod",
  "layer": "k_score",
  "value": 0.812,
  "baseline": 0.871,
  "delta_pct": -6.77,
  "confidence": 0.94,
  "window": "7d",
  "recommended_action": "open_recall_review",
  "first_seen": "2026-05-20T08:14:00Z"
}

A signal does not auto-roll back. It opens a recall review in the lifecycle queue and pages the on-call governance owner. Rollback is always a human decision; the signal exists to make sure the decision happens.

Thresholds

Each layer has a per-artifact threshold pinned at compile time. The defaults are conservative and tuned per recipe; override in spec.toml:

[govern.drift]
input_psi_max = 0.25
output_kl_max = 0.18
k_score_floor = 0.84
window         = "7d"
require_human  = true

Rollback path

  1. Drift signal fires → recall review opens in Lifecycle.
  2. Reviewer inspects the receipts that triggered it, then chooses: keep, roll back to prior artifact, or schedule re-distill.
  3. Rollback issues a routing-table swap (zero downtime, no pod restart) and writes an ARTIFACT_ROLLBACK row to the audit log.
