Run & Govern / Overview

Serve the artifact. Monitor it. Prove what it does.

Run & Govern is the surface that takes a signed .kolm from the compile pipeline, boots it on the runtime that best fits the host hardware, and gives you the evidence a buyer's security review will accept. Three layers: runtime (passport, auto-detect, deploy generators), governance (lifecycle, evidence DAG, assurance case), and operations (drift, cost displacement).

Runtime

Runtime passport →Per-(runtime, target) capacity row pinned to every artifact. Tested or estimated, never invented. Serve auto-detection →Given the artifact + host shape, pick llama.cpp / vLLM / mlx / ollama. Pure decision table. Docker Compose →Hand-rolled YAML with healthcheck, GPU reservations, optional TLS proxy block. Kubernetes manifests →ConfigMap, PVC, init Job, Deployment, Service, HPA in one apply. vLLM config →JSON for vllm serve --config-file. Validates round-trip before emit. Air-gap bundle →Real tar.gz with artifact, runtime installer, offline verifier, sha256 manifest.

Governance

Artifact lifecycle →created -> signed -> deployed -> monitored -> superseded / revoked / archived. Append-only history. Evidence DAG →Capture / eval / teacher / student / runtime / signature / policy / rights. Cycle-free by construction. Assurance case →The trust packet a procurement team can attach to a third-line review.

Operations

Drift detection →Fallback-rate delta + distribution distance + volume ratio against a baseline window. Cost displacement →Per-namespace savings from local routing vs the frontier counterfactual.

Reference

CLI reference →Every kolm serve / passport / evidence / assurance / drift / cost verb. HTTP API reference →The routes the modules expose under /v1/runtime, /v1/lifecycle, /v1/evidence, /v1/assurance, /v1/drift, /v1/cost. FAQ →Ten questions a buyer's risk team will ask, with the actual answer.

The shape

Three modules carry the contract; everything else is built around them.

src/runtime-passport.js        # R-1 schema + validators + estimator
src/artifact-lifecycle.js      # R-2 state machine, append-only history
src/serve-autodetect.js        # R-3 routing table, builds spawn invocation
src/serve-metrics-sidecar.js   # R-3 /metrics + /health Prometheus sidecar
src/deploy-generators.js       # R-4 docker / k8s / vllm config / air-gap tar
src/evidence-dag.js            # R-5 nodes + edges + cycle check
src/evidence-store.js          # R-5 per-artifact persistence + node index
src/assurance-case.js          # R-6 claims + controls envelope
src/assurance-case-pdf.js      # R-6 PDF renderer (pdfkit, lazy import)
src/drift-detector.js          # R-7 SPC ladder (2 sigma warn, 3 sigma alert)
src/cost-displacement.js       # R-8 baseline vs actual per receipt

Design rules

  • Never invent. A runtime passport row with status: "tested" requires real measurements. A cost-displacement row with no rate-card entry returns partial:N_unrepriced instead of a guess.
  • Caveats over silence. Every claim in the assurance case carries a limitations string that names what is not covered.
  • Append-only audit trails. Lifecycle history, evidence DAG, and signature blocks are all write-once.
  • Tenant scoping at the lake boundary. Drift detector and cost displacement use findByTenant; cross-tenant queries are structurally impossible.
