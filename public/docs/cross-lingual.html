Open-source AI workbench

Cross-lingual distillation.

Eleven languages auto-detected per capture. Distribution endpoint reports the per-language mix; auto-balance suggests mixture weights when a language drops below the configured floor; synthetic translation fills the underrepresented slots without burning teacher tokens on a typo. Per-language K-Score lands on the artifact manifest. Stamped w833-v1.

Supported languages

The detector is character-and-script-based, optimized for short prompt-response pairs (the typical capture row). Eleven language codes are recognized; an undetectable row resolves to unk.

The detector is in src/lingual-detect.js and the set is exported as SUPPORTED_LANGS_W833. Detector version is stamped w833-v1; future versions will increment and the version is recorded on every detection envelope.

Distribution endpoint

The distribution endpoint reads tenant captures for a namespace, runs the detector over each row, and returns per-language counts plus the underrepresentation flag against an optional target_ratio.

GET /v1/lingual/distribution

curl -H "Authorization: Bearer $KOLM_API_KEY" \
  "https://kolm.ai/v1/lingual/distribution?namespace=support-bot⌖_ratio=0.05"

# -> {
#   "ok": true,
#   "namespace": "support-bot",
#   "by_lang": { "en": 4823, "es": 412, "fr": 28, "ja": 7, "unk": 130 },
#   "underrepresented": ["fr","ja"],
#   "target_ratio": 0.05,
#   "version": "w833-v1"
# }

The query is bounded to 5000 rows per call (recent first); ample for the auto-balance suggestion. For full-corpus aggregation use /v1/lake/export and group by language locally.

Auto-balance

When a language drops below the floor, auto-balance suggests mixture weights for the next training run. The suggestion is a sampler-weight per language; multiply by your batch size to get the per-language batch share.

POST /v1/lingual/mixture/auto-balance

curl -X POST https://kolm.ai/v1/lingual/mixture/auto-balance \
  -H "Authorization: Bearer $KOLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "namespace": "support-bot",
    "target_langs": ["en","es","fr","ja"],
    "floor": 0.05
  }'

# -> {
#   "ok": true,
#   "namespace": "support-bot",
#   "distribution": { ... per-language counts ... },
#   "suggested": { "weights": { "en": 0.35, "es": 0.30, "fr": 0.20, "ja": 0.15 } },
#   "version": "w833-v1"
# }

The suggested weights are inverse-frequency biased toward the underrepresented languages, with a floor so a near-zero language is not given 90% of the batch (which would collapse the dominant languages). Pass an explicit target_langs list to clamp the suggestion to the languages you actually serve.

Synthetic translation

For underrepresented languages with too few real captures, the synthesize endpoint asks a teacher to translate existing rows. Synthetic rows are tagged so the trainer can elect to down-weight or exclude them.

POST /v1/lingual/synthesize

curl -X POST https://kolm.ai/v1/lingual/synthesize \
  -H "Authorization: Bearer $KOLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "namespace": "support-bot",
    "target_lang": "ja",
    "count": 50,
    "teacher": "anthropic/claude-sonnet-4-5"
  }'

# -> { "ok":true, "rows":[ ...synthetic translations... ], "version":"w833-v1" }

By default the synthesized rows are returned in the response BUT not auto-written to the capture lake. The operator explicitly POSTs them to /v1/capture/log if they want them persisted. This avoids accidental teacher-spend on a typo'd target_lang.

Per-language K-Score

When the trainer ships an artifact, the per-language K-Score is computed across the evaluation slice and lands on the manifest. The manifest endpoint reads the block back so dashboards do not need to re-derive.

GET /v1/lingual/manifest/:artifact_id

curl -H "Authorization: Bearer $KOLM_API_KEY" \
  https://kolm.ai/v1/lingual/manifest/art_8a3f...

# -> {
#   "ok": true,
#   "artifact_id": "art_8a3f...",
#   "per_lang_kscore": {
#     "en": { "n_eval": 200, "kscore": 0.87 },
#     "es": { "n_eval": 50,  "kscore": 0.81 },
#     "fr": { "n_eval": 20,  "kscore": 0.74 }
#   },
#   "version": "w833-v1"
# }

When the artifact has no language-tagged eval rows, the manifest endpoint returns {ok:false, error:'no_manifest_found'} rather than fabricating a per-language KSCORE from zero rows.

CLI

The CLI exposes kolm lingual distribution, kolm lingual auto-balance, and kolm lingual synthesize wrapping the routes above.

kolm lingual distribution --namespace support-bot --target-ratio 0.05
# -> pretty-prints per-language counts and underrepresented set

kolm lingual auto-balance --namespace support-bot --floor 0.05 \
  --target-langs en,es,fr,ja
# -> suggested sampler weights

kolm lingual synthesize --namespace support-bot --target-lang ja --count 50 \
  --teacher anthropic/claude-sonnet-4-5
# -> synthetic rows printed; operator pipes through `kolm capture log` to persist
