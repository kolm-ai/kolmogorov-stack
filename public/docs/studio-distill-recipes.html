Studio / Recipes

Recipes are the spec. spec.toml is the contract.

A recipe is a TOML file that tells the distill driver everything it needs: which teachers to call, which judge to use, which student base, what the loss looks like, what the eval looks like. Recipes are reproducible: hand the same spec.toml + the same seed to the same hardware and you get the same artifact hash.

The spec.toml file

# trinity-support-500.toml — the canonical Trinity recipe
[meta]
name = "trinity-support-500"
description = "Support agent distilled from Claude Haiku 4.5 + GPT-4o-mini + DeepSeek V3"
authors = ["kolm.ai"]
license = "Apache-2.0"
created = "2026-05-26"

[student]
base = "Qwen/Qwen2.5-7B-Instruct"
context_length = 8192

[teachers]
council = [
  { id = "anthropic/claude-haiku-4-5", weight = 0.45 },
  { id = "openai/gpt-4o-mini",         weight = 0.30 },
  { id = "deepseek/deepseek-v3",       weight = 0.25 },
]

[judge]
id = "google/gemini-2.5-pro"
rubric = "support-rubric-v2"
min_score = 0.75

[data]
source = "captures"             # or "synth", or "file"
namespace = "support"
since_last_distill = true
min_pairs = 400
max_pairs = 2000

[loss]
method = "distillm-2"
seq_kd_weight = 0.7
skl_weight    = 0.2
srkl_weight   = 0.1

[train]
adapter = "lora"
lora_r = 16
lora_alpha = 32
lora_dropout = 0.05
target_modules = ["q_proj","k_proj","v_proj","o_proj","gate_proj","up_proj","down_proj"]
batch_size = 1
grad_accum = 8
max_len = 1536
epochs = 3
lr = 1.5e-4
seed = 42

[eval]
holdout_pct = 0.10
axes = ["asks_first_question","clarifies","on_policy","faithful","concise"]

List the catalogue with kolm distill --list-recipes or fetch JSON with kolm distill --list-recipes --json.

Authoring your own

  1. kolm distill --recipe support-trinity-500 --dump > my-support.toml — gives you the canonical recipe to edit.
  2. Swap the [student] base, the [teachers] council, the [judge] rubric to match your task.
  3. kolm distill --recipe my-support.toml --dry-run — validates schema + estimates cost + estimates VRAM. No teacher calls yet.
  4. kolm distill --recipe my-support.toml --output my-support.kolm — runs for real.

Schema validation

Every spec.toml is validated against the JSON schema at https://kolm.ai/spec/kolm-format-v1.json. The validator catches typos in axis names, missing weights, and inconsistent context lengths between teacher and student.

Caveats. The eval axes you list under [eval].axes must exist in the judge's rubric. If you reference an unknown axis the dry-run fails before any teacher call is made.

Recipe hash & reproducibility

The compiled .kolm embeds recipe_hash = sha256(canonical_toml). Two artifacts with the same recipe_hash + same seed + same teacher versions + same student commit are byte-identical — kolm verify --reproduce my.kolm re-runs the recipe and asserts the produced hash matches.

Distill →The pipeline that consumes a recipe. Teachers →What goes into [teachers]. Judges →What goes into [judge].
