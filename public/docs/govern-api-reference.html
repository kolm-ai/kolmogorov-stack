Run & Govern / Reference

HTTP API reference.

REST routes the Run & Govern modules expose under /v1/runtime, /v1/lifecycle, /v1/evidence, /v1/assurance, /v1/drift, and /v1/cost. Every route is tenant-scoped via the API key on Authorization: Bearer ks_.... Every response is JSON with a top-level {ok, version, ...} envelope. Errors return non-2xx status with a structured {ok: false, error, hint} body.

Authentication

Authorization: Bearer ks_4b7bc3b...

# Or via environment variable in client SDKs:
KOLM_API_KEY=ks_4b7bc3b...
KOLM_BASE_URL=https://kolm.ai     # default

Sign up at /signup to mint a key. Keys carry their tenant binding so cross-tenant queries are structurally impossible.

/v1/runtime

VerbPathReturns
GET/v1/runtime/passport/:artifactArray of runtime_passport rows attached to the artifact.
POST/v1/runtime/passport/:artifact/estimateStamps a status:estimated row from {runtime, target_id, model_params, quant_bytes_per_param}.
POST/v1/runtime/passport/:artifact/recordStamps a status:tested row from a real measurement.
POST/v1/runtime/autodetectBody: {artifact_id, host_probe}. Returns the runtime decision plus invocation arguments.
POST/v1/runtime/deployBody: {artifact, target, runtime?, ...}. Returns the rendered config (compose / k8s yaml / vllm json) or air-gap bundle metadata.

/v1/lifecycle

VerbPathReturns
GET/v1/lifecycle/:artifact{state, history: [{from, to, ts, reason}]}.
POST/v1/lifecycle/:artifact/transitionBody: {to, reason?}. 200 on valid; 409 on invalid transition with {from, to, valid_next}.
GET/v1/lifecycle/:artifact/historyAppend-only array of transition records.

/v1/evidence

VerbPathReturns
GET/v1/evidence/:artifactFull DAG {spec, nodes, edges}.
GET/v1/evidence/:artifact/trace/:node{node, ancestors: [...], edges: [...]}.
GET/v1/evidence/:artifact/descend/:node{node, descendants: [...], edges: [...]}.
POST/v1/evidence/:artifact/revokeBody: {node, reason?}. Returns {revoked, needs_review}.
POST/v1/evidence/:artifact/add-nodeBody: {node, edges?}. Validates and appends.
GET/v1/evidence/lookup/:nodeReverse index: {artifact_id} or 404.

/v1/assurance

VerbPathReturns
POST/v1/assurance/buildBody: {artifact_id, workspace_id?}. Returns full kolm-assurance-case-1 envelope.
POST/v1/assurance/validateBody: {envelope}. Returns {ok, reasons}.
POST/v1/assurance/renderBody: {artifact_id, format: "pdf"|"json"}. Returns application/pdf binary or JSON.

/v1/drift

VerbPathReturns
GET/v1/drift?namespace=<ns>&lookback_days=7&baseline_days=30The full drift envelope (signals, status, recommendation, details).
POST/v1/drift/enableBody: {namespace}. Marks the namespace as monitored (required for the assurance case _claimDriftMonitoring to flip to implemented).
POST/v1/drift/disableBody: {namespace}. Removes the namespace from active monitoring.
GET/v1/drift/namespacesList of all enabled drift namespaces for the tenant.

/v1/cost

VerbPathReturns
GET/v1/cost/displacement?namespace=<ns>&period_days=30Per-namespace cost displacement envelope.
GET/v1/cost/summary?namespace=<ns>30-day roll-up: savings, cumulative savings, payback period.

Response shape

Successful responses carry HTTP 200 and a JSON body with a top-level ok: true:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "ok": true,
  "version": "r8-v1",
  "tenant_id": "tenant_4d3d8db6c820",
  "namespace": "support",
  "savings_usd": 1678.25,
  "ok_status": "computed"
}

Errors carry a non-2xx status and a structured body. The error field is a stable enum the SDKs branch on; hint is human prose for the operator.

HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "ok": false,
  "error": "invalid_period_days",
  "hint": "period_days must be a positive number <= 3650",
  "version": "r8-v1"
}

HTTP status codes

StatusMeaning
200OK.
400Bad input (missing required field, invalid value).
401Missing or invalid Authorization header.
402Plan does not include the requested feature (SSO, SAML, audit log export).
404Artifact, namespace, or evidence node does not exist.
409Invalid state transition (lifecycle) or DAG cycle (evidence).
422Validation failed (envelope shape, passport schema).
429Rate limit. Backoff and retry.
503Feature exists but is not yet implemented in this build (gated rollout).

curl examples

# Fetch the runtime passport for an artifact
curl -H "Authorization: Bearer $KOLM_API_KEY" \
  https://kolm.ai/v1/runtime/passport/qwen3-8b-support-q4km

# Compute drift for a namespace
curl -H "Authorization: Bearer $KOLM_API_KEY" \
  "https://kolm.ai/v1/drift?namespace=support&lookback;_days=7&baseline;_days=30"

# Build the assurance case
curl -X POST -H "Authorization: Bearer $KOLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"artifact_id":"qwen3-8b-support-q4km"}' \
  https://kolm.ai/v1/assurance/build
Caveats. The HTTP routes are 1:1 with the CLI verbs. If a verb is documented in the CLI reference but missing here, that route is gated behind a future plan tier and will return HTTP 503 with {error: 'not_yet_implemented'} for non-entitled keys. Track the changelog for promotion.
