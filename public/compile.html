kolm compile · the one user-facing primitive

One command. Four engines. One signed artifact.

kolm compile takes anything you can describe, "answer my emails in my voice", "spot bugs in my React PRs", "find every photo from a beach trip", plus the data that defines what you mean, and emits a portable .kolm file that does that thing, anywhere, signed end-to-end.

The contract.

# task only $ kolm compile "summarize this folder" --data ./notes # task + examples (one example per line, JSONL) $ kolm compile "classify intent" --examples ./examples.jsonl # task + corpus + base model override $ kolm compile "code review my Python style" \ --data ./projects \ --base-model qwen2.5-coder-7b-instruct-q4_0 # output: a .kolm file in ~/.kolm/artifacts/ ✔ ~/.kolm/artifacts/job_abc.kolm (2.3KB)

What runs underneath.

stage 1

recall.start → recall.done

If --data was given, qmd indexes the folder (BM25 + vector + RRF + reranker). Top-k chunks are pulled for the task and prepended to every Distill call so the artifact knows your specifics.

stage 2

distill.start → distill.done

Synthesize a verifier from your examples. K-sample a frontier model. Verify each candidate. Accept the winner. P(correct) ≥ 1 − (1 − p·v)k. Generator-Verifier asymmetry made shippable.

stage 3

decompose.start → decompose.done

The Recipe registry slice that covers this task's structured-token subset. Public registry compounds: every successful compile auto-merges new draft patterns.

stage 4

package.start → package.done

Single zip: manifest.json · model.gguf pointer · recipes.json · lora.bin · index.sqlite-vec · evals.json · signature.sig. K-score patched into the manifest. HMAC-SHA256 over canonical JSON, anchored to Kolmogorov's public registry.
