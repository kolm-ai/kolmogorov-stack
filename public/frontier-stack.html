kolm.ai  /  frontier stack

Every technique kolm uses, at every step.

The working inventory of methods the compiler reaches for. Each has a one-sentence purpose and a link to the implementation or the paper. Entries tagged default run unless you override them; the rest are opt-in.

01 · DESCRIBE

Tell kolm what you want, in plain english.

The compile starts from a written spec. We translate it into a structured task graph, a tool schema if there are tools, and an output spec the K-score gate will hold the artifact to.

Spec parser default

Natural-language description → structured task graph with input shape, output shape, and acceptance rules.

/research/provenance-data-generation

Tool schema synthesis default

If the task uses tools we infer JSON-schema signatures from the description and validate at compile time and at every call.

/research/function-calling-training

Output spec extraction default

Structured outputs are pulled from the spec into a JSON-schema the gate uses for shape conformance scoring.

/research/output-specs

Guarded prompts opt-in

Insert constitutional clauses, refusal templates, or persona constraints before distillation. Stored on the recipe, not the prompt.

/research/guarded-prompts
02 · DISTILL

Borrow knowledge from a frontier teacher.

A teacher model writes the data the student will learn from. The cheapest and most predictable lever in the whole pipeline; the rest of the stack is largely consequence of doing this part well.

Teacher trace default

Teacher solves seed prompts with rationales. Both the answers and the reasoning are kept.

/research/teacher-trace

Synthetic expansion default

Seed prompts get rephrased, recombined, and stretched by the teacher into a balanced, deduplicated training corpus.

/research/synthetic-data

Response distillation default

Soft-label distillation against teacher logits where the teacher exposes them; falls back to hard-label SFT otherwise.

/research/response-distill

Frontier prompt cache default

Distillation calls re-use the teacher prompt cache. Reads cost about 10% of writes. Surfaces as cache_hit on every receipt.

/research/prompt-cache

Rationale distillation opt-in

Student is trained to reproduce the teacher reasoning as well as the answer. Larger compute, better OOD.

/research/rationale-distill

Tool-call traces opt-in

Teacher records full tool dialogues; student learns the agent loop, not just the final response.

/research/tool-traces
03 · ALIGN

Align the student to the spec, not just the corpus.

Supervised fine-tuning gives capability; preference alignment gives behavior. We pick the objective by task: pairwise preferences for classifiers, KTO for sparse-reward, GRPO for multi-step reasoning, online-DPO when judges are fast.

SFT default

Standard cross-entropy fine-tuning on the distilled corpus. Always the first pass.

/research/sft

DPO default

Direct preference optimization on pairwise teacher comparisons. Stable, cheap, well-understood.

/research/dpo

ORPO opt-in

Reference-free preference loss. Lower VRAM. Useful when reference model checkpoints would not fit.

/research/orpo

GRPO opt-in

Group-relative policy optimization for multi-step tasks. Used when the K-score includes a per-step component.

/research/grpo

KTO opt-in

Kahneman-Tversky objective. Works on unpaired "good/bad" labels when you cannot afford pairwise comparisons.

/research/kto

Online-DPO opt-in

Preferences are sampled from a live judge during training. Higher cost, sharper alignment, used on K-score-critical tasks.

/research/online-dpo

LoRA / QLoRA adapters opt-in

Parameter-efficient training when the base model is shared across tasks or when GPU minutes matter more than fidelity.

/research/adapter-merging
04 · EVALUATE

Hold the artifact to the spec.

Every compile ends with a gate. K-score is the headline number; underneath it is a mixture of judges, a process-reward model when sequences matter, and an eval-set drift check that catches when the holdout has aged out.

K-score default

Composite quality score in [0,1]. Mix of pairwise judge wins, output-spec conformance, and OOD generalization. Per-artifact, never global.

/k-score

Mixture of judges default

A panel of frontier judges scores every eval sample. We use median to filter judge bias; tied votes route to a tiebreaker.

/research/judges

Held-out task split default

A fraction of the seed prompts is held back from distillation and used only at gate time. Replenished at every compile.

/research/holdout

Eval-set drift default

If the holdout starts looking too much like the corpus, the gate refuses to grade and asks for fresh seeds.

/research/eval-drift

Process reward model opt-in

For multi-step reasoning we train a per-step reward and score sequences, not endpoints. Adds compile time, catches silent partial credit.

/research/prm

Adversarial probes opt-in

Red-team prompts for safety, jailbreaks, prompt injection, leakage. Pass-rate is a gate component when configured.

/research/probes
05 · SIGN

Pack and seal.

The artifact is one signed file. The signature anchors the recipe, the weights, the gate result, and an SBOM so the operator can prove what they are running and where every byte came from.

.kolm wire format default

Deterministic zip of recipe, weights or adapter, gate result, SBOM, and license. Fully described in the spec.

/spec/rs-1

4-ring Ed25519 receipt default

Pre-image → derived → execution → seal. Each ring depends on the previous. Verifier checks all four offline.

/research/receipt-chain

SBOM default

Source corpus hashes, model base, library versions, and license posture, embedded in the .kolm and downloadable.

/research/sbom

Tenant-rotated key default

Ed25519 private key is per tenant. Rotated on schedule with a 90-day verify-only window for old receipts.

/research/receipt-chains
06 · RUN

Inference, anywhere.

The artifact does not assume a runtime. Pick the one that fits the deployment. The bridge produces the same receipt regardless of the engine underneath.

vLLM default

The default server for CUDA-capable hosts. Paged-attention KV cache, continuous batching, sharded tensor parallel.

vllm-project

sglang opt-in

RadixAttention prefix cache and structured-output decoding. We reach for it when sequence outputs are dominant.

sgl-project

TensorRT-LLM opt-in

NVIDIA TRT-LLM kernels for high-throughput single-host serving. Used when latency budget is tightest.

NVIDIA/TensorRT-LLM

MLX opt-in

Apple Silicon serving. The artifact is the same; the runtime is local.

ml-explore/mlx

llama.cpp opt-in

GGUF on CPU or edge GPU. Slower; the choice when there is no other choice.

ggerganov/llama.cpp

Customer-hosted bridge default

The runtime that talks to your tenants. Produces the receipt, enforces the gate, holds your Ed25519 private key.

/research/byoc-deploy-modes
FRONTIER STACK

Every technique that ships in a .kolm.

Quantization shrinks the weights. Distillation borrows from frontier teachers. Compilation and inference optimization make the artifact fast on real hardware. Merges fuse capabilities from multiple bases into one student. Every method below cites its lineage and is honest about whether it is first-class in kolm compile today or still experimental.

QUANTIZATION

Shrink the weights without losing the score.

Quantization paths from production-default to experimental. Quant choice is recorded in the receipt and can be replayed offline. See /compile for the verb, /recipes for ready-made recipes, and /benchmarkscompare for K-score vs bit-width sweeps.

2022 / GPTQ

GPTQ (Hessian-aware PTQ)

One-shot post-training quantization that uses approximate second-order information (per-layer Hessian) to choose quantization grid points. Frantar, Ashkboos, Hoefler, Alistarh 2022. Strong INT4 baseline; still the reference for 4-bit weight-only quantization.

FIRST-CLASS in kolm compile --quant=gptq
2023 / AWQ

AWQ (activation-aware weight quantization)

Protects the salient 1% of weights identified by activation magnitude; quantizes the rest aggressively. Lin, Tang, Tang, Yang, Xiao, Dang, Gan, Han 2023. Often pairs better than GPTQ with serving stacks like vLLM and TGI on Llama-class architectures.

FIRST-CLASS in kolm compile --quant=awq
2023 / bitsandbytes

bitsandbytes NF4 / FP4

4-bit normal-float and float quantization with optional double quantization. Underpins QLoRA (Dettmers, Pagnoni, Holtzman, Zettlemoyer 2023). The reference path for DeepSeek-R1 32B INT4 on a single RTX 5090: 61.0 GB BF16 down to 17.9 GB NF4 in 125 seconds.

FIRST-CLASS in kolm compile --quant=bnb-nf4
2022 / SmoothQuant

SmoothQuant

Migrates activation quantization difficulty into the weights via a per-channel scaling transform, enabling INT8 weight + INT8 activation. Xiao, Lin, Seznec, Wu, Demouth, Han 2022. Useful when activation outliers block low-bit weight-only quantization.

FIRST-CLASS in kolm compile --quant=smoothquant
2023 / GGUF

GGUF + imatrix

llama.cpp's portable quantized weight format with k-quant variants (Q4_K_M, Q5_K_S, Q6_K) and importance-matrix-aware quant levels for outlier preservation. Gerganov and contributors. The standard target for CPU and edge GPU deployment of distilled artifacts.

FIRST-CLASS in kolm compile --target=gguf
2024-2026 / DEFAULT

Unsloth Dynamic 2.0

Per-tensor dynamic quantization that scans each layer and picks 4-bit, 5-bit, 6-bit, or 8-bit by sensitivity rather than applying a single bit-width uniformly. Maintained by the Unsloth team and community. The default path in kolm distill when the user does not pin a quant scheme.

FIRST-CLASS / default in kolm distill
2024 / KV CACHE

TurboQuant KV-cache quantization

4-bit and 8-bit quantization of the K and V tensors at serve time, lowering memory for long-context inference without touching the weights. Distinct from weight quantization above; stackable with any of them. Used by vLLM and TensorRT-LLM under the FP8-KV / INT4-KV flags.

FIRST-CLASS in kolm run --kv-quant=int4
2024 / BITNET

BitNet b1.58

Ternary weights (-1, 0, +1) trained from scratch as a quantization-aware architecture, not a PTQ pass. Microsoft Research 2024 (Ma, Wang, Ma, Wang, Wang, Huang, Dong, Wang, Xue, Wei). Pairs with the low-bit specialist distill mode; the K-score floor decides whether the run survives.

EXPERIMENTAL in kolm distill --mode=low-bit-specialist --bit=1.58
2026 / ANGELSLIM

AngelSlim 1.25-bit

Sub-2-bit quantization toolkit published by Tencent in 2026, pushing below BitNet's ternary floor with calibration-set-aware lookup tables. Lineage and reproducibility still being validated externally; kolm exposes it behind a flag so the receipt records its experimental status.

EXPERIMENTAL in kolm compile --quant=angelslim
DISTILLATION

Borrow capabilities from a frontier teacher.

Distillation paths. See /distill for the full mode picker and /docs/distillation for the walkthrough. Lineage cited per method; every receipt records which teacher signed which row.

2015 / FOUNDATION

Soft-target knowledge distillation

The original. Hinton, Vinyals, Dean 2015. Student learns to match teacher's softmax over a temperature, not just the argmax. Every modern distillation path traces here; kolm's response-distill default falls back to hard labels only when the teacher will not expose logits.

FIRST-CLASS / default in kolm distill
2019 / DISTILBERT

Layer-wise distillation

Sanh, Debut, Chaumond, Wolf 2019. In addition to soft targets, match teacher hidden states and attention maps at corresponding layers. Costs more per step but raises retention on encoder-style tasks; kolm wires it on when the student backbone is smaller than half the teacher.

FIRST-CLASS in kolm distill --layer-match=auto
2025 / REASONING

Reasoning / CoT-trace distillation

Capture the teacher's full chain-of-thought, train the student to reproduce both the trace and the answer. The DeepSeek-R1-Distill family 2025 is the canonical reference; on RTX 5090 the 32B variant quantizes to 17.9 GB INT4 with the reasoning intact. See /recipes for the working sample.

FIRST-CLASS in kolm distill --mode=reasoning
ON-POLICY

On-policy / self-distillation

Student generates rollouts; teacher (or a separate grader) scores them; only the positives flow into the SFT batch. Lineage: the RLHF reward-model line plus the more recent self-distill papers that close the loop without a full RL stage. Cheaper than RLHF, sharper than vanilla SFT on multi-step tasks.

FIRST-CLASS in kolm distill --mode=on-policy
MULTI-TEACHER

Multi-teacher fusion

Per-token best-of across multiple teacher logits. Pair a reasoning teacher (DeepSeek-R1), a code teacher (Qwen2.5-Coder), and a generalist (Claude Opus) into one student. The runner records which teacher won each token so the receipt is auditable end to end.

FIRST-CLASS in kolm distill --mode=multi-teacher
AGENTIC

Agentic workflow distillation

Replay captured multi-step LangGraph or CrewAI runs. Compile the orchestrator plus the tool dialogues into one forward pass on a single student. Removes the network round-trips between steps; lineage is the 2025-2026 agent-distill literature, not an invention of ours.

FIRST-CLASS in kolm distill --mode=agentic
COMPILATION + INFERENCE

Make the artifact fast where it runs.

Quantizing a model is half the story; the other half is the kernel and the scheduler. The runtime path ships the following. See /docs/runtime for which engine kolm picks per device tier.

2022 / FLASHATTN

FlashAttention

IO-aware exact attention that tiles the softmax and avoids materializing the full N x N matrix. Dao, Fu, Ermon, Rudra, Re 2022. The kernel that made long-context tractable; every modern serving stack ships a variant.

FIRST-CLASS / default kernel in kolm run
2023-2024 / FA-2 / FA-3

FlashAttention-2 / FlashAttention-3

Dao 2023 (FA-2: better parallelism across heads and warps). Shah, Bikshandi, Zhang, Thakkar, Ramani, Dao 2024 (FA-3: asynchrony and FP8 on Hopper). FA-3 is the path for sub-millisecond per-token attention at high batch on H100-class hardware.

FIRST-CLASS in kolm run --engine=vllm
2023 / VLLM

PagedAttention

KV-cache memory management modelled on OS virtual memory: cache lives in non-contiguous pages, scheduler allocates lazily. Kwon, Li, Zhuang, Sheng, Zheng, Yu, Gonzalez, Zhang, Stoica 2023. The reason vLLM hits 24x throughput at long context vs naive serving. kolm's default CUDA runtime.

FIRST-CLASS / default in kolm run --engine=vllm
2023 / TORCH.COMPILE

torch.compile

TorchDynamo + Inductor: graph-capture and codegen for PyTorch models. PyTorch team 2023. Lifts the small-batch latency floor that hand-tuned kernels still leave on the table; kolm opts it in when the target is single-host single-GPU and batch size is below 8.

FIRST-CLASS in kolm run --compile
2023 / SPEC-DECODE

Speculative decoding

Small draft model proposes N tokens, target model verifies in parallel. Leviathan, Kalman, Matias 2023; Chen, Borgeaud, Irving, Lespiau, Sifre, Jumper 2023. Lossless throughput multiplier when draft and target agree often. Distinct from MTP, which removes the draft model entirely.

FIRST-CLASS in kolm run --speculate=draft
2025-2026 / MTP

MTP (multi-token prediction)

Architectural option in the student head: predict N tokens per forward, accept the consistent prefix. Native speculative decoding without an external draft model. The 2025-2026 Alibaba and DeepSeek lines are the public references; the kolm runner refuses MTP for student families that have not validated the head.

EXPERIMENTAL in kolm distill --mode=mpt
MERGES

Fuse capabilities from multiple bases.

Merging is the cheapest way to combine domain experts when you cannot afford a full re-train. kolm uses mergekit; the merge config and source SHAs are pinned in the receipt. See /forge for the merge picker.

TIES

mergekit TIES

Trim, Elect Sign, and Merge. Yadav, Tam, Choshen, Raffel, Bansal 2023. Resolves sign conflicts between fine-tunes by majority vote per parameter; preserves task-specific deltas that vanilla averaging would cancel.

FIRST-CLASS in kolm distill --merge=ties
DARE

mergekit DARE

Drop And REscale. Yu, Yu, Yang, Lin, Cheng, Zhao 2024. Randomly drops a fraction of delta parameters and rescales the rest; surprisingly robust at preserving multi-task capability when stacked with TIES.

FIRST-CLASS in kolm distill --merge=dare
SLERP

SLERP and task arithmetic

Spherical linear interpolation between two checkpoints plus task-arithmetic over deltas. Ilharco, Ribeiro, Wortsman, Schmidt, Hajishirzi, Farhadi 2022 on task vectors. Pairs cleanly with the merge-first distill mode.

FIRST-CLASS in kolm distill --merge=slerp

Merge lineage credit where it is due: the Gryphe MythoMax era proved community merges could match labs on creative writing benchmarks; mlabonne's public playbooks (Maxime Labonne, huggingface.co/mlabonne) standardized the recipe for the rest of us. The mergekit library by Charles Goddard is the underlying tool.

STACKS

Where the gains compound.

Individual techniques are well-studied. The frontier is in stacking them. Canonical combinations the compiler resolves on request, each pinned in the receipt so an auditor can replay the exact pipeline.

STACK / QAD

Quantization-Aware Distillation

Quantize the student during the distill loop, not after. The student learns to compensate for quant error while still consuming teacher signal; recovers significantly more K-score than PTQ-after-SFT at the same bit-width. The honest path for sub-3-bit targets.

FIRST-CLASS in kolm distill --quant-aware
STACK / META

Distill, merge, quant, MTP

The current meta. Distill multiple specialists from one teacher set; mergekit-TIES them into one base; Unsloth-Dynamic-2.0 quantize the merge; train an MTP head for speculative-free throughput. The full receipt records every step's source SHA.

FIRST-CLASS in kolm distill --mode=merge-first,mpt --quant=dynamic-2.0
STACK / 2026 FRONTIER

Reasoning, Dynamic, Speculative

DeepSeek-R1-style CoT-trace distillation into a 7B-14B student, Unsloth Dynamic 2.0 4-bit, plus speculative decoding with a 1-2B draft of the same family. Tractable single-host, preserves the reasoning shape, and serves at the latencies a chat UX expects.

FIRST-CLASS in kolm distill --mode=reasoning --quant=dynamic-2.0 --speculate=draft
STACK / EXTREME

Prune, distill, quant

Structured prune the base first (LLM-Pruner family), then distill against the dense teacher to restore K-score on the pruned scaffold, then quantize. Used when the deploy target is edge-class hardware; the K-score floor decides whether the pipeline ships or aborts.

EXPERIMENTAL in kolm distill --prune --mode=reasoning --quant=bnb-nf4
CANONICAL RANKING

Most-impactful overall.

Ranked by counterfactual: which methods, if removed from the literature, would force a complete redesign of the open-source serving stack? Compiled from the inventory above. Not a vendor list.

  1. 01
    GGUF / llama.cpp — the portable runtime that put local LLMs on every laptop, phone, and Raspberry Pi. Gerganov and the llama.cpp community.
  2. 02
    FlashAttention — the IO-aware kernel that made long-context tractable. Dao, Fu, Ermon, Rudra, Re 2022; FA-2/3 by Dao and Shah et al.
  3. 03
    GPTQ / AWQ + bitsandbytes — the three quantization families that took 4-bit serving from research curiosity to production default. Frantar et al. 2022; Lin et al. 2023; Dettmers et al. 2023.
  4. 04
    DeepSeek-R1 distill family — proved CoT-trace distillation works at 1.5B-70B scales, with open weights and an open recipe. 2025.
  5. 05
    vLLM / PagedAttention — the scheduler and memory manager that lifted serving throughput an order of magnitude at long context. Kwon et al. 2023.
  6. 06
    Unsloth Dynamic 2.0 + MTP — per-tensor dynamic bit-widths and native multi-token heads, the 2026 baseline for "make it small and fast without a draft model." Unsloth team and community.
  7. 07
    mergekit — took TIES, DARE, and SLERP from papers to a community-standard toolchain. Charles Goddard and contributors; popularized by mlabonne.
  8. 08
    KV-cache quantization — the inference-time companion to weight quantization; the reason 128k context fits on one card.
  9. 09
    Hinton 2015 KD — the foundation paper. Every method above traces back to soft targets and temperature.
  10. 10
    BitNet b1.58 — first credible 1-bit-class architecture from scratch, opening the path that AngelSlim and the 2026 sub-2-bit line continue. Microsoft Research 2024.
