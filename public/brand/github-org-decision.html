GitHub org: kolm-ai.

The repository lives at github.com/kolm-ai/kolmogorov-stack. Migration from the original org is complete. Old clone URLs continue to work via GitHub's built-in redirect.

Decision made kolm-ai is the canonical GitHub org. CI, badges, and SDK install paths all resolve here.

Why kolm-ai

The chosen org name mirrors the product domain kolm.ai. Short, searchable, one step from the brand. The disambiguation banner on every page (Kolm therapeutics, Kolm band, Kolm engines, Petter Kolm) already exists to make the product brand unambiguous in search.

  • Memorability. Identical to the domain. Easy to remember, easy to share.
  • Discoverability. One search step from kolm.ai resolves to the repo.
  • Brand alignment. Same word as the product. No translation step for users.
  • Migration cost. Low. GitHub auto-redirects old URLs. CI configs and badges were swept in W893.

Current status

kolm-ai is the canonical org. No further rename is planned.

SDKs install from pip install git+https://github.com/kolm-ai/kolmogorov-stack@main#subdirectory=sdk/python. The C and Rust SDKs resolve their READMEs against the same path. Old sneaky-hippo/* URLs continue to work via GitHub's redirect for the foreseeable future, but new docs prefer the new path.

Migration record

What we did, in order:

  1. Reserved the org name. Created the empty org on GitHub and verified ownership.
  2. Mirrored the repo. Pushed the existing repo as a new remote under kolm-ai with full history.
  3. Updated SDK install paths. Python, C, and Rust SDK READMEs now point at the new URL.
  4. Swept CI and badges. GitHub Actions uses: references and badge image URLs updated.
  5. Brand-anchor span. No change needed; the existing disambiguation lists Kolm therapeutics, Kolm band, Kolm engines, Petter Kolm.
  6. Announced in the changelog. Single dated entry with the redirect details.
  7. Kept the old org alive. Old org continues to point at the new one. GitHub's redirect is reliable; the safest path is to keep both alive indefinitely.
