GitHub org rename decision.

The repository currently lives at github.com/sneaky-hippo/kolmogorov-stack. We have been weighing a rename to a more professional org name. This page is the internal decision matrix, published for honesty. No rename has been performed and none is scheduled before Q3 2026.

Decision pending No rename has happened. Existing URLs, clones, and CI pipelines are unaffected.

Candidates under consideration

Three options are on the table. We are not removing the current org; if a rename happens, GitHub's built-in redirect handles old URLs and clone remotes for the foreseeable future.

  • sneaky-hippo (current). The personal org where the repo was born. Memorable as a username, weak as a company brand.
  • kolm-ai. Mirrors the product brand kolm.ai. Short, searchable, aligns with the domain.
  • kolmogorov-ai. Mirrors the project name "kolmogorov-stack" and the underlying Kolmogorov complexity framing. Long, harder to type, but distinctive.

Decision matrix

Each dimension scored on a relative scale across the three candidates. This is not a vote; it is a way to make the trade-off legible before we commit one way or the other.

Decision status

Pending Q3 2026. No rename has been performed.

The current org name sneaky-hippo remains canonical for clones, badges, CI, and SDK install URLs. The Python SDK still installs from pip install git+https://github.com/sneaky-hippo/kolmogorov-stack@main#subdirectory=sdk/python and the C and Rust SDKs still resolve their READMEs against the same path.

We will revisit this decision after the W708 legal foundation lands (entity status determines whether "kolm-ai" carries trademark exposure) and after Q3 2026 incorporation. Until then, treat any "kolm-ai/*" or "kolmogorov-ai/*" URL you see in the wild as not yet real.

Migration plan if we rename

This is the rough sequence we would follow. Outline only - no work has started.

  1. Reserve the target org name. Create the empty org on GitHub and verify ownership; this prevents squatters from grabbing the name during deliberation.
  2. Audit hardcoded URLs. Grep the repo for the literal string sneaky-hippo across CI, READMEs, SDK install instructions, docs site, frontend, and changelog entries. Each hit needs an explicit decision: rewrite, leave as historical reference, or redirect.
  3. Mirror, do not rename in place. Push the existing repo as a new remote under the new org with the same history. Keep the old remote alive for one full release cycle as a read-only mirror.
  4. Update SDK install paths. Cut a coordinated release of the Python, C, and Rust SDKs that points install paths at the new URL. Old install commands continue to work via GitHub's auto-redirect, but new docs prefer the new URL.
  5. Sweep CI and badges. Update GitHub Actions uses: references, badge image URLs, OG cards, and any signed-by claims in artifacts that reference the org slug.
  6. Update the brand-anchor span. If we choose kolm-ai, no anchor change is needed (the existing disambiguation already lists Kolm therapeutics, Kolm band, Kolm engines, Petter Kolm). If we choose kolmogorov-ai, the anchor stays as-is - it disambiguates the product brand kolm.ai, not the org name.
  7. Announce in the changelog. Single entry, dated, with the redirect details and a one-line summary of why.
  8. Keep the old org pointing at the new one indefinitely. Do not delete. GitHub's redirect is reliable but not eternal; the safest path is to keep both alive forever.

This page is honesty, not announcement

If you arrived here looking for a rename announcement, there is none. The product brand stays kolm.ai. The org name stays sneaky-hippo until we publish a changelog entry that says otherwise. Pull requests against the existing org are still the right path.
