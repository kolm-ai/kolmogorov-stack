Methodology · reproducible by construction

Every row is a signed claim.

The leaderboard is not a marketing surface. K-score is a computed quantity with a published formula; benchmark scores are recorded from rerunnable evaluation packs; receipts bind both to a content-addressed artifact. Hand a row to an auditor and they can re-derive it offline.

K-score

How K-score is computed

0.40 accuracy + 0.15 size + 0.15 latency + 0.15 cost + 0.15 coverage, normalized against the task pack. The same artifact will produce the same K-score on the same pack on any hardware. The formula, the eval packs, and the gates are open.

Read the K-score spec →
Benchmarks

Replays are reproducible

Aider, SWE-Bench, GPQA, LiveCodeBench, HLE: each pack is content-addressed and pinned. Re-running the same .kolm against the same pack hash gives the same score, modulo documented stochastic decoding seeds. The receipt records the seed.

Browse benchmark packs →
Receipts

Every row carries a signed receipt

The recipe, base SHA-256, teacher id, quantization scheme, K-score, eval-pack cid, and artifact_cid are JCS-canonicalized (RFC 8785), SHA-256-hashed, and Ed25519-signed by the publishing tenant. Tampering with any field breaks the signature.

Publish · one command

Beat one of these rows.

Compile your own .kolm with a recipe you can defend, sign it with your tenant key, and push to the public leaderboard. Authors of artifacts in the top three of any category share in the per-pull revenue from the registry; details in the community page.

Publish to the leaderboard

One CLI command. The publish step runs the K-score gate, signs the receipt, and pushes the .kolm to the public registry under your namespace. The board re-ranks within minutes.

Sync · mirrors, not lock-in

Push to Hugging Face or GitHub.

A .kolm is a content-addressed object. The same artifact can sit on the kolm registry, on a Hugging Face repo, and in a GitHub release at the same time. Pushing somewhere new does not break the signature; pulls verify against the original Ed25519 key.

Mirror to Hugging Face

HF Hub

Creates the repo if missing, pushes the .kolm plus a README pointing to the K-score receipt. The HF model card embeds the signature fingerprint so anyone pulling from HF can still verify offline.

$ kolm push hf --repo my-org/qwen-7b-reason
Integration details → Verify after pull

Mirror to GitHub release

GitHub

Drops the .kolm as a release asset on the repo of your choice. The release notes embed the recipe; the asset checksum matches the receipt. Re-pull from any CI runner and verify before deploying.

$ kolm push github --repo my-org/specialists --tag v1.0.0
