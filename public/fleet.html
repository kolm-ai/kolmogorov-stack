The fleet problem

If you’ve felt any of these…

You are not running one model anymore. You are running a portfolio — specialists per customer, per feature, per language, per release train — and the tooling never caught up.

Model X drifted last week and nobody noticed.

The input distribution shifted, the K-Score quietly dropped four points, and the first signal was a customer ticket. No alert fired because nothing was watching that namespace.

You have 40 variants and no way to compare them.

Different teachers, different recipes, different training pairs, different release dates. Picking which one to promote next is a spreadsheet exercise rebuilt every quarter.

The eval ran on Tuesday, but only on five of them.

Eval suites get wired to whichever model the on-call cares about. The other thirty-five sit there with last month’s numbers next to next month’s traffic mix.

A new version shipped and nobody can say what changed.

Training pairs, hyperparams, base model SHA, eval delta — the answer lives in three Slack threads and one engineer’s laptop.

What kolm fleet does

Four primitives. Every model in the fleet, every release, every namespace.

The fleet ledger is the source of record. Every inference, distill, eval, and deploy lands on it as a signed receipt that any auditor can re-verify offline.

Per-model receipts

Every inference signed

Each call emits an Ed25519 receipt attributable to the model SHA, the distill spec that produced it, and the training data hash. Replay later, prove the answer, drop into an audit packet.

kolm verify rcpt_01KYC1ZVTGDCW3FX06JQSC
Drift detection

Watch the input distribution

kolm fleet drift compares this week’s prod inputs against the prior window per namespace. Configurable threshold; alert fires before the quality regression reaches the customer.

kolm fleet drift --namespace support --window 7d
Cross-model eval

One suite, N models

Run the same eval set across every model in the fleet. The output is an agreement matrix — pairwise disagreement counts, K-Score per model, holdout pass/fail per case.

kolm fleet eval --suite support-v3 --all
Version diff

What actually changed between v1 and v2

kolm fleet diff modelA@v1 modelA@v2 prints the delta: added/removed training pairs, hyperparam changes, base SHA, eval agreement on the shared holdout. Hash-pinned, reproducible.

kolm fleet diff support-router@v1 support-router@v2
