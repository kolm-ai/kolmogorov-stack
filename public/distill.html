Distill · step 2 of 4 · capture to quantized specialist

Distill a frontier model into a quantized specialist.

A data engineer points the distill verb at a namespace of captured rows. The runner clusters them, generates teacher labels from Claude Opus 4.7 or any configured endpoint, trains a smaller student (Qwen2.5, Llama-3.3, DeepSeek-R1-Distill), and gates promotion on a real K-score against a frozen holdout. Out the back: a quantized .kolm artifact you can verify, sign, and ship.

One verb walks the whole pipeline

Distillation is seven engineering steps; the CLI hides none of them but you do not have to invoke them one by one. kolm distill from-captures reads from a namespace, runs the pipeline, and writes an artifact only if the K-score gate clears.

# Distill the captured rows in the 'support-triage' namespace
$ kolm distill from-captures support-triage \
    --teacher claude-opus-4-7 \
    --student Qwen2.5-7B-Instruct \
    --min-pairs 1000

Teacher endpoints

The teacher is the model you trust enough to label your rows. Hosted frontier APIs cover the easy case; bring-your-own self-hosted endpoints cover regulated and air-gapped paths where no traffic should leave your network.

The teacher honesty contract is the same either way: every label is attributed in the receipt with the endpoint, the model id, and the request hash.

The seven-step pipeline

Distillation is mining, augmentation, active selection, sanitization, seed materialization, teacher pass, train pass, and score. Every step writes a receipt; if any step fails the gate, the run aborts and nothing is promoted.

step 1 · mine

Cluster captures

Group by template hash. The dominant cluster becomes the training spine.

step 2 · augment

Widen the cluster

Paraphrase and generate edge cases without changing the cluster semantics.

step 3 · select

Active selection

Down-sample to the rows that best cover the decision surface of the cluster.

step 4 · sanitize

Scrub forbidden tokens

Run the configured redactor over every row. Fail loud if a PHI pattern hits.

step 5 · seeds

Deterministic dataset

Write seeds.jsonl with a stable hash. Re-runs reproduce.

step 6 · teach

Teacher label pass

Hit the teacher endpoint per row. Resume-aware if the run is interrupted.

step 7 · train

LoRA or full FT

Train the student against the seeds. INT4 quantize on the way out.

gate · score

K-score gate

Score on the frozen holdout. Promote only if the verdict is ship.

Resume and teacher fallback

Long teacher passes fail; networks drop; rate limits hit. The runner is designed to resume cleanly from the last checkpoint without re-paying the teacher for rows it already labeled.

resumeLive

--resume-from <run_id>

Picks up at the last persisted step. The progress log progress.jsonl in the run directory is the source of truth; the runner replays it and continues. Tenant-fenced so a run cannot be hijacked across boundaries.

fallbackLive

--teacher-fallback

If the primary teacher refuses (rate limit, 503, malformed response), the runner walks KOLM_DISTILL_TEACHER in order. Every attempt is recorded in teacher_attempts[] in the run receipt so audit can see exactly which teacher labeled which row.

honest scopeLive

Real eval, not aspirational

The K-score is computed against a real frozen holdout that the runner constructs from your captures. The split is deterministic; the score in the receipt is the number a third party would reproduce on the same seed.

refuseLive

--min-pairs floor refuses thin corpora

Below 1000 rows the runner refuses with exit code 2. A specialist trained on a hundred prompts overfits and the K-score gate will not let it ship anyway; the floor saves you the GPU bill.

Quantize matrix (real numbers on RTX 5090)

MODES

Every mode you'd reach for. Stackable.

Pick one, pick several. The compiler composes them into a single signed artifact with a reproducible spec. Each mode is honest about its research lineage; nothing here is invented by us alone.

REASONING

Reasoning / CoT trace distillation

Capture full step-by-step traces from teacher. Student learns the reasoning shape, not just the final answer. The DeepSeek-R1 distill family is the canonical reference; the original chain-of-thought distillation lineage traces to Hinton et al. 2015 on soft targets.

$ kolm distill --mode=reasoning --teacher=deepseek-r1 --student=qwen2-5-7b
# student inherits the trace, not just the label
ON-POLICY

On-policy / self-distillation

Student generates. Teacher grades. Student learns from its own corrected outputs. Lineage: the RLHF family (reward-model grading on student rollouts) and the more recent self-distill papers that close the loop without a separate RL stage.

$ kolm distill --mode=on-policy --student=qwen2-5-7b --grader=claude-opus-4-7
# student rolls out, grader scores, only positives flow back
MULTI-TEACHER

Multi-teacher

Per-token best-of across multiple teacher logits. Claude reasoning, DeepSeek math, and Qwen coding fused into one student. The runner records which teacher won each token so the receipt is auditable.

$ kolm distill --mode=multi-teacher --teachers=claude,gpt,deepseek --student=qwen2-5-14b
# per-token best-of, attribution preserved
AGENTIC

Agentic workflow distillation

Capture multi-step LangGraph or CrewAI runs. Compile the orchestrator plus tool calls into one forward pass on a single student. Reduces inference cost on workflows that repeat, and removes the network round-trips between steps.

$ kolm distill --mode=agentic --capture=~/.kolm/lake/workflows
# lineage: 2025-2026 agent-distill papers; not our invention
MERGE-FIRST

Merge-first

Run mergekit TIES / DARE / SLERP on multiple bases or teachers, then distill the merged result. Lineage: mlabonne and the mergekit community. The merge config and source SHAs are recorded in the receipt.

$ kolm distill --mode=merge-first --merge=ties --models=gemma-4-31b,qwen2-5-14b
# mergekit recipe pinned, source hashes locked
MTP

MTP (multi-token prediction)

Architectural option in the student: predict N tokens per forward. Native speculative decoding without an external draft model. Currently best on Qwen3.6 bases; the runner refuses for student families that have not validated MTP heads.

$ kolm distill --mode=mpt --student=qwen2-5-14b
# N=4 default; student head predicts ahead
LOW-BIT

Low-bit specialist

Aggressive quantization with K-score safety floor. The runner walks 4.0 to 2.0 to 1.58-bit and stops the moment K-score drops below the floor you set. Cited lineage: BitNet-b1.58 and the SpQR family for outlier-aware low-bit. The receipt records the final bit-width and the floor.

$ kolm distill --mode=low-bit-specialist --bit=1.58 --kscore-min=0.85
# walks 4.0 to 1.58; aborts if K-score floor breached
STACK THEM

One verb, many modes, one signed artifact.

Combine reasoning, multi-teacher, and MTP on a 14B Qwen base with 4-bit dynamic quantization and a 0.92 K-score floor. The compiler resolves the spec, runs the pipeline, and signs the artifact. Same .kolm format you'd get from a single-mode run.

$ kolm distill --teacher=claude-opus-4-7 --student=qwen2-5-14b --mode=reasoning,multi-teacher,mpt --bit=4.0 --quant=dynamic-2.0 --kscore-min=0.92
# spec.json records every mode; receipt records every gate

SYNTHETIC BOOSTER

When captures are sparse, augment honestly.

DISCLOSED IN RECEIPT

Teacher-generated traces, flagged

When the captured corpus is below the floor for a robust holdout, kolm can ask the teacher to generate additional traces seeded from the existing cluster centroids. The booster never silently mixes synthetic and real: every synthetic row is tagged in seeds.jsonl and the receipt reports synthetic_ratio, the teacher endpoint, and the prompt hash that produced the augmentation.

The K-score holdout is held out before augmentation. Synthetic data cannot leak into evaluation. If the receipt shows synthetic_ratio > 0, downstream consumers can refuse on policy.

$ kolm distill from-captures support-triage --augment-synthetic=0.4 --teacher=claude-opus-4-7
# receipt will record synthetic_ratio=0.40 and the augment prompt hash
HONESTY CONTRACT

No silent passthrough

The booster refuses to run if the augmentation would push the synthetic share above --max-synthetic-ratio (default 0.5). A run that exceeds the cap aborts before the teacher pass, so you do not pay for tokens you would not be able to ship.

If you need a fully real corpus, set --no-synthetic and the runner falls back to refusing thin namespaces below the --min-pairs floor.

booster off by default · always disclosed when on
