Distill · step 2 of 4 · capture to quantized specialist

Distill a frontier model into a quantized specialist.

A data engineer points the distill verb at a namespace of captured rows. The runner clusters them, generates teacher labels from Claude Opus 4.7 or any configured endpoint, trains a smaller student (Qwen2.5, Llama-3.3, DeepSeek-R1-Distill), and gates promotion on a real K-score against a frozen holdout. Out the back: a quantized .kolm artifact you can verify, sign, and ship.

One verb walks the whole pipeline

Distillation is seven engineering steps; the CLI hides none of them but you do not have to invoke them one by one. kolm distill from-captures reads from a namespace, runs the pipeline, and writes an artifact only if the K-score gate clears.

# Distill the captured rows in the 'support-triage' namespace
$ kolm distill from-captures support-triage \
    --teacher claude-opus-4-7 \
    --student Qwen2.5-7B-Instruct \
    --min-pairs 1000

Teacher endpoints

The teacher is the model you trust enough to label your rows. Hosted frontier APIs cover the easy case; bring-your-own self-hosted endpoints cover regulated and air-gapped paths where no traffic should leave your network.

TeacherEndpointPath
claude-opus-4-7api.anthropic.comHosted frontier. Default if ANTHROPIC_API_KEY is set.
claude-sonnet-4-6api.anthropic.comFaster, cheaper Anthropic teacher; good for high-volume distill.
gpt-4.5api.openai.comHosted OpenAI teacher. Default if OPENAI_API_KEY is set.
gemini-2.5-progenerativelanguage.googleapis.comHosted Google teacher. Default if GEMINI_API_KEY is set.
vllm://<host>Your networkSelf-hosted vLLM endpoint. Configure with --teacher-url.
sglang://<host>Your networkSelf-hosted SGLang endpoint. RadixAttention prefix caching helps cost.
tgi://<host>Your networkSelf-hosted Text Generation Inference endpoint from Hugging Face.

A regulated team running on-prem can chain --teacher vllm://gpu-01 to a vLLM server hosting Llama-3.3-70B locally; no Anthropic or OpenAI call ever fires. The teacher honesty contract is the same either way: every label is attributed in the receipt with the endpoint, the model id, and the request hash.

Student bases

The student is the smaller model you want to ship. The base must have a permissive license and a tokenizer the distill runner knows about. The defaults below cover roughly 0.5 GB to 18 GB on disk after INT4 quantization.

Pick a student by the inference budget at the runtime; the distill runner will not let you compile a base it does not have a tokenizer for. See runtimes for which targets each student lands on cleanly.

The seven-step pipeline

Distillation is mining, augmentation, active selection, sanitization, seed materialization, teacher pass, train pass, and score. Every step writes a receipt; if any step fails the gate, the run aborts and nothing is promoted.

step 1 · mine

Cluster captures

Group by template hash. The dominant cluster becomes the training spine.

step 2 · augment

Widen the cluster

Paraphrase and generate edge cases without changing the cluster semantics.

step 3 · select

Active selection

Down-sample to the rows that best cover the decision surface of the cluster.

step 4 · sanitize

Scrub forbidden tokens

Run the configured redactor over every row. Fail loud if a PHI pattern hits.

step 5 · seeds

Deterministic dataset

Write seeds.jsonl with a stable hash. Re-runs reproduce.

step 6 · teach

Teacher label pass

Hit the teacher endpoint per row. Resume-aware if the run is interrupted.

step 7 · train

LoRA or full FT

Train the student against the seeds. INT4 quantize on the way out.

gate · score

K-score gate

Score on the frozen holdout. Promote only if the verdict is ship.

Resume and teacher fallback

Long teacher passes fail; networks drop; rate limits hit. The runner is designed to resume cleanly from the last checkpoint without re-paying the teacher for rows it already labeled.

resumeLive

--resume-from <run_id>

Picks up at the last persisted step. The progress log progress.jsonl in the run directory is the source of truth; the runner replays it and continues. Tenant-fenced so a run cannot be hijacked across boundaries.

fallbackLive

--teacher-fallback

If the primary teacher refuses (rate limit, 503, malformed response), the runner walks KOLM_DISTILL_TEACHER in order. Every attempt is recorded in teacher_attempts[] in the run receipt so audit can see exactly which teacher labeled which row.

honest scopeLive

Real eval, not aspirational

The K-score is computed against a real frozen holdout that the runner constructs from your captures. The split is deterministic; the score in the receipt is the number a third party would reproduce on the same seed.

refuseLive

--min-pairs floor refuses thin corpora

Below 1000 rows the runner refuses with exit code 2. A specialist trained on a hundred prompts overfits and the K-score gate will not let it ship anyway; the floor saves you the GPU bill.
