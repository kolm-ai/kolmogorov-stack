Subprocessors

Every vendor, every region.

A subprocessor is a third-party service that processes data on behalf of kolm. The default install of the open-source compiler has zero subprocessors because it runs on your hardware. This page lists every vendor the hosted gateway and the registry touch, what each one handles, what region it sits in, and whether it can ever see customer PHI under a signed BAA.

What never leaves the customer environment

A short, complete list. If something is on this list, no subprocessor sees it under the default install.

Stays on the customer side

Captures
Prompts, completions, tool calls, retrieval contexts, and trace bodies are written to ~/.kolm/events on the customer filesystem. Never replicated to any vendor on this page.
Distill datasets
Teacher-student pairs generated by kolm distill live next to the captures they came from. Never indexed by any vendor on this page.
Model weights
Base weights, distilled weights, INT4 and INT8 quantizations stay on customer object storage. The .kolm artifact is a zip the customer controls.
Eval pack runs
K-score evaluations execute in-process on the customer machine or VPC. Scores and axes are computed locally and signed locally.
Receipt content
If a CID is published, only the content-addressed hash is mirrored to the registry. The receipt object itself, including signature, lives wherever the customer chooses.

Self-host reduces this list to zero

A reminder for the regulated reader. The open-source compiler is a single binary plus a small set of Python and Node SDKs. None of the vendors above is required to compile, run, sign, or verify a .kolm artifact.

Compile offline

Run kolm distill and kolm compile on a workstation with no outbound network. The CLI accepts a --offline flag that fails closed on any vendor attempt.

Verify offline

Drop a .kolm artifact into verify-prod in the browser. Or run kolm verify ./model.kolm on a disconnected machine. No vendor is involved.

Self-host the gateway

Run the gateway on your own infrastructure with a Helm chart, a Docker compose file, or a single binary. See self-host for the operator guide.

How we add a new subprocessor

For the enterprise plan, we give 30 days advance notice for any new vendor that processes customer data. Free and pro plans get the change log on the date of effect.

Subscribe to advance notice

Email privacy@kolm.ai with the subject line subprocessor-notice. We add the requesting address to the notification list and send a single email per change.

Object to a new subprocessor

Enterprise customers can object to a new vendor in writing. If we cannot offer a path that does not use the new vendor, the customer may terminate the affected service with a prorated refund.

How to reach us

For subprocessor questions, residency clarifications, or vendor-list change notifications.

Privacy desk

Email privacy@kolm.ai. Acknowledged within 2 business days. The privacy desk is the canonical channel for any question about the vendor list on this page.

Data Protection Officer

Required for EU subjects. Email dpo@kolm.ai. The DPO is independent of the product and engineering teams and can escalate to company officers.
