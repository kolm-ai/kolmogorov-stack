Manifest field · verifiable · on every artifact

Teacher source on every artifact.

Every .kolm artifact stamps manifest.teacher_source with one of three values: open-weights, proprietary, or mixed. The stamp is reproducible. Anyone with the file can re-derive it. Downstream consumers do not have to take your word for what teacher trained your model.

01The three values
teacher_source

open-weights

Llama-3, Qwen, Mistral, DeepSeek, or any other open-weight teacher. Inference outputs ride the same license as the weights. No API-side restriction on what the distilled artifact can do.

teacher_source

proprietary

Anthropic Claude, OpenAI GPT, Google Gemini, or another hosted-API teacher. The artifact carries a verify-your-TOS note. You and your consumers should read the latest provider terms before redistribution.

teacher_source

mixed

The distill corpus combined captures from both open-weight and proprietary teachers. Default posture is the stricter (proprietary). The manifest enumerates which teachers contributed which fraction.

02Verify any artifact

The receipt that kolm verify emits shows the teacher source plain. Anyone you hand the artifact to can run the same command and get the same answer; there is no proprietary path inside the verifier.

$ kolm verify ./support-redactor.kolm
artifact   : support-redactor.kolm
sha256     : a91d4f0c...8b7c
k_score    : 0.97
recipes    : 1
teacher    : qwen2.5-7b-instruct
teacher_source : open-weights
license    : apache-2.0
signature  : ed25519:valid
ok         : true

If the field reads proprietary, the verifier also prints a one-line reminder to consult the originating provider's terms. The artifact still verifies cryptographically; the note is informational.

03How the stamp is set

At distill time, kolm inspects each teacher you reference. Teachers on the open-weight allowlist (Llama, Qwen, Mistral, DeepSeek, plus any registered local weights file) yield open-weights. Hosted API teachers (Claude, GPT, Gemini, and any registered API endpoint with non-permissive output terms) yield proprietary. Any combination yields mixed.

The stamp is hash-bound into the artifact. You cannot post-edit the manifest without invalidating kolm verify. The provenance survives copy, mirror, and re-bundling.

04Why this matters

When a procurement reviewer asks "what model trained this," you want the answer to be a one-line CLI command, not an email thread. When a downstream customer changes their AI policy, you want them to be able to audit your shipped artifacts in place without re-running your distill pipeline. The teacher-source stamp is the smallest honest disclosure that does both jobs.
