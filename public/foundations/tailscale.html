Tailscale recipe

Tailscale gives every machine you own a stable, private address. kolm pushes compiled artifacts to a git remote that lives on that same tailnet. No port forwarding. No public mirror. No third party in the data path.

Why pair them

The artifact a kolm compile produces is a file. Files want a home. A git remote is the right home (versioned, hashed, distributed), but a git remote needs to be reachable from every box that will pull the artifact. The reachability problem is what Tailscale solves: every device gets a stable hostname on a private WireGuard mesh, and SSH or HTTPS to that hostname works the same way from your laptop, your phone, a cluster node, or a workshop forklift.

Pair them and the seam is one verb: kolm sync push artifact.kolm --git git@hub:org/private-mirror.git where hub is a tailnet hostname. The compile machine pushes; every other machine in the tailnet pulls; no traffic ever leaves the mesh.

One-time setup

  1. Install Tailscale on the compile box and each consumer box (tailscale.com/download). Run tailscale up. Make a note of each device's tailnet hostname.
  2. Pick one device to host the git remote. A small VM is fine. Install git, create a bare repo: git init --bare /srv/kolm/mirror.git. Add a user with SSH access from your tailnet.
  3. On the compile box: ssh-keygen -t ed25519, then ssh-copy-id user@hub.tailnet-name.ts.net. Verify with ssh user@hub.tailnet-name.ts.net true.
  4. Run kolm sync status --git git@hub.tailnet-name.ts.net:/srv/kolm/mirror.git to confirm kolm can reach the remote. First run clones the empty repo into a temp workdir.

The daily flow

# 1. compile on the box with the GPU
kolm compile spec.json --out artifact.kolm

# 2. push to the tailnet-private mirror
kolm sync push artifact.kolm --git git@hub.tailnet-name.ts.net:/srv/kolm/mirror.git \
 --subdir prod/$(date +%Y%m%d) \
 --message "prod recipe v3, K-score 0.91"

# 3. on any consumer box (laptop, phone via Termius, edge VM)
kolm sync pull --git git@hub.tailnet-name.ts.net:/srv/kolm/mirror.git
kolm run /tmp/kolm-sync-*/prod/20260518/artifact.kolm "your input"

What this gets you

  • Every device on the tailnet has the same compiled artifact within seconds of a push.
  • Receipt chain travels with the artifact. kolm verify on the consumer machine checks the chain locally, no network call to a registry.
  • Branch per tenant if you want per-customer isolation. git switch tenant-foo on the consumer side pulls a different artifact.
  • Air-gap-friendly: if the tailnet has no public exit, the entire flow happens on the WireGuard mesh.
kolm does not bundle Tailscale and we do not need root on your tailnet to make this work. The integration is one CLI verb pointed at a git URL; everything else is whatever git auth and tailnet config you already have.

Failure modes worth knowing

Stale clone. kolm caches the clone at $TMPDIR/kolm-sync-<hash>. If the hub repo is force-pushed, the next sync may need a manual rm -rf of the workdir before kolm sync pull works. This is the standard git rule, not a kolm rule.

SSH key per box. Each consumer box needs its own SSH key authorized on the hub. Resist the temptation to share one key across the fleet; the hub's auditedKeys log becomes useless if you do.

Tailnet ACLs. If you tighten tailnet ACLs after the initial setup, make sure the consumer boxes still have egress to the hub on port 22 (or whatever you use for SSH).
