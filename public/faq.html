General

What is kolm?

kolm is an open-source AI compiler. It takes captured input/output examples, distills them through a teacher model into a smaller student model, scores the result, and emits a signed .kolm artifact you can run on your own hardware. The full case is on the why-kolm page; the principles behind the design are in the manifesto.

What does "compiler" mean in this context?

A compiler takes a source artifact and turns it into a runnable artifact through a deterministic, verifiable pipeline. We borrow the word from systems engineering. Source = your captures + a teacher + a base model. Output = a content-addressed .kolm file with a K-score, an eval pack hash, and a signed receipt.

How is this different from fine-tuning?

Fine-tuning is a step inside the compile path; not the whole thing. The compiler also captures, scores against a held-out eval pack, quantizes, packages, and signs. Each step is reproducible. Each output carries the evidence chain. See compare for the side-by-side with adjacent stacks.

How do I start?

Install the CLI, run `kolm signup`, run `kolm distill` against the bundled sample captures, then `kolm score` on the output. The full path is documented in quickstart; the whole loop runs in under ten minutes on a developer laptop.

For developers

Which CLI do I install?

`npm i -g @kolm/cli`, `pip install kolm`, or `brew install kolm`. There is also a curl-based installer for environments without a package manager. All four install paths land the same binary; the source is in the GitHub repo.

Which language SDKs are first-class?

Six: Node.js, Python, MCP, VS Code, C (stb-style single header), and Rust. C and Rust target embedded and CLI integrators; the others target hosted apps and dev tools. See /sdks for the full matrix.

Can I use kolm without signing up for an account?

Yes. The CLI runs entirely locally for development. An account is only required when you want hosted features: the registry, hosted bake-offs, or production-tier rate limits on the gateway. Anonymous usage will work indefinitely.

How do I integrate kolm with my existing inference stack?

If you run vLLM, SGLang, TGI, or TRT-LLM, point the kolm gateway at your existing endpoint and turn on capture; nothing else changes. If you serve directly from your application, swap your model load call for `kolm.load(artifact_path)` and the verifier runs at load time. See runtimes.

For engineers (platform & SRE)

What does the runtime contract look like?

The runtime refuses to load an artifact whose signature fails verification, whose eval pack hash is not present, or whose K-score is below the configured floor. All three checks are at load time; once loaded, the artifact is referenced by CID throughout. The default policy refuses anything below K=80.

How do I gate CI on K-score?

`kolm score artifact.kolm --evals my-pack --min 80` exits with code 1 if the artifact does not score above the floor. Wire it into your CI step; the build fails on a sub-floor artifact and the receipt is uploaded as a CI artifact.

How does kolm handle drift in production?

The runtime emits per-call confidence and the K-score baseline travels with the artifact. The compiler watches for divergence, alerts when production K diverges from baseline by more than 3 points, and runs `kolm distill --since 90d` to rebuild from rolling captures. See runtimes for the alert wiring.

What happens if the kolm.ai control plane goes down?

Nothing in your runtime. The control plane handles registry, account, and bake-off APIs. The artifacts you have already pulled run unchanged; the verifier runs locally; the per-call receipt path does not touch our control plane. Telemetry retries when the control plane recovers.

For data scientists

How do I author my own eval pack?

An eval pack is a JSONL file plus a manifest that names the per-axis weights and the input/output schema. The compiler accepts any JSONL pack that conforms to the kolm-eval-pack-1 schema. We ship reference packs for prior auth, claims adjudication, SOAP notes, and FWA triage.

What if my K-score is low?

Read the per-axis breakdown. Low Faith means teacher mismatch or insufficient captures. Low Cover means distribution skew. Low Calibration usually means distill temperature too high. Low Honesty means the refusal probe is hitting a hole in your captures. Each axis has a documented failure-mode playbook on the k-score page.

Can I distill on a reasoning teacher (CoT)?

Yes. Chain-of-thought transfers cleanly when the student is in the same family as the teacher's reasoning trace style. Phi-4-Mini-Reasoning pairs particularly well with Claude Opus 4.7; DeepSeek-R1-Distill-Qwen-32B is already trained on R1 traces and accepts further specialization without losing its reasoning surface.

How do I prevent eval pack contamination during distillation?

The compiler holds out the eval pack at distill time and asserts no leakage. If any cases in the eval pack appear in the distill captures, the build fails with `eval_contamination:true`. The check is content-hash based; you cannot defeat it by rewording.

For security & compliance

Is kolm HIPAA-eligible?

Yes, under the BYOC and on-prem deployment patterns. We sign a BAA before any data touches our infrastructure. PHI does not leave your VPC under the recommended deployment pattern. See the healthcare page for the full Security Rule mapping.

Is kolm SOC 2 certified?

SOC 2 Type I is available now; Type II is in progress. The report is shared under NDA via the security page or directly through your account team.

What does the receipt actually contain?

Artifact CID, eval pack hash, teacher id and version, K-score and per-axis scores, weight vector, compiler version, build timestamp, and an Ed25519 signature over the manifest. Per-call receipts add input hash, decision, and stated confidence. Every field is named in the receipt schema, which is in the open source.

How long are receipts retrievable?

For HIPAA workloads, we retain receipt metadata under your record-keeping policy (typically seven years). The artifact and the eval pack persist under your registry retention; per-call receipts are content-addressed and survive as long as the underlying storage holds them.

Open source & licensing

What is the license?

Apache-2.0 for the compiler, the runtime, the verifier, the K-score math, the eval pack schema, and all six SDKs. Some hosted-only features in the registry are closed-source SaaS; you can run kolm end-to-end without those.

Can I fork kolm?

Yes. The license permits it. If you fork the compiler, your artifacts will carry a different compiler-id in the receipt and downstream verifiers will treat them as such. We will not pursue your fork; if you want to upstream improvements, the PR queue is open.

Do you accept community contributions?

Yes. CONTRIBUTING.md in the repo explains the workflow. Most accepted PRs target the eval-pack catalog, the SDK ergonomics, and the runtime adapters. Compiler core changes go through a longer review with two maintainers; the test floor is strict.

How do I reach the maintainers?

GitHub issues for bugs and feature requests. Discord for community questions. Email for sensitive disclosures (security@kolm.ai). For commercial conversations, the contact form routes to the right team.
