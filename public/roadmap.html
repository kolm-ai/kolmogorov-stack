Roadmap · honest signal · not a press release

What we are building next.

An engineer, an architect, or a procurement reviewer can read this page and decide whether to bet on kolm for a quarter that has not started yet. Items move between Now, Next, Later, and Researching as evidence arrives. We do not commit to dates we cannot keep, and we do not list anything in Now that does not already have an open pull request.

How to read this page

Four columns; the rules for what belongs in each are written down so a contributor can move a card without asking permission.

columnNow

Now: active sprint

An open branch with code already merged into staging or about to merge. Every Now card has a wave number and a developer pair on it this week. If the wave slips, we move the card; we do not hide it.

columnNext

Next: this quarter

Scoped, sized, and ordered. The honest scope note on each card explains what shipping looks like. A card moves up to Now only after design review and a paired engineer commit.

columnLater

Later: this year

Believed in. Not yet sized. The card lists the hardware, the regulatory dependency, or the upstream library that has to ship for the work to start. Honest signal, not a commitment.

columnResearching

Researching: open question

We have a hypothesis, a paper, and a prototype, and we are not yet convinced the technique is the right fit for the kolm contract. A card in this column is a signal that we have read the literature, not a promise.

policyStable

Cards move; they do not vanish

If a card slips from Now back to Next, the slip is annotated with the reason. If a card moves from Later to Researching, the regression is annotated. The roadmap is a real changelog of intent, not a marketing surface.

policyStable

The receipt is the contract

When a roadmap card ships, it ships with a signed .kolm artifact and a receipt CID. The card moves to the changelog with the receipt embedded. No card ships without that receipt.

N00

Now · active sprints

May 2026

Three lanes with code in the air this week. Every card lists who is paired on it and what shipping looks like.

complianceIn flight

SOC 2 Type II observation

Observation window opened with the SOC 2 Type I closer. We are running the control evidence collection in CI, not in a spreadsheet, so the next audit pulls from a real source of truth.

Why
Procurement at regulated payors will not move past pilot without Type II.
Ships
Final Type II report under NDA, ETA Q4 2026.
Scope
Honest scope: observation only; pre-audit gap remediations land as separate waves.
quantizeIn flight

RTX 5090 INT4 32B verified shipping path

DeepSeek-R1 32B distilled to 17.9 GB INT4 in 125 seconds with a 11.5 tok/s run on the same card. The wave lands the matrix as a reproducible script so any developer with a 5090 can replicate.

Why
Engineers want one canonical "is this real" reproduction script per size.
Ships
scripts/quantize-matrix.sh in the public repo with output JSON.
Scope
Five sizes: 0.5B, 3B, 7B, 14B, 32B. 70B is in Later.
federalIn flight

FedRAMP Moderate pilot

Two federal pilots in GovCloud, both regulated by FedRAMP Moderate. The pilots run on the air-gapped deployment path. We do not yet claim authorization.

Why
Federal procurement requires Moderate before a contract clock can start.
Ships
Pilot evidence package, then an SSP draft.
Scope
Honest scope: pilot, not authorized. We will say so until the ATO is in hand.
N01

Next · this quarter

Q3 2026

Sized, ordered, and scoped. Each card has a backing GitHub issue and a pair of engineers tagged to it. None of these have started yet.

runtimeNext

ARM64 native runtime

Cross-compiled native runtime for Linux ARM64 and macOS ARM64. The verifier and the CLI are already ARM64-clean; this wave brings the inference runtime up to parity so a developer on an M-series Mac or a Graviton instance can serve a .kolm artifact without an x86 emulator.

Why
An architect deploying on Graviton, Ampere, or an M3 dev box gets the same path as on x86.
Ships
kolm-runtime-aarch64 binary in the release pipeline.
Scope
CPU and unified-memory paths; CUDA on ARM is out of scope.
quantizeNext

Apple Silicon MLX quantize matrix

A second quantize matrix backed by MLX for the M-series GPU, parallel to the bitsandbytes matrix on x86 plus CUDA. The matrix lists the same four sizes (0.5B, 3B, 7B, 32B) with M-series wall-clock and tok/s numbers.

Why
Engineers on Apple Silicon want the same honest matrix as RTX 5090 users.
Ships
scripts/quantize-matrix-mlx.sh with output JSON.
Scope
M2 Pro through M4 Max. M1 family on best-effort.
federatedNext

Federated learning gradient aggregation v1

Build on the W461 approval-row sharing path: aggregate quantized gradient updates across tenants who opt in. Differential-privacy budget is per-tenant; no raw gradients cross the tenant boundary.

Why
Healthcare and finance teams want the K-score floor lift that federation can produce without exposing data.
Ships
POST /v1/federated/aggregate-gradients route plus a CLI sub-command.
Scope
Aggregation only; signed-by-tenant gradients are in Later.
auditNext

Signed audit log streaming to S3-compatible stores

The CLI already writes a structured JSON audit log. This wave streams it to an S3-compatible store (AWS S3, GCS, Azure Blob, S3-compatible storage, MinIO) with object-lock and an Ed25519 detached signature per object.

Why
Compliance reviewers want immutable, signed audit trail without a SIEM in the loop.
Ships
kolm audit ship --target s3://... with object-lock attestation.
Scope
S3 API surface only; Splunk and Datadog stay in the optional sink list.
distillNext

Self-hosted teacher honest fallback

The distill loop already supports a teacher chain (Anthropic, OpenAI, your vLLM). This wave makes the fallback honest end to end: if a teacher returns a refusal or a malformed structured output, the row is annotated and skipped instead of poisoning the dataset.

Why
A bad teacher row poisons the student model in silence. The honest path is to skip and log.
Ships
distill_skip_reason field in every row that did not survive teacher inference.
Scope
Refusal, malformed structured output, rate-limit timeout.
gatewayNext

Plan-aware budget caps on the hosted gateway

The gateway already meters and bills per namespace. This wave lets a tenant owner set a hard cap per namespace and a soft cap per developer key, so a runaway agent loop does not produce a surprise invoice.

Why
Procurement at enterprise tenants will not sign without a hard cap.
Ships
kolm namespace set-cap and kolm key set-cap with sub-second enforcement.
Scope
Soft cap warns; hard cap rejects. No overage flow.
N02

Later · this year

2026

Believed in. Not sized. The card names what has to ship upstream (hardware, regulation, library) before we can start.

quantizeLater

70B INT4 on H100

The matrix tops out at 32B for now because that fits comfortably on a single RTX 5090 in INT4. A 70B INT4 path needs an H100 (80 GB) or an A100 (80 GB) to clear the BF16 load step. We have access; the wave needs scheduling time.

Why
An engineer with H100 access deserves the same one-line script as a 5090 owner.
Blocked on
H100 reservation calendar.
Honest
We will not pre-announce a date.
quantizeLater

FP4 / FP6 when hardware lands

FP4 is a Blackwell-and-later feature. FP6 is research-grade on current hardware. We will add a column to the quantize matrix when an FP4-capable card is reachable and stable in CI, and again when FP6 has a stable inference kernel.

Why
Smaller weight format means more model fits per dollar of GPU.
Blocked on
Stable FP4 kernels in upstream inference engines.
Honest
No date until kernels are upstream.
data-residencyLater

EU residency tier

A hosted-gateway tier that pins captures, receipts, and registry entries to an EU region. Compiler workers already run in your VPC; this is about the optional hosted services that today default to us-east-1.

Why
EU customers under GDPR want a tier without an SCC redline.
Blocked on
EU-region observability stack parity.
Honest
No date until parity is in CI.
gatewayLater

Hosted gateway plan with K-score billing

A pricing tier where a developer pays per K-score point delivered, not per token. The hosted gateway today bills per token; K-score billing requires the bake-off to run on every request, which is a real cost shift.

Why
A team paying for outcome should pay for outcome, not for input.
Blocked on
Bake-off per-request budget below 50 ms.
Honest
The unit-economics math is not yet in our favor on small models.
runtimeLater

Wasm runtime for browser-side inference

A wasm-side runtime for sub-1B distilled models so a browser tab can serve the compiled .kolm without a network round trip. The compiler already targets wasm; the runtime needs SIMD-tuned kernels.

Why
A developer wants a demo that works offline in a browser.
Blocked on
Wasm SIMD performance ceiling.
Honest
Bigger models will not fit in a tab; this is the small-model path.
enclaveLater

Multi-tenant trusted enclave reference deployment

The W460 TEE attestation embed wave landed the receipt half. The deployment half (a reference Terraform that spins a multi-tenant SNP or TDX enclave with measured boot pinned into the receipt) is bigger work.

Why
Defense and pharma teams want the enclave as a deployable, not as documentation.
Blocked on
Stable measured-boot artifact for SNP and TDX in upstream cloud images.
Honest
Pilot-grade only until a cloud vendor ships a measured-boot SLA.
N03

Researching · open question

Signal, not commitment

A hypothesis and a prototype, not a roadmap commitment. Listed here so engineers and researchers know we have read the paper.

decodingResearch

Variational speculative decoding

Speculative decoding with a learned draft head, variationally bounded against a teacher. Initial micro-benchmarks show a 1.4x wall-clock win on a single 7B model; we need to verify the win holds under a real K-score gate.

Why
Decoding is the dominant cost for an INT4-served model.
Open Q
Does the K-score on faithfulness survive the bound? Unverified.
quantizeResearch

SpQR post-training quantize

SpQR adds sparse outlier preservation on top of an INT4 base. Single-pass quality numbers look better than NF4+double on 7B math. Has not been verified on 32B or on tool-use tasks.

Why
Better K-score at the same weight budget is the prize.
Open Q
Does the SpQR sparse layer stay honest under speculative decoding?
distillResearch

MoE distillation

Distilling a Mixture-of-Experts teacher (e.g. Mixtral, DBRX, Qwen-MoE) into a dense student is doable but tricky. Routing is the hard part; naive averaging across experts loses the gating signal.

Why
MoE teachers will dominate frontier in 2026 and 2027.
Open Q
Do we ship per-expert distill plus a learned router, or single-shot averaged distill?
enclaveResearch

Multi-tenant trusted enclave

Beyond the single-tenant SNP/TDX deployment, we are tracking research on sealed multi-tenant enclaves with per-tenant key separation that survives a host compromise.

Why
Co-tenancy with regulator-friendly isolation lowers per-tenant cost.
Open Q
How do we honest-scope "per-tenant key" against a side-channel attacker?
routingResearch

K-score-aware gateway routing

The gateway today routes on latency and cost. We are prototyping a router that picks an artifact based on its K-score on a per-request topic embedding, so a high-stakes call gets the highest-K-score model and a low-stakes call gets the cheapest.

Why
Dynamic K-score routing produces the best cost-per-K-score-point.
Open Q
Does the per-request embedding step blow the latency budget?
evalResearch

Adversarial eval pack synthesis

Most eval packs are hand-curated. We are testing an automated synthesis loop that adversarially produces eval rows that a candidate artifact gets wrong, then quarantines those rows for the next compile cycle.

Why
The cheapest way to harden a K-score is to find the gaps automatically.
Open Q
Adversarial loops collapse to nonsense without a reward model; can we ground in the receipt chain?

How we commit (and how we do not)

The roadmap is one of the things we are most honest about. The rules below are not aspirations; they are enforced by the same release-verify gates that block a shipped wave.

What we commit to

Every shipped wave
Carries a signed .kolm receipt; the receipt CID is on the changelog entry.
Honest column scope
A Now card has an open pull request; a Next card has a sized issue; a Later card has a named dependency.
Slip annotation
If a card slips a column, the slip is annotated with the reason and a target window.
Honest scope on every card
The "Scope" line on each card tells you what is in and what is out so you do not over-buy.
Honest path to verify
A developer can reproduce any shipped wave from the public repo. A reviewer can verify any signed receipt offline.

What we will not promise

Dates we cannot keep
A "Q4 2026" on a Later card is the target window of the engineering review, not a commitment to a customer.
Vapor features
If a feature is not in Now, Next, Later, or Researching, it is not on the roadmap. The roadmap is exhaustive for the things we are willing to publish.
Roadmap rewrites
We do not rewrite this page to make a missed wave look like a planned one. The honest history lives in the changelog and in the git log.
