Precisions · 1.0 → 8.0 bit

Every precision tier. One command.

From ternary BitNet to FP8, the forge speaks every quantization scheme worth shipping. Pick the target precision; the compiler picks the kernel. K-score-gated so the receipt is honest even at 1.58 bit.

1.0bit

BinaryNet weights

Sign-only weights with FP16 scales. Extreme density, narrow safety margin. K-score gate trips early on most tasks; ideal for classifier heads.

binary
1.25bit

QTIP variant

Trellis-coded quantization. Sub-2-bit with calibration-free decoding. Holds K-score better than BinaryNet on instruction-following.

ternary+
1.58bit

BitNet b1.58

Ternary weights {-1, 0, +1}. Approximately 6× smaller than NF4. Projected RTX 5090 footprint: 32B class fits in roughly 6.4 GB.

ternary
2.0bit

EXL2 / AQLM

Vector-quantized two-bit codebooks. EXL2 for llama.cpp paths; AQLM for transformer paths. K-score holds on most reasoning tasks at 32B+.

2-bit
2.25bit

QuIP#

Lattice quantization with codebook compression. Slightly larger than AQLM, materially better K-score on math and code.

2-bit+
3.0bit

GPTQ-3 / GGUF Q3_K

Per-group three-bit with mixed-precision outliers. Practical floor for most production deployments under a strict K-score gate.

3-bit
4.0bit

NF4 + double-quant default

NormalFloat 4 with double quantization (bitsandbytes 0.49.2). The forge default. Verified: DeepSeek-R1 32B 61.0 GB → 17.9 GB in 125.3s.

NF4
8.0bit

SmoothQuant / LLM.int8

Per-channel INT8 with activation smoothing. Near-lossless K-score on the toughest evals; preferred when accuracy budget is zero.

INT8
FP8

FP8 E4M3 + per-tensor scales

Native Hopper / Blackwell FP8 path. Compiles to TRT-LLM and vLLM with hardware-native kernels. Best wall-clock on H100/H200/B200.

E4M3
NVFP4

NVFP4 block-scaled

NVIDIA Blackwell-native four-bit floating point with shared exponents. Higher throughput than NF4 on B200; falls back to NF4 elsewhere.

Blackwell
Dyn

Dynamic 2.0 · per-layer adaptive

Layer-by-layer precision search: 2-bit for safe layers, 4-bit for sensitive ones, 8-bit for outlier-heavy projections. The forge picks the schedule by K-score sensitivity.

mixed
KV

TurboQuant KV · 2-bit with rotary preservation

Quantizes the KV cache, not the weights. Cuts memory at long context (32k+) by 4×. Composes with every weight-precision above.

Modes · stackable

Stack any modes. The recipe is the artifact.

Each mode is a flag, not a fork. A single kolm build can combine reasoning traces, on-policy self-distillation, multi-teacher fusion, agentic compilation, and TIES merges. The recipe is recorded in the receipt; an auditor can re-derive every choice.

  1. reasoning

    CoT traces

    The teacher emits chain-of-thought traces alongside answers; the student learns the reasoning pattern, not just the verdict. Distilling DeepSeek-R1 reasoning into Qwen2.5-7B holds 11+ tok/s on a 5090 with K-score parity.

    $ kolm build qwen2.5-7b --mode reasoning --teacher deepseek-r1

  2. on-policy

    self-distill

    The student generates a candidate; the teacher grades it; the student learns from its own mistakes rather than from teacher-only outputs. Closes the train/inference distribution gap; what the student saw at train time matches what it emits at serve time.

    $ kolm build qwen2.5-7b --mode on-policy --teacher claude-opus-4-7

  3. multi-teacher

    fused · cost-traced

    Claude, DeepSeek, GPT, and Gemini score each candidate. Per-token best-of fuses their judgments. The student inherits the best of every teacher rather than the bias of any one. Honest cost-trace per teacher recorded in the receipt.

    $ kolm build qwen2.5-7b --mode multi-teacher --teachers claude,deepseek,gpt,gemini

  4. agentic

    multi-step · collapsed

    A LangGraph or CrewAI run, with its tool calls and intermediate plans, is captured and compiled into one forward pass. Multi-hop reasoning collapses into a single student inference. Trace is preserved end-to-end.

    $ kolm build agent-spec.json --mode agentic --framework langgraph

  5. merge-first

    mergekit · TIES / DARE / SLERP

    TIES, DARE, SLERP, and Model-Stock from mergekit operate on multiple teachers before distillation begins. The student then distills from the merged teacher; you bake the ensemble into the weights rather than carrying it at inference.

    $ kolm build qwen2.5-7b --mode merge-first --merge ties:llama-3.3-70b,deepseek-r1

  6. MTP

    multi-token · 2–4 per pass

    Multi-token prediction trains the student to emit 2–4 tokens per forward pass, with speculative decoding built into the same weights. No separate draft model. Materially higher tok/s with no K-score loss on instruction-following.

    $ kolm build qwen2.5-7b --mode mtp --mtp-heads 4

  7. low-bit-specialist

    1.58 / 2.0 · gate-enforced

    Extreme quantization (BitNet b1.58, AQLM, QuIP#) with the K-score safety floor enforced at compile time. If the gate fails, the build refuses to sign. You only ship a sub-2-bit artifact when the receipt proves it earned the gate.

    $ kolm build qwen2.5-7b --mode low-bit-specialist --precision 1.58

Provenance · signed by default

Every artifact ships with a receipt.

A forge build is not a binary blob; it is a content-addressed object with a verifiable lineage. Hand the .kolm to an auditor; they can re-derive every claim offline.

  1. Ed25519 signed manifest

    Live

    Every receipt is signed with the tenant Ed25519 key on compile. The public-key fingerprint is embedded; key rotation does not invalidate old receipts.

    K-score spec

  2. JCS-canonical SHA-256

    Live

    Receipt JSON is JCS-canonicalized (RFC 8785) and hashed before signing. Editing any field — the K-score, the teacher id, the precision — breaks the signature deterministically.

    Verify a sample

  3. Reproducible from spec

    Live

    Same spec + same base model SHA-256 + same eval-pack cid = same artifact_cid. The receipt records what is inside the determinism envelope (seeds, scheme) and what sits outside (BLAS kernel, CUDA driver).
