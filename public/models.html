Runtimes

Where the .kolm artifact actually runs.

llama.cpp + GGUF

The default runtime for laptop, edge, and CPU-friendly deployments. The compiler emits GGUF directly; llama.cpp loads it; the .kolm sidecar lives next to the weights and is verified at load time.

Targets: Mac, Linux, WindowsGPU: CUDA, Metal, ROCm, VulkanBest for: developer laptop, edge, CPU-only

vLLM

Throughput-oriented GPU serving. PagedAttention plus the kolm load-time verifier; the runtime refuses unverified artifacts by default. Pairs cleanly with the LoRA adapter format kolm emits.

Targets: Linux + NVIDIAGPU: A100, H100, L40S, RTX 4090/5090Best for: production GPU serving

TensorRT-LLM

NVIDIA-specific runtime when the latency profile demands the absolute floor. The compiler exports the TRT-LLM engine; the .kolm receipt covers the full conversion graph; verification is at load time.

Targets: Linux + NVIDIAGPU: Hopper, Ada, BlackwellBest for: latency-critical production

MLX (Apple Silicon)

Apple-native runtime. The compiler emits MLX-format weights for M-series chips. The same artifact metadata travels; verification works identically; throughput is excellent on M4 Max and M4 Ultra.

Targets: macOS 14+GPU: Apple Silicon M1+Best for: developer laptop on Mac

ONNX Runtime

Cross-platform inference for mobile and embedded targets. The compiler exports INT4 ONNX with the kolm receipt sidecar. The verifier ships as a small C library for embedded contexts.

Targets: iOS, Android, Linux, WindowsGPU: CPU + GPU per platformBest for: mobile, embedded, cross-arch

SGLang / TGI

Alternate GPU runtimes for teams already invested in them. The compiler emits the formats they expect; the verifier wraps the loader; per-call receipts are emitted by the kolm gateway in front of the runtime.

Targets: Linux + NVIDIA / AMDGPU: as configuredBest for: existing inference stack adopters
