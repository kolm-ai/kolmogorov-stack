CLI reference

The kolm command, end to end.

Every verb the CLI accepts. The CLI is the canonical surface; SDKs wrap subsets. The source is in the open-source repo.

kolm signup
Create a kolm.ai account from the terminal. Prompts for email; opens the browser to confirm; writes credentials to ~/.kolm/config.json.
kolm login / logout
Sign in to an existing account; sign out clears the local credentials. The credentials are scoped per tenant and per API key.
kolm whoami
Print the current authenticated tenant, plan, and rate-limit posture. Useful in CI to confirm the right key is in scope.
kolm capture
Route a model call through the kolm gateway so the prompt, response, and metadata are captured into the local store. Captures are the input to distillation.
kolm distill
Distill a student model from captures. Flags: --teacher, --base, --captures, --epochs, --temperature, --resume-from. Output: a candidate weight set ready for compile.
kolm compile
Take a distilled candidate, quantize to the target precision, package as a .kolm artifact with metadata + receipt + Ed25519 signature. Flags: --target, --precision, --attestation-report.
kolm score
Compute the K-score against the held-out eval pack. Embeds the score in the receipt. Flags: --evals, --teacher-cache, --min (exits non-zero if score is below the floor).
kolm verify
Validate a .kolm artifact against its signature, eval pack hash, and teacher attestation. Prints the receipt as JSON. Exit 0 only if every check passes.
kolm run
Run the artifact locally against an input. Loads the runtime adapter automatically based on the artifact's target. Useful for smoke tests and ad-hoc inference.
kolm bakeoff
Score N artifacts on the same eval pack and pick a winner. Flags: --artifacts, --evals, --metric. Used in production gating when comparing iterations.
kolm drift
Compare a fresh capture window against the artifact's training distribution. Emits a drift score; triggers a re-distill alert when the score crosses the configured threshold.
kolm doctor
Sanity-check the local install: GPU, drivers, Python, Node, disk, network, credentials. Emits a structured report and an actionable hint list when something is missing.

Runtimes

Where the .kolm artifact actually runs.

Full runtime matrix lives on the runtimes page. Quick reference: GGUF + llama.cpp for laptop and edge, vLLM for production GPU serving, TensorRT-LLM for latency-floor production, MLX for Apple Silicon, ONNX for mobile and embedded, SGLang and TGI as alternates. Every artifact ships verifier sidecars for each target.

# Load the same .kolm in three runtimes
kolm run cm-claims-v4.kolm --target gguf --input "$(cat case.txt)"
kolm run cm-claims-v4.kolm --target vllm --input "$(cat case.txt)"
kolm run cm-claims-v4.kolm --target trt-llm --input "$(cat case.txt)"

# Each invocation runs the verifier first; refuses an unverified artifact.

The kolm docs surface reflects the kolm voice: every claim is grounded, every flag is verifiable against the installed CLI through `kolm --help`, every receipt can be replayed against `kolm verify`, every artifact carries a tenant-scoped binder roster row that the user can inspect through `kolm inspect`. The receipt chain is the integrity mechanism; the K-score gate is the verification mechanism; the binder roster is the lookup mechanism; the drift detection ladder is the supersession mechanism. The standing constraint is honest scope: every page distinguishes what kolm ships today from what its competitors ship from what is still in the planning wave. The honest-scope statement on every page is the brand contract.

W206 docs extension. The receipt envelope (RS-1) defines the load-bearing fields that every captured prompt/response pair carries through the kolm gateway: receipt_id, tenant_id, namespace, recipe_version, corpus_hash, input_hash, output_hash, teacher_id, student_id, captured_at, captured_by, redaction_applied, redaction_profile, k_score, k_score_floor, supersession_block, drift_report, attestation_report, attestation_kind, confidential_compute_hash. The artifact bundle (.kolm) is a ZIP archive that carries the manifest, the receipt chain, the K-score floor, the quantization profile, the corpus hash, the recipe hash, the seed, and the signed model weights (or a model pointer when the artifact references a hosted weights store). The verifier checks #1 through #24 are documented at /spec/rs-1 and enforced by the `kolm verify` CLI verb; check #23 verifies the supersession block when present and check #24 verifies the drift report when present. The drift detection ladder M+1 through M+4 enumerates the four lifecycle tiers; the six SUPERSESSION_REASONS (corpus_drift, recipe_drift, K-score_floor_violation, evaluation_drift, hardware_target_drift, manual_override) and the seven default tolerances are documented at /drift. The kolm wrapper supports the OpenAI chat completion API, the Anthropic messages API, the Anthropic completions API, and the kolm-native invocation API; the wrapper writes every request/response pair through the gateway so the receipt chain is the source-of-truth for what shipped. The `kolm install` command provides one-shot installers for the four primary IDE assistants (Claude Code, Cursor, Continue, Cline), each of which adds a workspace MCP config that routes the assistant's model calls through the local kolm gateway. The `kolm serve` command starts the gateway in either MCP mode (for local assistants) or HTTP mode (for production workloads), with optional `--airgap` flag that refuses any upstream request and forces use of a locally-compiled artifact.
