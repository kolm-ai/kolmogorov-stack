01Install

Three install paths.
Same binary.

The CLI is a single Go binary. Pick the package manager you already use. Every release is signed and reproducible from source at github.com/sneaky-hippo/kolmogorov-stack.

Node · macOS · Linux · Windows

npm

# global install
npm i -g @kolmogorov/kolm
kolm --version
# kolm 0.1.x
Curl · macOS · Linux

curl

# single command
curl -fsSL https://kolm.ai/install.sh | sh
kolm --version
# kolm 0.1.x
Homebrew · macOS · Linux

brew

# tap and install
brew install kolmogorov/tap/kolm
kolm --version
# kolm 0.1.x
02First compile

Five steps. Five minutes.

From kolm login to a signed artifact you can run on a phone. No GPU, no Docker, no MLOps. The only thing that costs money is the frontier API call you would have made anyway.

01

Sign in.

Sign up takes an email. The backend issues a kolm API key. Paste it into the CLI once.

kolm login
› opens https://kolm.ai/signin
› paste api key · saved to ~/.kolm/credentials
✓ signed in as you@company.com
02

Bring your frontier key.

Anthropic, OpenAI, Google, Mistral, Together, Fireworks, Bedrock, Azure, or your own gateway. Set once. Never persisted server-side.

export ANTHROPIC_API_KEY=sk-ant-...
# or
kolm config set teacher.provider anthropic
kolm config set teacher.model    claude-opus-4-7
03

Compile.

One command. Pick a template, point at your data, set your gate. The orchestrator does distill, decompose, train, evaluate, and package.

kolm compile "summarise tickets in our voice" \
    --examples ./tickets.jsonl \
    --base     qwen2.5-3b \
    --gate     0.70
04

Inspect the artifact.

kolm inspect opens the file: K-score, tests pass-rate, signature chain, every layer that went into it.

kolm inspect ./out/support.kolm
✓ manifest    rs-1 v0.1
✓ k-score     0.94   (gate 0.70)
✓ tests       97.4% pass · n=412
✓ hmac chain  anchored to registry
05

Run it. Or serve it.

Run locally on the same machine. Or expose it as an MCP server, an HTTP endpoint, or a stdio process for any agent.

kolm run   ./out/support.kolm "hi, my login is broken"
kolm serve ./out/support.kolm --mcp
# or import directly
import { run } from "@kolmogorov/sdk"
const reply = await run("./support.kolm", input)
03Concepts

Six ideas you keep forever.
Everything else is wiring.

If you understand these six, you understand kolm. The rest of the docs are reference material you read once and grep later.

Artifact

A single signed .kolm zip: model, LoRA, recipes, recall index, tests, manifest, signature. Reproducible byte-for-byte.

/anatomy →

Recipes

Deterministic draft tokens extracted from observed model behavior. The compute moat. Speculative decoding without a draft model.

/atlas →

K-score

Harmonic average of accuracy, size, latency, cost, coverage. One number. Below the gate, the artifact does not ship.

/k-score →

Registry

Public append-only registry of recipe drafts and artifact hashes. Every compile contributes back. Future compiles get cheaper.

/atlas →

Recall

Multimodal sqlite-vec index packed inside the artifact. Text, image, audio, video, PDF. Grounding ships with the model.

/anatomy#recall →

Serve · MCP

kolm serve --mcp exposes any artifact as a Model Context Protocol server. Drop into Claude, Cursor, Zed, or any MCP client.

/serve →
04API reference

Every endpoint.
Auth is one cookie.

Sign in once with an API key. The server sets kolm_session as an httpOnly cookie. Every other call rides that cookie. The CLI does it for you; the SDK does it for you; fetch() does it for you with credentials: "include".

POST
/v1/signup

Body: { email, name }. Backend auto-issues an api_key. No password, ever.

POST
/v1/signin

Body: { api_key }. Sets kolm_session httpOnly cookie. Same as /v1/session/login.

GET
/v1/account

Returns your account, current API key, and a snapshot of recent compile jobs.

POST
/v1/compile

Body: { task, examples?, evals?, base?, gate? }. Returns { job_id }. The job streams via SSE on the same id.

GET
/v1/compile/:job_id

Status, logs, K-score, and a download URL when ready. The manifest.k_score field is the official ship gate.

GET
/v1/registry/export

The public deterministic-draft pack. Pre-shaped for llama.cpp and vLLM. Cached at the edge for 60 seconds.

GET
/v1/health

Live status. Auth required. Public /health is a single boolean — no internal state.

GET
/sdk.js

Browser-native SDK. Versioned at /sdk-<sha>.js with SRI hashes published at /sdk-versions.json.

05Open spec · RS-1

The format is open.
The receipts are open.

RS-1 is the public, MIT-licensed specification of the .kolm artifact, the manifest schema, and the receipt chain. Implementable from scratch in any language. Read it. Fork it. Ship a different runtime.

RS-1 · core

The .kolm artifact format

Zip layout, layer ordering, manifest fields, signature placement. What every conforming runtime must accept.

rs-1.md github
RS-1 · manifest

manifest.json schema

JSON Schema for the manifest at the root of every artifact. Task, examples hash, base, K-score, layer hashes.

manifest-v0.1.json github
RS-1 · receipts

Receipt chain schema

HMAC chain from compile-input to artifact to inference-output. Anchored to the public registry. On-chain optional.

receipt-v0.1.json github
