Meta-demo / The assistant

kolm-assistant-1.5b - the AI that runs this site.

The bot answering questions on kolm.ai is itself a .kolm artifact. We distilled it with our own gateway from a Claude + GPT-4o teacher council, sealed the weights with a passport, and shipped it as a 1 GB GGUF that runs on a laptop. The CLI you would use to build it is the CLI we used to build it.

1.5Bparams Q4_K_M~1.0 GB GGUF Qwen2.5-1.5B-Instructbase, Apache-2.0 teacher council: Claude + GPT-4o

How it was made.

A four-step recipe we documented because every other kolm artifact takes the same shape. Base model, scoped corpus, teacher distillation through the gateway, then a sealed quant ladder. Nothing in the pipeline is bespoke to this build.

  • Base. Qwen2.5-1.5B-Instruct (Apache-2.0). Small enough to run on a CPU; coherent enough to answer about kolm without a heavy reasoning chain.
  • Corpus. ~900 training pairs scraped from cli/kolm.js verb inventory, data/error-catalog.json, and every docs/** page. Balanced across 10 buckets (docs, CLI help, error fix, workflow, casual, guardrail, concept, pricing, hardware, deflection).
  • Teacher council. Claude (primary) + GPT-4o (secondary) called through kolm gateway with a strict no-hallucinated-CLI system prompt. Pairs where the teachers disagreed (Jaccard < 0.85) went to a manual queue and never made it into training.
  • Compile and quant. LoRA r=16, 3 epochs on a consumer GPU. Quantized to GGUF Q4_K_M with an importance matrix from the holdout. Sealed with a compile-passport.json chaining training -> bench -> gate -> weight hashes.

Total compile cost was a small fraction of what a single month of frontier API calls runs at the same volume - the point of this exercise is that the artifact is the product, not the call budget.

The hard gates it passed.

Compile blocks publish unless two gates clear: K-Score above the policy threshold on a held-out eval, and zero hallucinated CLI verbs across the entire holdout. Both gates are recorded in the compile passport and verifiable via kolm verify.

Try it.

Ask the assistant anything about kolm - CLI verbs, K-Score, the compile pipeline, how distillation differs from quantization. Each reply ships with a passport hash and a verify link so you can audit the model that produced it.

Pre-launch the widget routes to the public docs endpoint; once that is online, every reply also feeds back into the capture lake for the next compile cycle. Use the CLI for unlimited turns: kolm chat.

Verify it.

Every .kolm artifact is sealed with a passport - a JSON manifest that hashes the training data, the training config, the resulting weights, and the deployed model. Each layer references the one below. Break any link and the chain fails verify.

01 Training data hash. SHA-256 of the deterministic stratified split (training-passport.json). Every pair is traceable to a seed row, a teacher dispatch ID, and an inter-teacher Jaccard score.
02 Training config hash. SHA-256 of (student base, LoRA rank, epochs, learning rate, quant target, holdout policy). Locked into the passport.
03 Output weight hash. SHA-256 of the merged .gguf file. The bench and gate steps run against this exact hash; if the file drifts, the gate report invalidates.
04 Deployed model hash. The hash served to your prompt. Returned as passport_hash on every reply envelope. Click through to /v1/verify/<hash> for the signed chain.

$ curl https://kolm.ai/v1/verify/8d6c4b9c1369fba9

The three-layer fallback chain.

The same assistant answers in your CLI, on your docs page, and inside your authenticated account. Layer 1 wins on latency, layer 2 keeps the experience online when you don't have the weights yet, and layer 3 is a frontier escape hatch capped at one cent per turn.

Implementation lives in src/assistant-client.js - same client backs the CLI kolm chat, the docs widget on this page, and the in-app assistant the moment you sign in.

Open source, top to bottom.

No part of this pipeline is private. The base model is Apache-2.0, the corpus generator is Apache-2.0, the eval suite is Apache-2.0, the weights are Apache-2.0, and the compile passport is checked into the repo. If you want to re-distill against a different teacher council or against your own private corpus, the recipe is the same recipe.

Why ship this at all?

Because every other vendor on this market answers questions about their product with a wrapper around someone else's frontier model. The pitch is small models distilled for your data, your hardware, your compliance envelope - so the meta-demo had to be that the product page itself is answered by a small model distilled for kolm's data, our hardware, our compliance envelope. If it doesn't work for us, it doesn't work for you.

Any rough edge you find here is a rough edge you can also find when you compile your own assistant tomorrow. Caveats are documented in the passport.
