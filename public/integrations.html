Build-pipeline gates

2 adapters

Reusable workflow files. Drop into your existing pipeline. The gate is honest: the build fails when the artifact is not ready to ship.

Live GitHub Actions Drop-in workflow. Runs kolm score --min and uploads the receipt. Sample in the repo. actions/runner Live GitLab CI Sample .gitlab-ci.yml that mirrors the GitHub action behavior. Same exit-code semantics. gitlab-runner
.github/workflows/kolm-score.yml · minimal gate
name: kolm-score
on: [push]
jobs:
  score:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @kolm/sdk
      - env:
          KOLM_API_KEY: ${{ secrets.KOLM_API_KEY }}
        run: |
          kolm verify ./dist/cm-claims-v4.kolm
          kolm score ./dist/cm-claims-v4.kolm --evals ks-finance-2026-q2 --min 80
      - uses: actions/upload-artifact@v4
        with:
          name: kolm-receipt
          path: ./dist/*.kolm.receipt.json

The contract every adapter honors

Eighteen adapters, one contract. A developer or engineer who reads one integration page can predict the shape of every other one.

install

One install line, one env var

stable

Every adapter page leads with a single install command and the env vars that need to be set. KOLM_API_KEY is the only required one; KOLM_BASE_URL when self-hosted. The CLI writes the key to ~/.kolm/credentials on kolm login, so most adapters work without manual env wiring.

verify

Verify step in the install path

stable

Every adapter ships a verification step. For IDE plugins it is the status-bar reading the tenant. For CI it is a no-op kolm whoami in the pipeline. For gateway proxies it is a single captured row appearing in the local store. If the verify step does not pass, the install did not work.

envelope

Honest envelope passes through

stable

Every adapter preserves the {ok, code, error, data} envelope. No adapter hides a 4xx behind a success message. The OpenAI and Anthropic SDK adapters surface kolm errors next to the upstream provider errors so a developer sees both layers.

scope

Honest scope per adapter

stable

If an adapter does not yet cover a kolm verb (e.g. the Cloudflare binding is verify-only; it does not compile), the adapter page says so explicitly. No silent gaps; no "coming soon" without a date. The /changelog records every adapter capability change.
