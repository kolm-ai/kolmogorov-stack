kolm in your stack. One config change.

Point your existing SDK at https://kolm.ai/v1. The OpenAI client, the Anthropic client, LangChain, Vercel AI, LiteLLM, your editor proxies – one line each. Captures, redaction, and signed receipts come along for free.

The nine copy-paste recipes.

One line of config per ecosystem. Each card links to a focused recipe page with a Python + JS snippet you can paste into your app today.

OpenAI SDK

one line

Python OpenAI(base_url=...) or JS new OpenAI({ baseURL: ... }). Streaming, tool calling, vision – all work unchanged.

OpenAI(base_url="https://kolm.ai/v1", api_key="ks_...")

Anthropic SDK

one line

Python Anthropic(base_url=...) or JS new Anthropic({ baseURL: ... }). The Messages API behaves the same.

Anthropic(base_url="https://kolm.ai/v1/capture/anthropic", api_key="ks_...")

OpenRouter

routing preserved

HTTP-Referer + X-Title headers travel end to end and stamp the receipt so app attribution stays correct.

default_headers={"HTTP-Referer": ..., "X-Title": ...}

LangChain

drop-in

ChatOpenAI(base_url=...) in Python, ChatOpenAI({ configuration: { baseURL } }) in JS. LangGraph + LangServe work too.

ChatOpenAI(base_url="https://kolm.ai/v1", api_key="ks_...")

Vercel AI SDK

next.js

createOpenAI({ baseURL }) from @ai-sdk/openai. Streaming, server actions, edge runtime – all preserved.

createOpenAI({ baseURL: "https://kolm.ai/v1", apiKey })

LiteLLM

router

One config.yaml entry. Fallbacks, retries, budgets work the same. Each model alias becomes a kolm destination.

api_base: https://kolm.ai/v1

Agents & editors

If your agent speaks MCP, it can drive kolm. If your editor speaks Cursor rules or Claude Code skills, it can drive kolm.

Claude Code (MCP)

shipped

Run kolm install claude-code --apply and the installer writes mcpServers.kolm into ~/.claude/settings.json. Claude can then synthesize, run, and inspect recipes as MCP tools over stdio - no manual JSON editing.

npm i -g github:sneaky-hippo/kolmogorov-stack
kolm install claude-code --apply
# writes ~/.claude/settings.json mcpServers.kolm: { command: "kolm", args: ["serve","--mcp"] }

Cursor

shipped

Compile on every push. Gate the build on K-score. Publish the artifact as a build attachment.

GitHub Actions

shipped

Composite action at .github/actions/kolm-compile. Drops into any workflow; uploads the .kolm as a build artifact.

 with:
 task: 'classify support tickets'
 examples: ./labels.jsonl
 api-key: ${{ secrets.KOLM_API_KEY }}

Agent frameworks

First-party LangChain and LlamaIndex source adapters live in packages/. Same .kolm artifact, same receipt chain, just wired in the framework your agent already lives in. Every call returns the receipt alongside the text so audit, drift detection, and replay work the same as in the bare CLI.

LangChain.js

source package

Drop-in KolmLLM that satisfies the LangChain LLM interface. invokeWithReceipt(prompt) returns { text, receipt } so chains keep the receipt chain intact. Subprocess mode for local .kolm files, HTTP mode for hosted artifacts.

git clone https://github.com/sneaky-hippo/kolmogorov-stack
npm i ./kolmogorov-stack/packages/langchain-kolm
import { KolmLLM } from '@kolm/langchain';
const llm = new KolmLLM({ artifactPath: './ticket-classifier.kolm' });
const { text, receipt } = await llm.invokeWithReceipt('refund #1234');

LangChain (Python)

source package

Same interface, Pythonic. invoke_with_receipt(prompt) returns a tuple. Works in LangGraph nodes, LangServe handlers, and plain scripts.

git clone https://github.com/sneaky-hippo/kolmogorov-stack
pip install -e ./kolmogorov-stack/packages/python-langchain-kolm[langchain]
from kolm_langchain import KolmLLM
llm = KolmLLM(artifact_path='./ticket-classifier.kolm')
text, receipt = llm.invoke_with_receipt('refund #1234')

LlamaIndex.TS

source package

Implements the LlamaIndex LLM protocol. complete(prompt) returns { text, raw } where raw.receipt carries the kolm receipt chain. Ready to plug into a QueryEngine.

git clone https://github.com/sneaky-hippo/kolmogorov-stack
npm i ./kolmogorov-stack/packages/llamaindex-kolm
import { KolmLLM } from '@kolm/llamaindex';
const llm = new KolmLLM({ baseUrl: 'https://kolm.example', apiKey: 'k_...' });
const r = await llm.complete('summarize this ticket');

LlamaIndex (Python)

source package

Python LlamaIndex adapter exposing the same invoke_with_receipt interface as the LangChain one. Lets QueryEngines and SubQuestion agents speak to a compiled .kolm without losing the audit trail.

git clone https://github.com/sneaky-hippo/kolmogorov-stack
pip install -e ./kolmogorov-stack/packages/python-llamaindex-kolm[llamaindex]
from kolm_llamaindex import KolmLLM
llm = KolmLLM(artifact_path='./classifier.kolm')
r = llm.complete('summarize this ticket') # r.raw.receipt is the kolm receipt

Workflow automation

No-code triggers and downstream wiring through the HTTP API. Zapier and Make.com do not have native Kolm apps yet; use their generic webhook and HTTP modules.

Zapier

webhook

Zapier has no native Kolm app yet. Use Webhooks by Zapier to call the Kolm HTTP API. Output payload includes text, k_score, and the full receipt chain so Zaps can branch on quality.

# Webhooks by Zapier:
POST https://kolm.ai/v1/run
authorization: Bearer $KOLM_API_KEY
{ "artifact": "ticket-classifier", "input": "{{trigger.body}}" }
docsdiscussions

Make.com

HTTP module

Make.com has no native Kolm module yet. Use Make's generic HTTP module to call the Kolm API directly. The body shape below is the supported integration contract.

# Make.com HTTP module:
POST https://kolm.ai/v1/run
authorization: Bearer $KOLM_API_KEY
{ "artifact": "ticket-classifier", "input": "{{1.payload}}" }
